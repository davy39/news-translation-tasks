Par Alexander Petkov

Ce guide est la référence absolue pour quiconque souhaitant choisir le bon chemin de carrière en programmation en 2018.

Et laissez-moi être clair sur un point :

Ce n'est pas un article sur les « langages les plus tendance » qui lance des noms et des mots à la mode.

**Il s'agit d'une revue objective et pratique de l'état actuel, des tendances et des prédictions sûres pour un avenir proche de l'industrie informatique au début de 2018.**

Il est basé sur des données statistiques provenant de diverses sources fiables et est le résultat d'une période de deux semaines de recherche approfondie.

L'objectif de cet article est de vous aider à prendre une décision éclairée sur les langages recherchés par le marché de l'informatique en 2018. Il vous aidera à décider quelle niche vaut la peine d'être maîtrisée et laquelle vous aidera à commencer (ou à continuer) une [**carrière réussie et rentable en tant que développeur logiciel**](https://www.myfirstitjob.com/career-shift/).

Lorsqu'ils essaient de décider quel langage de programmation offre le meilleur chemin de carrière, les développeurs recherchent généralement ces quatre traits :

1. Un salaire élevé
2. La popularité — de nombreuses offres d'emploi, variété
3. La tendance à devenir plus demandé à l'avenir
4. De préférence facile à apprendre et agréable à utiliser

Pour chaque langage présenté dans la liste, vous verrez :

1. Une brève introduction pour vous familiariser avec le langage, les types d'options de carrière qu'il peut offrir et ce qui est intéressant à son sujet
2. Des diagrammes de popularité du langage selon [TIOBE](https://www.tiobe.com/tiobe-index/) et [GitHub Octoverse](https://octoverse.github.com/)
3. Une comparaison des salaires basée sur les données de [Indeed.com](https://www.indeed.com/) et [l'enquête 2017 de StackOverflow](https://insights.stackoverflow.com/survey/2017#technology-top-paying-technologies-by-region)
4. La tendance de la demande au cours des cinq dernières années selon [Google Trends](https://trends.google.com/trends/)
5. Un court extrait de code pour vous donner une idée visuelle de la syntaxe du langage

À la fin, j'ai inclus quatre langages moins connus qui ont gagné en popularité rapidement au cours des dernières années et qui devraient être très demandés en 2018.

Commençons directement avec le premier choix.

### 1. JavaScript

JavaScript est le langage le plus populaire selon [l'enquête annuelle de StackOverflow](https://insights.stackoverflow.com/survey/2017#technology-programming-languages), avec 62,5 % des répondants affirmant l'utiliser.

C'est sans aucun doute le langage dominant du web et la croissance de JavaScript au cours de la dernière décennie a été immense. Pourquoi ? Il suffit de regarder autour de vous et de compter le nombre d'appareils connectés au web que vous pouvez voir. Le cofondateur de Stack Overflow et figure populaire de la programmation [Jeff Atwood](https://en.wikipedia.org/wiki/Jeff_Atwood) a famously [dit](https://blog.codinghorror.com/the-principle-of-least-power/) :

> Toute application qui peut être écrite en JavaScript finira par être écrite en JavaScript.

En gardant cela à l'esprit, il est sûr de dire qu'il n'y aura pas de pénurie d'opportunités en JavaScript en 2018 et au-delà.

Les connaissances en JavaScript alimentent de nombreux frameworks front-end tels qu'Angular, React, Ember, Backbone, et autres, ainsi que l'environnement d'exécution Node.js, qui vous permet d'exécuter JavaScript sur le backend avec une haute efficacité.

Trois des quatre [frameworks les plus populaires](https://insights.stackoverflow.com/survey/2017#technology-frameworks-libraries-and-other-technologies) pour 2017 selon StackOverflow sont basés sur JavaScript.

Si vous aimez voir immédiatement les résultats de votre travail en action, par exemple en créant des composants web interactifs, JavaScript est une bonne idée pour vous. Gardez à l'esprit qu'une carrière en JavaScript implique également que vous devriez être à l'aise avec HTML et CSS, qui sont essentiellement ce dont les pages web sont faites.

Des outils comme [Apache Cordova](https://cordova.apache.org/) ou [React Native](https://facebook.github.io/react-native/) permettent d'utiliser JavaScript pour les applications mobiles. Il est même possible de se lancer dans le développement de jeux ou d'applications de bureau avec des projets comme [Electron](https://electronjs.org/).

### 2. Swift

Swift est un langage de programmation relativement nouveau, publié par Apple en 2014. Il s'agit d'un langage pour développer des applications natives iOS ou macOS.

Il est considéré comme une amélioration en termes d'utilisabilité et de performance par rapport à Objective-C — le langage utilisé pour les systèmes d'exploitation iOS et macOS d'Apple.

Swift est généralement apprécié par les développeurs, se classant à la 4ème place dans la liste des [langages les plus aimés](https://insights.stackoverflow.com/survey/2017#technology-most-loved-dreaded-and-wanted-languages) pour 2017, après avoir été [n°1 en 2015](https://insights.stackoverflow.com/survey/2015#tech-super). Si vous choisissez une carrière avec Swift, il y a de fortes chances que vous l'apprécierez.

Si vous souhaitez vous lancer dans le développement mobile, vous devriez définitivement considérer Swift comme un chemin de carrière bien rémunéré. En général, les applications iOS se sont avérées [plus rentables](http://bgr.com/2016/07/20/ios-vs-android-developers-profits-app-store-google-play/) que les applications Android.

### 3. Python

Python est un langage polyvalent que l'on peut trouver presque partout aujourd'hui. Vous le trouverez dans les applications web, les applications de bureau, les serveurs réseau, l'apprentissage automatique, les outils multimédias et bien plus encore.

Il est utilisé par des grands acteurs comme la NASA ou Google, où le créateur de Python [Guido van Rossum](https://en.wikipedia.org/wiki/Guido_van_Rossum) a été employé pendant environ 8 ans, écrivant principalement... du Python.

Le code Python est propre, lisible et bien structuré. L'indentation correcte n'est pas seulement pour la beauté ici — elle détermine l'exécution du code.

Les frameworks de développement web basés sur Python comme [Django](https://en.wikipedia.org/wiki/Django_(web_framework)) et [Flask](https://en.wikipedia.org/wiki/Flask_(web_framework)) ont gagné de plus en plus en popularité. De plus, le langage est largement équipé de bibliothèques de qualité pour l'apprentissage automatique et l'analyse de données comme [Scikit-learn](https://en.wikipedia.org/wiki/Scikit-learn) et [Pandas](https://en.wikipedia.org/wiki/Pandas_(software)).

Dans l'ensemble, les chemins de carrière avec Python sont variés et là pour rester. C'est un bon choix pour les développeurs débutants, car il est de haut niveau et facile à lire et à comprendre.

### 4. Java

Java est sans doute le langage de programmation le plus populaire, car 90 % des entreprises du [Fortune 500](http://fortune.com/global500/list) l'utilisent largement. Son slogan célèbre « write once, run anywhere » capture l'une des clés qui rend Java si précieux — sa puissante machine virtuelle Java (JVM), qui le rend compatible multiplateforme.

Les chemins de carrière populaires avec Java sont développeur backend, développeur big data, ingénieur systèmes embarqués ou développeur Android. Bien que ce ne soit pas le langage le plus « tendance » du moment, Java est si largement utilisé que nous pouvons presque garantir qu'il ne disparaîtra pas dans la prochaine décennie et au-delà.

Pour cette raison, vous pouvez être confiant qu'il y a de nombreuses offres d'emploi en Java, tant dans votre ville qu'à distance, ce qui ne peut pas être dit pour certains des langages moins populaires de cette liste. Ainsi, si vous êtes à l'aise avec Java, ce n'est qu'une question de temps avant que vous ne trouviez la bonne place.

### 5. C++

C++ est un langage très efficace et flexible, créé pour la première fois en 1985. Il est resté très demandé en raison de ses performances, de sa fiabilité et de la variété des contextes dans lesquels vous pouvez l'utiliser.

De nombreux grands systèmes ont été créés et maintenus avec succès en utilisant C++, y compris des entreprises comme Microsoft, Oracle, PayPal et Adobe.

Une carrière avec C++ implique généralement le développement d'applications de bureau, en particulier pour des tâches intensives en performance. Bien que plutôt complexe, se familiariser avec C++ vous donnera une compréhension plus approfondie du fonctionnement des langages, par exemple en offrant des facilités pour la manipulation de la mémoire de bas niveau.

### 6. Ruby

Ruby est l'un des langages de programmation les plus aimés. Il est conçu pour être convivial et facile à utiliser par les développeurs, car même son propre slogan est « [le meilleur ami d'un programmeur](https://www.ruby-lang.org/en/) ».

Ruby est un langage de haut niveau qui vise à accomplir beaucoup avec peu de lignes de code propre et lisible. Cela demande parfois un effort significatif « sous le capot », ce qui rend Ruby relativement plus lent en termes d'efficacité par rapport à d'autres langages populaires — mais il booste définitivement votre productivité.

Un code Ruby bien écrit ressemble presque à des phrases écrites en anglais simple. C'est un excellent choix pour le premier langage à apprendre, car les débutants l'adoptent généralement rapidement et l'apprécient tout au long du processus.

Ruby est principalement utilisé pour son framework le plus populaire — [Ruby on Rails](http://rubyonrails.org/). Ruby on Rails est un framework web qui encapsule toutes les idées de Ruby dans un outil puissant pour le web. La grande productivité atteinte avec Rails en fait un [choix courant pour les startups](https://medium.com/@RubyGarage/what-are-the-benefits-of-using-ruby-on-rails-for-your-startup-f3d6f6b9938d) qui visent un bon départ.

Une carrière avec Ruby on Rails est définitivement un excellent choix, car le framework est « [optimisé pour le bonheur des programmeurs](http://rubyonrails.org/doctrine/#optimize-for-programmer-happiness) » et est également bien rémunéré. Le seul inconvénient majeur de choisir une carrière avec Ruby est que le langage n'est pas tout à fait aussi populaire que les langages mentionnés ci-dessus. Ainsi, les offres d'emploi avec Ruby peuvent être limitées dans certains endroits.

### 7. Rust

Rust est sponsorisé par Mozilla et a été élu [le plus apprécié par les développeurs](https://insights.stackoverflow.com/survey/2017#most-loved-dreaded-and-wanted) pour une deuxième année consécutive dans les enquêtes de StackOverflow. Il s'agit d'un langage relativement nouveau, apparu pour la première fois en 2010, qui a déjà gagné une popularité remarquable et devrait encore s'améliorer à l'avenir.

Rust est un langage compilé qui est souvent comparé à C, tant en termes de cas d'utilisation que de performance. La principale différence est que Rust est sûr en mémoire. L'une des fautes les plus courantes que l'on peut trouver dans le code C sont les pointeurs pendants, les débordements de tampon ou tout autre type d'erreurs de mémoire. Rust est créé dans le but d'éviter celles-ci — le langage rend littéralement _impossible_ pour vous de commettre de telles erreurs, car elles sont détectées lors de la compilation (avant que le code ne soit jamais exécuté).

Pour un débutant, Rust peut être un peu difficile à apprendre, car il insiste sur diverses règles pour atteindre la sécurité de la mémoire. Cependant, les développeurs expérimentés [l'adorent](https://medium.com/mozilla-tech/why-rust-is-the-most-loved-language-by-developers-666add782563), et il est tout à fait possible que dans les prochaines années, Rust soit très demandé.

En ce qui concerne le salaire, Rust est considéré comme bien rémunéré, [se classant deuxième mondial](https://insights.stackoverflow.com/survey/2017#technology-top-paying-technologies-by-region) dans l'enquête 2017 de StackOverflow.

Une autre raison de choisir Rust est que le langage est développé par un nom bien connu dans le domaine de l'informatique, Mozilla.

### 8. Elixir

Elixir est un autre nouveau langage, apparu pour la première fois en 2011, qui a immédiatement gagné en popularité.

Elixir a été inspiré par [Erlang](https://en.wikipedia.org/wiki/Erlang_(programming_language)), un langage développé dans les années 1980 par Ericsson et considéré comme l'un des meilleurs outils pour la forte concurrency. L'auteur d'Elixir, José Valim, a lui-même [dit](http://www.creativedeletion.com/2015/04/19/elixir_next_language.html) qu'il aimait tout dans Erlang, mais voyait aussi des possibilités d'amélioration. Le plus grand inconvénient d'Erlang pour les développeurs est la syntaxe souvent étrange et l'utilisabilité, ainsi que le manque de gestion intuitive des paquets.

Ainsi, Elixir apparaît — combinant des aspects de Ruby, un langage et un écosystème très conviviaux pour les développeurs, avec ceux d'Erlang.

Elixir est principalement utilisé pour le développement web, et les options de carrière sont généralement bien rémunérées mais limitées. La popularité du langage a augmenté d'année en année, donc si votre ville compte diverses entreprises informatiques, les développeurs Elixir peuvent être très recherchés.

En termes de salaire, Elixir est généralement bien rémunéré, se classant à la [troisième place mondiale](https://insights.stackoverflow.com/survey/2017#technology-top-paying-technologies-by-region) dans l'enquête 2017 de StackOverflow.

### 9. Scala

Scala signifie Scalable Language, et est l'une des nombreuses tentatives de « réécrire Java » tout en améliorant ses inconvénients. Le code Scala est compilé pour s'exécuter sur la machine virtuelle Java (JVM).

Nous pouvons convenir en toute sécurité que Scala est déjà un succès, étant donné que de grandes entreprises comme LinkedIn, Twitter et The Guardian [l'utilisent](http://www.scala-lang.org/old/node/1658) dans leurs bases de code.

Scala a la réputation d'être [un langage complexe](https://stackoverflow.com/questions/2573899/why-is-scala-very-complex) à apprendre pour un développeur junior. Mais ceux qui franchissent la courbe d'apprentissage profitent probablement d'une grande carrière, car les postes ouverts pour les développeurs Scala apparaissent définitivement de plus en plus.

Scala se classe haut dans les enquêtes sur les salaires, faisant partie du [top dix](https://insights.stackoverflow.com/survey/2017#technology-top-paying-technologies-by-region) mondial et partageant la première place aux États-Unis selon le rapport 2017 de StackOverflow.
