Le développement piloté par les tests est quelque chose que chaque développeur de logiciels devrait mettre en œuvre dans ses projets. 

Le succès de l'utilisation du [TDD](https://www.guru99.com/test-driven-development.html), cependant, dépend grandement de la productivité du développeur lors de la mise en œuvre du code et des tests d'application.

C'est là que les outils d'automatisation entrent en jeu.

Ci-dessous se trouve une liste de 10 frameworks/plateformes de test automatisé recommandés et un bref résumé de leurs fonctionnalités.

## [Selenium](https://www.selenium.dev/)

![Image](https://lh3.googleusercontent.com/fOEtWbGlSP_hzAkDaSowqKEUiz9l6e8l9nGNuVuZhXhCJRVGSEuA59oWIWyP387ndCrEsuB1K2Z3_c2quLaM1OB5iaLreOLydfaTiPlmhlAvj_OnS-jIJUhUBAg0bSZh1A)

Selenium est, sans conteste, le framework de test automatisé le plus populaire. Il s'agit d'un framework basé sur le navigateur qui fonctionne sur différents systèmes d'exploitation et navigateurs.

Il dispose de Selenium IDE, une extension Chrome et Firefox qui permet l'enregistrement et la lecture des interactions utilisateur. Pour les tests à grande échelle (c'est-à-dire, exécutés sur plusieurs machines), Selenium Grid est l'outil de choix.

L'outil le plus populaire du framework est Selenium Webdriver, qui est une collection de liaisons spécifiques à chaque langage et permet la création de tests de régression basés sur le navigateur.

## [Perfecto.io](https://www.perfecto.io/)

![Image](https://lh4.googleusercontent.com/DplVGhU1kNQJVWhayvQ4h738H_M97iYitDoxAaDFCRDTwlZffOg9KlyhuJM2QW1HvpJrAOvX8yhlyuufjiNc_GnoqUw7nZE2IEZIvjTFSKhVwZz57KENe4XXlRoGNAY9ww)

Perfecto est un framework de test pour les applications web et mobiles. Il est livré en tant qu'outil SaaS et permet aux ingénieurs de test d'exécuter leurs tests depuis n'importe où.

L'outil de test web permet des tests parallèles d'applications sur différents systèmes d'exploitation et navigateurs. Il permet l'exécution d'un grand volume de tests quotidiens, et les tests sont 50 % plus rapides que tout autre framework.

L'outil de test mobile permet aux ingénieurs de tester à la fois sur des émulateurs et des appareils réels. Tout comme l'outil de test web, il permet un grand volume d'exécutions de tests quotidiens (10 000 exécutions).

## [Serenity](http://www.thucydides.info/#/)

![Image](https://lh4.googleusercontent.com/TQ1Vqg7502EEaW8P8S9cYHz_1ebaXSqA95ilO6KzMe13vR_CK29WH_jZESyU2EutJD6XOV63R0PbnR7_NtqKgS59Rh7nk6iuA8YlcIlCFq3heOeN3b4K06Zs8_eLPSFeww)

Le slogan de Serenity BDD est « Automated Acceptance Testing with Style ». Cela est dû au fait que l'angle unique de Serenity est d'aider les testeurs à rédiger des rapports de test et une documentation de classe mondiale.

Les tests commencent par la création d'histoires utilisateur et de critères d'acceptation. Serenity automatise ensuite les critères d'acceptation. Les tests sont automatiquement décomposés en étapes qui les rendent plus lisibles. Les développeurs peuvent facilement exécuter ces tests par rapport aux exécutions réelles de l'application.

Une fois les tests terminés, Serenity crée des rapports détaillés qui incluent des captures d'écran des tests et toutes les informations pertinentes telles que les messages d'erreur et les temps d'exécution.

## [Cypress](https://www.cypress.io/)

![Image](https://lh4.googleusercontent.com/q6KHdpPaoPhHi5JKT5HA4oH_00gVtIuCW8JdPITzA7VxGZhL1GbFp3g5dxH9roP6WRU4wys8jQLWyH0_Depy1o4FblwzNweaEiCeS6AmlifxVFWE07IZlJX5VNM1M_Zvpg)

Cypress est un excellent outil à utiliser pour les tests frontend ou de bout en bout [automatisés](https://www.perfecto.io/blog/what-is-test-automation).

L'installation de Cypress dans un projet est aussi simple que l'exécution d'une commande `npm install cypress` ou `yarn add cypress`. Les développeurs JavaScript habitués aux outils de test unitaire tels que [Jest](http://jestjs.io/) ou [Mocha](https://mochajs.org/) trouveront la rédaction de scripts de test Cypress un jeu d'enfant.

Son tableau de bord rend les tests plus puissants et plus rapides en permettant de regrouper les tests par type de navigateur, environnement, type de package, etc. La fonction de parallélisation permet aux développeurs d'exécuter plus de tests et de tester plus de fonctionnalités facilement.

## [Lambda Test](https://www.lambdatest.com/)

![Image](https://lh5.googleusercontent.com/7w_SznSrnJ3RbUpAjsWmHuuHZmJYCRIJpncsvXuedQrBix8lp80YKqBDADgSjNKYdoY6a2q6gmeyPeQ3vDwG8y9BxP8iABYLJvpWq8d9_ATFXmdLhvCpW7Lz6XFmDrEWCg)

LambdaTest est un logiciel leader de test automatisé pour les applications de bureau et web.

Il permet des tests en direct et interactifs d'applications et de sites web hébergés publiquement ou localement.

LambdaTest permet également aux développeurs d'exécuter des scripts de test Selenium avec son Browser Testing Grid. Il s'agit d'une suite très puissante et polyvalente qui s'intègre également avec des outils CI/CD tels que Jenkins, Circle CI et Travis CI.

La partie la plus intéressante est qu'il permet des tests géolocalisés, ce qui signifie que les applications web peuvent être testées pour leurs performances en fonction de l'endroit dans le monde où elles sont accessibles.

## [TestProject.io](https://testproject.io/)

![Image](https://lh3.googleusercontent.com/AFtWUpNPNMMP382wf89DoDKiWgtJn8FFNY6NEYKT2DQ-w4PAJlw72WaByvEWHatg0hA6JZxYVSnnN0VovjB7swkEu_XstmWNnSiWaNzn-eCN2THVed57P0j4ZcLxcm6kEw)

La communauté de TestProject est probablement l'une des plus grandes parmi celles de la communauté des tests. Il s'agit d'un framework de test end-to-end et API hébergé dans le cloud.

En tant que projet open source, les utilisateurs ont un accès facile aux mises à jour et peuvent même participer à ses améliorations. Il est prêt à l'emploi avec Selenium avec toutes ses dépendances pré-packagées.

Avec sa myriade d'outils d'intégration, il permet des tests sur différents navigateurs, et même dans Docker.

L'une des meilleures choses à propos de TestProject est qu'il permet aux non-codeurs de réaliser des tests grâce à son enregistreur de tests sans script.

## [Katalon Studio](https://www.katalon.com/homepage/?pk_abe=AB_testing_Homepage_08_2020&pk_abv=layout2)

![Image](https://lh4.googleusercontent.com/Etrs1mFEa-U-0wofZ-Vw-Cmn0G9dlZbA1DcMjs1M2hDXVK3qBe2VJ4X-o1KKH-6dUoxLHdm-LknJPbinYRpWBlBIQTtYM9Qf8nf0hNTYDQrbNYzud_0ZQg-4wLNoTiBWEg)

Une autre suite de test API, web et mobile open source est Katalon Studio. Il s'agit de l'une des suites de test les plus complètes de toute la liste, avec des fonctionnalités telles que l'enregistrement, la génération automatique de scripts de test et des intégrations puissantes.

Il s'agit d'un framework de test facile à démarrer (oui, même pour les non-codeurs) mais suffisamment puissant pour évoluer.

Katalon Studio fonctionne bien avec les configurations CI/CD existantes. Par exemple, il est facile de l'intégrer avec les outils de test continu de Github ou Gitlab. Cela en fait un excellent choix pour les équipes Agile.

## [OpenTest](https://getopentest.org/)

![Image](https://lh6.googleusercontent.com/ht1yDKuYIf2V3aQakjQqZSKeIPIfUrOr5b_sowtlr7f2l-h7Hr3_bHcX41fZoLDBNGJXWhaY8Gd7RUAiCAZMYGoGA7Ra63WwSWrO9iwxeu7mjisKyZ5dANIoq97eq_jIiw)

OpenTest est un autre outil open source qui automatise les tests pour les API, les applications web, mobiles et de bureau. Il est principalement utilisé pour les tests fonctionnels.

Il présente encore de nombreuses limitations car certains scénarios de test ne sont pas encore supportés. Cependant, il est très adapté aux débutants et aux non-codeurs, car il utilise des mots-clés en anglais simple pour écrire des actions de test.

Parmi ses fonctionnalités les plus notables, on trouve le test web avec Selenium, le test mobile avec Appium, les tests pilotés par mots-clés, les tests parallèles et les tests pilotés par les données.

## [AccelQ](https://www.accelq.com/)

![Image](https://lh3.googleusercontent.com/HLpAj_Qg-Prqnp6Zq5i9WtahnDrBQjclRAknTaCiS_20fc40jEUDWgnu1gt5M6mpt4sfYtQkMGwnazK95RRTU8LRyk9jajQ4OkeSKptTgaP8sq14cDttfsb7MqtHY30CMw)

AccelQ est une plateforme de test automatisé sans code basée sur l'intelligence artificielle. À ce titre, elle est principalement utilisée pour le développement Agile, car elle permet une construction rapide et une introduction facile des changements dans le projet.

La plateforme permet les tests d'API et de bout en bout. L'épine dorsale AI fournit des analyses prédictives qui aident à accélérer la génération de scripts de test. Elle dispose d'une interface utilisateur intuitive et facile à utiliser qui permet même aux débutants de commencer rapidement.

## Conclusion

Le choix se réduit, sans surprise, à votre cas d'utilisation particulier.

Cependant, étant donné les capacités étendues de chacun de ces outils, il est compréhensiblement difficile de n'en sélectionner qu'un seul. La belle chose à propos de beaucoup d'entre eux étant open source est que les équipes de développement peuvent en utiliser un pour un projet particulier, et un autre pour un projet différent.
