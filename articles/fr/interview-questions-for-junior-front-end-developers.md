Le but de cet article est de familiariser les nouveaux développeurs Front-End avec les questions théoriques. Si vous commencez tout juste dans le monde du développement, il est probable que vous vous sentiez nerveux lors d'un entretien pour un poste de développeur. À cause de cela, certaines questions relativement basiques peuvent sembler plus difficiles qu'elles ne le sont en réalité. Une bonne façon de contrer ce scénario est d'être bien préparé.

## **Questions**

Sans plus attendre, voici quelques questions que vous pourriez rencontrer :

### **HTML/CSS**

1. Dans le contexte du style CSS, pouvez-vous expliquer ce que signifie `display-style: inline-block` ?
2. Si vous avez un élément `<ul>` avec plusieurs éléments `<li>`, comment feriez-vous pour disposer ces éléments en ligne en utilisant HTML et CSS ?
3. Dans le style CSS, quand utiliseriez-vous les `float` ? Comment fonctionnent-ils ?
4. Quand utiliseriez-vous : `clear: both;` ?
5. Qu'est-ce que Bootstrap ? Quand l'utiliseriez-vous ? Comment fonctionne son système de grille ?
6. Expliquez la notion de design responsive et son fonctionnement. Utilisez un exemple si vous le souhaitez.
7. Qu'est-ce que le DOM ? Que savez-vous à son sujet ?
8. Qu'est-ce que SASS ? Quels sont les avantages de son utilisation ?

### **JavaScript**

1. Qu'est-ce qu'ECMAScript ?
2. Qu'est-ce que jQuery et à quoi sert-il couramment ?
3. Quelle est la différence entre `==` et `===` en JS ?
4. Pouvez-vous énumérer tous les types de données JavaScript ?
5. Qu'est-ce qu'une fonction de rappel (callback) en JS et quand en utiliseriez-vous une ?
6. Quelle est la différence entre `undefined` et `null` ?
7. Qu'est-ce qu'une réponse asynchrone ?
8. Que signifie JSON ? À quoi sert-il ?
9. Comment expliqueriez-vous une requête API ?
10. Avec quelles API avez-vous travaillé ?
11. Comment enverriez-vous/convertiriez-vous en format JSON ?
12. Quelle est la différence entre `GET` et `POST` lors de la spécification d'une requête AJAX ?

### **Logiciels/Outils**

1. Avez-vous travaillé avec Photoshop en tant que développeur front end ? Dans quel but ?
2. Quel système de gestion de versions avez-vous utilisé ? Pourriez-vous décrire à quoi ressemblerait un flux de travail dans ce système ?

### **À propos de vous**

1. Quel a été le projet le plus difficile sur lequel vous avez travaillé ? Pourquoi était-il difficile ? Si c'est une application/un algorithme, pouvez-vous expliquer comment il fonctionne ?
2. Comment restez-vous à jour avec les actualités technologiques ?
3. Sur quels projets parallèles avez-vous travaillé ?
4. Y a-t-il des langages ou des frameworks que vous aimeriez apprendre à l'avenir ? Comment vous y prendriez-vous pour vous les enseigner ? Pourquoi avez-vous choisi ces langages/frameworks ?
5. Quelles sont les compétences et qualités les plus importantes qu'un développeur front end junior devrait avoir ?
