Aujourd'hui, GitHub a annoncé que la prochaine version de leur API utilisera une nouvelle technologie développée par Facebook appelée GraphQL.

GraphQL pourrait éventuellement remplacer le type d'API le plus courant aujourd'hui, l'API REST.

Dans la documentation pour [la nouvelle version de leur API](https://fcc.im/2qOBK97), GitHub a expliqué :

> « GitHub passe à GraphQL pour la v4 de notre API car cela offre une flexibilité significativement plus grande pour nos intégrateurs. La capacité à définir précisément les données que vous voulez — et _uniquement_ les données que vous voulez — est un avantage puissant par rapport aux endpoints de l'API REST v3. »

> « GraphQL est une nouvelle façon de penser la construction et l'interrogation des API. Plutôt que de construire plusieurs requêtes REST pour récupérer les données qui vous intéressent, vous pouvez souvent faire _un seul appel_ pour obtenir les informations dont vous avez besoin. »
