Par David Kopal

Le but de cet article est de souligner l'importance des fondamentaux de JavaScript pour chaque développeur front-end. Je vais vous expliquer pourquoi vous devriez avoir une bonne connaissance du JavaScript pur ([Vanilla](https://en.wikipedia.org/wiki/Vanilla_software)). Cela signifie JavaScript sans aucun framework ou bibliothèque supplémentaire.

Je mentionnerai également quelques ressources qui m'ont aidé à apprendre ces bases.

Une autre raison derrière cet article est que de nombreux développeurs web en herbe tendent à sauter l'apprentissage des concepts fondamentaux de JavaScript tels que le hoisting, les closures ou les prototypes. Ils passent directement aux frameworks les plus populaires comme React ou Angular 2. Je vais vous montrer pourquoi cette approche n'est pas la plus pratique.

### Tout le monde veut des développeurs avec des connaissances en ${addTheHotFramework}...

Alors, y a-t-il une raison pour laquelle vous devriez vous donner la peine d'apprendre Vanilla JavaScript ?

![Image](https://cdn-media-1.freecodecamp.org/images/3asPo97aLp4ixpSc9jAE1hRjES1QXOhngwaf)

Il est difficile de devenir un développeur compétent sans connaître les bases du langage lui-même. Tout comme vous devez comprendre les principes fondamentaux du droit avant de pouvoir vous lancer dans un domaine particulier du droit. [Cette comparaison](https://ideas.ataccama.com/i-stopped-being-a-lawyer-became-a-developer-and-its-awesome-5311e8d74882) est purement accidentelle. ?

Je comprends que la majorité des développeurs web en herbe veulent être embauchés le plus rapidement possible. Je le voulais aussi.

Il semble beaucoup plus facile de suivre un cours intensif sur les bases de JavaScript, de se lancer dans un framework à la mode, de créer une liste de tâches ([laisser un chiot mourir](https://medium.freecodecamp.com/every-time-you-build-a-to-do-list-app-a-puppy-dies-505b54637a5d)), de la télécharger sur GitHub, puis de commencer à chercher un emploi.

### ...mais investir du temps dans Vanilla JavaScript portera ses fruits à long terme

Ne vous méprenez pas ici. Je ne suis pas biaisé contre les frameworks JavaScript de quelque manière que ce soit. C'est exactement le contraire. Beaucoup d'entre eux vous aident à écrire un code plus lisible et maintenable. Ils vous permettent d'écrire un code plus abstrait qui est souvent plus facile à déboguer.

Mais l'écosystème JavaScript évolue assez rapidement. De nouveaux frameworks sont créés. De nouvelles fonctionnalités sont ajoutées à ceux existants. Et surtout, beaucoup des frameworks les plus populaires d'aujourd'hui seront éventuellement remplacés, comme Angular 1.

Dans de telles circonstances, pensez-vous toujours que la connaissance d'un framework JavaScript particulier est suffisante pour un développeur web ?

Ou est-il préférable de comprendre comment fonctionne le langage sur lequel toutes ces bibliothèques et frameworks sont basés sous le capot ?

![Image](https://cdn-media-1.freecodecamp.org/images/vJM8nOjyJ4tX1Xcvq2-ADbKzbQw2oL2XQpBz)
_Oui, vous avez raison, c'est la deuxième option._

Si vous possédez une solide connaissance des fondamentaux de JavaScript, la seule chose dont vous avez besoin pour vous familiariser lorsque vous commencez à travailler avec un nouveau framework est sa **syntaxe**. Les règles de base seront les mêmes puisque sous toutes les couches d'abstractions, c'est toujours du bon vieux JavaScript.

Si votre connaissance est limitée à un seul framework JavaScript, vous aurez du mal à en apprendre un nouveau. Différents frameworks sont souvent basés sur un ensemble différent de principes JavaScript. À long terme, vous passerez plus de temps à essayer de comprendre divers frameworks et à déboguer le code que vous écrivez avec eux.

Tous les frameworks et bibliothèques JavaScript sont inévitablement basés sur son cœur Vanilla.

Cela devrait convaincre quiconque que la connaissance du JavaScript pur est un must à long terme. C'est un prérequis pour tout développeur réussi, surtout pour celui qui travaille principalement dans l'écosystème JavaScript.

![Image](https://cdn-media-1.freecodecamp.org/images/kZZHHwhgLCosFsZbR3mrRLHTGJe56FtGOeLl)

### Expérience personnelle

Il n'y a pas si longtemps, j'ai regardé en arrière mon [parcours](https://ideas.ataccama.com/i-stopped-being-a-lawyer-became-a-developer-and-its-awesome-5311e8d74882#.v3xurb9v5) d'avocat à développeur web. Cela fait déjà 18 mois que j'ai écrit ma première fonction JavaScript, et mon 10ème mois en tant que développeur front-end professionnel.

Je me souviens encore à quel point il était difficile pour moi de découvrir la bonne façon d'apprendre JavaScript, puisque je n'avais aucune expérience préalable en programmation. J'ai essayé (et j'essaie toujours) de nombreuses façons différentes de devenir un apprenant efficace. Certaines d'entre elles étaient plus gratifiantes, d'autres moins.

Surtout, je ne peux pas insister assez sur l'aide qu'il m'a été de mettre l'accent sur l'apprentissage du JavaScript pur dès le début. **Les frameworks sont venus plus tard.**

![Image](https://cdn-media-1.freecodecamp.org/images/j91ro5XgX9NRTDACXLnH-kXyQJWwJyJm8d7W)
_[crédit photo](https://www.keepcalm-o-matic.co.uk/p/keep-calm-and-learn-javascript/" rel="noopener" target="_blank" title=")_

Actuellement, je travaille avec une pile [React](https://facebook.github.io/react/)-[Redux](http://redux.js.org/). Même ainsi, je rencontre souvent un problème que je peux résoudre grâce à ma connaissance de Vanilla JavaScript. Ces bugs seraient beaucoup plus difficiles à résoudre avec seulement la connaissance d'un framework particulier.

Apprendre React ou Angular 2 ne vous apprendra pas que les objets sont passés par référence ou comment fonctionnent les closures. Essayer de comprendre ces concepts sous les abstractions apportées par le framework est beaucoup plus difficile. Cela rend les concepts simples de JavaScript plus difficiles à comprendre.

De plus, si vous travaillez avec [JSX](https://facebook.github.io/react/docs/jsx-in-depth.html) (React, Vue, Inferno) ou [TypeScript](https://www.typescriptlang.org/) (Angular 2), vous avez une autre couche d'abstraction par-dessus.

Si vous voulez comprendre comment ces frameworks fonctionnent sous le capot, vous devez d'abord savoir comment JavaScript lui-même fonctionne.

Vous pouvez **tester vos connaissances** en JavaScript pur en lisant le code source de votre framework préféré. Non seulement cela vous donnera une idée de son fonctionnement sous le capot, mais cela vous enseignera également beaucoup sur sa logique, que vous pourrez utiliser dans votre propre code. Vous verrez que les fonctions du framework qui semblent faire beaucoup de magie dans votre application ne sont que des compositions de concepts de base de JavaScript.

### Donnez-moi quelque chose que je peux apprendre

La question que vous pourriez vous poser maintenant est « Quelles sont les bonnes ressources qui peuvent m'aider à acquérir une connaissance décente de Vanilla JavaScript ? ».

Il existe des tonnes de cours et de livres sur JavaScript et ses frameworks. Pourtant, peu d'entre eux visent à vous enseigner Vanilla JavaScript de manière exhaustive. La plupart d'entre eux se spécialisent dans une technologie JavaScript particulière.

Mais il y en a encore quelques bons...

![Image](https://cdn-media-1.freecodecamp.org/images/Qtxt3WWxeUrTgdoPY3FTqQVoj-e8Pblt8mdS)

[Eloquent JavaScript](http://eloquentjavascript.net/) vous enseignera non seulement les bases de JavaScript, mais aussi les bases de la programmation en général. Si vous êtes déjà un développeur avancé, ce livre vous fournira une nouvelle perspective sur JavaScript et sur ses principes fondamentaux.

Une autre excellente ressource est la série de livres [You Don't Know JavaScript](https://github.com/getify/You-Dont-Know-JS) (YDKJS) de Kyle Simpson. Kyle sait vraiment comment enseigner. Ses explications des concepts avancés de JavaScript sont adaptées aux débutants et il les couvre en profondeur. Les titres mêmes des livres vous donneront un indice sur ce que vous allez apprendre. « Up & Going », « Scope & Closures », « this & Object Prototypes », « Types & Grammar », « Async & Performance », « ES6 & Beyond ». Il y a aussi un nouveau 7ème livre sur [la programmation fonctionnelle en JavaScript](https://github.com/getify/Functional-Light-JS).

L'une des grandes choses à propos de Eloquent JavaScript et YDKJS est que **vous pouvez les obtenir gratuitement** (vérifiez les liens fournis). Mais si vous les trouvez vraiment utiles, n'oubliez pas de soutenir les auteurs en les achetant.

Si vous êtes plus une personne de cours vidéo, vous pouvez regarder [les cours en ligne de Kyle](https://frontendmasters.com/kyle-simpson/). Je suppose que la meilleure chose à faire est de les regarder en complément de ses livres puisque les sujets sont les mêmes. Cependant, ces cours sont payants.

Un autre cours vidéo que je trouve utile est [Javascript: Understanding the Weird Parts](https://www.udemy.com/understand-javascript/) par Anthony Alicea. Il explique ce qui se passe sous le capot de JavaScript de manière accessible. Il couvre des concepts avancés tels que l'héritage prototypal, la programmation fonctionnelle et les chaînes de portée.

### Apprenez Vanilla JavaScript, maintenant

Si vous investissez votre temps précieux dans l'apprentissage de Vanilla JavaScript, vous ne le regretterez pas. Non seulement il contient le mot **vanilla**, mais il aura un impact bénéfique sur vos compétences en programmation.

Pour moi, le meilleur compromis est que l'apprentissage du JavaScript pur vous apportera beaucoup plus à long terme que d'investir du temps dans un framework particulier. Les frameworks ne sont que des raccourcis. Sous le capot, c'est toujours du JavaScript.

Vous verrez par vous-même dans une situation où un framework que vous utilisez a un comportement inattendu. Dans un tel cas, vous serez obligé d'enquêter sur ce bug en parcourant le code source. Ai-je mentionné que bien que de nombreux frameworks manquent de documentation décente, ils ont un code assez complexe ? Mais bon, un jeu d'enfant, non ? Vous avez sûrement déjà passé beaucoup de temps à apprendre Vanilla JavaScript. Ou pas encore ?

Si vous ne deviez retenir qu'une seule chose de cet article :

Gardez toujours à l'esprit que la connaissance de Vanilla JavaScript vous aidera à devenir un bien meilleur développeur. Point.
