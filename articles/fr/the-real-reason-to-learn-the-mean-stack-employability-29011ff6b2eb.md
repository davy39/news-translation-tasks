De plus en plus de bootcamps (comme Hack Reactor) et de plateformes de cours en ligne (comme Khan Academy) passent à JavaScript comme langue d'enseignement.

JavaScript a du sens comme première langue car vous devrez l'apprendre de toute façon. Vous ne pouvez pas faire grand-chose sur le web sans lui, donc les gens passent de plus en plus directement à JavaScript et sautent Ruby, Python et PHP. JavaScript se targue également de la communauté de développeurs la plus active. Mais il y a une troisième raison pour laquelle vous devriez apprendre JavaScript dès le départ : l'employabilité.

Les employeurs ne sont pas seulement en manque de talents JavaScript  ils sont en manque d'ingénieurs familiers avec des technologies spécifiques basées sur JavaScript. Et ces technologies sont précisément celles qui composent la pile MEAN  MongoDB, Express.js, Angular.js et Node.js.

Avant de commencer, notez que ces données proviennent d'Indeed.com, le plus grand agrégateur d'offres d'emploi. Nous utilisons le volume d'offres d'emploi comme proxy pour le volume d'emplois. Nous nous sommes concentrés uniquement sur les solutions non propriétaires comme Oracle, Microsoft et IBM.

### Comparaison des frameworks de développement web Model-View-Controller

* Node.js et Express.js  le serveur web et le framework de développement web minimaliste qui alimentent la pile MEAN. Node.js a été released il y a moins de 5 ans, mais il a déjà été adopté par LinkedIn, Walmart, SAP et Microsoft.
* Rails  le populaire framework de développement web basé sur Ruby conçu pour aider les développeurs à construire rapidement des applications conventionnelles lourdes en serveur. La majorité des bootcamps de codage se concentrent encore sur Rails. Fun fact : Rails a été [inventé par un pilote de course danois](http://en.wikipedia.org/wiki/David_Heinemeier_Hansson).
* Django  le framework de développement web le plus populaire de Python. Un autre fun fact : Django a été [inventé dans un journal au Kansas](http://en.wikipedia.org/wiki/Lawrence_Journal-World).
* Laravel  Actuellement le framework de développement web PHP le plus populaire, bien que la communauté PHP ne se soit pas vraiment rallié à un framework de manière aussi universelle que les communautés Ruby et Python.

### Comparaison des bases de données NoSQL

Le prochain composant de la pile que nous allons examiner est la base de données. Commençons par comparer les bases de données NoSQL (non relationnelles) les plus populaires.

* MongoDB  La base de données de type Document Store utilisée par la pile MEAN. Mongo est également populaire avec les applications Rails, et est la base de données par défaut pour le projet Meteor.js.
* Cassandra  Une base de données à colonnes larges créée chez Facebook et ensuite open sourced vers Apache.
* Neo4j  La base de données de type Graph la plus populaire.
* Couchbase  Une autre base de données de type Document Store, conçue pour mettre l'accent sur la vitesse.

![Image](https://cdn-media-1.freecodecamp.org/images/0*52WzJq_y4DR0sdQA.jpg)

La demande des employeurs pour MongoDB est celle qui croît le plus rapidement parmi les bases de données NoSQL. En termes absolus, MongoDB est également en tête:

Mais vous vous demandez probablement comment cela se compare aux bases de données SQL. Jetons un coup d'œil:

MongoDB croît beaucoup plus vite que les bases de données SQL les plus populaires. Mais si nous regardons le marché de l'emploi en termes absolus, il reste encore un long chemin à parcourir:

Il est clair que les bases de données NoSQL représentent encore une part relativement petite du marché total pour les talents en bases de données. Une interprétation courante du nom NoSQL est Not Only SQL, signifiant que les bases de données NoSQL doivent être utilisées comme un complément, plutôt qu'un substitut, des bases de données SQL traditionnelles. C'est une raison majeure pour laquelle nous introduisons les bases de données relationnelles et SQL avant d'introduire MongoDB.

### Frameworks JavaScript Front End

Plusieurs frameworks JavaScript Model-View-Controller ont émergé au cours des trois dernières années. Ceux-ci permettent aux développeurs de manipuler des éléments DOM côté client. Cela permet des applications à page unique, où le navigateur n'a jamais à naviguer vers une nouvelle page.

* Angular.js  Le A de la pile MEAN. Google (et par extension YouTube) ont investi beaucoup d'argent et de talents en ingénierie pour améliorer Angular.js.
* React.js  Développé par Facebook, avec un accent sur les performances.
* EmberJS  Fournit une structure MVC côté backend, tout en utilisant des outils conventionnels comme Handlebars.js et jQuery côté front end.
* Backbone.js  Le framework JavaScript MVC original, et toujours assez populaire. Il a une grande communauté et une bonne documentation, y compris quelques cours sur Code School.

![Image](https://cdn-media-1.freecodecamp.org/images/0*E9BXqsSO98Owh6Cl.jpg)

Comme vous pouvez le voir, Angular.js croît rapidement, tandis qu'Ember.js et Backbone.js semblent avoir atteint un plateau. React.js est si nouveau qu'il n'apparaît même pas sur le graphique, mais j'ai entendu parler de certaines entreprises, comme Khan Academy, qui l'utilisent. Et en termes absolus, Angular.js rattrape rapidement Backbone.js:

![Image](https://cdn-media-1.freecodecamp.org/images/0*XhvrctkyWC2JYkDm.jpg)

Mais, bien sûr, ce graphique omet le 800 livres gorille du monde JavaScript Front End, jQuery:

![Image](https://cdn-media-1.freecodecamp.org/images/0*H_Iwc3QWXCdXPSBT.jpg)

Pour le plaisir, comparons le pourcentage total d'offres d'emploi mentionnant jQuery au pourcentage d'offres d'emploi mentionnant JavaScript lui-même:

![Image](https://cdn-media-1.freecodecamp.org/images/0*pAaTjwFZnWJu89go.jpg)

Incroyablement, jQuery obtient presque autant de mentions que JavaScript. Il y a, à mon avis, 3 explications possibles à cela:

1. jQuery est si intuitif à utiliser que les web designers moins techniques continuent à l'utiliser au lieu de passer à des frameworks front-end plus puissants comme Angular.js. Ces web designers ont beaucoup d'applications héritées utilisant jQuery, et embauchent des gens pour les maintenir.
2. Les personnes non techniques des RH créant des offres d'emploi pensent que jQuery et JavaScript sont synonymes, ou que jQuery (sorti en 2006) est une forme plus tendance de JavaScript (sorti il y a presque 20 ans), donc ils listent jQuery à la place.
3. De nombreux employeurs ne nécessitent pas le travail lourd que fournissent les frameworks JavaScript front-end de pointe. Ou, plus probablement, ils échouent à comprendre à quel point ces frameworks sont puissants et comment ils peuvent produire des expériences supérieures.

Quelle que soit la cause, le message est clair. Pour maximiser votre employabilité, il est absolument essentiel que vous appreniez jQuery, même si ces autres frameworks peuvent faire toutes les choses que jQuery peut faire et plus encore. Beaucoup du succès de jQuery peut probablement être attribué à sa facilité d'utilisation. À bien des égards, apprendre jQuery est même plus facile qu'apprendre JavaScript, et beaucoup des concepts que vous appliquerez avec jQuery se transposent lorsque vous apprenez un framework MVC comme Angular. En résumé:

* Chacun des composants individuels de la pile MEAN émerge comme le gagnant de sa catégorie respective.
* Du point de vue de l'employabilité, il est logique de se concentrer sur la pile MEAN dans son ensemble. Si votre employeur veut utiliser, par exemple, Backbone.js au lieu d'Angular.js, vous serez déjà familier avec l'un et pourrez facilement apprendre l'autre.
* MongoDB est la bonne base de données NoSQL sur laquelle se concentrer, mais il sera payant de se familiariser avec les bases de données relationnelles et SQL lui-même.
* Apprenez jQuery. Pratiquez jQuery beaucoup. Mettez-le en évidence sur votre profil LinkedIn et votre CV.
