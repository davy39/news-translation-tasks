Par Markus Hatvan


Vous ne vous en rendez peut-être pas compte, mais vous utilisez une forme de logiciel open source chaque jour.

Chaque fois que vous lancez une application sur votre téléphone ou un programme sur votre ordinateur, vous profitez du code que quelqu'un a écrit gratuitement.

WordPress, le plus grand et le plus connu des systèmes de gestion de contenu, est utilisé par [38 % de tous les sites web](https://w3techs.com/technologies/details/cm-wordpress) dans le monde. Il est open source et gratuit.

Linux alimente [30 % de tous les sites web](https://w3techs.com/technologies/details/os-linux) dans le monde. Il est open source et gratuit.

Ce ne sont là que deux exemples parmi une myriade de projets créés pour résoudre un problème ou servir un cas d'utilisation.

Ces projets sont devenus très populaires car des communautés se sont formées autour d'eux. Ils ont tenté de livrer le meilleur produit possible en termes d'expérience utilisateur, de stabilité, de sécurité, et plus encore.

Mais ce n'est pas seulement l'utilisation des projets open source qui est en hausse. La participation au mouvement open source dans son ensemble est également en croissance.

Selon le rapport [State of the Octoverse](https://octoverse.github.com/) de GitHub, sur plus de 40 millions de développeurs sur GitHub, 10 millions de nouveaux utilisateurs ont rejoint en 2019 seulement !

Le mouvement open source se développe rapidement et vous devriez en faire partie aussi.

Chaque fois que je vois une nouvelle mise à jour pour mon système d'exploitation ou de nouvelles versions de logiciels que j'utilise activement tous les jours, cela me fait sourire. J'apprécie l'idée que les produits s'améliorent et deviennent plus sophistiqués en continu.

Ressentez-vous la même chose ?

Voici pourquoi je suis convaincu que vous devriez commencer à contribuer aux logiciels open source dès maintenant.

## Vous pouvez apprendre beaucoup du code source

Puisque le code source des projets open source est disponible pour que chacun puisse le lire, cela signifie qu'un grand nombre de développeurs peuvent tester et améliorer un projet.

Les développeurs peuvent signaler des problèmes de confidentialité ou de sécurité, mettre à jour la documentation et améliorer le code source selon les dernières normes de développement web en permanence.

Surtout lorsque vous parcourez le code de projets avec des centaines, voire des milliers de contributeurs, vous pouvez acquérir une immense connaissance des meilleures pratiques et de la qualité du code.

Non seulement la révision du code elle-même est une expérience d'apprentissage, mais aussi la structure et la hiérarchie des dossiers dans les grands projets sont bien pensées et fonctionnent bien à long terme.

## Vous travaillerez avec les personnes les plus intelligentes

Comparé à une entreprise qui a un nombre limité d'employés pour travailler sur les demandes de fonctionnalités et les corrections de bugs, vous avez les esprits les plus brillants travaillant dans le développement open source.

Dans mon imagination, je le vois comme une intelligence collective, capable de résoudre chaque problème qui survient.

Plus il y a de personnes qui rejoignent une communauté, mieux un projet peut évoluer. Cela peut être comme une ruche bourdonnante, où vous pourriez avoir des demandes de tirage (pull requests) vers une base de code de la part d'utilisateurs du monde entier 24h/24 et 7j/7, sans interruption.

Un bon exemple est l'éditeur de code bien connu [Visual Studio Code](https://github.com/microsoft/vscode) qui est devenu très populaire et compte un total de 1 200+ contributeurs sur GitHub.

Vous ne verrez pas un seul jour sans demande de tirage sur GitHub et les cycles de publication mensuels apportent toujours de nouvelles fonctionnalités incroyables.

Lorsque vous participez à un projet et soumettez une demande de tirage, vous recevrez des commentaires extrêmement utiles de la part de mainteneurs très expérimentés. Vous pourrez ensuite implémenter ces commentaires pour grandir en tant que développeur.

## Votre propre code pourrait être utilisé mondialement

Puisque certains projets de développement de logiciels sont utilisés par des millions d'utilisateurs quotidiennement, il peut être très gratifiant de voir votre propre code aider autant de personnes.

J'ai écrit des règles de lint pour des projets JavaScript appelées [eslint-plugin-unicorn](https://github.com/sindresorhus/eslint-plugin-unicorn) et [svelte](https://github.com/sveltejs/svelte). C'est une grande satisfaction de savoir que ma demande de tirage améliorera la qualité du code de nombreux développeurs dans le monde entier.

D'après mon expérience personnelle, il est également motivant de recevoir des commentaires positifs sous forme de remerciements.

## Les projets open source sont inclusifs

Un grand avantage des logiciels open source gratuits est que personne n'est exclu de l'utilisation du produit parce qu'il ne peut pas se le permettre.

Bien que certains projets open source coûtent de l'argent pour être utilisés, la plupart ne le font pas.

De plus, lorsque vous contribuez à un projet sur GitHub, beaucoup des grands dépôts ont un code de conduite. Ceux-ci garantissent que chaque contributeur se sente bienvenu et accepté dans un projet.

## Les projets commencent à devenir durables

Le principal objectif d'une entreprise est de devenir rentable, ce qui conduit souvent à des décisions discutables. Mais les logiciels open source se concentrent sur la résolution des besoins de leurs utilisateurs comme priorité absolue.

La plupart des projets sont entièrement soutenus par des bénévoles, et les mainteneurs de projets ne verront malheureusement jamais de récompense financière. Mais il existe aujourd'hui de grandes façons d'aider à rendre ces projets durables.

Avec des sites comme [OpenCollective](https://opencollective.com/) ou [GitHub Sponsors](https://github.com/sponsors), vous pouvez faire un don pour accélérer le développement des projets que vous aimez.

Personnellement, je pense qu'il serait formidable que chaque entreprise fasse un don, même modeste, aux projets de logiciels open source, car elles profitent de ces outils quotidiennement. Un tel soutien réduirait le stress pour de nombreux mainteneurs et certains pourraient même en faire leur travail à temps plein.

## Comment contribuer à l'open source

Contribuer au développement open source semble plus effrayant que cela ne l'est en réalité. Il existe de nombreux projets sur GitHub qui encouragent les nouveaux contributeurs et les débutants à agir en étiquetant les problèmes comme "Good first issue", "Beginner friendly" ou "Help wanted".

Vous ne savez pas par où commencer ?

Demandez-vous : quelle est une application que vous aimez utiliser tous les jours et à laquelle vous aimeriez contribuer ?

Cela peut être aussi simple que de rechercher cette application sur GitHub et de parcourir les problèmes ouverts.

Ce n'est pas obligatoirement une contribution de code, vous pouvez aussi aider en créant une demande de tirage pour mettre à jour la documentation, corriger les fautes de frappe que vous trouvez, ou en faisant une révision de code approfondie.

Le fichier `README.md` d'un projet inclut généralement une section sur la façon de contribuer.

Si vous décidez de contribuer à un projet, je vous recommande de lire mon article sur [Contribuer aux projets open source de la bonne manière](https://markushatvan.com/blog/contributing-to-open-source-projects-the-right-way). C'est un guide détaillé étape par étape sur le processus de contribution.

Je l'ai écrit pour qu'il soit très accessible aux débutants, alors ne vous inquiétez pas d'être submergé. Vous serez en mesure de trouver votre premier projet et de soumettre une contribution en un rien de temps !

## Conclusion

Il m'a toujours impressionné que tout le monde dans le monde puisse rejoindre un projet de logiciel open source et y travailler.

Et les logiciels open source ne fonctionnent que par un effort collaboratif. Le but est de produire le meilleur produit ou service sans compromettre des facteurs importants comme la stabilité, la sécurité ou la confidentialité des utilisateurs.

J'espère que vous comprenez l'importance des logiciels open source et que vous appréciez leurs avantages. Peu importe vos raisons de redonner à la communauté open source, sachez que vous êtes grandement apprécié !

De nombreux projets ne peuvent prospérer qu'avec le soutien et les contributions de développeurs comme vous.
