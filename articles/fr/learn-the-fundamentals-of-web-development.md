Les nouveaux développeurs web essaient souvent d'apprendre tout à la fois sans maîtriser une seule compétence. Mais avant d'apprendre le dernier framework JavaScript, assurez-vous de connaître les fondamentaux du HTML et du CSS.

Colby Fayock est un développeur front-end expérimenté et un contributeur prolifique de freeCodeCamp. Il enseigne à d'autres à coder depuis de nombreuses années.

Colby a récemment donné une conférence à distance sur la manière d'améliorer vos compétences en développement web lors de la conférence Byteconf React 2020. Et nous avons collaboré avec Byteconf pour publier la conférence de Colby.

Dans sa conférence, Colby souligne que les développeurs doivent comprendre les fondamentaux – ou cette ignorance les empêchera toujours de réaliser le potentiel de leurs projets.

Par exemple, sans quelques connaissances de base en HTML, vous pourriez involontairement exclure des personnes de l'utilisation de votre site web en raison d'une mauvaise accessibilité.

Et si vous manquez d'une compréhension complète du CSS, vous pourriez opter pour l'utilisation de bibliothèques inutiles qui peuvent alourdir votre site web et ralentir les vitesses de chargement de vos pages.

![Image](https://www.freecodecamp.org/news/content/images/2020/10/external-content.duckduckgo.com.gif)

Dans sa conférence thématique Mario Bros., Colby couvre de nombreux fondamentaux du HTML et du CSS que tous les développeurs web doivent connaître.

Il discute également de certaines stratégies de base en HTML et CSS que vous pouvez immédiatement utiliser pour une meilleure accessibilité, un code plus simple et un meilleur référencement.

Même si vous pensez maîtriser les fondamentaux, il ne peut pas faire de mal de faire un rappel et de vérifier les lacunes dans vos connaissances.

Vous pouvez regarder [la conférence complète sur la chaîne YouTube freeCodeCamp Talks](https://youtu.be/3xq96I9IUXw) (22 minutes de visionnage). Et pendant que vous y êtes, cliquez sur le bouton 'S'abonner' pour ne manquer aucune des grandes conférences que nous publions là-bas.
