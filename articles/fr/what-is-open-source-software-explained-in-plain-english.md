Au cours de votre parcours en programmation, vous avez peut-être entendu le terme "Open Source". Mais que signifie-t-il réellement ?

Dans cet article, je vais expliquer ce qu'est un logiciel Open Source et parler de ses différences avec les autres types de logiciels. Je discuterai également des avantages des logiciels Open Source et de la manière de commencer à contribuer à des projets. 

## Qu'est-ce que l'Open Source ?

Un logiciel Open Source est un code qui est publiquement disponible pour que les gens puissent le consulter, le modifier et le partager. Un exemple de logiciel open source serait [freeCodeCamp](https://www.freecodecamp.org/).

Si je voulais consulter le code du programme de freeCodeCamp, je pourrais le voir sur [la page GitHub de freeCodeCamp](https://github.com/freeCodeCamp/freeCodeCamp).  

![Image](https://www.freecodecamp.org/news/content/images/2021/08/Screen-Shot-2021-08-17-at-12.21.53-AM.png)

Si je trouve un bug dans l'un des défis, je peux lire les [directives pour les contributeurs](https://contribute.freecodecamp.org/#/index) et proposer une correction. Le logiciel Open Source me permet de contribuer à la base de code de freeCodeCamp. 

Il existe [des millions de projets open source](https://github.com/open-source) parmi lesquels choisir et contribuer. 

## D'où vient le terme "Open Source" ?

Il y a plus de vingt ans, le terme "open source software" a été inventé par la cofondatrice de la Foresight Institute, Christine Peterson. Elle voulait inventer ce terme comme moyen d'attirer les nouveaux venus dans cet espace. 

Le nom original devait être "free software", mais il y avait beaucoup de confusion sur ce que "free" signifiait réellement. Afin de clarifier le problème, Peterson a proposé un changement de nom pour "open source software" à la place. 

## En quoi l'Open Source est-il différent des autres types de logiciels ?

Maintenant que nous comprenons la définition de base du logiciel open source, examinons un autre type de logiciel. 

### Qu'est-ce qu'un logiciel propriétaire ?

Contrairement aux logiciels open source, seuls les auteurs originaux des logiciels propriétaires peuvent modifier leur code. Les utilisateurs de ce type de logiciel doivent accepter de ne pas le modifier de quelque manière que ce soit, sauf s'ils ont la permission des propriétaires de le faire. 

Des exemples de logiciels propriétaires incluent iTunes, Microsoft Windows et Adobe Photoshop.  

## Que sont les licences Open Source ? 

 Le code open source est publié sous l'une des licences suivantes. Cela vous indique ce que vous pouvez faire avec le code et quelles permissions, le cas échéant, vous devez utiliser. 

Voici une liste des licences open source les plus populaires. 

* [Licence Apache 2.0](https://opensource.org/licenses/Apache-2.0)
* [Licence BSD 3-Clause "Nouvelle" ou "Révisée"](https://opensource.org/licenses/BSD-3-Clause)
* [Licence BSD 2-Clause "Simplifiée" ou "FreeBSD"](https://opensource.org/licenses/BSD-2-Clause)
* [Licence Publique Générale GNU (GPL)](https://opensource.org/licenses/gpl-license)
* [Licence Publique Générale GNU "Lesser" (LGPL)](https://opensource.org/licenses/lgpl-license)
* [Licence MIT](https://opensource.org/licenses/MIT)
* [Licence Publique Mozilla 2.0](https://opensource.org/licenses/MPL-2.0)
* [Licence de Développement et de Distribution Commune](https://opensource.org/licenses/CDDL-1.0)
* [Licence Publique Eclipse version 2.0](https://opensource.org/licenses/EPL-2.0)

Examinons l'une de ces licences plus en détail. 

### Licence MIT

Cette licence est apparue dans les années 1980 et comporte très peu de restrictions. Avec cette licence, vous pouvez faire presque tout ce que vous voulez avec le code, tant que vous conservez le copyright et la notice de licence. 

[choosealicense.com](https://choosealicense.com/) fournit cette description pour la licence MIT :

> Elle permet aux gens de faire presque tout ce qu'ils veulent avec votre projet, comme créer et distribuer des versions closed source.

Certains projets utilisant cette licence sont Ruby on Rails, Node.js, React et Angular. 

Si vous êtes intéressé à en apprendre davantage sur l'une des autres licences populaires mentionnées, veuillez cliquer sur l'un des liens de cette liste. 

## Quels sont les avantages des logiciels Open Source ? 

Il y a de nombreux avantages à utiliser et à travailler avec des logiciels open source. L'un des avantages est le contrôle que vous avez sur le code que vous décidez d'utiliser. 

Les programmeurs peuvent examiner le code et décider quelles parties du code ils veulent modifier. Les utilisateurs réguliers peuvent utiliser ce logiciel comme ils le souhaitent sans que quelqu'un leur dise de l'utiliser d'une manière spécifique. 

Un autre avantage est tout ce que vous pouvez apprendre en utilisant le code d'un projet open source. Les développeurs de tous horizons peuvent étudier le code des projets open source ainsi que partager leur travail avec les autres. 

Les logiciels open source sont également considérés comme plus sécurisés que les logiciels propriétaires. Puisque le code est librement disponible, les développeurs peuvent trouver des problèmes, faire des mises à jour et corriger ces bugs plus rapidement que dans les logiciels propriétaires.   

L'open source dispose également d'une très grande communauté derrière lui. En conséquence, il existe de nombreuses ressources utiles pour les débutants souhaitant rejoindre une communauté open source et commencer à contribuer. 

## Comment commencer à contribuer aux projets Open Source

Il existe de nombreuses ressources qui enseignent les bases de la contribution aux projets open source. En voici quelques-unes pour vous aider à commencer :

* [Comment contribuer aux projets Open Source – Un guide pour débutants](https://www.freecodecamp.org/news/how-to-contribute-to-open-source-projects-beginners-guide/)
* [Le guide ultime de l'Open Source](https://www.freecodecamp.org/news/the-ultimate-guide-to-open-source/)

Vous pouvez rechercher sur GitHub des étiquettes marquées "help wanted", "first timers only" ou "good first issue". Une fois que vous avez trouvé un projet correspondant à votre stack technique, assurez-vous de lire les directives de contribution du projet.  

Et si vous avez besoin d'aide pour commencer avec Git ou GitHub, ces guides peuvent vous aider :

* [Qu'est-ce que GitHub ? Qu'est-ce que Git ? Et comment utiliser ces outils de développeur](https://www.freecodecamp.org/news/what-is-github-what-is-git-and-how-to-use-these-developer-tools/)
* [Cours accéléré sur Git et GitHub](https://www.freecodecamp.org/news/git-and-github-crash-course/)

Contribuer à des projets open source est un excellent moyen de développer davantage vos compétences et de travailler avec d'autres développeurs. C'est aussi une autre façon de gagner de l'expérience si vous cherchez à entrer dans l'industrie technologique. 

Cela peut sembler intimidant au début, mais vous pouvez commencer par quelque chose de petit comme corriger une faute de frappe dans la documentation. 

Voici une liste de sites qui peuvent vous aider à trouver des projets auxquels contribuer :

* [good first issue](https://goodfirstissue.dev/)
* [CodeTriage](https://www.codetriage.com/)

## Conclusion

Un logiciel Open Source est un code qui est publiquement disponible pour que les gens puissent le consulter, le modifier et le partager. Le terme "open source software" a été inventé par la cofondatrice de la Foresight Institute, Christine Peterson.

Avec les logiciels propriétaires, seuls les auteurs originaux du logiciel peuvent modifier leur code. Les utilisateurs de ce type de logiciel doivent accepter de ne pas le modifier de quelque manière que ce soit, sauf s'ils ont la permission des propriétaires de le faire. 

Les logiciels open source présentent de nombreux avantages, notamment un large soutien communautaire, des guides utiles pour commencer à contribuer, l'apprentissage par l'étude du code des autres et l'acquisition d'expérience en travaillant avec d'autres développeurs. 

Si vous êtes intéressé à commencer à contribuer à un projet, recherchez sur GitHub des étiquettes marquées "help wanted", "first timers only" ou "good first issue" et trouvez un projet qui vous intéresse.
