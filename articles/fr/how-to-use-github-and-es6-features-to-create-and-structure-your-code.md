Par M. S. Farzan

Si vous passez des tutoriels de codage à la création de vos propres projets JavaScript, vous voudrez vous familiariser avec l'utilisation d'un système de contrôle de version comme [GitHub](https://github.com/) et profiter des [fonctionnalités ES6](https://www.w3schools.com/js/js_es6.asp) pour nettoyer votre code et rationaliser votre processus de développement.

Dans cette nouvelle vidéo, nous allons parcourir le clonage d'un dépôt existant depuis GitHub, la création de vos propres dépôts locaux et distants, l'utilisation du gestionnaire de paquets node (NPM) pour installer des dépendances, la création d'une base de code rationalisée en utilisant les classes ES6, les modules import/export et les fonctions fléchées, et l'ajout, la validation et le poussage des modifications vers GitHub.

De plus, nous allons parler de la structure de fichiers de base pour un projet [Phaser 3](https://phaser.io/) !
