Dans [l'article d'Ariel sur freeCodeCamp](https://medium.freecodecamp.org/e1045057468f), il expose trois règles clés que vous devriez suivre si vous voulez devenir un maître de Git.

Les règles sont :

* Règle #1 : Créez un dépôt Git pour chaque nouveau projet
* Règle #2 : Créez une nouvelle branche pour chaque nouvelle fonctionnalité
* Règle #3 : Utilisez les Pull Requests pour fusionner le code dans Master

Ariel explique pourquoi vous devriez suivre ces règles et comment elles sont devenues des meilleures pratiques au fil du temps.

Lisez l'article complet d'Ariel [sur la publication Medium de freeCodeCamp](https://medium.freecodecamp.org/e1045057468f) (7 minutes de lecture).
