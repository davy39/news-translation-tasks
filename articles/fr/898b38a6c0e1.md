# Pourquoi les images deviennent floues lors du redimensionnement

Nous n'avions aucun problème de netteté des images dans les premières années d'Internet, car nous n'avions pas besoin d'afficher des sites web sur des appareils mobiles.

Aujourd'hui, nous observons une croissance énorme de la navigation web sur mobile. Tellement énorme que, cette année, elle a même dépassé le trafic sur les ordinateurs de bureau.

La plupart des clients essaient de créer des interfaces réactives pour leurs sites web afin qu'ils soient facilement accessibles depuis n'importe quel type d'appareil. Et il ne semble pas étrange que de nombreux développeurs de mise en page aient commencé à utiliser

```
img {    width: 100%;    height: auto;}
```

pour leurs images. Ainsi, elles fonctionnent de manière transparente sur n'importe quelle résolution de fenêtre de navigateur, en s'adaptant à l'espace disponible.

Conceptuellement, tout semble bien. Les designers créent de belles grandes images, et les développeurs utilisent ces images pour chaque appareil. Dans certains cas, les développeurs créent plusieurs versions de la même image, afin qu'une image plus petite soit téléchargée pour les écrans mobiles, et une image plus grande pour les ordinateurs de bureau.

Malheureusement, Chrome ne veut pas redimensionner les images de la manière dont le font les bons éditeurs d'images. Au lieu de cela, il produit une image qui n'a pas la même netteté, ou une netteté similaire, que l'image initiale.

La raison principale de cela peut être la performance. Lorsqu'une page web contient de nombreuses images, et que le processeur de l'appareil n'est pas très performant, un traitement supplémentaire des images pourrait entraîner des retards plus importants dans le rendu de la page, donc Chrome omet ce processus car il n'est pas crucial pour les utilisateurs finaux.

Ce que les développeurs doivent faire maintenant est d'activer la propriété webkit auto-sharping :

```
img {    image-rendering: -webkit-optimize-contrast;}
```

Voici ce que nous obtiendrions si elle était activée :

![Image](https://cdn-media-1.freecodecamp.org/images/vCERaeq5p6YpYfYvMi2kv2YKAns1Wl-7gGBn)

![Image](https://cdn-media-1.freecodecamp.org/images/51RdQBlurbGD0Qy8VgDVK2eClSFIgxyxmNJX)

Ainsi, avant que le navigateur le plus populaire au monde n'implémente un bon algorithme de sous-échantillonnage d'images, nous pouvons utiliser la propriété webkit-optimize-contrast, qui permet aux visiteurs de nos sites de profiter de nos photos, de discerner les détails des articles dans notre boutique en ligne, et d'apprécier la lisibilité de nos captures d'écran et diagrammes. Mais soyez prudent, dans certains cas, cela peut entraîner des résultats pixelisés.
