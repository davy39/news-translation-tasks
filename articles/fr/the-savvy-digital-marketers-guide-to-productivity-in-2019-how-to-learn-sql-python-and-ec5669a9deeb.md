Par Trevor Fox

**La réponse est « Oui. »** Vous en êtes arrivé là, et vous êtes dans une bonne position. Vous êtes prêt à faire le premier pas audacieux vers l'acquisition d'une nouvelle compétence précieuse et, mieux encore, vers l'indépendance en informatique. Peu importe ce que vous choisissez, tant que vous avancez, vous vous remercierez en un rien de temps.

J'ai commencé à apprendre à coder il y a six ans après une carrière professionnelle ratée dans le baseball, et je maintiens que c'était l'une des meilleures décisions que j'aie jamais prises. À l'époque, je travaillais dans une agence web en tant que généraliste du marketing digital. Je passais mon temps à bidouiller des feuilles de calcul jusqu'à ce qu'elles ressemblent à l'automatisation que je recherchais, ou je me retrouvais dans des impasses lorsque je me retrouvais à fixer de petites zones de texte qui me permettaient de personnaliser un peu de code dans un CMS.

![Image](https://cdn-media-1.freecodecamp.org/images/Ahy7c-JOIZrlmRJyxjhGdaXFy8YSNcpCquze align="left")

J'ai été surpris et reconnaissant de la rapidité avec laquelle j'ai pu appliquer ce que j'avais appris. C'est ce que je veux vous aider à faire aujourd'hui.

[J'ai commencé avec Javascript](https://trevorfox.com/learn-digital-marketing-technical-skills/), je suis passé à Python, et enfin à SQL. J'ai suivi ce chemin en raison du cadre que j'expose ci-dessous. Pour la plupart, cependant, il n'y a pas de mauvaise réponse. Apprendre *comment* le code fonctionne — en tant qu'ensemble d'instructions qui disent à un ordinateur comment vous voulez qu'il fonctionne — est assez universel. Apprendre un langage de script vous aidera à apprendre d'autres langages, et il en va de même pour les langages de requête comme SQL. Alors ne vous inquiétez pas, les coûts de changement sont faibles et le seul temps perdu est le temps d'attente pour commencer.

Pour tirer le meilleur parti de votre parcours d'apprentissage, la valeur réside entièrement dans l'applicabilité. Donc la première question est, **quel problème voulez-vous résoudre avec le code ?** La nécessité est un puissant moteur et vous maintiendra concentré. Commencer avec un objectif en vue est la bonne façon de procéder.

### **Application : Quels problèmes rencontrez-vous ?**

Chacun de ces langages de programmation a été initialement conçu pour résoudre un problème particulier. Pour Javascript, il s'agissait de [faire fonctionner des applications dans un navigateur](http://2ality.com/2011/03/javascript-how-it-all-began.html) ; pour Python, il s'agissait en fait de [l'administration des systèmes](http://effbot.org/pyfaq/why-was-python-created-in-the-first-place.htm), et pour SQL, il s'agissait de [requêter des ensembles de données relationnelles](https://docs.oracle.com/cd/B13789_01/server.101/b10759/intro001.htm).

Aujourd'hui, Javascript et Python ont évolué bien au-delà de leurs origines pour englober tout, des applications web aux matériels embarqués. SQL reste bien... un langage de programmation très spécialisé (mais très utile).

Bien que les applications se soient élargies, chaque langage de programmation a une valeur plus immédiate dans certains domaines que dans d'autres. Et vous serez plus motivé à apprendre si vous savez que vous avez un bon problème à résoudre.

Je me place ici du point de vue d'un marketeur digital ou d'un analyste digital qui dépend des données pour réussir dans son travail — car c'est là que ces langages de programmation seront immédiatement utiles. Alors, considérez votre accès aux données :

1. Avez-vous accès à une base de données relationnelle ?
   
2. Devez-vous accéder, interpréter, manipuler et déplacer des données ?
   
3. Devez-vous suivre les analyses de site web ?

### Un Cas Pour SQL

Si vous avez répondu « Oui » à la question 1 et « Non » aux autres, alors la réponse est simple : apprenez SQL. Tout le monde ne travaille pas dans un environnement où il peut utiliser un entrepôt de données pour l'analyse (même si c'est [plus facile que jamais de configurer un entrepôt de données](https://panoply.io/)), donc si vous vous retrouvez à exporter répétitivement des listes à partir du marketing, du commerce électronique ou des applications en demandant à l'IT, à un DBA ou à des ingénieurs d'extraire des données d'une base de données ou d'un entrepôt de données pour votre analyse, dépêchez-vous et apprenez SQL ! Vous me remercierez dans quelques semaines.

Je recommande [le cours de SQL de Duke](https://www.coursera.org/learn/analytics-mysql) sur Coursera pour passer de zéro à une compétence SQL utilisable en moins de deux mois. Avec ce cours et quelques livres sur PostgreSQL, j'ai appris assez pour enregistrer [cette démonstration avec des données de réseau publicitaire.](https://www.youtube.com/watch?v=FvVzxUffrlw)

Il existe même de bonnes applications Android/iPhone qui vous aideront à apprendre la syntaxe par la répétition. Le cours que je recommande ci-dessous ([Manipulation de données à grande échelle : Systèmes et algorithmes](https://www.coursera.org/learn/data-manipulation)) pour Python aborde également SQL, donc c'est un double coup. Et [Python Anywhere](https://www.pythonanywhere.com/) propose également MySQL hébergé, donc c'est un double-double coup !

La décision entre Python et Javascript est un peu plus délicate. Ce sont deux des langages de programmation les plus populaires au monde, et tous deux disposent de tonnes de ressources pour approfondir autant que vous le souhaitez — principalement gratuitement !

### Un Cas Pour Python

Si vous avez répondu « Oui » à la question 2, je recommanderais Python. Il est simple à apprendre et offre une excellente base pour apprendre N'IMPORTE QUEL langage de programmation. Vous pouvez l'utiliser pour accéder et interagir avec des API, ce qui peut considérablement augmenter l'échelle de vos activités marketing. Par exemple, Facebook dispose d'une API pour les publicités Facebook qui vous permet de créer et de gérer des campagnes de manière programmatique. C'est comme une baguette magique pour le marketing !

**MAIS** vous constaterez que vous devez apprendre plus que les bases pour faire quoi que ce soit d'intéressant. Python s'exécute sur votre ordinateur ou un serveur, vous devez donc apprendre *comment* *exécuter* le code en plus de *comment écrire* le code.

Si vous êtes prêt à vous y mettre et à emprunter ce chemin, je recommande de commencer par [Introduction à la programmation Python](https://www.udacity.com/course/introduction-to-python--ud1110) d'Udacity. C'est parfait pour les marketeurs digitaux car il introduit le concept et vous apprend à construire un crawler web *très* simple (pensez à Googlebot). Après cela, [Manipulation de données à grande échelle : Systèmes et algorithmes](https://www.coursera.org/learn/data-manipulation) (Semaine 1) de Coursera vous montre comment tweeter en accédant à l'API Twitter avec Python. Amusant (et effrayant) !

Lorsque vous voulez essayer des choses pratiques, consultez [Automate the Boring Stuff](https://automatetheboringstuff.com) pour des projets intéressants que vous pouvez faire sur votre ordinateur, ou [Python Anywhere](https://www.pythonanywhere.com/) ou [AWS Lambda](https://aws.amazon.com/lambda/) pour commencer à exécuter Python dans le cloud !

Le moyen le plus rapide d'appliquer Python au marketing digital est d'utiliser [JuPyteR](http://jupyter-notebook-beginner-guide.readthedocs.io/en/latest/what_is_jupyter.html) pour l'analyse de données, mais cela a encore une courbe d'apprentissage, alors attention. [Python pour l'analyse de données](https://www.amazon.com/Python-Data-Analysis-Wrangling-IPython/dp/1449319793) est un excellent moyen de commencer dans cette direction.

### Un Cas Pour Javascript

Si vous avez répondu « Oui » à la question 3 et que vous êtes un as de la logique, alors vous l'avez deviné, apprenez Javascript. Bien que Javascript puisse être utilisé pour créer de nombreuses applications web intéressantes (j'ai enfin créé une application pratique avec D3 pour la [recherche de mots-clés](https://keywordclarity.io/)), il peut également être utile en quelques semaines si vous souhaitez configurer ou améliorer Google Analytics sur un site web.

La plupart des CMS rendent assez facile l'édition d'au moins un peu de code Javascript, et Google Tag Manager permet d'insérer du code Javascript pour le suivi. Javascript est également la façon dont les outils de test A/B comme Optimizely et Google Optimize modifient la disposition d'une page web. Si vous gérez un compte Google Ads, vous pouvez même utiliser Javascript pour mettre à jour et maintenir vos annonces de manière programmatique. Saviez-vous que [Google Apps Script](http://googleappscripting.com/) utilise Javascript ?!

Le fait qu'il puisse être plus facile de commencer à voir les avantages de l'apprentissage du code avec Javascript ne signifie pas que c'est un langage plus facile à apprendre en général. Entre les callbacks, la version de Javascript de la programmation orientée objet, et les milliards de bibliothèques, versions et frameworks, il peut y avoir quelques passages difficiles, mais persévérer vous aidera à comprendre comment fonctionnent les navigateurs et le web. Cela changera complètement votre monde !

J'ai appris, *vraiment appris*, les fondamentaux de Javascript sur [freeCodeCamp](https://www.freecodecamp.org/) et j'ai pu l'appliquer avant même de terminer la première certification. Je n'ai pas besoin de vous dire comment naviguer dans votre intérêt pour Javascript ; leur programme fait un excellent travail à cet égard. J'ajouterais seulement que consulter [les vidéos Javascript the Good Parts](https://vimeo.com/8691412) accélérera votre compréhension du monde Javascript.
