Par Thu Nghiem

Il existe d'excellentes feuilles de route pour les développeurs web. Mais certaines vous obligent à résoudre une "énigme" avant même de commencer, car il y a plusieurs choix que vous devez faire.

Lorsque j'ai commencé à apprendre le développement web, j'aurais souhaité trouver un développeur expérimenté qui me dirait ce qu'il avait fait pour devenir développeur web. Malheureusement, je n'ai trouvé personne.

Maintenant que je travaille dans l'industrie depuis 4 ans, je veux partager avec vous comment je commencerais à apprendre le développement web en 2021. Dans cet article, je vais vous montrer chaque étape que vous pouvez suivre, depuis le niveau débutant jusqu'à devenir développeur.

Je commencerai par quelques **Outils Essentiels** que vous devriez connaître, puis je couvrirai les **Langages de Programmation** et les **Bibliothèques/Frameworks** que vous devez apprendre pour devenir un développeur Front-end ou Full-stack.

Je parlerai également de quelques **projets exemples** que vous pouvez construire pour pratiquer vos compétences.

Vous pouvez également trouver [une feuille de route mise à jour](https://learn.devchallenges.io/tutorial/web-developer-roadmap-2021) sur mon site web.

## Outils que vous devez connaître pour devenir développeur web

![Image](https://www.freecodecamp.org/news/content/images/2020/12/Screenshot-2020-12-07-at-15.02.06.png)

Je vais supposer que vous êtes un débutant. Dans ce cas, il y a quelques outils que vous devez connaître avant de vous lancer dans la programmation.

### VS Code (ou autre éditeur de code)

Tout d'abord, vous devriez apprendre à utiliser [**Visual Studio Code**](https://code.visualstudio.com/), qui est un éditeur de code source. C'est un outil gratuit et puissant.

Au début, je recommande d'apprendre quelques-uns des raccourcis de base et d'installer certaines extensions comme ESlint, Prettier ou Live Server.

Voici un [cours complet et gratuit](https://www.freecodecamp.org/news/learn-visual-studio-code-to-increase-productivity/) sur la chaîne YouTube de freeCodeCamp pour vous lancer.

### La ligne de commande

Ensuite, vous devez connaître la **[Ligne de Commande](https://developer.mozilla.org/en-US/docs/Learn/Tools_and_testing/Understanding_client-side_tools/Command_line)**. Vous devriez apprendre ce que c'est, et quelques commandes de base comme comment naviguer dans les répertoires, comment créer un nouveau répertoire ou comment créer un nouveau fichier.

Voici un excellent article sur [comment utiliser Bash](https://www.freecodecamp.org/news/linux-command-line-bash-tutorial/), la ligne de commande Linux. Et en voici un autre qui vous aidera à [utiliser la ligne de commande plus efficacement](https://www.freecodecamp.org/news/basic-linux-commands-bash-tips-you-should-know/).

### Contrôle de version - Git et GitHub

Peu importe ce que vous faites, en tant que développeur, vous devez connaître **[Git](https://git-scm.com/)**. Git est un système de contrôle de version utilisé pour suivre les modifications. Il est généralement utilisé avec **GitHub**, qui est une plateforme d'hébergement de code.

Au début, apprendre Git peut être écrasant, donc vous devez simplement connaître quelques bases comme comment créer un nouveau _Dépôt_, comment _cloner_ un projet, comment faire un nouveau _commit_, et comment _pull_ et _push_ les nouvelles modifications.

L'une des meilleures façons de pratiquer Git est de travailler en équipe. Là, vous devez savoir comment créer une nouvelle _branche_, comment faire des _pull requests_, et comment résoudre les _conflits_.

Voici un excellent [cours accéléré sur Git et GitHub](https://www.freecodecamp.org/news/git-and-github-crash-course/) sur la chaîne YouTube de freeCodeCamp pour vous lancer avec le contrôle de version.

### Outils de design - Figma

Le dernier outil est **[Figma](https://www.figma.com/)**. Figma est un outil de design et est gratuit pour les particuliers. Mais ici, je veux parler de l'utilisation de Figma en tant que développeur.

En tant que développeur, vous pourriez recevoir des designs de la part des designers de votre équipe. Avec Figma, vous pouvez inspecter le code des éléments et mesurer la mise en page. Par conséquent, vous devez savoir comment lire un design Figma, comme obtenir la bonne couleur, typographie ou espacement.

Voici un article sur [différents outils de design, y compris Figma](https://www.freecodecamp.org/news/adobe-xd-vs-sketch-vs-figma-vs-invision/). Et voici un tutoriel amusant sur [la création de croquis 3D dans Figma](https://www.freecodecamp.org/news/creating-realistic-3d-objects-in-figma-carton-box-example-f674c21c3452/).

D'accord, ce sont donc les outils que vous devez connaître lorsque vous commencez. **Vous n'avez pas besoin de tout savoir**, mais assurez-vous de comprendre les bases afin de pouvoir vous améliorer tout en apprenant à coder.

## Langages de programmation que les développeurs web devraient apprendre

![Image](https://www.freecodecamp.org/news/content/images/2020/12/Screenshot-2020-12-07-at-15.02.27.png)

### HTML et CSS

Ensuite, continuons avec les langages de programmation. Et commençons par parler des **Sites Web Responsive**.

Les sites web responsive sont des sites qui ont une belle apparence et sont utilisables sur tous les appareils ou tailles d'écran. Vous savez probablement à quel point il est important de construire un site web responsive, car les gens utilisent tant d'appareils différents de nos jours.

Alors, passons aux deux premiers langages que vous devez connaître pour construire un site web : **HTML** et **CSS**.

**HTML** signifie _Hypertext Markup Language_. Il est utilisé pour construire le squelette de votre site web. Le HTML n'est pas difficile à apprendre, mais vous pourriez vouloir porter plus d'attention aux formulaires HTML car ils seront fondamentaux à l'avenir.

**CSS** signifie _Cascading Style Sheets_. C'est un langage de balisage, mais je le considère également comme un langage de programmation. Le CSS n'est pas nécessairement difficile à apprendre, mais il est difficile à maîtriser.

Il y a quelques sujets sur lesquels vous voulez porter plus d'attention comme :

* **Modèle de boîte** - comment les marges, le remplissage et les bordures fonctionnent ensemble.
* **Unités CSS** - utilisées pour exprimer les longueurs (par exemple : rem, vh et vw).
* **Position** - spécifie le type de méthode de positionnement. Cela confond également beaucoup de gens, alors assurez-vous de passer du temps dessus.
* **Variables** - ou **Propriétés personnalisées** sont des entités qui peuvent être réutilisées dans un document. C'est ma fonctionnalité préférée en CSS. Elles rendent le travail avec CSS si agréable et vous pouvez créer des thèmes avec seulement quelques lignes de code.
* **Requête média** - décide ce qu'il faut montrer sur différentes tailles d'écran. Elles sont un composant clé du [design responsive](https://developer.mozilla.org/en-US/docs/Web/Progressive_web_apps).
* **Animation** - permet à un élément de changer d'un style à un autre. Si vous savez comment utiliser l'animation correctement, cela rend votre site unique. Sinon, cela rendra votre site web peu professionnel, alors soyez prudent.
* **Flexbox, CSS Grid** - utilisés pour construire des mises en page responsive (j'ai oublié de mentionner cela dans la vidéo).

Voici [un manuel complet](https://www.freecodecamp.org/news/the-html-handbook/) qui vous apprendra toutes les bases du HTML. Et voici un [cours complet sur le CSS](https://www.freecodecamp.org/news/learn-css-in-this-free-6-hour-video-course/) sur la chaîne YouTube de freeCodeCamp qui vous aidera à commencer à embellir vos sites.

Lorsque vous connaissez les bases du HTML et du CSS, l'étape suivante est de construire quelques sites web de base. Par exemple, vous pouvez essayer une _Page d'accueil_, un _Formulaire_ comme une page de connexion ou une page de paiement. Vous pouvez même construire un _Portfolio_. Vous pouvez trouver des projets exemples sur [DevChallenges.io](https://devchallenges.io/paths/responsiveWebPaths).

### Déploiement de site web

Une fois que vous avez votre site web, vous devez le mettre en ligne pour que les gens puissent le voir. Le déploiement est le processus de déploiement de votre code sur une plateforme d'hébergement.

Auparavant, c'était beaucoup plus difficile à faire. Mais maintenant, c'est super facile et vous pouvez utiliser des outils comme [GitHub Pages](https://pages.github.com/), [Netlify](https://www.netlify.com/), ou [Vercel](https://vercel.com/).

Voici un [cours YouTube complet](https://www.freecodecamp.org/news/how-to-put-a-website-online-guide-to-website-creation-custom-domain-and-hosting/) sur la façon de mettre votre site en ligne qui couvre tout le processus du début à la fin.

### JavaScript - Fondamentaux

D'accord, le [prochain sujet est **JavaScript**](https://www.freecodecamp.org/learn/javascript-algorithms-and-data-structures/basic-javascript/). JavaScript est un langage de programmation populaire et est largement utilisé pour le développement web, entre autres.

Vous devrez apprendre certaines des fonctionnalités de base du langage comme les **[Types de Données](https://www.freecodecamp.org/news/javascript-data-types-typeof-explained/)**, les **[Boucles](https://www.freecodecamp.org/news/javascript-loops-explained-for-loop-for/)**, et les [Conditionnelles](https://www.freecodecamp.org/news/what-in-the-world-is-a-javascript-conditional-for/).

Ensuite, il y a des sujets sur lesquels vous voudrez vous pencher plus profondément.

Tout d'abord, nous avons le **débogage**. C'est le processus de recherche et de correction des erreurs. Voici un [guide approfondi pour écraser les bugs](https://www.freecodecamp.org/news/the-beginner-bug-squashing-guide/) pour vous lancer.

Ensuite, il y a d'autres sujets comme les **Objets**, les **Primitives**, et les **Tableaux**. Surtout lorsque vous travaillez avec des tableaux, vous devez connaître les **[Méthodes de Tableaux](https://www.freecodecamp.org/news/javascript-map-reduce-and-filter-explained-with-examples/)** également.

Les **Fonctions** sont les principaux blocs de construction de votre programme. Assurez-vous donc de ne pas les négliger.

L'une de mes fonctionnalités préférées en JavaScript est la [déstructuration](https://www.freecodecamp.org/news/array-and-object-destructuring-in-javascript/) - elle est facile à écrire et rend le langage super puissant.

Comme en C#, Java, ou d'autres langages de programmation, en JavaScript moderne, nous avons également des **classes**. Celles-ci sont utiles lorsqu'il s'agit de [Programmation Orientée Objet et des principes SOLID](https://www.freecodecamp.org/news/solid-principles-explained-in-plain-english/).

Et peu importe à quel point vous êtes bon en programmation, vous aurez des erreurs dans vos scripts. Cela signifie que vous voudrez [en savoir plus sur la **gestion des erreurs** également](https://www.freecodecamp.org/news/how-to-write-error-messages-that-dont-suck-f31c53b64c3e/).

La programmation asynchrone est importante, surtout lorsque vous devez communiquer avec le serveur. Passez donc du temps à apprendre les [**Promesses** et **Async/Await**](https://www.freecodecamp.org/news/learn-promise-async-await-in-20-minutes/).

### JavaScript - Navigateur

Passons à la façon dont JavaScript est utilisé dans le navigateur.

Tout d'abord, vous devez savoir ce qu'est un _Document Object Model_ ou **DOM**. Ensuite, vous devez savoir comment obtenir des éléments, comment changer les classes, ou comment changer le style avec JavaScript.

Voici une bonne [introduction au DOM JS](https://www.freecodecamp.org/news/an-introduction-to-the-javascript-dom-512463dd62ec/), et voici un guide sur [comment manipuler le DOM](https://www.freecodecamp.org/news/learn-how-to-manipulate-the-dom-by-building-a-simple-javascript-color-game-1a3aec1d109a/) (vous apprendrez en construisant un projet).

Vous devez également apprendre les différents **[Événements d'Interface Utilisateur](https://www.freecodecamp.org/news/javascript-dom-events-onclick-and-onload/)** comme le clic, le survol de la souris, le clic de la souris, etc.

Et vous voudrez également porter plus d'attention aux **[Formulaires](https://www.freecodecamp.org/news/build-and-validate-beautiful-forms-with-vanilla-html-css-js/)** en JavaScript, car ils ont de nombreux événements et propriétés.

### JavaScript - Autres fonctionnalités

L'**[API Fetch](https://www.freecodecamp.org/news/javascript-fetch-api-tutorial-with-js-fetch-post-and-header-examples/)** est un sujet important. Elle vous permet d'envoyer des requêtes réseau aux serveurs. Cela est utile, par exemple, lorsque nous devons soumettre un formulaire ou obtenir les informations d'un utilisateur.

Un autre sujet important est le **[Stockage de Données](https://www.freecodecamp.org/news/how-to-store-data-in-web-browser-storage-localstorage-and-session-storage-explained/)** dans le navigateur. Ici, vous devez savoir quelles sont les différences entre les Cookies, LocalStorage et sessionStorage.

D'autres sujets moins importants lorsque vous commencez sont les expressions régulières, les composants web et les websockets.

Les **[expressions régulières](https://www.freecodecamp.org/news/regular-expressions-demystified-regex-isnt-as-hard-as-it-looks-617b55cf787/)** sont utilisées pour rechercher et remplacer du texte. Les **composants web** sont une nouvelle chose, mais vous devriez absolument les vérifier.

Enfin, nous avons les **[Websockets](https://www.freecodecamp.org/news/beginners-guide-to-websockets/)**. Ils sont utiles lorsque vous devez avoir des échanges de données continus comme dans les applications de chat.

Donc, après avoir appris JavaScript, vous pourriez vouloir passer du temps à [apprendre **TypeScript**](https://www.freecodecamp.org/news/learn-typescript-data-types-from-zero-to-hero/). J'adore TypeScript car il me donne une sensation de sécurité lors de l'écriture de code.

TypeScript fournit une typographie statique, qui vous permet de détecter les erreurs plus tôt dans le processus de débogage. Il vous fait également gagner du temps car il trouve les bugs avant que le code ne s'exécute.

Voici un article utile sur les [types TypeScript](https://www.freecodecamp.org/news/a-mental-model-to-think-in-typescript-2/) pour vous aider à y penser de la bonne manière.

Vous pouvez vous arrêter ici et commencer à travailler sur des projets. Mais personnellement, je continuerais en apprenant au moins un framework. Ensuite, vous pouvez pratiquer JavaScript en même temps.

D'accord, alors passons à la suite.

## Bibliothèques et Frameworks à connaître

![Image](https://www.freecodecamp.org/news/content/images/2020/12/Screenshot-2020-12-07-at-15.02.35.png)

Pour devenir un développeur Front-end ou Full-stack, vous devez connaître certaines des bibliothèques et frameworks qui existent.

### Bibliothèques et Frameworks pour les développeurs Front-end

Alors, commençons par **Sass**, qui est un langage de script préprocesseur. J'utilise Sass dans presque tous mes projets. Il rend le CSS plus propre et plus rapide à développer.

Voici un [cours complet sur Sass](https://www.freecodecamp.org/news/give-your-css-superpowers-by-learning-sass/) qui vous apprendra à donner des superpouvoirs à votre CSS.

Ensuite, nous avons **NPM**, qui est un gestionnaire de paquets pour la programmation JavaScript. Cela vous permet d'installer rapidement différents paquets sur votre machine.

Voici un bon [guide pour débutants sur NPM](https://www.freecodecamp.org/news/what-is-npm-a-node-package-manager-tutorial-for-beginners/) qui vous aidera à commencer à télécharger des paquets immédiatement.

L'un de mes regrets est de ne pas avoir connu **[Contentful](https://www.freecodecamp.org/news/learn-react-contentful-and-netlify-by-building-a-beach-resort-website/)** plus tôt. Contentful est un système de gestion de contenu sans tête ou CMS. Il est différent d'un CMS traditionnel, car vous pouvez stocker des données dans Contentful et les utiliser pour votre front-end.

Et pour obtenir un emploi en front-end, vous devez connaître au moins un framework front-end. Personnellement, je choisirais **React**. Nous parlerons des autres frameworks plus tard.

En plus d'[apprendre les bases de React](https://www.freecodecamp.org/news/the-react-handbook-b71c27b0a795/), vous voulez également porter une attention particulière à la façon dont l'**État** est géré dans les applications React et à la façon dont les **Formulaires** fonctionnent dans React.

Une fois que vous avez appris quelques bases, vous pouvez solidifier vos compétences en construisant une série de projets avec React [dans ce cours YouTube](https://www.freecodecamp.org/news/solidify-your-react-skills-by-building-15-projects/).

Un autre regret que j'ai est de ne pas avoir connu **[Next.js](https://nextjs.org/)** plus tôt. Next.js est utilisé pour le _rendu côté serveur_ ou la _génération de sites web statiques_. Et oui, Next.js est encore assez nouveau, mais je crois vraiment que c'est la compétence que nous _devons_ avoir en tant que développeurs React.

Vous pouvez apprendre toutes les bases de Next.js dans [ce manuel approfondi](https://www.freecodecamp.org/news/the-next-js-handbook/).

Après avoir appris ces outils, vous pourriez également vouloir jeter un coup d'œil à **[Material UI](https://material-ui.com/)** qui est une bibliothèque de composants React, ou **[Tailwind CSS](https://tailwindcss.com/)** qui est un framework CSS qui vous aide à accélérer votre travail lors de la prototypage ou du freelance, par exemple.

Donc, après avoir appris JavaScript et un framework, vous devez **pratiquer** en construisant des projets.

Vous pouvez commencer par construire quelques composants réutilisables simples pour comprendre comment React fonctionne. Ensuite, vous pouvez construire des applications plus complexes comme une _Application de Quiz_ ou une _Application de Todo_.

Après cela, vous voudrez construire des applications encore plus difficiles comme un _Outil de Recherche d'Emploi_, un _Blog_, ou une _Page de Documentation_. Encore une fois, vous pouvez trouver des projets exemples sur [DevChallenges.io](https://devchallenges.io/paths/frontEndPaths).

### Développeur Full-stack

Maintenant, vous êtes prêt à postuler pour un emploi de développeur Front-end. Si vous voulez continuer pour devenir un développeur Full-stack, vous pouvez commencer par [apprendre **Node.js**](https://www.freecodecamp.org/news/nodejs-course/) et **[Express](https://www.freecodecamp.org/news/learn-express-js-in-this-complete-course/)_.

Ici, vous devez savoir comment construire une _API RESTful_ et vous pouvez utiliser **[MongoDB](https://www.freecodecamp.org/news/mongodb-crud-app/)** lorsque vous travaillez avec des bases de données. Ce qui est assez simple à apprendre lorsque vous connaissez déjà JavaScript.

Après cela, si vous voulez apprendre plus, vous pouvez [vous pencher sur **GraphQL**](https://www.freecodecamp.org/news/learn-how-to-use-react-and-graphql-to-make-a-full-stack-social-network/) qui est un langage de requête et de manipulation de données pour les API. Personnellement, je passerais également du temps à apprendre **PostgreSQL**. Comparé à MongoDB, c'est un peu plus difficile à apprendre car vous devez [apprendre SQL](https://www.freecodecamp.org/news/sql-recipes/) également.

Après avoir appris ces outils, vous pouvez pratiquer en construisant des applications comme un _Uploader d'Images_, une _Authentification_, ou un _Chat Room_. Vous pouvez également trouver ces projets sur [Devchallenges.io](https://devchallenges.io/paths/fullStackPaths).

Maintenant, vous êtes prêt à postuler pour un poste de Full-stack. Si votre travail nécessite que vous connaissiez **[Vue.js](https://www.freecodecamp.org/news/vue-js-full-course/)**, **[Angular](https://www.freecodecamp.org/news/angular-tutorial-course/)**, ou même **[Svelte](https://www.freecodecamp.org/news/the-svelte-handbook/)**, vous pouvez passer du temps à apprendre ces outils également. Cela ne devrait pas être trop difficile une fois que vous connaissez déjà React.
