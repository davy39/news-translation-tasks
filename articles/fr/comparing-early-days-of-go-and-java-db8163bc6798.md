Par Harshavardhana

Au cours des 2 dernières années, Golang (ou simplement « Go ») s'est élevé au rang des langages de programmation grand public comme Java, Python et C/C++. Des entreprises comme [Google, Docker, DigitalOcean et plus](https://github.com/golang/go/wiki/GoUsers) ont adopté Go pour construire leurs outils d'infrastructure cloud modernes.

Depuis sa première version, le **21 janvier 1996**, Java a dépassé toutes les attentes. Des applications mobiles aux logiciels financiers critiques, Java est aujourd'hui omniprésent. En comparant le rythme de croissance de Go avec celui de Java, certains enthousiastes spéculent sur le fait que Go pourrait devenir le prochain Java ou du moins atteindre un statut équivalent.

Alors que Java existe depuis près de deux décennies, Go est relativement jeune. La première version stable n'est apparue qu'il y a 3 ans. Il peut être prématuré de comparer Go à Java à ce stade, mais il existe des données préliminaires pour comparer les débuts et les tendances actuelles des deux langages.

### Les débuts de Java et Go

#### Java

**Histoire du langage**

> James Gosling, Mike Sheridan et Patrick Naughton ont lancé le projet de langage Java en juin 1991. Java était à l'origine conçu pour la télévision interactive, mais il était trop avancé pour l'industrie de la télévision par câble numérique de l'époque. Le langage s'appelait initialement Oak, d'après un chêne qui se trouvait à l'extérieur du bureau de Gosling. Plus tard, le projet a été nommé Green et a finalement été rebaptisé Java, d'après le café Java.

En 1991, Java était définitivement en avance sur son temps. Au cours des 4 premières années, Java s'est réincarné plusieurs fois avant d'atteindre sa forme finale. Java a été fortement influencé par Objective-C, Smalltalk et Modula. L'objectif de Java était d'être un langage orienté objet moderne, portable, haute performance, robuste et sécurisé. Il a trouvé ses fondamentaux dans la juste proportion pour être sophistiqué et simple en même temps. L'approche du bytecode de Java et l'implémentation de la machine virtuelle étaient tout simplement brillantes.

#### Go

**Histoire du langage**

> Robert Griesemer, Rob Pike et Ken Thompson ont commencé à concevoir Go chez Google à la fin de 2007, pour répondre aux problèmes de développement logiciel auxquels Google était confronté.

Go a également mis les mêmes quatre ans à mûrir, passant du concept à la version 1, similaire à Java et à la plupart des autres langages. Go a été fortement influencé par Modula, Pascal, Oberon et Newsqueak. Il est né de la frustration envers les langages de programmation système existants. Go a été conçu pour être un langage amusant et en même temps productif, pratique, expressif et puissant. Les programmeurs pouvaient sentir comment Google avait influencé les choix de conception et les outils de Go, mais qui ne voudrait pas hériter des meilleures pratiques de Google. Les points forts de Go sont les goroutines efficaces, les canaux et la compilation rapide.

#### **Projets open source populaires en Java**

Java est populaire dans les mondes open source et propriétaire. La liste des projets et des entreprises utilisant Java est sans fin. C'est l'un des langages modernes les plus éprouvés et matures de notre époque. La plus grande force de Java est la disponibilité de développeurs logiciels expérimentés et de haute qualité dans le monde entier. Voici un aperçu des projets open source notables en Java.

* Junit
* Eclipse
* Spring
* Solr
* Hudson et Jenkins
* Hadoop
* Android
* ElasticSearch
* Apache Commons

#### **Projets open source populaires en Go**

Puisque Go est encore dans les premiers jours de son adoption, sa liste n'est en rien comparable à celle de Java. Cependant, l'adoption que Go a déjà reçue à ce stade par des organisations comme Google, Dropbox, MongoDB, la BBC et SoundCloud est remarquable. La tendance actuelle dans l'industrie est la cloudification de la pile logicielle d'entreprise. Go se développe rapidement comme le langage de choix pour les projets d'infrastructure et d'outils cloud. Voici une liste de projets open source notables en Go.

* Docker
* Kubernetes
* Deis
* CoreOS
* InfluxDB
* Packer, Consul — par Hashicorp

Java a dépassé les attentes et est déjà en déclin. Il diminue progressivement depuis 2005. Peu de données ont été collectées publiquement avant 2005, mais cela suffit pour faire une comparaison avec Go.

#### **Tendance de croissance de Go**

Go connaît une croissance phénoménale depuis 2011. Alors que Java est en déclin, Go connaît une croissance rapide. Cependant, si vous superposez les graphiques, vous remarquerez que Go a encore un long chemin à parcourir.

#### Maturité

Sans conteste, Java est l'un des langages de programmation les plus matures aujourd'hui. Toute version supérieure à la version 3 est considérée comme « prête pour l'heure de grande écoute » dans le monde de l'entreprise. Java est déjà en route vers la version 9. Il est pris en charge par tous les principaux systèmes d'exploitation et architectures matérielles.

Versions majeures de Java, avec leurs dates de sortie :

* JDK 1.0 (21 janvier 1996)
* JDK 1.1 (19 février 1997)
* J2SE 1.2 (8 décembre 1998)
* J2SE 1.3 (8 mai 2000)
* J2SE 1.4 (6 février 2002)
* J2SE 5.0 (30 septembre 2004)
* Java SE 6 (11 décembre 2006)
* Java SE 7 (28 juillet 2011)
* Java SE 8 (18 mars 2014)

Comparé à Java, Go est encore dans sa phase d'enfance. La dernière version stable de Go « v1.6 » est sortie il y a seulement quelques semaines. Go a une approche communautaire collaborative dynamique et des cycles de sortie fréquents, environ tous les 6 à 8 mois. Il est facile de prévoir qu'avec cette approche, Go atteindra la maturité plus rapidement que Java.

Versions majeures de Golang, avec leurs dates de sortie :

* go1 (sorti le 28/03/2012)
* go1.1 (sorti le 13/05/2013)
* go1.2 (sorti le 01/12/2013)
* go1.3 (sorti le 18/06/2014)
* go1.4 (sorti le 10/12/2014)
* go1.5 (sorti le 19/08/2015)
* go1.6 (sorti le 17/02/2016)

### Conclusion

Il est trop tôt pour dire si « Go » remplacera un jour Java ou atteindra même son niveau d'omniprésence, mais il est certain que Go laissera une empreinte durable dans la catégorie des langages de programmation à usage général. Il est susceptible de remplacer davantage d'environnements Python, Ruby, NodeJS, Perl, C, C++ avant de faire une quelconque entaille notable dans le paysage Java.
