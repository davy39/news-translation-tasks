L'une des formes de codage les plus populaires de la dernière décennie a été la création d'applications, ou apps, qui fonctionnent sur des appareils mobiles.

Aujourd'hui, il existe deux grandes catégories d'appareils mobiles : ceux qui fonctionnent sous iOS et ceux qui fonctionnent sous Android.

## Systèmes d'exploitation mobiles

### iOS (iPhone, iPad, Apple TV)

iOS est un système d'exploitation mobile développé et distribué par Apple Inc. Il a été initialement lancé en 2007 avec l'iPhone, et fonctionne aujourd'hui sur de nombreux autres appareils, y compris l'iPod Touch, l'iPad et l'Apple TV.

iOS partage des racines similaires avec le système d'exploitation macOS utilisé dans les ordinateurs Apple modernes.

### Android

Android est un système d'exploitation mobile open source basé sur Linux. Android a été développé par l'Open Handset Alliance, qui était dirigée par Google et comportait des contributions de nombreuses autres entreprises.

Bien qu'initialement développé pour les téléphones, Android alimente aujourd'hui tout, des tablettes et montres intelligentes aux appareils intelligents comme les réfrigérateurs.

## Comment développer des applications mobiles

De manière similaire à la façon dont les appareils mobiles peuvent être divisés en deux catégories générales, il existe deux façons de penser au développement moderne d'applications mobiles : le développement natif ou le développement multiplateforme.

### Développement natif

Cette méthode de développement d'une application mobile décrit sa construction pour un système d'exploitation mobile spécifique en utilisant des outils spécialisés.

Pour développer des applications pour iOS, vous devez coder en Objective-C ou Swift dans XCode, un IDE (environnement de développement intégré) exclusif à macOS.

Les applications pour les appareils Android sont développées en Java en utilisant le SDK Android (kit de développement logiciel), qui fonctionne sur Windows, macOS et Linux.

Il existe un certain nombre de raisons pour développer des applications mobiles natives pour chaque système d'exploitation mobile, notamment la vitesse, la sécurité et la fiabilité. Le gros inconvénient est que, si vous souhaitez développer une application pour iOS et Android, vous devez pratiquement construire toute l'application deux fois.

### Développement multiplateforme

Ce type de développement utilise un framework pour construire une application une fois, puis crée des versions pour iOS et Android. Voici une liste de certains des frameworks les plus populaires :

**Ionic**

Ionic est un framework open source utilisé pour développer des applications mobiles. Il fournit différents outils et services pour construire une interface mobile avec une apparence et une sensation natives. Le framework Ionic nécessite un wrapper natif pour pouvoir fonctionner sur les appareils mobiles.

**Apache Cordova**

Anciennement connu sous le nom de PhoneGap, Apache Cordova est un framework de développement mobile initialement créé par Nitobe. Il est maintenant propriété d'Adobe Systems. Avec ce framework, le développeur n'a pas besoin de connaître Java ou Swift, juste le HTML, CSS et JavaScript de base.

#### **React Native**

React Native est un framework de développement d'applications mobiles multiplateformes par Facebook. Comme React, React Native permet aux développeurs de construire des applications mobiles en utilisant JSX, une variante de JavaScript. Bien qu'il ait une courbe d'apprentissage plus raide que d'autres frameworks comme Apache Cordova, React Native permet aux développeurs d'utiliser plus de fonctionnalités natives de l'appareil comme la caméra sans plugins supplémentaires.
