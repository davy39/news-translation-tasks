Par Dennis Bruijn / @0x1ad2

#### une liste de plugins Atom dont je ne peux pas me passer

J'ai récemment changé de [Sublime Text 3](https://www.sublimetext.com/) à [Atom Editor](https://atom.io/), et je dois dire que cela a accéléré mon processus de développement, surtout la partie de production de code front-end impeccable.

### Qu'est-ce qu'Atom Editor ?

**Comme indiqué sur leur page d'accueil :** Atom est un éditeur de texte moderne, accessible, mais hackable jusqu'au noyau — un outil que vous pouvez personnaliser pour faire n'importe quoi, mais aussi utiliser de manière productive sans jamais toucher un fichier de configuration.

![Image](https://cdn-media-1.freecodecamp.org/images/FzUdknZtJpNByYAOBSJMFsXabQNgzjqDzupn)

### Qu'est-ce qui m'a fait passer à Atom Editor ?

Tout d'abord, il est soutenu par [Github](https://github.com/). Github est une entreprise prospère qui se concentre sur les besoins des développeurs en matière de contrôle de version et de développement de produits.

> _Github dispose de nombreuses ressources pour investir dans le processus de développement d'une application. Il a la capacité de créer un produit stable rapidement et de l'améliorer au fil du temps._

Deuxièmement, il est hackable jusqu'au noyau, ce qui nous permet (développeurs) de créer de nouveaux packages pour tout, de la thématique et de la synchronisation à la manière dont nous écrivons des caractères à l'écran.

Il est également inspectable par l'inspecteur Chrome car il est fait avec [Electron](http://electron.atom.io/), un framework multiplateforme pour créer des applications de bureau avec JavaScript, HTML et CSS, également par Github.

![Image](https://cdn-media-1.freecodecamp.org/images/7ppw8NPegDtdcVBN5kClp3eB-bedxtfUnloU)

### La liste

#### [Minimap](https://atom.io/packages/minimap)

Minimap vous fournit un aperçu du code source complet ; c'est pratique pour avoir une vue d'ensemble du fichier que vous éditez.

![Image](https://cdn-media-1.freecodecamp.org/images/gjvgwSdZhRPIAm8q0-tm3hTALuoRq2FJ38hW)

#### [Minimap cursor line](https://atom.io/packages/minimap-cursorline)

Il est bon de savoir où vous vous trouvez dans votre fichier, cursor line vous permet de voir où vous êtes dans votre fichier. Il se déplace avec vous si vous déplacez des lignes.

![Image](https://cdn-media-1.freecodecamp.org/images/dIdEsuPcibmLZV0fgx-eRpaZm7wxzikCPE9E)

#### [Pigments](https://atom.io/packages/pigments)

Pigments analysera les fichiers sources dans vos répertoires de projet à la recherche de couleurs et construira une palette avec toutes celles-ci. Il met également en surbrillance les couleurs dans le fichier, par exemple une feuille de style pour faciliter l'utilisation des couleurs.

![Image](https://cdn-media-1.freecodecamp.org/images/R3cmaQzL0mOrX4gXPOvYvkMopR1Z7cRSIa38)

#### [Emmet](https://atom.io/packages/emmet)

Emmet vous permet de générer facilement du HTML basé sur la syntaxe Emmet. Par exemple, vous souhaitez créer une liste non ordonnée avec trois éléments de liste qui contiennent chacun un paragraphe.

![Image](https://cdn-media-1.freecodecamp.org/images/Yw9A4SxHxwZlQTBSpQ2Xgx9dfIbMJYIbdeTR)

#### [Double tag](https://atom.io/packages/double-tag)

Double tag vous permet de modifier à la fois les balises HTML de début et de fin en même temps. Cela est extrêmement utile lors de l'édition de gros fichiers avec beaucoup de HTML.

![Image](https://cdn-media-1.freecodecamp.org/images/ExSof-hdyFP9uK38zN6C83uPXhmkEqqFd-E0)

#### [Linter](https://atom.io/packages/linter)

Linter vous permet de vérifier votre code, les linters généreront des avertissements et des erreurs basés sur l'analyse de code en temps réel. Vous pouvez installer différents Linters, presque chaque langage en a un : HTML, CSS, SCSS, JavaScript, TypeScript, et même certains frameworks comme Twitter Bootstrap.

![Image](https://cdn-media-1.freecodecamp.org/images/wWnJOOkyVYPCmkbZ1GFg59-1GhEv15xpEGV5)

#### [Beautify](https://atom.io/packages/atom-beautify)

Beautify vous permet de formater votre HTML, CSS, JavaScript, PHP, Python, Ruby, Java, C, C++, C#, Objective-C, CoffeeScript, TypeScript, Coldfusion, et SQL avec un raccourci **Ctrl-Alt-B** ou **Cmd-Alt-B**

![Image](https://cdn-media-1.freecodecamp.org/images/KsFhh4yIkJTxbllMO0yJ3mb2ZeFw-wvcg8iG)

#### [Auto close HTML](https://atom.io/packages/autoclose-html)

Auto close vous donne la possibilité de fermer automatiquement vos balises HTML. Cela sera également utile lors de l'édition de gros fichiers avec beaucoup de HTML.

**_Note:_** _Vous n'avez pas besoin à la fois d'Emmet et d'Auto close HTML, cette fonctionnalité est déjà incluse dans Emmet._

![Image](https://cdn-media-1.freecodecamp.org/images/dc-z58BnSJIUtUJ06Tmx4TSVWOe9ecutduq2)

#### [Fuzzy finder](https://github.com/atom/fuzzy-finder)

Fuzzy finder vous permet de trouver et d'ouvrir des fichiers rapidement avec un raccourci **Ctrl-T** ou **Cmd-T**

![Image](https://cdn-media-1.freecodecamp.org/images/hCQfXdplAJhxGRtHaez-2fo52J35v2-0R1zU)
_Fuzzy finder est inclus par défaut, vous n'avez pas besoin de l'installer_

#### [Color picker](https://atom.io/packages/color-picker)

Color picker vous permet de choisir des couleurs dans vos fichiers SCSS, LESS et CSS. Il lit actuellement les formats de couleur suivants : HEX, HEXa, RGB, RGBa, HSL, HSLa, HSV, HSVa, VEC3 et VEC4.

![Image](https://cdn-media-1.freecodecamp.org/images/axdtM-WuHrgj8Y8KQWmtW6f5BqQII-OBREGx)

#### [Highlight selected](https://atom.io/packages/highlight-selected)

Highlight selected vous permet de mettre en surbrillance les mots sélectionnés dans le fichier que vous éditez. Vous pouvez utiliser **Ctrl-U** ou **Cmd-U** pour sélectionner plusieurs mots.

![Image](https://cdn-media-1.freecodecamp.org/images/LErNEMAXeqnyg47pbEUupdWasLmiPE4bjVZH)

#### [GIT diff](https://atom.io/packages/git-diff)

GIT diff marque les lignes dans la gouttière de l'éditeur qui ont été ajoutées, modifiées ou supprimées depuis le dernier commit.

![Image](https://cdn-media-1.freecodecamp.org/images/o27NUGzsvt2uPmccbGaBkKI8NB6mvpxONAtZ)
_GIT diff est inclus par défaut, vous n'avez pas besoin de l'installer_

#### [Sort lines](https://atom.io/packages/sort-lines)

Triez vos lignes par ordre alphabétique.

![Image](https://cdn-media-1.freecodecamp.org/images/27u4LG9Pwgbiy9uv6sfRuySRe4AhYaaM7m-P)

#### [Last cursor position](https://atom.io/packages/last-cursor-position)

Last cursor positions conserve un historique des positions passées du curseur pour revenir en arrière en utilisant **Alt-moins(-)**

![Image](https://cdn-media-1.freecodecamp.org/images/xYCbyYrCurV4QTIE5POJt5mz6-Ho2VcgJxjy)

#### [Code peek](https://atom.io/packages/code-peek)

Code peek vous permet de jeter un coup d'œil et d'éditer rapidement les fonctions contenues dans d'autres fichiers au lieu d'avoir à ouvrir le fichier séparément.

![Image](https://cdn-media-1.freecodecamp.org/images/2cL2WXyNMJePYj54UaQCSqF4UQgrCVYc9YlN)

#### [Package sync](https://atom.io/packages/package-sync)

Package sync vous permet de synchroniser les packages installés entre plusieurs ordinateurs. Cela est utile si vous travaillez sur différents environnements de travail.

![Image](https://cdn-media-1.freecodecamp.org/images/X-qWZrrd7y-wrRxdCJzX6FBn5q-RchWUIOVk)
_Vous pouvez synchroniser vos extraits, votre keymap et d'autres paramètres_

Voici mon fichier [packages.json](https://gist.githubusercontent.com/0x1ad2/3dcc76bd77334753c9993f2be38ebac3/raw/05b0eac18ae85b9ae346c45b3c0e74ff287db808/packages.json), copiez-le pour installer rapidement les packages mentionnés ci-dessus (et plus).

[Matthew Leonard](https://medium.com/@matt1222?source=responses---------0-) se demandait comment l'utiliser, voici donc un petit guide

* Installer le [plugin](https://github.com/atom-community/sync-settings) via la gestion des packages
* Créer un token Github avec les bons accès
* Créer un fichier Gist
* Définir les tokens et la dernière partie de l'URL du fichier Gist dans les paramètres du package
* Utiliser Ctrl-Shift-P (ou Cmd-Shift-P) et taper sync pour sauvegarder et restaurer les packages

#### [WakaTime](https://atom.io/packages/wakatime)

Parce que qui ne veut pas suivre son temps par projet ? ☺️

WakaTime vous permet de suivre votre temps par projet, les langages de programmation utilisés, le temps connecté et bien plus encore. Tout ce que vous avez à faire est d'installer le plugin et de récupérer une clé API depuis [https://wakatime.com/](https://wakatime.com/)

![Image](https://cdn-media-1.freecodecamp.org/images/lgMBJA39EqsaR1f6xE5jDbPe7dS2x-AH1cSm)

> Un grand merci à tous les développeurs de packages qui rendent ma vie beaucoup plus facile !

### Développement JavaScript

#### [Atom TypeScript](https://atom.io/packages/atom-typescript)

Pour tous ceux qui utilisent TypeScript pour le développement JavaScript, je recommande d'installer Atom TypeScript. Le seul plugin TypeScript dont vous aurez jamais besoin ♡

Quelques-unes des fonctionnalités sont

* Autocomplétion
* Analyse des erreurs en temps réel
* Informations de type au survol
* Compilation à la sauvegarde

#### [Atom TernJS](https://atom.io/packages/atom-ternjs)

TernJS fournit une intelligence de code JavaScript pour Atom avec Tern. Ajoute la prise en charge de ES5, ES6 (JavaScript 2015), Node.js, jQuery & Angular. Extensible via des plugins. Utilise le fournisseur de suggestions par autocomplete-plus.

![Image](https://cdn-media-1.freecodecamp.org/images/3hiUdyManoQNxGbJWRyHkCaqBuuQzZ90BW3Y)
