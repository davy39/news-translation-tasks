Par Jennifer Bland

En tant que développeur, vous devriez envisager de contribuer à des logiciels open source. De nombreux employeurs potentiels verront d'un bon œil ces contributions.

Mais lorsque vous êtes un nouveau développeur, il peut sembler que contribuer à l'open source dépasse vos compétences actuelles. Je vais donc vous montrer comment ignorer ces sentiments de doute et comment vous pouvez commencer à contribuer à l'open source dès maintenant.

### Comment j'ai appris Node.js

J'ai découvert Node.js pour la première fois lors de mon bootcamp de codage. Le bootcamp était spécialisé en JavaScript. Node.js fournit un back-end pour un projet écrit en JavaScript. Donc, si vous avez appris JavaScript en front-end, vous pouvez facilement transférer vos compétences en back-end avec Node.js.

Une fois diplômée de mon bootcamp de codage, j'étais motivée à améliorer mes compétences en codage le plus rapidement possible. La seule façon de le faire était d'écrire autant de code que possible pour m'améliorer. C'est alors que j'ai rejoint freeCodeCamp.

Sur une période de six mois, j'ai complété les quatre certifications originales de freeCodeCamp dans le programme de freeCodeCamp. Lorsque je suis arrivée à la section axée sur la construction de projets, j'ai utilisé JavaScript et Node.js en back-end. Après avoir complété toutes les certifications, j'ai travaillé sur l'un des premiers projets à but non lucratif de freeCodeCamp. Le projet concernait [Crimes Committed Against Women](https://youtu.be/9-WgWY2B10E).

### Contribuer à l'Open Source

Le projet à but non lucratif sur lequel j'ai travaillé avec Danny Libin peut être [trouvé ici](https://github.com/freeCodeCamp/Conference-for-Good). Danny et moi étions tous deux de nouveaux développeurs. Danny est pharmacien et a décidé d'apprendre à coder en parallèle. Nous avons tous deux appliqué nos nouvelles compétences en programmation pour créer ce projet. freeCodeCamp a pris notre code original et l'a converti en un projet open source appelé [Meeting for Good](https://github.com/freeCodeCamp/meeting-for-good).

Tout le monde peut contribuer à ce code open source. N'oubliez pas que deux développeurs juniors ont écrit le code initial. Donc, même si vous vous considérez comme un développeur junior, vous avez le même niveau de compétences que nous avions. La bonne nouvelle est que freeCodeCamp a d'autres codes open source auxquels vous pouvez contribuer. [Consultez leur dépôt GitHub](https://github.com/freeCodeCamp) pour trouver un projet open source. freeCodeCamp dispose également d'un excellent [guide sur la contribution à l'open source](https://github.com/freeCodeCamp/how-to-contribute-to-open-source).

Pour contribuer à tout code open source, visitez leur dépôt sur GitHub. Le code aura un onglet intitulé "Issues". Ce sont des problèmes que les gens ont identifiés avec le code. Ils peuvent également être de nouvelles fonctionnalités que les mainteneurs du code souhaiteraient ajouter. Vous pouvez passer en revue tous ces problèmes pour trouver quelque chose à faire pour contribuer.

La plupart des logiciels open source auront un fichier nommé "CONTRIBUTING.MD". Ce fichier expliquera précisément comment vous pouvez contribuer. Lisez les instructions et suivez-les exactement.

### Contribuer à Node.js

Mon travail de jour consiste à travailler en tant que développeur JavaScript. Tous les emplois que j'ai eus depuis que j'ai appris à coder ont utilisé Node.js en back-end. J'ai enseigné plusieurs ateliers de trois heures sur Node.js, Express.js et MongoDB pour Women Who Code Atlanta. Donc, puisque j'étais déjà familière avec Node.js, je me suis intéressée à contribuer à son code source.

Vous pouvez consulter les "Contributors" de n'importe quel dépôt sur GitHub et découvrir qui contribue au code source. De cette façon, j'ai appris que [Franziska Hinkelman](https://twitter.com/fhinkel) et [Myles Borins](https://twitter.com/MylesBorins) étaient des contributeurs principaux. Je leur ai envoyé un email pour leur demander comment m'impliquer dans la contribution à Node.js.

Myles a mentionné qu'ils allaient lancer un programme de mentorat. Il m'a donné le lien direct pour m'inscrire au programme de mentorat avant qu'il ne soit annoncé. J'étais probablement la toute première personne à m'inscrire.

Franziska a répondu à mon email. Elle était prête à travailler avec moi pour compléter ma première contribution. En raison de nos emplois du temps, nous n'avons pas pu planifier une réunion avant quelques semaines plus tard. À la date et à l'heure prévues, nous nous sommes réunis via Google Hangouts. Franziska m'a montré comment consulter les "Issues" et en trouver celles étiquetées "Good First Issue".

![Image](https://cdn-media-1.freecodecamp.org/images/Y05XeGdlHKxRwXubeuLnxP7f6gZrrn7W0dn8)
_Cliquez sur "good first issue" pour trouver des problèmes auxquels vous pouvez contribuer_

Franziska et moi avons trouvé un problème facile. Elle a expliqué ce qui causait le problème. Nous avions besoin de plus d'informations de la part de la personne qui avait ouvert le problème. De là, j'ai contacté cette personne pour obtenir les informations requises. Une fois que je les ai eues, j'ai pu résoudre le problème. Ensuite, j'ai pu soumettre une demande pour clôturer le problème. Ce fut ma première contribution open source à Node.js.

### Le programme de mentorat Node.js

Le comité Node.js recherche davantage de personnes pour contribuer au code source de Node.js. Actuellement, il y a plus de 600 problèmes en attente de contributions. Pour impliquer davantage de personnes, le comité Node.js a lancé un programme de mentorat Node.js.

Ce programme est ouvert à toute personne intéressée à contribuer aux logiciels open source. Ce programme est ouvert à tous, quel que soit leur niveau de compétence actuel. Les développeurs juniors et seniors sont les bienvenus.

Actuellement, le programme est en version bêta. Ils ont sélectionné quatre personnes pour être les premiers mentorés du programme bêta. Heureusement pour moi, j'étais l'une des quatre personnes choisies. J'ai été sélectionnée en raison de la contribution que j'ai faite avec l'aide de Franziska.

Lorsque le programme sera lancé, il durera six mois. Pendant cette période, un groupe de mentorés sera associé à un mentor. Le mentor leur montrera comment contribuer à Node.js. Les mentorés devront appliquer ce qu'ils apprennent et contribuer au code source. Après six mois, la classe sera diplômée. Les mentorés seront alors éligibles pour devenir mentors dans la prochaine classe afin de travailler avec leur propre groupe de mentorés.

Si vous êtes intéressé à rejoindre le programme de mentorat, [vous pouvez vous inscrire ici](https://github.com/nodejs/mentorship).

Je crée une série de vidéos détaillant mon expérience avec la version bêta du programme de mentorat Node.js. J'ai des vidéos montrant mes contributions, ainsi que des enregistrements de toutes mes réunions avec mon mentor. Vous pouvez [les regarder ici](https://www.youtube.com/watch?v=ig5Yx4OSNGo&list=PLjJlQgygxY7e_7FMBvkUFkUwIPC5vg5OL). Je tiens également un journal de mon expérience dans le programme que vous [pouvez lire ici](https://github.com/nodejs/mentorship/issues/87).
