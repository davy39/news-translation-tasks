## Pourquoi vous avez besoin d'une PWA n'est pas la question. Voyons pourquoi vous n'en avez peut-être PAS besoin

Ma boîte de réception a été remplie de questions concernant les PWA après mes deux derniers articles. 

Mon article "[Conseils pratiques sur le développement d'applications web progressives](https://www.freecodecamp.org/news/practical-tips-on-progressive-web-app-development/)" a partagé les avantages réels de la technologie PWA, et mon article "[Applications web progressives vs natives](https://keenethics.com/blog/progressive-web-apps-vs-native-which-to-choose-and-when)" a encore renforcé la conviction de nombreuses personnes qu'une application web progressive est exactement ce dont elles ont besoin.

Je suis exceptionnellement heureux d'avoir réussi à les convaincre que les PWA sont l'avenir des applications web. Je soutiens la technologie PWA comme une voie que l'industrie du développement logiciel devrait prendre à l'avenir. Mais les applications web progressives ne sont pas une panacée, pas une solution universelle.

Puisque j'ai déjà expliqué _ce que signifie PWA_ et _pourquoi vous avez besoin_ d'une _PWA_, aujourd'hui, je vais terminer le travail et montrer _pourquoi vous n'avez peut-être PAS besoin_ d'une _PWA_.

Faire le choix entre utiliser ou non une PWA est plus facile que vous ne le pensez. Il suffit de répondre à quatre questions simples et vous verrez le bon chemin vous-même.

## 1. Avez-vous des développements prêts à l'emploi ?

La vitesse et la simplicité du développement d'applications web progressives ont été mentionnées parmi les principaux avantages des PWA. Cependant, cet avantage peut être annulé par le fait que vous avez déjà certains développements prêts à l'emploi pour une application mobile. 

Convertir une application mobile prête à l'emploi en une application web progressive ne peut pas se faire avec quelques clics simples. Au lieu de cela, une application web progressive doit être développée à partir de zéro. Si vous avez déjà traversé le processus long et compliqué du développement d'applications natives, pourquoi le jeter et tout recommencer depuis le début ?

Bien sûr, votre application mobile peut être obsolète ou inefficace. Dans ce cas, vous devez choisir entre la relancer avec l'aide d'une entreprise de développement d'applications mobiles ou construire une application entièrement nouvelle en utilisant la technologie PWA.

Par exemple, [Pinterest](https://www.pinterest.ca/) – l'un des exemples les plus brillants d'entreprises ayant mis en œuvre la PWA – a décidé de choisir les deux options. Ils ont développé une application web progressive mais n'ont pas oublié l'application mobile traditionnelle non plus.

![Image](https://www.freecodecamp.org/news/content/images/2019/09/Progressive-web-app.png)

_**Idée clé** : L'approche PWA est une excellente solution si vous n'avez pas de développements prêts à l'emploi. Si vous avez déjà une version mobile native ou multiplateforme de votre application, il sera plus facile et plus productif de les terminer plutôt que de construire une nouvelle application à partir de zéro._

## 2. Prévoyez-vous de vous concentrer uniquement sur les appareils mobiles ?

L'idée des fonctionnalités PWA est d'améliorer l'expérience d'un utilisateur accédant à une page web depuis un appareil mobile. Jusqu'à ce que la technologie PWA soit introduite, les entreprises souhaitant cibler à la fois le web et le mobile avaient deux options :

* adapter une page web aux appareils mobiles
* développer une application mobile à partir de zéro

La première option est maladroite et peu pratique du point de vue de l'utilisateur. La deuxième option est coûteuse en ressources du point de vue de l'entreprise. 

L'avancement des PWA a apporté le juste milieu : la commodité pour l'utilisateur et l'efficacité pour l'entreprise. Si vous ne prévoyez pas d'entrer dans le créneau du web ou du bureau mais choisissez de vous concentrer uniquement sur les applications mobiles, il n'est pas nécessaire de construire une PWA. 

Bien sûr, lors du développement d'applications web progressives, vous gagnez car vous développez une application pour Android et iOS. Cependant, c'est exactement ce que vous pouvez réaliser avec le développement multiplateforme avec Ionic, Cordova ou React Native.  

Pendant ce temps, Pinterest a développé une application web progressive pour augmenter leur plateforme web et leur application mobile déjà existantes.

![Image](https://www.freecodecamp.org/news/content/images/2019/09/Progressive-web-app--4-.png)

_**Idée clé** : Les PWA fonctionnent mieux pour les entreprises ciblant à la fois les plateformes web et mobiles. Si vous choisissez de vous concentrer sur une seule de celles-ci, il n'y a aucune raison de choisir le développement d'applications web progressives._

## 3. Êtes-vous réticent à laisser vos utilisateurs accéder à l'application via un navigateur web ?

Une autre différence essentielle entre les applications mobiles et les applications web progressives réside dans le fait que les premières sont téléchargées depuis le store, tandis que les secondes – via un lien partagé ou directement depuis un site web comme illustré ci-dessous.

![Image](https://www.freecodecamp.org/news/content/images/2019/09/Progressive-web-app--3-.png)

Les goûts diffèrent. Si vous pensez que travailler avec l'App Store ou Google Play est plus facile que de lutter contre la concurrence SEO sur Google ou d'inspirer les utilisateurs à partager vos liens, vous devriez vous sentir libre de choisir le développement d'applications mobiles. 

Si vous pensez qu'il est plus difficile d'être présenté dans les résultats de recherche en tête des Stores par rapport aux résultats de recherche Google, optez pour la PWA. De plus, si votre site web a déjà un public cible suffisamment large d'utilisateurs dévoués, les applications web progressives fonctionneraient mieux pour vous.

---

_**Idée clé** : Si vous voulez que vos utilisateurs accèdent à l'application via l'App Store ou Google Play plutôt que via un navigateur, ou si l'optimisation pour les stores d'applications (ASO) semble plus simple que l'optimisation pour les moteurs de recherche (SEO), vous devriez opter pour une application mobile._

## 4. Avez-vous besoin de fonctionnalités avancées des appareils mobiles ?

Bien que les PWA puissent fonctionner avec la géolocalisation, de nombreuses fonctionnalités avancées des appareils mobiles restent hors de leur portée. Les applications web progressives ne fonctionnent pas avec les capteurs de proximité, les contrôles avancés de la caméra, l'enregistrement audio ou vidéo, la numérisation des empreintes digitales, le NFC ou même le Bluetooth. 

Demandez à votre équipe de développement la liste des fonctionnalités PWA qu'ils ne pourront pas implémenter. Si vous ne pouvez pas vous passer d'au moins une de ces fonctionnalités, vous devriez opter pour le développement mobile.

Par exemple, l'application web progressive de Pinterest ne propose pas la fonction de prise de photos, qui est présente dans leur application mobile.

![Image](https://www.freecodecamp.org/news/content/images/2019/09/Progressive-web-app--2--1.png)

_**Idée clé** : L'incompatibilité avec les fonctionnalités natives des appareils peut vous empêcher de développer une PWA. Bien que ce problème puisse être résolu à l'avenir, pour l'instant, vous devriez opter pour les applications mobiles._

## Pour conclure

Avant de demander une PWA, vous devez décider si vous avez vraiment besoin de l'approche PWA. Commencez par répondre à quatre questions simples :

1. Avez-vous des développements prêts à l'emploi, comme une application mobile native ?
2. Voulez-vous vous concentrer uniquement sur les appareils mobiles ? Pas de plateformes de bureau ou web ?
3. Êtes-vous réticent à laisser vos utilisateurs accéder à l'application via un navigateur web ?
4. Avez-vous besoin de fonctionnalités avancées des appareils mobiles ? Allez-vous utiliser la numérisation des empreintes digitales, le NFC ou le Bluetooth ?

Si vous répondez "oui" à la majorité de ces questions sur les applications web progressives, alors, probablement, vous n'avez pas besoin d'une application web progressive. Une application mobile couvrira pleinement tous vos besoins.
