Les bases de données NoSQL peuvent parfois sembler déroutantes et écrasantes, en partie à cause de leur flexibilité.

C'est pourquoi nous avons préparé un cours vidéo de 3 heures pour vous aider à comprendre exactement ce qu'est une base de données NoSQL, ainsi que les différents types disponibles pour vous. 

À la fin de ce cours, vous aurez construit 4 bases de données basées sur les 4 types principaux, et vous aurez pratiqué vos apprentissages en réalisant des projets. 

Mais d'abord, commençons par les bases.

## Qu'est-ce que NoSQL ?

La première chose que vous devez savoir est que NoSQL est une **approche** de la gestion des bases de données.

Elle est considérée comme super flexible car elle permet une variété de modèles de données, tels que 'clé-valeur', 'document', 'large colonne ou tabulaire' et 'graphe'.

![Image](https://www.freecodecamp.org/news/content/images/2021/09/nosql-types.jpg)

Ce sont les 4 types que nous examinerons de près dans le **cours vidéo**, ainsi que la nouvelle tendance émergente des bases de données Multi Modèle. 

Avec chaque plongée en profondeur sur les 4 types de bases de données NoSQL, nous aborderons chaque apprentissage comme une explication, un exemple et un exercice – les 3 E – afin de bien comprendre le sujet que nous discutons.

## Comment fonctionnent les bases de données ?

Les bases de données ont plusieurs couches. La première couche est une interface, ou en d'autres termes, une plateforme visuelle où vous pouvez visiter et interagir avec les données. C'est là que vous trouverez le format, le langage et le transport. 

![Image](https://www.freecodecamp.org/news/content/images/2021/09/Screenshot-2021-09-14-at-19.17.49.png)

Dans ce cours vidéo, l'interface que nous allons utiliser s'appelle Datastax Astra Database management system. C'est là que nous créerons les 4 types de bases de données pour les parties exemple et exercice. 

DataStax Astra DB est une base de données autoscalable en tant que service, construite sur Apache Cassandra, conçue pour simplifier le développement d'applications natives du cloud. 

Parce qu'elle est construite sur Apache Cassandra, vous nous verrez utiliser le Cassandra Query Language, ou CQL, à quelques reprises dans ce cours. CQL offre un modèle proche de SQL dans le sens où les données sont placées dans des tables contenant des lignes de colonnes. Ces langages sont la manière dont nous interagissons avec les données dans notre base de données.

La couche suivante d'une base de données est la couche d'exécution. C'est là que nous analysons les requêtes entrantes, provenant de notre interface. Elle est également utilisée comme analyseur et dispatch.

Et enfin, nous avons la couche de stockage, où l'indexation des données se produit.

![Image](https://www.freecodecamp.org/news/content/images/2021/09/database.jpg)

L'utilisation de Datastax Astra nous permettra de créer les 4 types de bases de données pour ce tutoriel, donc je n'aurai pas à m'inscrire à des systèmes de gestion de bases de données séparés pour chaque section. Mais vous n'êtes pas obligé de l'utiliser. Il existe littéralement des dizaines et des dizaines de choix, alors n'hésitez pas à faire votre choix. 

## Commençons !

Maintenant que vous savez quels types de bases de données NoSQL nous allons apprendre, ainsi que le fonctionnement des bases de données, commençons à en apprendre davantage sur chacun d'eux en détail.

Voici les sujets que ce cours couvrira :
