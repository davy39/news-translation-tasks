Chaque mois d'octobre, Digital Ocean et GitHub envoient des t-shirts Hacktoberfest gratuits à des milliers de personnes à travers le monde.

J'ai reçu des t-shirts Hacktoberfest ces trois dernières années, et je les porte avec fierté.

Dans ce court article, je vais vous montrer comment obtenir un t-shirt Hacktoberfest — même si vous n'avez jamais codé auparavant.

L'essentiel est le suivant : si vous pouvez ouvrir 5 pull requests d'ici le 31 octobre — et je vais vous expliquer comment faire — vous pourrez recevoir votre propre t-shirt Hacktoberfest gratuitement chez vous.

### Étape n°1 : S'inscrire au Hacktoberfest

Vous avez besoin d'un compte GitHub. Si vous n'en avez pas encore, vous pouvez en [créer un gratuitement en quelques minutes](https://github.com/join).

Ensuite, rendez-vous sur la [page d'inscription au Hacktoberfest et suivez les instructions](https://hacktoberfest.digitalocean.com/sign_up/register). Cela ne prend qu'un instant.

### Étape n°2 : Apprendre à créer une pull request

Les pull requests sont un moyen de contribuer avec vos modifications de code à un projet open source. Elles utilisent un outil appelé Git, qui fonctionne sur une plateforme de codage social appelée GitHub.

GitHub est le centre de l'univers open source et héberge des projets comme Linux et React.js.

Cela peut prendre des mois aux développeurs pour devenir assez bons pour contribuer à certains de ces projets. Mais la communauté freeCodeCamp a facilité la tâche des débutants complets pour contribuer à notre projet open source.

**Vous pouvez contribuer à freeCodeCamp directement dans votre navigateur sur GitHub.** Vous n'avez rien besoin d'installer sur votre ordinateur. Vous n'avez même pas besoin de connaître un langage de programmation. Il vous suffit de choisir un article que vous souhaitez aider à améliorer.

Voici un court gif montrant comment faire cela :

![Image](https://cdn-media-1.freecodecamp.org/images/gLqsWdwnqZhfNaUmda0apuCMUJa1pLQwpF9S)
_Un Gif montrant comment faire une contribution open source à un article, directement dans votre navigateur._

Les étapes sont :

1. Explorez le [dépôt freeCodeCamp](https://github.com/freeCodeCamp/freeCodeCamp/tree/master/guide) et choisissez un article que vous souhaitez aider à améliorer.
2. Ouvrez le fichier index.md de ce dossier en double-cliquant dessus.
3. Cliquez sur le symbole du stylo dans le coin supérieur droit pour le modifier.
4. Apportez vos modifications. Vous pouvez consulter ce [pense-bête Markdown de base](https://github.com/adam-p/markdown-here/wiki/Markdown-Cheatsheet) si vous voulez voir comment ajouter des images ou des vidéos. Vous n'avez même pas besoin de connaître le HTML.
5. Faites défiler vers le bas et décrivez vos modifications dans le message de commit.
6. Assurez-vous que le bouton radio « Create a new branch for this commit and start a pull request » est sélectionné.
7. Cliquez sur « Commit Changes ».
8. Sur la page suivante, cliquez sur « Create Pull Request ».
9. Lisez la liste de contrôle pour vous assurer que vous avez tout fait, cochez les cases, puis soumettez.

Nous effectuerons des tests automatiques pour nous assurer que vos modifications n'ont rien cassé. Ensuite, l'un de nos mainteneurs vous donnera son avis sur votre article. Une fois que tout semblera correct, nous fusionnerons votre pull request.

Nous déploierons éventuellement votre contribution sur [freeCodeCamp.org](https://www.freecodecamp.org), où des millions de personnes pourront la lire et s'y référer.

![Image](https://cdn-media-1.freecodecamp.org/images/Y0y2AetCAV6WwEpIrCkMx6tPlojSc3Kvqc6L)

Contribuer à freeCodeCamp est un bon moyen de s'initier à la contribution open source. Vous pouvez également rejoindre notre [salon de discussion des contributeurs](https://gitter.im/FreeCodeCamp/Contributors) pour passer du temps avec d'autres contributeurs et poser des questions.

De plus, nous avons récemment traduit l'intégralité du programme de freeCodeCamp dans plusieurs langues mondiales majeures.

### Étape n°3 : OK — maintenant, faites 4 pull requests de plus

Vous pouvez faire quatre pull requests supplémentaires pour freeCodeCamp, ou pour n'importe quel projet open source de votre choix. Voici [un guide plus détaillé sur la contribution à l'open source](https://github.com/freeCodeCamp/how-to-contribute-to-open-source), si vous vous sentez l'âme d'un aventurier.

### Étape n°4 : Vérifiez si vous êtes qualifié

Une fois inscrit au Hacktoberfest, vous pouvez suivre votre progression sur le [site Web du Hacktoberfest](https://hacktoberfest.digitalocean.com).

### Étape n°5 : Attendez que votre t-shirt Hacktoberfest 2018 arrive par la poste

Les années précédentes, les t-shirts sont arrivés en novembre ou décembre, selon la distance à laquelle vous habitez de San Francisco.

L'équipe de Digital Ocean devrait finir par vous contacter pour vous demander votre taille de t-shirt et votre adresse de livraison. (Gardez à l'esprit qu'ils envoient plus de 10 000 t-shirts, ce processus prendra donc un certain temps.) Et oui, ils expédient à l'international gratuitement.

### Questions fréquemment posées sur le Hacktoberfest

#### Quels types de pull requests comptent pour gagner le t-shirt Hacktoberfest ?

Toute pull request effectuée sur un dépôt public sur GitHub comptera. La pull request doit contenir des commits que vous avez personnellement effectués vous-même — et non des commits automatisés provenant de bots.

#### Et si mes pull requests ne sont pas acceptées ?

Même si vos pull requests ne sont pas acceptées, elles devraient tout de même compter pour les 5 pull requests nécessaires pour gagner le t-shirt. La seule exception serait si le mainteneur du projet choisit de marquer votre pull request comme « invalide ». Ils peuvent le faire s'ils estiment que votre pull request est peu qualitative ou si elle contient du plagiat.

#### Que se passe-t-il si je ne fais pas au moins cinq pull requests avant le 31 octobre à minuit ?

Tant que vous vous êtes inscrit au Hacktoberfest, vous recevrez tout de même quelques autocollants gratuits par la poste. Il vaut donc toujours la peine de s'inscrire, même si vous êtes trop occupé pour participer.

#### Existe-t-il des événements Hacktoberfest en personne auxquels je peux assister pour obtenir de l'aide pour mes pull requests ?

Il y a des événements partout dans le monde. Voici [une liste à jour](https://docs.google.com/spreadsheets/d/1yjPG_WoF_6ZkrJF7pHZjAdr0ztt-23fNK9DFgNZvB80/edit#gid=0).

Vous pouvez également [assister à un événement de groupe d'étude freeCodeCamp dans votre ville](https://study-group-directory.freecodecamp.org).

### Célébrez le Hacktoberfest avec la communauté open source mondiale

Jusqu'à présent ce mois-ci, nous avons déjà fusionné des centaines de pull requests sur le [dépôt de freeCodeCamp](https://github.com/freecodecamp). Et nous ne faisons que commencer.

J'espère que vous contribuerez à la communauté open source, que vous apprendrez de nouvelles choses et que vous vous amuserez au passage !
