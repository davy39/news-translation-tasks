## Automatiser les flux de travail avec Blueprints, C++ et Python

**Chaque métier comporte des tâches et des processus répétitifs qui peuvent être automatisés.** Ces tâches peuvent absorber une partie importante de votre temps.

Quelqu'un ayant seulement des connaissances de base en scripting peut créer un script capable de réduire le temps d'exécution de ces tâches à un strict minimum. À long terme, ce gain de temps se transforme en heures supplémentaires utilisables pour un travail plus productif.

Apprendre à automatiser les choses est donc une **compétence inestimable**. Une compétence qui peut être acquise dans des secteurs différents et plus spécifiques, puis appliquée à d'autres secteurs plus généraux.

Surtout compte tenu des temps difficiles actuels, l'apprentissage d'une nouvelle compétence qui aura un impact significatif sur votre employabilité est vital.

Dans cet article, nous voulons décrire l'importance de l'automatisation lors du travail avec des logiciels en temps réel comme l'Unreal Engine.

## Pourquoi l'Unreal Engine est intéressant

L'Unreal Engine d'Epic est l'une des applications de jeu et de 3D en temps réel les plus populaires du marché. Il est utilisé pour la création de contenus de divertissement tels que des jeux et des installations interactives.

Des disciplines comme la réalité virtuelle et augmentée nécessitent l'utilisation de processus sophistiqués qui, comme tout autre pipeline, peuvent être optimisés grâce à l'automatisation.

De plus, le marché du jeu vidéo et de la 3D en temps réel est en croissance continue, ce qui augmente la demande de talents.

Selon une étude menée par [Burning Glass Technologies](https://www.burning-glass.com/), une société d'analyse du marché du travail, les salaires moyens des développeurs Unreal ont **augmenté de 22 % l'année dernière**, et les **salaires des artistes de 51 %**.

### Pourquoi utiliser les Blueprints pour le flux de travail et même la programmation de jeux

En plus de l'interface de programmation générale en C++, l'Unreal Engine propose un **système de programmation graphique appelé Blueprints**. Ils exposent l'intégralité des fonctionnalités sous forme de code C++, ce qui signifie que tout, y compris les fonctionnalités de jeu, peut être programmé sans inconvénients majeurs en termes de performances.

L'interface graphique nous permet de composer rapidement des entités dans un flux pour créer des fonctionnalités. En regardant un exemple qui définit le matériau d'un static mesh actor, nous pouvons voir que le Blueprint est facilement lisible.

![Image](https://www.freecodecamp.org/news/content/images/2020/04/utility.jpeg)
_Script Blueprints pour définir un matériau pour un static mesh actor_

### Pourquoi utiliser le C++ pour le flux de travail et la programmation de jeux

L'approche « native » du codage pour l'Unreal Engine est le C++. Il est utilisé pour créer la logique de jeu, simplifier la création de niveaux et les flux de travail, et améliorer le pipeline de développement. Il est plus complexe et difficile à apprendre que les Blueprints, mais peut apporter un boost supplémentaire en termes de temps d'exécution et de performance.

Son **avantage en termes de performance** en fait le langage de choix pour les opérations essentielles de bas niveau telles que le rendu et la physique dans le développement de jeux. Pour les développeurs qui maîtrisent déjà le C++, c'est un ajout pratique de pouvoir automatiser des processus sans apprendre directement un langage supplémentaire.

### Pourquoi utiliser Python pour l'optimisation des flux de travail

Par rapport aux Blueprints ou au C++, Python est l'un des **langages standards de facto utilisés pour automatiser les tâches**. Il est simple à apprendre, compréhensible et extrêmement polyvalent puisqu'il peut s'exécuter sur presque n'importe quelle plateforme sans effort supplémentaire.

Epic liste Python comme l'une des compétences requises, par exemple, pour les Data Pipeliners dans leur **[Creator’s Field Guide to Emerging Careers in Interactive 3D](https://epicgames.ent.box.com/s/n12ixy53l8cknz73npimsr54frkvm72c).**

L'**Unreal Engine dispose d'un support complet pour le scripting Python**. Malheureusement, il n'est pas adapté au scripting en temps réel et en jeu, mais peut seulement être utilisé pour le scripting de l'Unreal Editor. La simplicité de Python en fait cependant une option incroyable pour le prototypage rapide de l'automatisation de pipeline.

Epic eux-mêmes tentent de promouvoir l'usage de Python via la [documentation](https://docs.unrealengine.com/en-US/Engine/Editor/ScriptingAndAutomation/Python/index.html), les [docs de l'API](https://docs.unrealengine.com/en-US/PythonAPI/index.html), et même un [webinaire enregistré](https://m.youtube.com/watch?v=0guOMTiwmhk) sur le scripting Python pour l'Unreal Engine.

## Écrire notre propre script de nettoyage de projet automatisé en utilisant Python

Les projets de grande envergure peuvent vite devenir désordonnés. Avoir un script sous la main capable d'effectuer un nettoyage en examinant tous nos assets sélectionnés et en les déplaçant automatiquement dans les dossiers appropriés nous aide à améliorer considérablement notre flux de travail.

Le schéma ci-dessous explique l'idée derrière le script.

![Image](https://www.freecodecamp.org/news/content/images/2020/05/WhatsApp-Image-2020-05-01-at-14.11.07.jpeg)

> À l'heure actuelle, la version de Python définie par défaut dans Unreal est Python 2.7. Si vous souhaitez utiliser Python 3, vous pouvez suivre le processus [décrit ici](https://docs.unrealengine.com/en-US/Engine/Editor/ScriptingAndAutomation/Python/index.html) pour passer à Python 3.

Pour commencer, nous devons d'abord importer la bibliothèque unreal dans notre portée. Une fois cela fait, nous pouvons créer des instances de classes élémentaires. Pour l'instant, nous n'avons besoin que de l' `[EditorUtilityLibrary](https://docs.unrealengine.com/en-US/PythonAPI/class/EditorUtilityLibrary.html?highlight=editorutilitylibrary)` qui nous permet d'obtenir une liste de tous les assets sélectionnés.

Nous pouvons obtenir le nombre d'assets sélectionnés en utilisant la méthode Python `len()` et utiliser la méthode de journalisation d'Unreal pour obtenir une sortie dans le Debug Log de l'Unreal Engine.

```python
import unreal

# créer des instances de classe unreal
editor_util = unreal.EditorUtilityLibrary()

# obtenir les assets sélectionnés
selected_assets = editor_util.get_selected_assets()
num_assets = len(selected_assets)

unreal.log(num_assets)
```

Le snippet de code ci-dessus affichera un nombre unique dans le `Debug Log`. Pour chacun des assets sélectionnés, nous voulons maintenant obtenir des informations telles que le nom de l'asset et sa classe.

L'objet Unreal `[ObjectBase](https://docs.unrealengine.com/en-US/PythonAPI/class/_ObjectBase.html?highlight=objectbase#unreal._ObjectBase)` possède plusieurs méthodes utilitaires pour obtenir le nom, la classe et d'autres attributs. Nous utiliserons les méthodes `get_fname()` et `get_class()`.

```python
for assets in selected_assets:
    # obtenir l'instance de classe et le nom en texte clair
    asset_name = asset.get_fname()
    asset_class = asset.get_class()
    
    unreal.log("{} - {}".format(asset_name, asset_class))
```

Cependant, cela ne nous donnera que la définition de la classe et non le nom en texte clair de la classe elle-même, que nous voulons utiliser pour la création de dossiers.

Pour obtenir le nom d'affichage au lieu de la définition de la classe, nous devons créer une instance de la `[SystemLibrary](https://docs.unrealengine.com/en-US/PythonAPI/class/SystemLibrary.html?highlight=systemlibrary)`. Sa méthode `get_class_display_name()` prend une définition de classe et renvoie le nom de la classe sous forme de `String`.

```python
# créer des instances de classe unreal
editor_util = unreal.EditorUtilityLibrary()
system_lib = unreal.SystemLibrary()

...

for assets in selected_assets:
    # obtenir l'instance de classe et le nom en texte clair
    asset_name = asset.get_fname()
    asset_class = asset.get_class()
    class_name = system_lib.get_class_display_name(asset_class)

    unreal.log("Name: {} - Class: {}".format(asset_name, class_name))

```

Maintenant, nous pouvons voir quelque chose comme "Name: NewMaterial - Class: Material" affiché dans notre log. C'est précisément le genre d'information dont nous avions besoin.

La dernière étape consiste à « renommer » nos assets vers un emplacement donné. Par exemple, chaque Material sera renommé en `"/Material/<Nom de l'Asset Material>"`, ce qui le déplacera dans les dossiers correspondants.

Pour « renommer » des assets, nous avons besoin d'une classe supplémentaire. La méthode `rename_loaded_asset()` fait partie de l'[`EditorAssetLibrary`](https://docs.unrealengine.com/en-US/PythonAPI/class/EditorAssetLibrary.html?highlight=editorassetlibrary), nous devons donc d'abord créer une instance de cette classe. En plus de cela, nous devons créer un nouvel emplacement vers lequel l'asset sera relocalisé.

Pour que cela reste plus indépendant de la plateforme, nous utiliserons le module `os` et sa méthode `path.join()`.

Une fois que nous avons créé la variable `new_path`, nous pouvons l'utiliser dans l'appel de la méthode `rename_loaded_asset()` pour relocaliser notre asset actuel.

```python
import os
import unreal

# créer des instances de classe unreal
editor_util = unreal.EditorUtilityLibrary()
system_lib = unreal.SystemLibrary()
editor_asset_lib = unreal.EditorAssetLibrary()

...

for assets in selected_assets:
    # obtenir l'instance de classe et le nom en texte clair
    asset_name = asset.get_fname()
    asset_class = asset.get_class()
    class_name = system_lib.get_class_display_name(asset_class)

    # assembler le nouveau chemin et déplacer l'asset
    new_path = os.path.join("/Game", class_name, asset_name)
    editor_asset_lib.rename_loaded_asset(asset_name, new_path)
    
    unreal.log("Moved {} to {}".format(asset_name, new_path))
```

En exécutant ce script dans l'Unreal Engine, le log vous fournira un message tel que : `"Moved NewMaterial to /Game/Material/NewMaterial"`.
En observant notre projet, nous pouvons maintenant voir que tous les assets sélectionnés ont été rangés dans des dossiers nommés selon leurs classes.

Comme vous pouvez le voir, créer un script de base est assez simple. Bien sûr, nous devons nous occuper de la gestion des erreurs, d'une journalisation appropriée et d'une interface utilisateur conviviale pour des outils plus sophistiqués, mais même des scripts simples peuvent faire gagner beaucoup de temps.
