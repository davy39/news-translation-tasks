[Hacktoberfest](https://hacktoberfest.digitalocean.com/) est un événement organisé par [DigitalOcean](https://www.digitalocean.com/) en partenariat avec de nombreuses organisations. Le but est de contribuer et de donner en retour aux projets open-source. Les organisateurs offrent de petites récompenses pour un travail bien fait, comme un t-shirt et des autocollants.

D'autres organisations participent également en offrant leurs propres récompenses. Vous pouvez trouver une liste communautaire des organisations offrant des incitations à contribuer sur [Hacktoberfest Swag](https://hacktoberfest-swag.com/).

Seules les contributions significatives compteront pour les 4 requises. Prenez votre temps et faites preuve de courtoisie envers les mainteneurs pour vous assurer que vos pull requests soient fusionnées. Plus d'informations à ce sujet dans [Évitons d'en faire un Spamtoberfest](#heading-evitons-den-faire-un-spamtoberfest) ci-dessous.

## Comment participer à Hacktoberfest

Il est important de noter que Hacktoberfest est destiné à apporter toute contribution de **qualité** aux projets open-source. Personne n'a dit qu'il fallait coder !

Tout le monde est capable d'apporter quelque chose :

* Auteurs
  
* Artistes
  
* Développeurs logiciels
  
* Rédacteurs techniques
  
* Traducteurs
  
### Projets recommandés pour contribuer à Hacktoberfest

Je ne vais pas lister explicitement des dépôts pour contribuer du code, car c'est assez facile à trouver. Tous les dépôts listés sur les liens suivants ont opté pour Hacktoberfest.

* [GitLab](https://gitlab.com/explore/projects?topic=hacktoberfest) (Tous les dépôts avec le sujet `hacktoberfest` sur GitLab.)
  
* [GitHub](https://github.com/topics/hacktoberfest) (Tous les dépôts avec le sujet `hacktoberfest` sur GitHub.)
  
Ci-dessous se trouvent des projets que je recommande si vous ne savez pas coder mais que vous souhaitez tout de même contribuer à des projets open-source (ou [Creative Commons](https://creativecommons.org/)). Cela n'inclut que les projets qui ont manuellement opté pour Hacktoberfest.

Vous ne devriez pas avoir besoin de connaître des langages de programmation ou du HTML pour contribuer à ces projets. Cependant, vous devrez savoir comment commiter votre travail avec Git, ce que nous aborderons plus tard.

Projets où vous devez simplement taper en anglais courant. Des connaissances en programmation et techniques seront utiles, mais aucune compétence spécialisée n'est requise.

Cela n'inclut que les dépôts qui maintiennent la documentation dans un format simple pour la plupart des utilisateurs, comme [Markdown](https://commonmark.org/help/).

* [GitHub Docs](https://github.com/github/docs) — Documentation pour GitHub.

#### Traductions

Projets nécessitant des traductions de texte de l'anglais vers d'autres langues.

Les projets utilisant des plateformes de traduction collaborative comme [Weblate](https://weblate.org/) ne sont pas inclus ici, car le traducteur ne crée pas la pull request. Malheureusement, Hacktoberfest ne considère que la personne ayant ouvert la pull request, et non les auteurs des modifications.

Assurez-vous de ne contribuer que des traductions revues par des humains — soit traduites par un humain, soit traduites par une machine mais vérifiées par un humain. Encore une fois, la qualité est importante.

* [Free Programming Books](https://github.com/EbookFoundation/free-programming-books/) — Traduire les fichiers de santé.
  
* [tldr-pages](https://github.com/tldr-pages/tldr) — Traduire les fiches mémo.

## Évitons d'en faire un Spamtoberfest

Malheureusement, Hacktoberfest s'est forgé une mauvaise réputation auprès de certains projets en raison du spam. Parfois, les contributeurs peuvent être un peu trop enthousiastes pour terminer Hacktoberfest et manquer le but. ^-^'

Depuis 2020, si un utilisateur fait 2 pull requests qui reçoivent une étiquette contenant le mot `spam` ou `invalid`, le contributeur sera banni de Hacktoberfest indéfiniment, y compris pour les événements futurs.

[Mindustry](https://github.com/Anuken/Mindustry) gère cela de la meilleure manière selon moi avec le [Hacktoberfest Hall of Shame](https://github.com/Anuken/Mindustry/pulls?q=label%3A%22hacktoberfest+hall+of+shame+%28invalid%29%22+).

Depuis 2021, Hacktoberfest fait plus pour lutter contre le spam en excluant les projets qui encouragent les pull requests sans valeur.

![Image](https://www.freecodecamp.org/news/content/images/2021/10/image-8.png align="left")

*L'objectif de donner en retour à l'open-source est compromis si la plupart des contributions vont simplement à des dépôts "Complete Hacktoberfest !".*

### Comment faire des contributions précieuses pour Hacktoberfest

* Vous avez un mois pour faire les 4 pull requests. Utilisez ce temps ! Elles ont plus de chances d'être fusionnées si vous faites moins de pull requests de bonne qualité.
  
* Ne supposez pas que chaque pull request que vous faites sera fusionnée avant la fin de Hacktoberfest. Essayez d'en faire plus que 4 si vous le pouvez !
  
### Pratiquer la courtoisie commune

Les mainteneurs de projets open-source ont aussi une vie. Ils n'ont pas toujours le temps de revoir toutes les pull requests, surtout vers la fin de Hacktoberfest.

Ne faites pas sentir mal les mainteneurs, ne les pressez pas et ne demandez pas d'attention. Travaillez simplement sur autre chose s'ils sont occupés.

## Comment commiter du travail et ouvrir une pull request

Maintenant que vous savez comment être un participant utile à Hacktoberfest et que vous avez appris quelques projets auxquels vous pourrez contribuer, il ne reste plus qu'à apprendre comment contribuer.

GitHub et GitLab offrent tous deux des moyens de simplifier la contribution pour les petites modifications. Nous n'allons pas les utiliser car il sera plus simple de montrer un seul ensemble d'instructions qui fonctionnera partout plutôt que de dépendre de fonctionnalités spécifiques à la plateforme.

Avant de continuer, choisissez un dépôt auquel vous souhaitez contribuer. Si vous êtes multilingue, tldr-pages sera le plus facile, sinon envisagez Free Programming Books.

Je vais utiliser Free Programming Books dans mes exemples.

### Comment installer Git

Tout d'abord, vous devez installer Git, qui est un système de contrôle de version. C'est l'outil que les projets utilisent pour gérer les modifications.

Les utilisateurs de Windows peuvent le télécharger depuis le site [git-scm](https://git-scm.com/). Les utilisateurs de Linux peuvent l'installer en utilisant leur gestionnaire de paquets préféré.

Une fois Git installé, vérifiez qu'il est correctement installé avec `git --version`.

Configurez maintenant votre nom et votre adresse e-mail. Le nom n'a pas d'importance, mais vous devez spécifier la même adresse e-mail que celle que vous utilisez pour Hacktoberfest.

```plaintext
git config --global user.name "Votre Nom"
git config --global user.email "votre@email.org"
```

Notez que l'adresse e-mail que vous spécifiez ici sera considérée comme une information publique une fois votre pull request fusionnée. Tout le monde peut voir les détails d'un commit, y compris les détails de l'auteur.

Par exemple, mon adresse e-mail apparaît dans la sortie lorsque je fais `git log` sur l'un de mes commits.

```sh
commit 9647912b202a57474b4cd0ce796126c462c1ecc0 (HEAD -> added-react-book, develop/added-react-book)
Author: Seth Falco <seth@falco.fun>
Date:   Tue Oct 5 14:04:25 2021 +0200

    added a react book from digitalocean
```

Si cela pose un problème, [GitHub](https://docs.github.com/en/account-and-profile/setting-up-and-managing-your-github-user-account/managing-email-preferences/setting-your-commit-email-address#setting-your-commit-email-address-on-github) et [GitLab](https://docs.gitlab.com/ee/user/profile/index.html#use-an-automatically-generated-private-commit-email) offrent une fonctionnalité pour masquer votre adresse e-mail en vous donnant une adresse jetable. Vous pouvez activer cette fonctionnalité et utiliser cette adresse e-mail dans `git config` à la place.

### Comment forker et cloner un dépôt

Ensuite, vous voudrez forker le dépôt. Vous ne serez pas autorisé à pousser des modifications directement vers le dépôt, alors à la place, vous créerez un fork, qui est une copie du projet sous votre contrôle.

![Image](https://www.freecodecamp.org/news/content/images/2021/10/image-10.png align="left")

*Le bouton "Fork" se trouve en haut à droite de la page du dépôt.*

Ensuite, clonez le dépôt localement. Cela télécharge une copie de tout ce qui se trouve dans le dépôt sur votre machine afin que vous puissiez commencer à travailler dessus.

Nous allons faire cela dans le terminal. Si vous êtes sous Windows, vous pouvez utiliser l'invite de commandes ou [PowerShell](https://docs.microsoft.com/en-us/powershell/).

Allez à un emplacement approprié sur votre système de fichiers, comme votre dossier `Téléchargements` ou `Documents`, puis exécutez ce qui suit :

* Clonez le dépôt sur lequel vous souhaitez travailler.
  
* Entrez dans le dossier dans le terminal.
  
* Liez le dépôt avec le fork sur votre compte.
  

```sh
git clone https://github.com/EbookFoundation/free-programming-books.git
cd free-programming-books
git remote add develop https://github.com/{VOTRE_NOM_DUTILISATEUR}/free-programming-books.md
```

### Comment travailler avec le projet

Maintenant que vous avez cloné le dépôt localement, vous pouvez commencer à travailler dessus. Pour la plupart des dépôts, il est préférable de l'ouvrir dans un éditeur de code comme [Visual Studio Code](https://code.visualstudio.com/), mais vous pouvez techniquement travailler avec n'importe quel éditeur de texte, y compris le Bloc-notes.

Ouvrez le dossier contenant le dépôt que vous avez cloné dans votre éditeur. Par défaut, il porte le nom du dépôt, par exemple, j'ai maintenant un dossier nommé `free-programming-books`.

Maintenant, vous pouvez apporter des modifications :

* Si vous n'êtes pas sûr de ce qu'il faut faire, vérifiez s'il y a quelque chose de spécifique dont ils ont besoin en regardant sous l'onglet "Issues" du dépôt.
  
* Alternativement, apportez toute modification *significative* que vous souhaitez.
  

Avec Free Programming Books, il s'agit d'un dépôt pour maintenir une liste de livres électroniques gratuits, donc nous pouvons ajouter plus de ressources.

[DigitalOcean](https://www.digitalocean.com/community/tags/book) a quelques excellents livres dans la section communauté de leur site web, donc nous pouvons simplement en ajouter un ou plusieurs au dépôt. J'ai choisi [How To Code in React.js](https://www.digitalocean.com/community/books/how-to-code-in-react-js-ebook) pour cet exemple.

Je peux aller à la section respective dans le projet qui est le fichier `books/free-programming-books-langs.md`, et ajouter la ressource sous la section `React` du document.

La ligne que j'ai ajoutée ressemble à ceci :

```md
* [How To Code in React.js](https://www.digitalocean.com/community/books/how-to-code-in-react-js-ebook) - Joe Morgan
```

### Comment commiter votre travail

Maintenant que vous avez apporté une modification, vous voudrez réellement commiter et pousser votre travail.

Vous n'avez fait la modification que *localement* jusqu'à présent — maintenant nous voulons envoyer une pull request. Cela signifie essentiellement que vous enverrez une demande au mainteneur du dépôt pour inclure vos modifications dans leur dépôt.

Dans votre terminal, nous allons exécuter ce qui suit, ce qui permettra de :

* Créer une nouvelle branche.
  
* Dire à Git que vous voulez préparer toutes les modifications à être sauvegardées.
  
* Commiter la modification avec un message décrivant ce que vous avez fait.
  
* Pousser les modifications vers votre fork.
  

```sh
git checkout -b added-react-book
git add .
git commit -m "added a react book from digitalocean"
git push develop added-react-book
```

### Comment créer une pull request

GitHub et GitLab laisseront un lien dans la sortie de la commande ci-dessus qui vous permettra d'ouvrir une pull request.

Si vous cliquez sur ce lien (vous devrez peut-être maintenir `CTRL` enfoncé), cela vous mènera à une page où vous pourrez fournir plus d'informations et soumettre votre pull request.

![Image](https://www.freecodecamp.org/news/content/images/2021/10/image-11.png align="left")

Remplissez simplement les détails demandés et cliquez sur "Create pull request". Il ne reste plus qu'à attendre que les mainteneurs la révise.

### Nettoyer le code

Dès que vous avez terminé une pull request, vous devez quitter votre branche et revenir à la branche principale du dépôt.

Exécutez les commandes suivantes dans votre terminal :

* Vérifiez si la branche principale est `main` ou `master`.
  
* Passez à la branche principale du dépôt.
  
* Mettez à jour votre copie locale pour qu'elle soit synchronisée avec le dépôt.
  

```sh
git branch
git checkout main
git pull
```

Recommencez à partir de l'étape "Comment travailler avec le projet" pour apporter plus de modifications.

Bonne chance avec vos pull requests !
