Par Sacha Greif

#### Tout comme Noël ou la grippe, l'enquête State of JavaScript revient chaque année. Mais contrairement à ces événements, c'est quelque chose à laquelle on peut vraiment se réjouir !

#### ➡️ [Répondez à l'enquête maintenant](https://stateofjs.com/)

Si vous n'êtes pas familier, l'enquête couvre tout ce qui concerne JavaScript et tente de répondre à deux questions principales :

* Quels sont les outils émergents qui enthousiasment le plus les développeurs ?
* Quelles sont les technologies éprouvées qui satisfont le plus les développeurs ?

Ces deux questions sont également importantes : vous voulez rester à jour avec les dernières bibliothèques pour ne pas être laissé pour compte, mais vous ne voulez pas non plus miser sur une nouvelle solution non éprouvée.

![Image](https://cdn-media-1.freecodecamp.org/images/1*h3MB-aEgYFTE4ARM5fckGA.png)
_[Résultats 2017 Front-end](https://2017.stateofjs.com/2017/front-end/results" rel="noopener" target="_blank" title=")_

Vous devriez définitivement consulter [les résultats de l'année dernière](https://2017.stateofjs.com/) pour voir les données que nous avons collectées. Avec plus de 20 000 entrées, nous avons réussi à capturer une bonne partie de l'écosystème. Et nous espérons faire encore mieux cette fois-ci !

### Des questions plus granulaires

Le principal changement cette année est que nous avons ajouté une logique supplémentaire à l'enquête : si vous mentionnez avoir utilisé une bibliothèque donnée, nous vous poserons une question de suivi pour savoir ce que vous aimez (ou n'aimez pas) à son sujet :

![Image](https://cdn-media-1.freecodecamp.org/images/1*WDoaJGJB3OqGqJlPw9SBjw.png)

Nous espérons que cela nous aidera à obtenir des données plus granulaires sur _pourquoi_ certaines technologies sont populaires.

### Au revoir CSS

Alors que l'écosystème évolue vers l'utilisation de JavaScript pour tout et que CSS devient un vestige du passé, nous avons décidé que nous pouvions supprimer en toute sécurité la section CSS de l'enquête.

…

Je plaisante bien sûr ! CSS est toujours aussi pertinent pour le développement web, mais nous ne voulions pas rendre l'enquête trop longue, alors nous avons dû faire des choix difficiles. Finalement, nous avons décidé de supprimer toutes les questions liées aux bibliothèques CSS.

Mais qui sait, CSS pourrait un jour avoir sa propre enquête !

### Nouvelles bibliothèques

Et bien sûr, nous avons ajusté les options par défaut proposées comme réponses pour refléter l'état actuel de l'écosystème.

Globalement, il n'y a pas eu trop de grands changements, nous avons principalement supprimé les options les moins populaires de l'année dernière et les avons remplacées par les meilleurs résultats des catégories « autres options » soumises par les utilisateurs pour chaque section.

Parmi les nouvelles entrées de cette année, on trouve [Preact](https://preactjs.com/), [Storybook](https://storybook.js.org/), et [Next.js](https://nextjs.org/), parmi d'autres.

### Les résultats

Les résultats devraient être publiés mi-novembre, et en fait, je donnerai un aperçu à [Dot JS à Paris](https://www.dotjs.io/) le 9 novembre. Venez dire bonjour si vous êtes dans les parages !

Si cela semble être une longue attente, c'est parce que nous voulons nous laisser suffisamment de temps pour examiner les données et trouver la meilleure façon de les présenter. Après tout, nous n'avons qu'une chance par an de bien faire les choses !

Eh bien, je pense que nous avons tout couvert, n'est-ce pas ? Je n'oublie rien, n'est-ce pas… ?

Oh oui !
