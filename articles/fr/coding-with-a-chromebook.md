Les Chromebooks sont géniaux. Ce sont des appareils relativement simples et peu coûteux qui exécutent [Chrome OS](https://en.wikipedia.org/wiki/Chrome_OS), un système d'exploitation basé sur Linux, allégé et développé par Google.

Bien qu'ils soient parfaits pour les personnes qui ont juste besoin d'un navigateur web pour effectuer des tâches de base, si vous vous lancez dans le développement avec uniquement un Chromebook, vous vous demandez peut-être s'il est préférable d'investir dans un PC.

Mais avec toutes les technologies basées sur le cloud et les récentes mises à jour de Chrome OS, vous avez beaucoup d'options. Nous allons passer en revue quelques-unes des solutions populaires ici.

## Solutions basées sur le cloud

Si vous êtes complètement nouveau dans le développement web, il y a bien sûr [freeCodeCamp.org](https://www.freecodecamp.org/). L'ensemble du programme peut être complété entièrement dans le navigateur et en utilisant des outils comme [CodePen](https://codepen.io/), [CodeSandbox](https://codesandbox.io/), [Glitch](https://www.freecodecamp.org/news/p/633c27ad-f2a2-4b7a-a56b-e85620d957dc/glitch.me), et [Repl.it](https://repl.it/) pour des projets plus complexes.

Même les développeurs expérimentés utilisent un ou plusieurs des sites listés ci-dessus pour le prototypage rapide et pour partager facilement leurs projets avec d'autres, tout cela gratuitement. Bien qu'ils puissent être plus lents qu'un environnement de développement natif, le fait que vous puissiez y accéder depuis n'importe quel appareil connecté à Internet et que tout soit sauvegardé sur leurs serveurs est un gros avantage.

## Linux pour Chromebooks

À partir de [Chrome OS v.69](https://9to5google.com/2018/09/18/google-chrome-os-69-stable-release/), vous pouvez activer Linux pour Chromebooks et installer une version bêta de l'interface Linux sur [certains Chromebooks](https://www.chromium.org/chromium-os/chrome-os-systems-supporting-linux). Bien que la liste des appareils pris en charge soit courte, la plupart, sinon tous les futurs Chromebooks, devraient [prendre en charge cette fonctionnalité](https://www.zdnet.com/article/all-chromebooks-will-also-be-linux-laptops-going-forward/).

Ce qui se passe essentiellement, c'est que Chrome OS exécute une version de Debian dans une machine virtuelle. Comme Debian est la base d'Ubuntu, une distribution/operating system Linux populaire, vous devriez pouvoir installer n'importe quoi sur votre Chromebook comme vous le feriez sur une machine Debian/Ubuntu.

Par exemple, si vous voulez installer Firefox, tout ce que vous avez à faire est d'ouvrir le terminal et d'entrer `sudo apt install firefox`.

Il y a cependant quelques inconvénients. Actuellement, cette fonctionnalité est en bêta, et l'accélération matérielle n'est toujours pas prise en charge dans les applications non-Chrome OS.

Des outils comme Firefox ou VSCode fonctionneront plus lentement que sur d'autres machines exécutant une distribution Linux comme Ubuntu nativement. Cela signifie également que le décodage vidéo est un peu plus lent, et la lecture peut en souffrir. De plus, les appareils comme les microphones et les webcams ne sont pas encore pris en charge.

Consultez [cette page](https://support.google.com/chromebook/answer/9145439?hl=en) pour plus d'informations sur ce qui n'est pas encore pris en charge.
