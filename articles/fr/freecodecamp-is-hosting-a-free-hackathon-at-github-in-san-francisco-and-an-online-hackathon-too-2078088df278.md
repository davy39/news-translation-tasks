## Qu'est-ce qu'un hackathon ?

Un hackathon est un événement où les gens rejoignent des équipes et codent ensemble, puis présentent les projets qu'ils ont construits. Ils durent généralement un ou deux jours et impliquent souvent un jury et des prix.

## Qu'est-ce que la JAMstack ?

La JAMstack est une manière simple de construire des applications web. JAM signifie JavaScript, APIs et Markup (HTML et CSS).

Si vous avez construit certains des projets du cursus freeCodeCamp, vous avez déjà utilisé la JAMstack.

La principale différence entre la JAMstack et d'autres piles de développement web comme LAMP et MEAN est la suivante : au lieu d'utiliser un serveur back-end, vous utilisez des APIs.

### Comment un grand événement comme celui-ci dans le centre-ville de San Francisco peut-il être entièrement gratuit ?

La plupart des hackathons ont des sponsors d'API qui souhaitent faire connaître leurs APIs auprès des développeurs. Ils peuvent également avoir des développeurs sur place pour répondre à vos questions sur leur API et vous aider à l'intégrer dans votre application web.

## Comment mettre en ligne votre première application web JAMstack

Ce tutoriel fonctionnera sur Mac, Linux et Windows.

Si vous utilisez Windows, je vous recommande d'utiliser le nouveau [Linux Bash Shell](https://www.howtogeek.com/249966/how-to-install-and-use-the-linux-bash-shell-on-windows-10/).

**Remarque :** lors de la configuration de toutes ces choses sur votre ordinateur, vous pouvez rencontrer des problèmes liés à votre système d'exploitation. Si vous êtes bloqué, lisez [ce fil de forum](https://forum.freecodecamp.org/t/freecodecamp-2018-jamstack-hackathon-at-github-in-san-francisco-on-october-27-28/221604) et si vous êtes toujours bloqué, demandez de l'aide en répondant au fil.

### Étape #1 : Installer Git

Git est un outil puissant de contrôle de version que la plupart des grands projets open source, y compris freeCodeCamp, utilisent pour collaborer sur des logiciels.

Vous pouvez [télécharger et installer Git ici](https://git-scm.com/download).

### Étape #2 : Installer npm, le gestionnaire de paquets de Node

Vous pouvez [télécharger et installer npm ici](https://nodejs.org/en/).

### Étape #3 : Installer VS Code, l'éditeur de code open source de Microsoft

Vous pouvez [télécharger et installer VS Code ici](https://code.visualstudio.com/download). Vous pouvez utiliser n'importe quel éditeur de code, mais celui-ci est un éditeur gratuit populaire.

### Étape #4 : Installer Gatsby.js

Ouvrez votre terminal (qui peut être appelé shell dans votre système d'exploitation) et tapez la commande suivante :

```
sudo npm install --global gatsby-cli
```

Vous devrez entrer votre mot de passe système, qui devrait être le même mot de passe que vous utilisez pour déverrouiller votre ordinateur depuis le mode veille.

### Étape #5 : Créer votre application JAMstack "hello world" en utilisant Gatsby

Exécutez la commande suivante :

```
gatsby new gatsby-site
```

### Étape #6 : Changer de répertoire vers celui de votre application et l'exécuter localement sur votre ordinateur

Exécutez la commande suivante :

```
cd gatsby-site && gatsby develop
```

Ouvrez maintenant un nouvel onglet dans votre navigateur. Tapez localhost:8000 dans la barre d'adresse de votre navigateur. Vous devriez voir quelque chose comme ceci :

![Image](https://cdn-media-1.freecodecamp.org/images/f7ZojazMYldhpR3tlIxNqHRRgeMJXzigkQ-Y)

### Étape #7 : Personnaliser le code de votre projet

Ouvrez votre projet dans VS Code ou l'éditeur de code que vous préférez.

Accédez à la page index.js et personnalisez son HTML.

![Image](https://cdn-media-1.freecodecamp.org/images/pcJaVk6YDZDLasXNKUnDXaHnefQ9w9V4pE36)

### Étape #8 : Valider votre code dans Git

Assurez-vous d'être dans le répertoire de votre projet dans votre terminal et tapez la commande suivante pour initialiser un dépôt Git :

```
git init
```

Ensuite, ajoutez tous les fichiers que Gatsby a créés à l'étape #5 en exécutant cette commande :

```
git add .
```

Et enfin, validez ces fichiers dans Git avec le message de validation "first commit" :

```
git commit --message "first commit"
```

### Étape #9 : Créer un compte GitHub

Vous pouvez [créer un compte GitHub gratuit en quelques minutes ici](https://github.com/join). (Choisissez l'option "Unlimited public repositories").

![Image](https://cdn-media-1.freecodecamp.org/images/vArrSWwVQDBN0Aaaz1AKveYqS8dBIBa4f3h0)

Vérifiez votre boîte de réception pour pouvoir vérifier votre compte GitHub.

### Étape #10 : Ajouter vos clés SSH à GitHub.

Les clés SSH sont un moyen d'accéder en toute sécurité à votre compte GitHub depuis votre terminal sans avoir besoin de vous authentifier avec un nom d'utilisateur et un mot de passe.

Exécutez cette commande dans votre terminal pour générer des clés SSH :

```
ssh-keygen
```

Le terminal vous demandera de choisir un emplacement de fichier. Appuyez simplement sur Entrée pour accepter l'emplacement de fichier par défaut. Si le terminal vous indique que vous avez déjà généré une clé SSH par le passé, appuyez sur "n" pour ne pas écraser votre clé existante.

Ensuite, exécutez cette commande pour copier la clé SSH publique de votre ordinateur dans votre presse-papiers :

```
pbcopy < ~/.ssh/id_rsa.pub
```

Ensuite, [cliquez ici pour accéder à la page des paramètres de clé SSH dans GitHub](https://github.com/settings/ssh/new) et collez-la.

![Image](https://cdn-media-1.freecodecamp.org/images/2u4QW2Kb5mZQDdFNN5y-VD6XWaA6dMrQQJtK)

### Étape #11 : Créer un dépôt sur GitHub

[Suivez ce lien pour créer un dépôt GitHub](https://github.com/new). Vous pouvez le nommer comme vous voulez, et la description n'a pas d'importance pour nos besoins. Tant que vous le rendez public, c'est complètement gratuit.

![Image](https://cdn-media-1.freecodecamp.org/images/OtE3F5i5ZaMZFlA14r74yaCHVbvEnJsH0AZX)

### Étape #12 : Définir votre dépôt GitHub comme origine de votre projet Git local et pousser votre code vers GitHub

Une fois que vous avez terminé l'étape #10, GitHub devrait vous envoyer vers une page qui ressemble à ceci, qui a une option pour pousser un dépôt existant depuis la ligne de commande.

![Image](https://cdn-media-1.freecodecamp.org/images/uIIOtJsacu7Ay6hAyMpSELxU0o2Op2nxSWMe)

Vous pouvez cliquer sur le bouton du presse-papiers pour copier les commandes dans votre presse-papiers. Ensuite, retournez à votre terminal et assurez-vous d'être toujours dans le répertoire de votre projet. Ensuite, collez ces commandes dans votre terminal.

### Étape #13 : Créer un compte Netlify

Suivez [ce lien pour accéder à la page d'inscription de Netlify](https://app.netlify.com/signup). Ensuite, connectez-vous en utilisant le compte GitHub que vous venez de créer.

### Étape #14 : Créer une application Netlify à partir de votre dépôt GitHub

Maintenant, [allez ici](https://app.netlify.com/start/repos) et choisissez GitHub comme votre fournisseur Git.

![Image](https://cdn-media-1.freecodecamp.org/images/6Vp21ezS3a10jjMyoleQ3csVoLQlI6OgvTij)

Vous devriez maintenant voir le dépôt GitHub que vous avez créé il y a quelques minutes, et pouvoir le sélectionner.

La configuration par défaut de Netlify devrait convenir. Vous pouvez donc simplement faire défiler vers le bas et cliquer sur le bouton "deploy site".

![Image](https://cdn-media-1.freecodecamp.org/images/p5SJcspDLwZe6a1byCL0tUCl0P5w3EhuXL7c)

### Étape #15 : Attendre que votre application web soit construite, puis la visiter sur internet et noter son URL publique.

Vous pouvez cliquer sur l'URL générée automatiquement par Netlify pour voir votre projet en direct sur le web.
