Docker est un outil DevOps puissant pour mettre vos applications dans des "conteneurs".

Vous pouvez exécuter ces mêmes conteneurs n'importe où - sur votre ordinateur portable, sur un serveur - même sur un réfrigérateur intelligent si vous le souhaitez.

Et partout où vous exécutez un conteneur, l'environnement - et toutes ses dépendances - sera identique.

Cela rend Docker idéal pour exécuter des applications distribuées.

Ce cours vous enseignera Docker à travers une série de leçons avec des animations et illustrations conceptuelles amusantes.

Et surtout, vous pourrez coder chez vous, directement dans votre navigateur. Le cours inclut un environnement où vous pouvez bricoler avec des conteneurs Docker. Vous n'avez pas besoin de démarrer vos propres serveurs ou même d'installer Linux vous-même.

### Voici un peu plus d'informations sur Docker du Guide freeCodeCamp, au cas où vous ne le connaîtriez pas. Dans tous les cas, vous devriez toujours suivre le cours.

Docker est un outil open-source qui vous donne la capacité de construire des conteneurs pour vos applications.

Il inclut un système d'exploitation, des bibliothèques et tout ce dont vous avez besoin pour exécuter votre application. Ensuite, vous pourrez déployer cette même application exacte - et son environnement entier et ses dépendances - sur n'importe quelle machine.

La structure légère de Docker vous permet d'exécuter plusieurs conteneurs sur la même machine. Une image de conteneur est un package léger, autonome et exécutable d'un logiciel qui inclut tout ce dont vous avez besoin pour l'exécuter.

### **Fonctionnalités principales de Docker**

Les conteneurs Docker exécutés sur une seule machine partagent le noyau du système d'exploitation de cette machine. Ils démarrent instantanément et utilisent moins de puissance de calcul et de RAM.

Les images Docker sont construites à partir de couches de système de fichiers et sont capables de partager des fichiers communs. Cela minimise l'espace que ces images occupent sur le disque et rend ces images beaucoup plus rapides à télécharger.

Les conteneurs Docker sont basés sur des standards ouverts et s'exécutent sur toutes les distributions *nix majeures (Unix, Linux, etc. - y compris MacOS), Microsoft Windows, et sur toute infrastructure incluant les VMs, le bare-metal, et dans le cloud.

Les conteneurs Docker isolent les applications les unes des autres et de l'infrastructure sous-jacente. Docker fournit une isolation par défaut forte, afin que vous puissiez limiter tout problème avec votre application à un seul conteneur au lieu d'une machine entière.

Les images de conteneurs sont des packages logiciels légers, autonomes et exécutables qui incluent tout ce dont vous avez besoin pour les exécuter : code, runtime, outils système, bibliothèques système et paramètres.

Les logiciels conteneurisés s'exécuteront toujours de la même manière, indépendamment de l'environnement.

Les conteneurs isolent le logiciel de son environnement. C'est-à-dire, les différences entre les environnements de développement et de staging.

En fin de compte, Docker aide à réduire les conflits entre les équipes exécutant différents logiciels sur la même infrastructure.

### **Une note sur les Conteneurs Vs. Machines Virtuelles**

* Les conteneurs virtualisent le système d'exploitation et les rendent plus portables
* Les machines virtuelles, en revanche, virtualisent le matériel.
* Les conteneurs sont une abstraction au niveau de la couche application qui regroupe le code et les dépendances ensemble. Les VMs sont une abstraction du matériel physique transformant un serveur en plusieurs.
* Encore une fois, [les conteneurs ne sont pas des VMs](https://blog.docker.com/2016/03/containers-are-not-vms/).
