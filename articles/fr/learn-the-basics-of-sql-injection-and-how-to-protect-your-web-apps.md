L'injection SQL est une technique de piratage courante utilisée pour récupérer ou détruire des données d'une base de données sans permission. Elle est considérée comme l'un des principaux risques de sécurité des applications web, il est donc important de comprendre comment s'en défendre.

Nous avons publié un cours sur la chaîne YouTube freeCodeCamp.org qui vous apprendra comment fonctionne l'injection SQL et comment vous assurer que vos applications ne sont pas vulnérables à cette attaque.

Sagar Bansal a développé ce cours. Sagar a créé de nombreux cours de sécurité à succès et a travaillé dans l'industrie pendant de nombreuses années.

Pour comprendre l'injection SQL, vous devez d'abord comprendre SQL. Ce cours commence par couvrir les bases de SQL, puis donne un aperçu des différents types d'injection SQL.

Vous apprendrez les techniques d'attaque courantes ainsi que les outils souvent utilisés pour les attaques par injection SQL. Vous apprendrez également les meilleures pratiques de l'industrie pour vous défendre contre ces attaques.

Voici les sections couvertes dans ce cours :

* Qu'est-ce que l'injection SQL
* Installation du laboratoire
* Bases de SQL
* Contournement d'injection classique
* Types d'injection SQL
* Injection SQL basée sur UNION
* Injection SQL basée sur les erreurs
* Injection SQL basée sur les booléens
* Injection SQL basée sur le temps
* Injection SQL semi-automatisée
* Injection SQL entièrement automatisée
* Se défendre contre les injections SQL
