Partager des liens sans image sociale peut transformer un contenu puissant en échec. Comment pouvons-nous tirer parti de l'espace que les réseaux sociaux nous offrent lorsque nous partageons notre travail acharné sur Github ?

Vous voulez passer directement au "comment" ? [Passez au "comment"](#heading-ajouter-une-image-a-votre-depot-github) !

%[https://www.youtube.com/watch?v=KRNiqCLlPNQ]

## Impact des images dans les réseaux sociaux

Tout flux de réseau social est un tourbillon de contenu qui peut être difficile à assimiler complètement. En balayant ou en faisant défiler les flux, vous serez confronté à du contenu basé sur du texte et à une tonne de médias.

![Image](https://www.freecodecamp.org/news/content/images/2020/02/twitter-feed-media-1.jpg)
_Flux Twitter avec médias sur TweetDeck_

Et c'est pour une bonne raison ! En parcourant l'exemple ci-dessus, qu'est-ce qui se démarque ? Pas le [retweet de Dust Particle](https://twitter.com/uiswarup/status/1230781530907324417) de [CodePen](https://twitter.com/CodePen) à gauche, mais l'AWS au milieu, le museau de chien en haut à gauche, et Al Pacino en haut à droite parmi les autres publications multimédias.

Les images, les gifs et les vidéos sont un bon moyen de fournir un contenu accrocheur. En prime, ils occupent généralement beaucoup plus d'espace dans le flux, ce qui augmente encore les chances que vous ayez l'occasion de les voir.

## Le minimum sur Github

Lorsque vous partagez vos projets Github sur les réseaux sociaux, l'option par défaut est assez terne. Prenez ce tweet comme exemple :

%[https://twitter.com/colbyfayock/status/1226204581824204804]

Github nous offre une simple [carte de réseau social](https://developer.twitter.com/en/docs/tweets/optimize-with-cards/overview/abouts-cards) qui se compose de notre avatar Github, du chemin de notre projet et de la courte description en haut de la page.

Maintenant, c'est bien qu'ils montrent une carte pour commencer et, soyons justes, que pourraient-ils montrer ? Mais ce tweet va être facilement ignoré dans les flux des gens entre les grandes images brillantes.

## Images de réseaux sociaux Github

Heureusement, Github nous offre un moyen simple et facile d'ajouter une image à chaque dépôt qui peut aider le contenu que nous partageons à mieux se vendre.

Avec un peu de travail créatif, nous pouvons télécharger une image qui tirera parti de l'espace dans les flux des gens et attirera l'attention que votre travail mérite.

![Image](https://www.freecodecamp.org/news/content/images/2020/02/canva-social-media-example-2.jpg)
_Exemple d'une image sociale utilisant Canva_

Avant de plonger, il convient de noter que vous devez avoir l'accès approprié pour pouvoir modifier les paramètres du dépôt afin de changer l'image. Si vous avez créé le dépôt, il y a de fortes chances que vous ayez cet accès.

## Trouver ou créer une image

Avant de télécharger une image, nous avons besoin d'une image en premier lieu. Vous pouvez choisir l'une des deux options : trouver une image ou en créer une nouvelle.

Si vous voulez opter pour la solution simple, vous pouvez chercher des images libres de droits qui sont assez faciles à trouver sur le web. Un de mes préférés est [Unsplash](https://unsplash.com/), car vous y trouverez généralement des images de haute qualité, mais il existe une tonne d'autres options que vous pouvez trouver avec [une simple recherche](https://www.google.com/search?q=free+stock+photos). Cependant, l'ajout de photos est généralement mieux adapté aux articles de blog et aux publications basées sur du contenu.

%[https://twitter.com/freeCodeCamp/status/1230190815605121024]

La meilleure option est de créer la vôtre. L'avantage de créer une image est que vous pouvez la personnaliser avec du texte en gros caractères pour attirer un peu plus d'attention dans le flux.

Il existe [une tonne d'outils gratuits](https://zapier.com/blog/graphic-design-tools-for-social-media-images/) qui vous permettent de créer une image en commençant par un modèle spécifique aux réseaux sociaux, vous n'avez donc pas besoin de dépenser de l'argent pour [Photoshop](https://www.adobe.com/products/photoshopfamily.html) pour y parvenir.

La seule exigence pour votre image est que sa taille doit être d'au moins 640×320px. Si possible, visez au moins 1280×640px pour vous assurer que l'image s'affiche en haute résolution dans les flux.

## Ajouter une image à votre dépôt Github

Une fois que nous avons notre image, il ne nous reste plus qu'à ajouter l'image à notre dépôt, ce qui est probablement la partie la plus simple.

Tout d'abord, accédez aux Paramètres de votre dépôt en utilisant la navigation par onglets en haut de la page. Pour rappel, vous devez être en mesure de modifier les paramètres pour changer l'image.

![Image](https://www.freecodecamp.org/news/content/images/2020/02/github-settings-page-1.jpg)
_Page des paramètres Github_

Ensuite, faites défiler vers le bas jusqu'à la section Aperçu social, où, si vous n'avez pas actuellement d'image définie, vous trouverez un grand rectangle vide avec un bouton d'édition dans le coin inférieur gauche.

![Image](https://www.freecodecamp.org/news/content/images/2020/02/github-upload-social-image.jpg)
_Télécharger une image sociale sur Github_

Cliquez sur le bouton Modifier, sélectionnez Télécharger une image, puis trouvez votre image sur votre ordinateur et sélectionnez le fichier.

Une fois sélectionné, votre fichier image sera téléchargé sur Github et défini comme votre image sociale !

![Image](https://www.freecodecamp.org/news/content/images/2020/02/github-social-media-image.jpg)
_Aperçu social sur Github_

## Prévisualisation de votre nouvelle image

Lorsque vous avez terminé de télécharger votre image, vous pouvez vous rendre sur votre plateforme sociale préférée et l'essayer. Par exemple, publier sur Twitter affichera désormais une grande image au lieu de votre petit avatar Github !

%[https://twitter.com/colbyfayock/status/1230604806324183046]

Comme vous pouvez le voir, ce tweet maximise l'espace disponible et fait passer le message sur ce dont il s'agit le projet.

## Quelques conseils lors de la création d'images

### Texte grand et simple

Il peut être facile de se laisser emporter par la surconception d'un graphique ou de ne pas prêter suffisamment attention à la taille de la police. Assurez-vous d'utiliser de grandes lettres avec une police facile à lire, afin que lorsque quelqu'un fait défiler votre publication, il puisse la comprendre et ne pas simplement la sauter.

### Ajoutez un logo

Avez-vous un logo pour votre projet ? Ou est-ce un plugin pour un outil spécifique ? Essayez d'ajouter un logo pour donner immédiatement un contexte sur ce dont il s'agit le projet !

### Maximisez la résolution

Personne n'aime regarder des images de mauvaise qualité. Tirez parti de tous ces pixels précieux et téléchargez une image qui se traduira par une haute qualité dans un flux. Github recommande particulièrement une taille d'image d'au moins 1280×640px.
