Par Anchal Nigam

Comme nous le savons tous, Angular est le mot le plus discuté dans le monde du développement frontend. Il a gagné beaucoup de popularité ces dernières années. Angular est devenu le choix de nombreux développeurs frontend et je fais partie d'entre eux.

Dans cet article, je vais mettre en lumière l'arrivée d'Angular et sa croissance rapide sur le marché. Tous les six mois, l'équipe Angular sort une nouvelle version d'Angular, le rendant plus puissant avec de nouvelles fonctionnalités. Je dois apprécier l'équipe Angular pour leurs efforts merveilleux.

Maintenant, vous devez être excité de savoir comment il a évolué, quelles nouvelles fonctionnalités ont été ajoutées dans les différentes versions. Maintenant, l'attente est terminée. Commençons !

Avant de continuer, je tiens à informer mes lecteurs que 'Angular 8' est la dernière version publiée par notre talentueuse équipe Angular.

### **Naissance du Héros (AngularJS) — 2010**

En 2010, la naissance du héros du marché actuel a eu lieu. Il était connu sous le nom d'AngularJS.

AngularJS est un framework JavaScript développé par Google. Il est utilisé pour créer des applications monopages (SPA). Attendez ! J'espère que vous savez ce qu'est une SPA. Si ce n'est pas le cas, demandez à Google et revenez, car c'est l'une des belles approches qu'Angular utilise, mais voici une brève introduction à la SPA (ceux qui savent peuvent sauter) :

SPA ou Single Page Application est une application dans laquelle tout est téléchargé en une seule fois. Tout le code nécessaire est téléchargé en une seule fois. Contrairement aux applications multipages, vous n'avez pas à demander une page web au serveur pour chaque requête et charger la page. Dans une SPA, le fichier index.html est téléchargé une fois avec tout le contenu, puis pour chaque changement d'URL, la page web existante charge dynamiquement le contenu demandé. Cette approche offre une meilleure expérience utilisateur lors du passage entre différentes pages et vous donne l'impression d'une application.

C'est tout pour la SPA. J'espère que cela vous donne une brève image. Revenons au sujet principal.

AngularJS permet aux développeurs de développer des applications web plus rapidement. Il utilise une technique de rendu côté client dans laquelle le rendu du contenu est pris en charge par le client (navigateur). Je ne vais pas entrer dans les détails des avantages et inconvénients de cette technique, mais le principal problème de cette technique est le SEO. Ce rendu utilisé par Angular est mauvais pour le SEO. Attendez ! Ne pensez pas qu'Angular est mauvais. Il y a un dialogue très populaire de Bollywood — « Picture abhi baki hai mere dost (Le film n'est pas encore terminé, mon ami) ». Je pense que ce dialogue convient à la situation actuelle. Je vais discuter pourquoi j'ai dit cela.

Maintenant, nous devons nous concentrer sur les fonctionnalités d'AngularJS. En voici quelques-unes :

1) **Data binding** — synchronisation automatique entre le modèle et la vue.

2) **Système d'injection de dépendances** — un modèle de conception dans lequel le système fournit les objets dépendants lorsqu'il crée l'objet.

3) **scope** — qui prend en charge le contrôleur et la vue.

4) **Services** — pour partager des informations entre différentes parties de l'application.

5) **Directives** — qui donnent des super pouvoirs au HTML. Par exemple, ng-model, ng-app.

6) **Contrôleurs** — le cœur de l'application où réside la logique.

7) **Template** — la vue qui donne des informations en utilisant notre contrôleur et notre modèle.

Cette liste est longue. Je ne vais pas entrer dans les détails d'AngularJS, mais dans AngularJS, les contrôleurs sont le cœur de l'application. AngularJS est venu avec de nombreuses fonctionnalités pour développer une application web puissante, mais a échoué à certains points comme la taille importante du bundle, les problèmes de performance, les problèmes de SEO, les problèmes de maintenabilité du code, mais cela ne signifie pas que c'est un échec total. Le data binding, les concepts d'injection de dépendances sont des succès d'Angular. Par conséquent, nous pouvons dire que c'est un demi-échec et un demi-succès.

Les imperfections d'AngularJS ont poussé l'équipe Angular à réécrire complètement le framework à partir de zéro. Un changement aussi important dans une nouvelle version d'un framework/bibliothèque n'avait jamais été rencontré sur le marché. La nouvelle version d'Angular est totalement différente d'AngularJS. Comment est-elle différente ? Comprenons-le en plongeant dans notre prochain sous-sujet.

### Angular 2 — 2016

La version ultérieure d'AngularJS est arrivée sur le marché en 2016. Elle n'était plus connue sous le nom d'AngularJS. Elle a obtenu le nom 'Angular'. La version 1.x d'Angular est connue sous le nom d'AngularJS. La version ultérieure après 2.x est connue sous le nom d'Angular. Lorsque j'ai rencontré ces deux mots dans mon parcours de développement web, j'ai littéralement pensé que c'étaient deux frameworks différents et après une recherche, j'ai découvert qu'Angular était la version mise à jour d'AngularJS. J'espère que mes lecteurs n'auront pas mal compris.

Angular 2 est arrivé sur le marché avec des changements drastiques. Le plus grand changement était l'introduction de TypeScript. TypeScript est un sur-ensemble de JavaScript avec des fonctionnalités supplémentaires comme le fait qu'il suit les concepts OOP, fortement typé. Les programmeurs venant du monde orienté objet le trouvent plus familier contrairement à JavaScript.

Les composants sont le cœur du monde Angular 2+. Angular a introduit divers packages pour réaliser des fonctionnalités de base et importantes comme le package de routage pour définir facilement les routes, le package http pour récupérer les données du serveur, le package d'animation pour les animations, etc.

Angular 2 propose également le concept de directives comme AngularJS. Les directives donnent des super pouvoirs à notre HTML comme *ngFor, *ngIf (directive structurelle) rendent notre HTML dynamique tandis que les directives d'attribut comme ngModel (pour le data binding bidirectionnel), ngStyle prennent en charge l'apparence et le comportement de notre DOM.

Une autre fonctionnalité qui n'a pas changé dans Angular est qu'il utilise également le système DI. Comme AngularJS, le système DI fournit les objets dépendants (dépendances) au composant.

Attendez ! J'ai oublié de vous parler de la merveilleuse aide qu'Angular fournit, à savoir angular-cli. C'est une aide précieuse qui nous aide à développer notre application plus rapidement, par exemple pour générer un composant, utilisez simplement 'ng g c nom-du-composant' ('g' est pour générer et 'c' est pour composant). C'est une autre fonctionnalité cool ajoutée dans Angular.

Ce n'est pas la fin. Je vous ai dit que le plus grand inconvénient d'AngularJS est qu'il est mauvais pour le SEO et souvenez-vous de ce célèbre dialogue que j'ai mentionné. Vous comprendrez dans quelques minutes pourquoi je l'ai utilisé.

C'est le plus gros inconvénient qu'AngularJS est mauvais pour le SEO. Lorsque vous regardez le code source de la page, vous pouvez voir qu'il n'y a rien (pas de HTML) ce qui fait que les crawlers pensent que le site web est inutile, c'est-à-dire sans aucune information. C'est très douloureux lorsque votre site web veut que les crawlers de Google les indexent et les fassent atteindre le sommet dans la recherche Google mais ne peut pas y parvenir. L'équipe Angular s'en est occupée et a introduit Angular Universal qui prend en charge le SEO de votre site web Angular. Il utilise le rendu côté serveur qui, à son tour, résout le problème de SEO.

Ce sont les fonctionnalités avec lesquelles Angular 2 est arrivé sur le marché.

### **Angular 4 — Mars 2017**

Après avoir lu ce sous-sujet, vous vous demandez peut-être pourquoi pas Angular 3 ? C'est une question courante qui peut venir à l'esprit de n'importe qui. Ne vous inquiétez pas ! Je vais vous le faire savoir. La raison pour laquelle il n'y a pas eu d'Angular 3 est due au package de routage. Le package de routage Angular était déjà distribué en version 3. Pour éviter la confusion, l'équipe Angular a publié la version 4 d'Angular. Je pense que maintenant mes lecteurs peuvent passer à la compréhension des fonctionnalités d'Angular 4.

Angular 4 est arrivé avec des corrections de bugs et d'autres nouvelles fonctionnalités et améliorations. La plus grande amélioration qui a été faite était dans la taille du bundle. Ils ont réduit la taille du bundle de 60 %, ce qui a rendu l'application plus légère et donc le temps de chargement de l'application a diminué.

Une autre chose qui a été faite était dans le package d'animation. Ils ont extrait la fonctionnalité d'animation dans un package séparé : @angular/animations.

Une autre amélioration était dans la directive structurelle. *ngIf est venu avec une partie else dans cette version.

C'est la brève introduction des fonctionnalités d'Angular 4.

### **Angular 5 — Novembre 2017**

Après six mois, l'équipe Angular est venue avec une autre nouvelle version, c'est-à-dire Angular 5. Cette version d'Angular 5 est à nouveau venue avec beaucoup de nouvelles fonctionnalités, améliorations et corrections de bugs.

Comme la principale préoccupation de chaque site web est le temps de chargement plus rapide. Angular en a tenu compte dans cette version également. Pour améliorer davantage les performances de l'application, ils ont introduit l'optimiseur de build — c'est un outil qui a été introduit pour réduire la taille du bundle. Il utilise la technique de tree shaking pour supprimer le code mort de l'application.

Des améliorations du compilateur ont également été faites, ce qui permet une reconstruction plus rapide de l'application.

Une autre fonctionnalité qui a été introduite était la clé de transfert d'état (TransferStateKey qui fait partie du package platform/browser). Vous vous demandez peut-être ce que c'est ? Quand l'utiliser ? Vous pouvez profiter de la beauté de cette fonctionnalité dans votre application si vous utilisez le SSR. Oui, si vous avez implémenté le SSR, alors vous devez utiliser la fonctionnalité de clé de transfert d'état. La raison pour laquelle je dis cela est que lorsque vous utilisez le SSR et que votre application fait une requête HTTP (ce qui est assez courant), cette requête va être invoquée deux fois, c'est-à-dire une fois sur le serveur et une autre sur le navigateur. Cela provoque un problème de scintillement (j'ai rencontré ce problème de scintillement dans mon application parce que la requête HTTP était invoquée deux fois). Grâce à la fonctionnalité de clé de transfert d'état. Cette fonctionnalité permet au navigateur d'utiliser la réponse de la requête HTTP qui est envoyée au serveur. Elle utilise la réponse que le serveur obtient de la requête HTTP. Comme son nom l'indique, le serveur transfère l'état de la réponse avec le HTML au navigateur. Ainsi, les deux requêtes HTTP peuvent être évitées.

Une autre amélioration qui a été faite était dans le package client HTTP. Dans cette version, HTTPClientModule est venu avec des améliorations comme l'utilisation de ce module, les développeurs n'ont pas à analyser la réponse en utilisant une carte. L'étape d'analyse n'est plus nécessaire. Supposons qu'il y ait une réponse de type non-JSON, alors vous pouvez spécifier cette réponse en utilisant responseType dans votre requête HTTP.

C'est tout pour les fonctionnalités d'Angular 5. Maintenant, le moment est venu de passer à la version Angular 6. Après six autres mois, Angular 6 est arrivé sur le marché avec plus de puissance.

### **Angular 6 — Mai 2018**

En mai 2018, juste après six mois, l'équipe Angular a publié une autre version d'Angular — Angular 6. Cette version est également venue avec beaucoup de nouvelles fonctionnalités. Je vais en lister quelques-unes.

Dans cette version, angular CLI a été mis à jour. De nouvelles commandes ont été introduites comme ng update. Pour mettre à jour les dépendances de votre projet Angular vers les dernières versions, vous pouvez l'utiliser. Par exemple :

> _ng update @angular/core_

Une autre amélioration qui a été faite était dans la bibliothèque RxJS appelée RxJS6. Les deux changements importants étaient :

1. RxJS6 a introduit une nouvelle structure de package interne.
2. L'utilisation des opérateurs.

La nouvelle structure de package interne implique des changements dans la façon d'importer les packages. Au lieu d'importer des packages liés, nous pouvons utiliser une seule importation dans celle-ci. Par exemple :

> _import {Observable} from 'rxjs/Observable';
import {Subject} from 'rxjs/Subject';
import 'rxjs/add/operator/map';_

Maintenant, avec rxjs6 :

> _import {Observable, Subject} from 'rxjs';
import {map} from 'rxjs/operators';_

L'utilisation des opérateurs a également changé dans Angular 6. Par exemple :

Ancienne version :

> _import 'rxjs/add/operator/map';
this.http.get(url).map((response)=>response.json)_

Nouvelle version :

> _import {map} from 'rxjs/operators';
this.http.get(url).pipe(map((data)=>data*2)_

J'espère que vous avez compris les changements qui ont été faits dans la bibliothèque RxJS.

Un autre changement est — angular-cli.json est remplacé par angular.json. Ce fichier définit la configuration du projet comme les styles, les scripts, les tests, le processus de build, etc. Dans angular.json, plus d'options de configuration sont ajoutées comme la configuration de plusieurs projets peut être faite.

Les autres améliorations sont — <ng-template> est maintenant disponible au lieu de <template>. Il y a un changement dans la façon de rendre les services disponibles pour l'utilisation, comme dans la version précédente, si nous voulons rendre un service disponible dans toute l'application ou dans un composant spécifique, nous devons le fournir dans le tableau des fournisseurs, mais dans cette version, dans le fichier de service lui-même, il y a une métadonnée 'providedIn' qui est disponible pour cela. Vous pouvez spécifier là la disponibilité des services. Par défaut, il rend le service disponible au niveau racine.

Une autre beauté qui a été ajoutée est un package 'angular element'. Ce package permet aux développeurs d'utiliser votre composant Angular dans un autre environnement (non-Angular) comme Vue.js. C'est une autre fonctionnalité intéressante qui vous permet d'utiliser votre composant Angular dans un autre environnement.

C'est tout pour les fonctionnalités d'Angular 6 que j'ai apprises. Maintenant, je vais discuter des fonctionnalités de la dernière version — Angular 7.

### **Angular 7 — Octobre 2018**

En octobre 2018, une autre version avec plus de beauté est arrivée sur le marché.

Les fonctionnalités qui ont été ajoutées étaient les invites CLI, le défilement virtuel, le glisser-déposer et à nouveau la réduction de la taille du bundle. Dans les invites CLI, angular-cli vous demande des options comme lorsque vous créez une nouvelle application en utilisant ng new nom-de-l'application. CLI vous demande si vous voulez ajouter un fichier de routage ou non, etc. Il y a également une propriété de budget ajoutée dans angular.json dans laquelle vous pouvez spécifier vos valeurs de budget maximum et minimum.

C'est tout pour les fonctionnalités d'Angular 7. Je sais que je n'ai pas mentionné toutes les autres fonctionnalités d'Angular 7 parce que je ne suis pas allé dans ces fonctionnalités. Par conséquent, je ne suis pas à l'aise pour en parler. Nous savons également que la version Angular 8 est sortie, mais pour la même raison de ne pas être allé dans les fonctionnalités d'Angular 8, je n'ai pas écrit à ce sujet.
