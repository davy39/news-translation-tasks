Choisir une carrière dans la technologie peut être délicat car il existe de nombreux chemins possibles. Il y a le développement web, le machine learning, la science des données, le DevOps, et bien d'autres.

Mais la bonne nouvelle est que vous n'avez pas nécessairement besoin d'être un **développeur** pour travailler dans le monde de la technologie.

Cet article est un guide pour les personnes qui ont du mal avec la transition vers la technologie. Il expliquera diverses carrières dans la technologie que vous pouvez envisager. Nous discuterons également de la manière de savoir si un chemin particulier vous convient ou correspond à votre personnalité.

Cela est basé sur mes opinions et mon expérience, mais j'espère que mes perspectives seront utiles.

# Carrières possibles dans la technologie

## Conception UI/UX

![Illustration UI/UX](https://www.freecodecamp.org/news/content/images/2022/07/Whats-the-Difference-Between-them.png align="left")

*Illustration UI/UX*

Si vous aimez l'esthétique et le style, alors la conception UI/UX pourrait être votre carrière.

### Qu'est-ce que la conception UI/UX ?

**UI** signifie Interface Utilisateur et fait référence aux écrans, pages et icônes visuelles avec lesquelles les utilisateurs interagissent sur une page web ou dans une application (boutons, icônes, etc.). Une UI bien conçue aide les non-techniciens et tous les utilisateurs finaux à interagir avec un produit ou un service en ligne.

**UX** signifie Expérience Utilisateur et implique d'aider un produit ou un service à améliorer son service client. Une bonne UX peut également aider à développer des followers pour la marque.

En tant que développeur UI/UX, votre travail consiste à concevoir et construire l'expérience utilisateur pour une marque ou une entreprise.

Un développeur UI/UX doit connaître et comprendre comment utiliser les éléments suivants :

* [**Wireframing**](https://www.freecodecamp.org/news/designing-a-website-ui-with-prototyping/) : Un wireframe est une mise en page de page web dépouillée de design visuel. Vous l'utilisez pour prioriser les éléments de la page en fonction des besoins des utilisateurs.
    
* [**Prototypage**](https://www.freecodecamp.org/news/designing-a-website-ui-with-prototyping/) : Un prototype est un échantillon ou une simulation d'un produit final utilisé pour tester et recueillir des commentaires. Les prototypes de faible fidélité peuvent être esquissés sur papier et ne permettent pas l'interaction de l'utilisateur. Les prototypes de haute fidélité sont généralement basés sur ordinateur et permettent l'interaction avec la souris et le clavier.
    
* **Maquette** : Une maquette permet aux concepteurs d'avoir une idée d'un modèle visuel réaliste d'une page web ou d'une application finale.
    
* **Flux utilisateur** : Un flux utilisateur est un moyen de cartographier chaque étape de l'utilisateur lors de l'utilisation d'un produit ou d'un service. Cela aide à améliorer l'expérience utilisateur.
    

### Quels outils dois-je connaître pour devenir développeur UI/UX ?

Si vous travaillez en tant que développeur UI/UX, vous utiliserez probablement des outils comme :

* [Adobe XD](https://www.adobe.com/products/xd.html)
    
* [Figma](https://www.figma.com/)
    
* [Photoshop](https://www.adobe.com/products/photoshopfamily.html)
    
* [Sketch](https://www.sketch.com/)
    

Il existe de nombreux outils, mais ceux-ci sont les plus populaires.

Un développeur UI/UX conçoit des sites web et des applications web et aide à améliorer l'expérience utilisateur avec les connaissances nécessaires mentionnées ci-dessus. Vous pouvez trouver de l'inspiration sur Behance ou Dribbble si vous souhaitez suivre ce chemin.

## Développement Frontend

![Illustration des outils Frontend](https://www.freecodecamp.org/news/content/images/2022/07/frontend-development-tools.png align="left")

*Illustration des outils Frontend*

Si vous aimez créer des sites web, vous devriez envisager le développement frontend.

Un développeur frontend s'assure qu'un visiteur peut facilement interagir avec une page web. Ils utilisent des outils de conception, divers frameworks et bibliothèques, et du code pour construire des sites web.

Les développeurs frontend sont également des développeurs côté client car ils construisent directement les éléments sur les sites web et les applications avec lesquels les utilisateurs et les clients interagissent.

Les principales technologies web utilisées par les développeurs frontend sont :

* HTML (HyperText Markup Language) :
    

**HTML** vous permet de construire le squelette des pages web et des sites web. Le langage de balisage définit le web. Vous pouvez le considérer comme la structure d'une maison car il contient tout le texte et définit chaque en-tête sur un site web.

* CSS (Cascading Style Sheets) :
    

**CSS** est responsable de rendre le site web beau. Vous pouvez le considérer comme la décoration d'une maison. Vous l'utilisez pour donner à vos pages de la couleur, des animations, et plus encore après que le HTML a affiché le texte.

* JavaScript :
    

**JavaScript** gère la réactivité du site web. Il donne vie à un site web, aide à le rendre fabuleux sur chaque appareil, et le rend plus interactif.

Vous aurez besoin de connaissances de base dans ces langages pour construire un site web unique et interactif avec succès. Connaître seulement une des compétences techniques mentionnées ne sera pas suffisant pour construire des applications de qualité de production car vous ne pourrez créer qu'un site web à moitié utile.

Parfois, les développeurs frontend travaillent main dans la main avec les développeurs UI/UX.

### Quels outils dois-je connaître pour devenir développeur Frontend ?

#### Un éditeur de code ou un IDE (Environnement de Développement Intégré)

Un éditeur de code/IDE est l'endroit où vous écrirez votre code, le déboguerez et prévisualiserez le code écrit.

Il existe de nombreux éditeurs/IDE parmi lesquels choisir, mais les plus populaires sont [Visual Studio Code](https://code.visualstudio.com/), [Atom](https://atom.io/), et [Sublime text editor](https://www.sublimetext.com/).

#### [Outils de développement Chrome](https://developer.chrome.com/docs/devtools/)

Vous utilisez les outils de développement Chrome pour déboguer des applications en temps réel sur Chrome. Ils vous donnent une compréhension et un accès aux informations internes importantes d'une application web. Les outils de développement aident également à comprendre comment optimiser le flux de chargement et ce que le navigateur fait maintenant.

#### [Git et GitHub](https://github.com/) pour le contrôle de version

Je sais ce que vous pensez – [qu'est-ce qu'un système de contrôle de version](https://www.freecodecamp.org/news/git-and-github-for-beginners/) ? C'est un système qui vous aide à réviser votre code en étapes et à suivre vos révisions (et celles des autres). Git est un système de contrôle de version, et GitHub est un service d'hébergement en ligne pour les dépôts Git.

Disons que vous travaillez sur une page de destination et que vous avez tout gâché dans le pied de page, et dans la base de code, il semble difficile à déboguer. Vous pouvez rapidement revenir au code précédemment validé sur votre dépôt et voir votre dernier changement. Cependant, vous ne pouvez voir votre validation précédente que lorsque vous validez dans votre dépôt.

GitHub aide les membres de l'équipe à travailler ensemble plus efficacement, et vous pouvez l'utiliser pour contribuer à des projets open source.

## Développement Backend

![Illustration des outils Backend](https://www.freecodecamp.org/news/content/images/2022/07/pexels-photo-276452-1.jpg align="left")

*Illustration des outils Backend*

Si vous aimez travailler avec et construire des choses qui dépendent de vous, vous devriez être un développeur backend.

Un développeur backend construit et maintient les technologies ou logiciels nécessaires pour alimenter les composants de développement côté client. Ils sont également appelés développeurs côté serveur.

Leur travail inclut :

* Construire/maintenir des bases de données
    
* Travailler avec des données et l'intégration d'applications
    
* Travailler avec des API
    
* Construire la logique principale de l'application
    

### Quels outils dois-je connaître pour devenir développeur Backend ?

Les développeurs backend utilisent presque les mêmes outils que les développeurs Frontend. Voici ce que vous trouverez utile :

* Éditeur de code/IDE : Cela est utilisé pour écrire du code, déboguer et prévisualiser les bases de code.
    
* [Outils de développement Chrome](https://developer.chrome.com/docs/devtools/) : Cela vous aide à déboguer votre code.
    
* [Git et GitHub](https://github.com/) : Cela vous aide à réviser votre code en étapes et facilite la construction d'équipe, la collaboration et la contribution open source. Tout développeur peut contribuer à votre projet si le dépôt est public.
    
* Serveur SQL ou NoSQL : SQL signifie Structured Query Language. Cela vous aide à interroger les bases de données et à interagir avec elles lors de la construction d'une application.
    
* Connaissance d'une base de données

## Analyse de données et Science des données

![Illustration des compétences en Science/Analyse des données](https://www.freecodecamp.org/news/content/images/2022/07/pexels-photo-265087.jpg align="left")

*Illustration des compétences en Science/Analyse des données*

Envisagez une carrière d'analyste de données si vous aimez le calcul, les mathématiques ou les statistiques.

L'analyse de données implique l'examen, la transformation et l'arrangement d'un ensemble de données donné de manière spécifique pour étudier ses parties et extraire des informations utiles. Elle traite de l'interprétation, de l'analyse et de la visualisation des données.

En termes profanes, l'analyse de données implique la collecte d'informations à partir de données pour aider les décisions commerciales.

La science des données traite les données à l'aide de l'apprentissage automatique, des algorithmes et des statistiques. C'est la partie statistique du domaine des données. Les scientifiques des données collectent, nettoient et interprètent des ensembles de données pour les enquêtes. Ainsi, la science des données implique l'analyse statistique des données.

Il est toujours préférable de commencer en tant qu'analyste de données puis de passer à la science des données car l'analyse de données est la base de la science des données. Bien qu'elles puissent être sur le même chemin, il y a une fine ligne entre la science des données et l'analyse de données.

Les données sont présentes dans toutes les industries, donc les scientifiques et analystes de données sont nécessaires partout.

#### Compétences techniques essentielles à avoir pour la Science des données :

* [MySQL](https://www.mysql.com/)
    
* [Langage de programmation Python](https://www.python.org/)
    
* Mathématiques
    
* Statistiques
    
* [Langage de programmation R](https://www.r-project.org/about.html)
    

### Outils que vous utiliseriez dans ce domaine :

* [Google Spreadsheets](https://www.google.com/sheets/about/)

## Gestion de produit

![Illustration des compétences en gestion de produit](https://www.freecodecamp.org/news/content/images/2022/07/pexels-photo-416405.jpeg align="left")

*Illustration des compétences en gestion de produit*

C'est pour vous si vous êtes doué pour organiser des produits et des équipes.

Un gestionnaire de produit gère et coordonne les projets de développement de produits. Le gestionnaire de produit est la voix du client au sein de l'équipe.

Ils s'assurent que le produit construit contient les fonctionnalités et exigences nécessaires pour satisfaire le client. Ils sont également responsables de la rédaction d'une feuille de route pour ce produit, de l'imagination à la mise sur le marché. Et ce sont eux qui décident quelles fonctionnalités inclure dans la v1 ou la V2.

Leurs responsabilités incluent :

* Comprendre les besoins des utilisateurs,
    
* Rédiger des feuilles de route pour le produit en cours de construction.
    
* Définir une vision pour les produits
    
* Développer des analyses concurrentielles
    
* Prioriser les fonctionnalités du produit
    

Et bien plus encore.

### Outils courants utilisés par les gestionnaires de produit :

* Il existe de nombreux outils pour le suivi et l'analyse des utilisateurs tels que [Pendo](https://www.pendo.io/) ou [Amplitude](https://amplitude.com/). Ces applications donnent un aperçu de l'activité des utilisateurs sur le logiciel ou le site web.
    
* [Product Plan](https://www.productplan.com/) est un logiciel de cartographie pour rédiger et gérer le plan de produit. Ils peuvent également gérer les problèmes de contrôle de version qui peuvent ralentir la progression de votre produit.
    
* Outils de sondage tels que [Typeform](https://www.typeform.com/surveys/) sont très pratiques. Les PM les utilisent pour réaliser des sondages clients ou utilisateurs afin d'aider à améliorer le produit selon les besoins des utilisateurs. Un gestionnaire de produit peut facilement suivre et analyser les résultats à l'aide de ces outils.
    
* Outils de gestion des fonctionnalités tels que [Split.io](https://www.split.io/) aident les équipes à activer rapidement des fonctionnalités spécifiques lorsque les utilisateurs les ont signalées. Cela est utile lors du lancement ou du processus de test d'une fonctionnalité ou d'un produit notable.
    
* Outils de création de diagrammes tels que [Visio](https://www.microsoft.com/en-us/microsoft-365/visio/flowchart-software) aident à créer le flux de travail de développement de produit et aident le gestionnaire de produit à structurer le parcours utilisateur sur le produit.

## Cybersécurité

![Illustration des compétences en cybersécurité](https://www.freecodecamp.org/news/content/images/2022/07/pexels-photo-5380664.jpeg align="left")

*Illustration des compétences en cybersécurité*

Si vous êtes passionné par la sécurité, envisagez une carrière dans la cybersécurité.

La cybersécurité implique tout, du chiffrement à la recherche et à la protection contre les logiciels malveillants ou les virus, en passant par le piratage pour trouver des vulnérabilités de sécurité dans un système afin qu'elles puissent être corrigées.

Un spécialiste de la cybersécurité protège les informations numériques pour les particuliers, les entreprises, les firmes et le gouvernement.

Leurs tâches incluent :

* Tester, analyser et mettre en œuvre des développements de systèmes de sécurité
    
* Gérer les vulnérabilités du système
    
* Répondre aux menaces et attaques de sécurité
    
* Développer des stratégies de prévention des menaces
    
* Rapporter directement aux administrateurs et aux exécutifs
    

### Outils utilisés par les analystes en cybersécurité :

* Outils de surveillance de la sécurité du réseau tels que [Splunk](https://www.splunk.com/) ou [Argus](https://argus-sec.com/). Ces outils suivent les menaces du réseau et analysent les données du réseau.
    
* Outils de chiffrement tels que [VeraCrypt](https://www.veracrypt.fr/code/VeraCrypt/) et [TrueCrypt](http://truecrypt.sourceforge.net/). Ces outils sont utilisés pour le chiffrement en brouillant le texte clair en code inaccessible aux utilisateurs non autorisés.
    
* Scanners de vulnérabilités tels que [Nikto](https://cirt.net/Nikto2) et [Burp Suite](https://portswigger.net/burp). Ces outils scannent les logiciels pour identifier les vulnérabilités de sécurité en utilisant l'[injection SQL](https://www.freecodecamp.org/news/what-is-sql-injection-how-to-prevent-it/) et le script inter-sites.

## Ressources humaines

![Illustration des compétences en ressources humaines](https://www.freecodecamp.org/news/content/images/2022/07/pexels-photo-5989935.jpeg align="left")

*Illustration des compétences en ressources humaines*

Si vous êtes doué avec les gens et aimez les aider à résoudre les conflits et les problèmes, vous pourriez envisager de devenir un spécialiste des ressources humaines. **RH** signifie Ressources Humaines.

La gestion des ressources humaines consiste à recruter des talents et à développer une main-d'œuvre solide pour une entreprise. Le département des ressources humaines identifie les compétences ou les ressources humaines dans une entreprise, évalue les candidats potentiels et embauche des talents. Ils sont également responsables de la publicité d'un poste pour une entreprise.

Un spécialiste des ressources humaines sélectionne, recrute, interviewe et place de nouveaux employés.

### Autres responsabilités des ressources humaines :

* Gérer les relations avec les employés
    
* Gérer la paie
    
* Gérer les avantages sociaux et la formation des employés
    
* Conseiller les dirigeants sur la planification stratégique
