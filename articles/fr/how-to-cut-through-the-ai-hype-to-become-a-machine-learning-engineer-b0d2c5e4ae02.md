Je suis sûr que vous avez entendu parler des incroyables applications d'intelligence artificielle qui existent - des programmes capables de battre les meilleurs joueurs de Go au monde aux voitures autonomes.

Le problème est que la plupart des gens se laissent emporter par le battage médiatique autour de l'IA, mélangeant les discussions techniques avec des débats philosophiques.

Si vous cherchez à percer le battage médiatique autour de l'IA et à travailler avec des modèles de données mis en œuvre de manière pratique, formez-vous pour devenir ingénieur en données ou ingénieur en machine learning.

Ne cherchez pas des applications intéressantes de l'IA dans des articles sur l'IA. Cherchez-les dans des tutoriels sur l'ingénierie des données ou le machine learning.

Voici les étapes que j'ai suivies pour construire [ce petit scraper amusant que j'ai construit pour analyser](https://github.com/Rogerh91/Springboard-DS2/blob/master/Springboard%20DS2%20Capstone%20Project%20-%20Diversity%20in%20Bootcamps.ipynb) la diversité de genre dans différents bootcamps de codage. C'est le chemin que j'ai pris pour faire des recherches pour le nouveau [bootcamp en ligne AI/ML avec garantie d'emploi de Springboard.](https://www.springboard.com/workshops/ai-machine-learning-career-track)

Voici un guide étape par étape pour entrer dans le domaine du machine learning avec un ensemble critique de ressources attachées à chacune.

### **1. Commencez à vous perfectionner en Python et en pratiques de développement logiciel**

![Image](https://cdn-media-1.freecodecamp.org/images/h5iAZ9X9kElc-pY9HEnd8jrPjSpDWoJQJeK-)

Vous voudrez commencer par adopter Python, le langage de choix pour la plupart des ingénieurs en machine learning.

Ce langage de script pratique est l'outil de choix pour la plupart des ingénieurs en données et des data scientists. La plupart des outils pour les données ont été construits en Python ou ont construit un accès API pour un accès Python facile.

Heureusement, la syntaxe de Python est relativement facile à apprendre. Le langage dispose de tonnes de documentation et de ressources de formation. Il inclut également le support pour tous types de paradigmes de programmation, de la programmation fonctionnelle à la programmation orientée objet.

La seule chose qui pourrait être un peu difficile à apprendre est l'indentation et l'espacement requis pour organiser et activer votre code. En Python, les espaces blancs comptent vraiment.

En tant qu'ingénieur en machine learning, vous travaillerez en équipe pour construire des applications complexes, souvent critiques. C'est donc le bon moment pour vous rafraîchir sur les meilleures pratiques en ingénierie logicielle.

Apprenez à utiliser des outils collaboratifs tels que Github. Prenez l'habitude d'écrire des tests unitaires complets pour votre code en utilisant des frameworks de test tels que nose. Testez vos API en utilisant des outils tels que Postman. Utilisez des systèmes CI tels que Jenkins pour vous assurer que votre code ne casse pas. Développez de bonnes compétences en révision de code pour mieux travailler avec vos futurs collègues techniques.

**Une chose à lire** : [Quel est le meilleur IDE Python pour la science des données ?](https://www.kdnuggets.com/2018/11/best-python-ide-data-science.html) Faites une lecture rapide pour comprendre quel ensemble d'outils vous voulez utiliser pour implémenter Python sur des ensembles de données.

J'utilise moi-même le [Jupyter Notebook](https://ipython.org/notebook.html), car il vient préinstallé avec la plupart des bibliothèques importantes de science des données que vous utiliserez. Il vient avec une interface interactive facile et propre qui vous permet de modifier votre code à la volée.

Jupyter Notebook vient également avec des extensions qui vous permettent de partager facilement vos résultats avec le monde entier. Les fichiers générés sont également super faciles à utiliser sur Github.

**Une chose à faire** : [Pandas Cookbook](https://github.com/jvns/pandas-cookbook) vous permet de bifurquer vers des exemples vivants du framework Pandas, l'une des bibliothèques de manipulation de données les plus puissantes. Vous pouvez rapidement travailler sur un exemple de la façon de jouer avec un ensemble de données à travers celui-ci.

### **2. Explorez les frameworks et la théorie du machine learning**

![Image](https://cdn-media-1.freecodecamp.org/images/18ks5ytRMEhiuUOamxannS29NaMlWKtMgbFw)

Une fois que vous jouez avec Python et que vous pratiquez avec, il est temps de commencer à regarder la théorie du machine learning.

Vous apprendrez quels algorithmes utiliser. Avoir une connaissance de base de la théorie derrière le machine learning vous permettra de mettre en œuvre des modèles avec facilité.

**Une chose à lire** : [Un tour des dix meilleurs algorithmes pour les débutants en machine learning](https://towardsdatascience.com/a-tour-of-the-top-10-algorithms-for-machine-learning-newbies-dde4edffae11) vous aidera à commencer avec les bases. Vous apprendrez qu'il n'y a pas de "repas gratuit". Il n'y a pas d'algorithme qui vous donnera le résultat optimal pour chaque paramètre, donc vous devrez plonger dans chaque algorithme.

**Une chose à faire** : Jouez avec le cours interactif [Cours gratuit de machine learning en Python](https://www.springboard.com/resources/learning-paths/machine-learning-python#!) - développez vos compétences en Python et commencez à implémenter des algorithmes.

### **3. Commencez à travailler avec des ensembles de données et à expérimenter**

![Image](https://cdn-media-1.freecodecamp.org/images/v2WtTkd-Vjoak44zWIBXPZqKs96UR7fxt4cT)

Vous avez les outils et la théorie sous la main. Vous devriez penser à faire de petits mini-projets qui peuvent vous aider à affiner vos compétences.

**Une chose à lire** : Jetez un coup d'œil à [19 ensembles de données publics gratuits pour votre premier projet de science des données](https://www.springboard.com/blog/free-public-data-sets-data-science-project/) et commencez à regarder où vous pouvez trouver différents ensembles de données sur le web pour jouer avec.

**Une chose à faire** : [Kaggle Datasets](https://www.kaggle.com/datasets) vous permettra de travailler avec de nombreux ensembles de données publics. Ce qui est cool avec cette collection, c'est que vous pouvez voir à quel point certains ensembles de données sont populaires. Vous pouvez également voir quels autres projets ont été construits avec le même ensemble de données.

### **4. Développez vos compétences en données avec Hadoop ou Spark**

![Image](https://cdn-media-1.freecodecamp.org/images/dIdLEMPzeON-s-bF99lWMOE2bJnS-FT32oFG)

Maintenant que vous pratiquez sur des ensembles de données plus petits, vous voudrez apprendre à travailler avec Hadoop ou Spark. Les ingénieurs en données travaillent avec des données de streaming, en temps réel, à l'échelle du téraoctet et parfois du pétaoctet. Améliorez vos compétences en apprenant à utiliser un framework de big data.

**Une chose à lire** : Cet article court [Comment Hadoop et Spark se comparent-ils ?](https://logz.io/blog/hadoop-vs-spark/) vous aidera à comprendre à la fois Hadoop et Spark et comment ils se comparent et contrastent l'un avec l'autre.

**Une chose à faire** : Si vous voulez commencer à travailler avec un framework de big data tout de suite, [Spark Jupyter notebooks hébergés sur Databricks](https://databricks-prod-cloudfront.cloud.databricks.com/public/4027ec902e239c93eaaa8714f173bcfc/346304/2168141618055109/484361/latest.html) offre une introduction de niveau tutoriel au framework, et vous permet de pratiquer avec des exemples de code de niveau production.

### **5. Travaillez avec un framework de deep learning comme TensorFlow**

![Image](https://cdn-media-1.freecodecamp.org/images/ltD0ZMVbQ0X1SRRgI7ZOM-vkZjt2Ydz592tj)

Vous avez terminé d'explorer les algorithmes de machine learning et de travailler avec les différents outils de big data disponibles.

Il est maintenant temps de vous attaquer au type puissant d'apprentissage par renforcement qui a été le centre des nouvelles avancées. Apprenez le framework TensorFlow et vous serez à la pointe du travail en machine learning.

**Une chose à lire** : Lisez [Qu'est-ce que TensorFlow ?](https://www.infoworld.com/article/3278008/tensorflow/what-is-tensorflow-the-machine-learning-library-explained.html) et comprenez ce qui se passe sous le capot en ce qui concerne ce puissant framework de deep learning.

**Une chose à faire** : [TensorFlow et Deep Learning sans un PhD](https://codelabs.developers.google.com/codelabs/cloud-tensorflow-mnist/#0) est un cours interactif construit par Google qui combine la théorie placée dans des diapositives avec des laboratoires pratiques avec du code.

### **6. Commencez à travailler avec de grands ensembles de données de niveau production**

![Image](https://cdn-media-1.freecodecamp.org/images/u2P-QdY4xFmpAfoxPI8AAHzc3Tn9DretbTrc)

Maintenant que vous avez travaillé avec des frameworks de deep learning, vous pouvez commencer à travailler vers de grands ensembles de données de niveau production.

En tant qu'ingénieur en machine learning, vous prendrez des décisions d'ingénierie complexes sur la gestion de grandes quantités de données et le déploiement de vos systèmes.

Cela inclurait la collecte de données à partir d'API et le web scraping, les bases de données SQL + NoSQL et quand vous les utiliseriez, l'utilisation de frameworks de pipeline tels que Luigi ou Airflow.

Lorsque vous déployez vos applications, vous pourriez utiliser des systèmes basés sur des conteneurs tels que Docker pour la scalabilité et la fiabilité, et des outils tels que Flask pour créer des API pour votre application.

**Une chose à lire** : [7 Façons de gérer les grands fichiers de données pour le Machine Learning](https://machinelearningmastery.com/large-data-files-machine-learning/) est un bon exercice théorique sur la façon de gérer les grands ensembles de données, et peut servir de liste de contrôle pratique des tactiques à utiliser.

**Une chose à faire** : [Ensembles de données Big Data disponibles publiquement](https://hadoopilluminated.com/hadoop_illuminated/Public_Bigdata_Sets.html) est une liste d'endroits où vous pouvez obtenir de très grands ensembles de données - prêts à pratiquer vos nouvelles compétences en ingénierie des données.

### **7. Pratiquez, pratiquez, pratiquez, construisez vers un portfolio et ensuite un emploi**

![Image](https://cdn-media-1.freecodecamp.org/images/OLbkTWtHUtUd4AZXSCgSe5La2-jFmtK2t3fq)

Enfin, vous en êtes arrivé à un point où vous pouvez construire des modèles de machine learning fonctionnels. La prochaine étape pour faire avancer votre carrière en machine learning est de trouver un emploi dans une entreprise qui possède ces grands ensembles de données afin que vous puissiez appliquer vos compétences chaque jour à un problème de machine learning de pointe.

**Une chose à lire** : [41 Questions essentielles d'entretien sur le Machine Learning (avec réponses)](https://www.springboard.com/blog/machine-learning-interview-questions/) vous aidera à pratiquer les connaissances dont vous avez besoin pour réussir un entretien en machine learning.

**Une chose à faire** : Allez et trouvez des meetups dédiés au machine learning ou à l'ingénierie des données sur [Meetup](http://meetup.com/) - c'est un excellent moyen de rencontrer des pairs dans le domaine et des responsables de recrutement potentiels.
