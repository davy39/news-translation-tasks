Matt Mullenweg, fondateur d'Automattic, a récemment offert [ce conseil](https://ma.tt/2016/02/getting-a-job-after-coding-bootcamp/) aux développeurs en herbe : « **Contribuez à l'open source.** »

Mullenweg — un décrocheur en sciences politiques dont le logiciel (WordPress) alimente désormais près d'un quart de tous les sites web — affirme que vous pouvez « compenser à 100 % un manque d'expérience professionnelle » en prouvant vos compétences grâce à des contributions open source.

> Si vous pouvez vous forger une position reconnue dans un projet open source, cela vous place dix pas devant les candidatures, même de personnes avec 20 ans d'expérience dans le domaine, au moins pour une entreprise comme Automattic, qui privilégie l'Open Source.

Et le centre incontesté de l'univers open source est GitHub. Certaines personnes sont allées jusqu'à dire que [les comptes GitHub ont remplacé les CV](http://code.dblock.org/2011/07/14/github-is-your-new-resume.html).

Ou, comme l'a plaisanté John Resig, le créateur de jQuery : « En matière de recrutement, je préférerai toujours un historique de commits GitHub à un CV. »

Rafael Rodriguez, mainteneur du Wiki de Free Code Camp, a trouvé un moyen de commencer à contribuer à l'open source en quelques minutes seulement, sans même quitter le site web de GitHub.

![Image](https://cdn-media-1.freecodecamp.org/images/1*tFd7RJUU8T9wVhZzoVQyIw.jpeg)

#### Ajouter un article au Wiki de Free Code Camp

Le wiki de Free Code Camp est rempli d'articles sur la technologie. Alors que les articles académiques visent à être détaillés et que la documentation technique vise à être précise, nos articles de wiki visent à être facilement compris par les nouveaux développeurs.

Voici quelques exemples de sujets que vous pouvez aborder sur notre wiki :

* une nouvelle bibliothèque JavaScript comme Redux
* une personne importante dans le monde du logiciel, comme [Dan Abramov](https://github.com/gaearon)
* comment déployer un site e-commerce WordPress
* comment installer Ruby on Rails sur Windows
* comment connecter un Raspberry Pi à un thermostat
* l'histoire de Git
* le chapitre local de Women Who Code dans votre ville

En gros, tout ce qui est lié à la technologie est acceptable. Nous privilégierons l'inclusion.

Tout d'abord, vérifiez [que l'article n'existe pas déjà](https://gitter.im/FreeCodeCamp/wiki) (si c'est le cas, vous pouvez nous aider à l'enrichir).

Et si vous ne trouvez pas d'idée spontanément, vous pouvez choisir un sujet parmi nos [demandes d'articles](https://github.com/FreeCodeCamp/wiki/issues) et écrire quelques paragraphes à ce sujet.

Nous accueillons également les traductions d'articles existants dans n'importe quelle langue.

Vous pouvez ensuite contribuer en suivant ces étapes :

![Image](https://cdn-media-1.freecodecamp.org/images/1*Akhl7oveuXo6Zb5QZYaOog.gif)
_Un enregistrement de moi en train de faire une contribution d'exemple en moins d'une minute._

1. Allez sur le [dépôt Wiki open source de Free Code Camp](http://www.github.com/freecodecamp/wiki). (Si vous n'avez pas encore de compte GitHub, [vous pouvez en créer un en environ 2 minutes](http://www.freecodecamp.com/challenges/create-a-github-account-and-join-our-chat-rooms).)
2. Cliquez sur le bouton « New File ». Cela créera automatiquement un fork (votre propre copie personnelle) du dépôt.
3. Nommez votre fichier « votre-nom-d-article-wiki.md » (le « md » signifie Markdown, une alternative populaire aux fichiers .txt simples).
4. Rédigez votre article. Si vous n'êtes pas familier avec Markdown, vous pouvez [écrire votre article ici](https://jbt.github.io/markdown-editor), puis le copier et le coller sur GitHub.
5. Faites défiler vers le bas et remplissez un message de commit (par exemple, « Create wiki article about the Minimax algorithm ») et ajoutez éventuellement une description. Si vous écrivez cet article en réponse à une demande d'article, vous devez le référencer dans la description comme suit : « Closes issue #177. »
6. Cliquez sur « Propose new file ».
7. Cliquez sur « Create pull request ».
8. Votre pull request sera déjà remplie à partir de votre message de commit et de votre description. Cliquez donc à nouveau sur « Create pull request » pour la finaliser.

Et voilà ! Vous avez créé un fichier, un commit et une pull request (et peut-être même fermé une issue au passage) !

Raphael ou un autre bénévole examinera votre pull request et pourrait vous demander d'apporter des améliorations supplémentaires à votre article. (Dans certaines situations, vous devrez peut-être cloner le dépôt localement pour fusionner des commits ou résoudre des conflits de fusion — nous vous aiderons si cela se produit.)

Si vous avez des questions, rendez-vous dans [notre salon de discussion Wiki](https://gitter.im/FreeCodeCamp/wiki) et demandez.

Une fois votre pull request acceptée, GitHub la fusionnera dans le dépôt Wiki de Free Code Camp, et cela ressemblera à ceci :

![Image](https://cdn-media-1.freecodecamp.org/images/1*vRobXj44BlOqzBx3RCmI-A.jpeg)

Une fois votre pull request acceptée, vous pouvez consulter votre profil GitHub. Voyez ce carré vert sur votre calendrier de heat map, et cette série ? Vous avez fait une contribution open source !

![Image](https://cdn-media-1.freecodecamp.org/images/1*-R0uTUD2lfsFZjTDL3qonQ.jpeg)

#### C'est le début de quelque chose de significatif.

Contribuer à notre wiki est un bon moyen de s'initier à l'open source.

Après avoir écrit quelques articles de wiki (et apporté des modifications à des articles existants), vous voudrez vous aventurer et voir quels autres projets vous pouvez aider.

Vous ne pourrez pas contribuer à la plupart de ces projets sans quitter le confort relatif de votre navigateur et du site web de GitHub. Mais ne vous laissez pas intimider par le processus de développement sur votre ordinateur local — [ce n'est pas si difficile](https://github.com/FreeCodeCamp/FreeCodeCamp/wiki/How-To-Fork-And-Maintain-a-Local-Instance-of-Free-Code-Camp).

En contribuant à des projets de plus en plus grands, vous deviendrez meilleur pour apprendre les bases de code suffisamment bien pour traquer les bugs et ajouter des fonctionnalités.

C'est ainsi que vous apprenez. C'est ainsi que vous grandissez en tant que développeur. Et c'est ainsi que vous prouvez à de futurs employeurs que vous pouvez collaborer avec d'autres personnes et écrire du code qui vaut la peine d'être utilisé.
