Vous pouvez obtenir une erreur 502 Bad Gateway lorsqu'il y a un problème avec le serveur du site web auquel vous essayez de vous connecter.
![502-error](https://www.freecodecamp.org/news/content/images/2022/04/502-error.png)

En termes plus techniques, le « 502 » dans l'erreur est un code d'état HTTP qui indique qu'un serveur a reçu une réponse invalide d'un autre serveur.

Il existe d'autres catégories d'erreurs 500, notamment :
- 501 – Non implémenté,
- 503 – Service indisponible
- 504 – Délai d'attente de la passerelle
- 505 – Version HTTP non supportée
Et bien d'autres.

Les causes de l'erreur 502 Bad Gateway incluent un serveur surchargé, une erreur dans le code et des noms de domaine pas encore propagés. Ainsi, l'erreur est souvent causée par le serveur d'un site web et non par vous en tant qu'utilisateur.

Mais parfois, le navigateur peut afficher cette erreur en raison de mises à jour en attente, de bloqueurs de publicités, d'extensions et de plugins de navigateur, ou même de problèmes de serveur DNS.

Dans cet article, je vais vous montrer 5 façons de corriger l'erreur 502 Bad Gateway afin que vous puissiez accéder à Internet à nouveau.

## Solution 1 : Actualiser la page
De nombreuses erreurs de serveur ne sont que temporaires, pas permanentes, et l'erreur 502 Bad Gateway ne fait pas exception.

Si vous obtenez cette erreur, la première chose à faire est d'actualiser la page après quelques minutes et de voir si le site web se charge à nouveau.

Pour actualiser Google Chrome, cliquez sur le bouton d'actualisation dans le coin supérieur gauche :
![ss1-6](https://www.freecodecamp.org/news/content/images/2022/04/ss1-6.png)

Edge propose également le même bouton d'actualisation au même endroit :
![ss2-7](https://www.freecodecamp.org/news/content/images/2022/04/ss2-7.png)

Si l'actualisation ne fonctionne pas, attendez quelques minutes et réessayez. Si l'erreur persiste, passez aux autres solutions de cet article.

## Solution 2 : Essayez d'accéder au site web sur un autre appareil
Votre propre appareil – qu'il s'agisse d'un ordinateur ou d'un téléphone – pourrait être la raison pour laquelle vous obtenez l'erreur 502 Bad Gateway. De plus, votre source de connexion Internet – routeur ou modem – pourrait en être la cause.

Puisque cela pourrait être la cause, essayez d'accéder au site web sur un autre appareil ou changez votre source de connexion Internet.

Vous pouvez également redémarrer vos appareils – ordinateur, téléphone mobile, routeur et modem.

## Solution 3 : Passer à un autre navigateur
Puisque la cause de l'erreur 502 Bad Gateway pourrait être due à un navigateur obsolète ou à des bloqueurs de publicités, vous devriez envisager de changer de navigateur et de voir si l'erreur persiste.

Par exemple, si vous utilisez le système d'exploitation Windows et que vous essayez d'accéder au site web avec Chrome et que vous obtenez l'erreur 502 Bad Gateway, passez à Edge et vice versa. Si vous êtes sur Mac et que vous obtenez l'erreur sur Chrome, envisagez de passer à Safari et vice versa.

Si vous avez Mozilla Firefox installé sur votre ordinateur, vous pouvez également passer à celui-ci. Vous pourriez obtenir une expérience différente car il n'est pas construit sur le même moteur Chromium que Edge et Google Chrome.

Si le site web se charge dans un autre navigateur, vous devrez peut-être mettre à jour le navigateur initial, le réinstaller ou vérifier vos extensions et bloqueurs de publicités.

## Solution 4 : Effacer le cache de votre navigateur
Vous pourriez obtenir l'erreur 502 Bad Gateway parce que votre navigateur a stocké des informations obsolètes sur le site web que vous essayez de visiter. Ainsi, effacer le cache de votre navigateur pourrait corriger l'erreur.

### Pour effacer le cache du navigateur Chrome, suivez les étapes ci-dessous
**Étape 1** : Cliquez sur les 3 points verticaux dans le coin supérieur droit :
![ss3-6](https://www.freecodecamp.org/news/content/images/2022/04/ss3-6.png)

**Étape 2** : Survolez Plus d'outils et sélectionnez Effacer les données de navigation :
![ss4-4](https://www.freecodecamp.org/news/content/images/2022/04/ss4-4.png)

**Étape 3** : Assurez-vous de sélectionner « Images et fichiers en cache » comme partie des données à effacer, puis cliquez sur le bouton « Effacer les données » :
![ss5-4](https://www.freecodecamp.org/news/content/images/2022/04/ss5-4.png)

### Pour effacer le cache de Microsoft Edge, suivez les étapes ci-dessous
**Étape 1** : Cliquez sur les 3 points horizontaux dans le coin supérieur droit :
![ss6-4](https://www.freecodecamp.org/news/content/images/2022/04/ss6-4.png)

**Étape 2** : Sélectionnez Paramètres :
![ss7-3](https://www.freecodecamp.org/news/content/images/2022/04/ss7-3.png)

**Étape 3** : Cliquez sur « Confidentialité, recherche et services ».

**Étape 4** : Sous « Effacer les données de navigation », cliquez sur le bouton « Choisir les éléments à effacer » :
![ss8-3](https://www.freecodecamp.org/news/content/images/2022/04/ss8-3.png)

**Étape 5** : Assurez-vous que les images et fichiers en cache sont sélectionnés, puis sélectionnez « Effacer maintenant » :
![ss9-3](https://www.freecodecamp.org/news/content/images/2022/04/ss9-3.png)

## Solution 5 : Vider le cache DNS
Les problèmes DNS pourraient être la raison pour laquelle vous continuez à obtenir l'erreur 502 Bad Gateway en essayant de visiter un site web. Ainsi, vider le cache DNS pourrait être une solution.

### Pour vider le cache DNS sur Windows 10, suivez les étapes ci-dessous :
**Étape 1** : Cliquez sur Démarrer, tapez « cmd », puis sélectionnez « Exécuter en tant qu'administrateur » à droite :
![cmd-admin-1](https://www.freecodecamp.org/news/content/images/2022/04/cmd-admin-1.jpg)

**Étape 2** : Tapez « ipconfig /flushdns » et appuyez sur `ENTRÉE`.
Vous devriez obtenir une réponse indiquant que le cache DNS a été vidé, comme celle ci-dessous :
![ss10-3](https://www.freecodecamp.org/news/content/images/2022/04/ss10-3.png)

Cela signifie que tous les caches seront effacés et que les sites web seront chargés fraîchement pour vous.

### Si vous utilisez Chrome, il a son propre cache DNS
Pour vider le DNS de Chrome, tapez `chrome://net-internals/#dns` dans la barre d'adresse et appuyez sur `ENTRÉE`, puis cliquez sur « Clear host cache » :
![ss11-2](https://www.freecodecamp.org/news/content/images/2022/04/ss11-2.png)

### Si vous utilisez Edge, il a également son propre cache DNS

Pour vider le DNS d'Edge, tapez `edge://net-internals/#dns` dans la barre d'adresse et appuyez sur `ENTRÉE`, puis cliquez sur « Clear host cache » :
![ss12-1](https://www.freecodecamp.org/news/content/images/2022/04/ss12-1.png)

## Conclusion
Bien qu'il soit peu probable que l'erreur 502 provienne du côté utilisateur (car il s'agit d'une erreur liée au serveur), vous pouvez toujours essayer certaines solutions de votre côté en tant qu'utilisateur. C'est pourquoi j'ai écrit cet article pour vous montrer comment vous pouvez corriger l'erreur de votre côté.
