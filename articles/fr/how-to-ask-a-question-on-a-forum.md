Poser une question sur un forum est un excellent moyen d'apprendre et de progresser en tant que développeur. Mais il est important de savoir comment poser de bonnes questions afin que les gens puissent vous aider efficacement. 

Dans cet article, je vais vous expliquer comment poser de bonnes questions sur un forum afin de recevoir des réponses utiles. 

## Assurez-vous de rechercher le problème d'abord

freeCodeCamp a mis au point la méthode [Lire - Rechercher - Demander](https://forum.freecodecamp.org/t/how-to-get-help-when-you-are-stuck-coding/19514). Avant de publier sur le forum, assurez-vous de lire les messages d'erreur que vous recevez. 

Ensuite, vous pouvez rechercher dans la documentation ou sur Google pour voir s'il existe une solution que vous pouvez utiliser pour vous aider. Si vous avez des questions sur des méthodes ou des propriétés spécifiques, assurez-vous de consulter d'abord la documentation. 

Veuillez éviter de faire des publications comme celle-ci :

![Image](https://www.freecodecamp.org/news/content/images/2021/12/Screen-Shot-2021-12-16-at-8.09.14-AM.png)

Cette question est vague et peut facilement être recherchée sur Google. 

![Image](https://www.freecodecamp.org/news/content/images/2021/12/Screen-Shot-2021-12-16-at-8.10.26-AM.png)

Si vous avez des questions spécifiques sur le fonctionnement d'une méthode, vous devez vous assurer de détailler cela dans votre publication. 

## Posez des questions spécifiques et fournissez des détails sur le problème

Lorsque vous avez besoin d'aide pour un problème, veuillez fournir autant de détails que possible afin que les gens sachent comment vous aider. 

Vous voulez éviter de faire des publications comme celle-ci :

![Image](https://www.freecodecamp.org/news/content/images/2021/12/Screen-Shot-2021-12-16-at-7.17.24-AM.png)

Cette publication ne fournit aucune explication sur ce avec quoi l'utilisateur a du mal, ce qui rend difficile pour les gens d'offrir de l'assistance. 

Si vous avez du mal avec un problème, assurez-vous d'inclure les éléments suivants :

* un titre descriptif lié au problème que vous rencontrez
* une brève description du problème que vous rencontrez
* une liste des messages d'erreur que vous voyez
* un lien vers le défi ou le projet sur lequel vous travaillez
* votre code de ce que vous avez essayé jusqu'à présent

### Exemple de question spécifique sur un forum

Maintenant, examinons un exemple de la façon de poser une question plus spécifique. Le titre de la publication ci-dessous est "J'ai du mal à créer une image réactive pour le projet de page hommage".

Ce titre descriptif permet aux gens de savoir avec quoi je lutte. Dans le corps de ma publication, je vais décrire le problème et fournir toutes les informations nécessaires.

![Image](https://www.freecodecamp.org/news/content/images/2021/12/Screen-Shot-2021-12-16-at-7.59.58-AM.png)

Dans cette publication, j'ai inclus le code complet du projet, une description du problème, un extrait de code de ce que j'ai essayé et les messages d'erreur que je reçois.

J'ai également fait savoir aux gens que j'ai d'abord essayé de rechercher le problème. 

Avec ce type d'informations, quelqu'un peut aller dans mon code de projet et réaliser que j'ai écrit par erreur `.img-div` au lieu de `#img-div`. 

## N'oubliez pas de formater votre code

Lorsque vous fournissez des extraits de code dans votre publication, n'oubliez pas de formater votre code afin que les gens puissent le lire. 

Veuillez lire [cette publication](https://forum.freecodecamp.org/t/forum-code-formatting/25574) sur la façon de formater votre code dans un forum. 

Si vous avez besoin d'aide pour formater votre code, veuillez contacter un modérateur de ce forum et il vous assistera. 

## Conclusion

Il est important de se rappeler que les gens veulent vous aider. Mais si vous ne fournissez pas assez d'informations, il devient difficile de vous assister.

J'espère que vous avez trouvé cet article utile, et bon codage.
