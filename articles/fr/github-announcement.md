Par Mehul Mohan

Il y a juste quelques jours, GitHub a écrit un article de blog déclarant qu'il est désormais gratuit pour les équipes. Voici l'article officiel du blog si vous êtes [intéressé](https://github.blog/2020-04-14-github-is-now-free-for-teams/). 

Dans cet article de blog, je ne vais pas discuter de ce qui a déjà été discuté par GitHub. Plutôt, je vais prendre une perspective différente et parler de pourquoi cela vous concerne en tant que développeur individuel.

## Microsoft ❤️ GitHub

GitHub a réussi à inclure beaucoup de fonctionnalités gratuites pour tous, qui étaient payantes auparavant. Comment cela s'est-il produit ? - [L'acquisition par Microsoft](https://blogs.microsoft.com/blog/2018/10/26/microsoft-completes-github-acquisition/). 

Microsoft a réalisé l'une de ses plus grandes acquisitions jamais réalisées - 7,5 milliards de dollars - pour un site d'hébergement de dépôts git, fin 2018. Il y a eu beaucoup de bruit dans de nombreuses communautés sur ce sujet. Les gens argumentent pourquoi cela était si mauvais pour l'open source, et comment "Microsoft va tuer GitHub". 

Mais je suppose que chaque trimestre ou presque, GitHub prouve à nouveau et encore que chaque personne qui a argumenté de cette manière a tort.

GitHub a depuis lors déployé un certain nombre de mises à jour mineures et majeures sur la plateforme. Ils ont introduit leur propre registre de packages, puis ont acquis npm cette année. Tout cela a été possible parce que GitHub a maintenant accès aux ressources et aux talents de Microsoft.

## Ce que GitHub a fait

Si vous regardez juste quelques années en arrière, GitHub était une plateforme qui avait :

1. Aucun dépôt privé gratuit (pour les comptes personnels et d'équipe)
2. Aucune action GitHub - support CI/CD
3. Un plan à 9 dollars par mois pour les équipes
4. Un nombre limité de fonctionnalités

Et c'était déjà un gros problème à l'époque. Mais depuis ce temps, GitHub a vraiment pris de la vitesse à la fois en termes de fonctionnalités sur la plateforme et de la manière dont il rivalise avec la concurrence de manière plus efficace. Et en ce moment, son concurrent le plus fort est GitLab.

GitLab est un autre service d'hébergement de dépôts git. Mais il vous permet soit de l'installer sur leur plateforme (comme GitHub) soit d'exécuter votre propre GitLab local. Il proposait également des plans comme GitHub, mais avec plus de fonctionnalités et des prix plus bas.

L'annonce faite hier peut être résumée par ces points :

1. GitHub a rendu les dépôts publics/privés gratuits pour tous - comptes individuels et d'organisation - tout comme GitLab.
2. GitHub a réduit les prix de son plan Team de 9 $ à 4 $ - le même que ce que GitLab propose.
3. Je ne savais pas grand-chose à ce sujet avant aujourd'hui, mais GitHub propose en fait quelque chose appelé GitHub Actions. Il vous permet de faire un tas de choses sympas avec vos dépôts GitHub, comme CI/CD - similaire à ce que GitLab propose.

## Pourquoi cela compte-t-il ?

À ce stade, GitHub s'est honnêtement mis au niveau de GitLab. Avec l'écosystème puissant de GitHub et le soutien de Microsoft, il y a peu de chances que vous souhaitiez choisir GitLab plutôt que GitHub pour une raison spécifique. Surtout si vous n'êtes lié à aucune des deux plateformes. Tant que vous ne vous inquiétez pas que Microsoft vole vos données.

Bien sûr, vous pouvez aussi choisir GitLab - c'est juste une question de choix. Mais le point ici est que, bien que GitLab gagnait en termes de fonctionnalités, les deux sont maintenant pratiquement égaux.

Cela signifie que GitHub est prêt à rivaliser. Et la concurrence crée l'innovation, ce qui profite toujours aux utilisateurs réguliers. Une concurrence saine est vraiment nécessaire pour l'amélioration de la technologie, et c'est ce que nous observons en ce moment. 

Pour vous, en tant que développeur individuel, cela signifie que vous serez mieux pris en charge tandis que GitHub se concentre sur l'apport d'une grande valeur à leurs clients entreprises et payants - les personnes dont ils tirent des revenus. 

Même si GitHub subit des pertes en ce moment, ils s'en sortiront parce que Microsoft les soutient. C'est un peu comme Amazon eCommerce qui se porte bien malgré les pertes car AWS les couvre bien.

## Conclusion

Les choses deviennent vraiment intéressantes ! J'ai personnellement utilisé CircleCI pour [codedamn - une plateforme pour les développeurs pour apprendre et grandir](https://codedamn.com), pour CI/CD. Mais j'ai découvert que je pourrais probablement tout centraliser en utilisant uniquement GitHub Actions ! 

Peut-être que je vais l'essayer plus tard cette semaine et partager avec vous comment vous pouvez le faire aussi. En attendant, continuez à programmer !
