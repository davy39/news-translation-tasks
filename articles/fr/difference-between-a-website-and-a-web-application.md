Il est généralement difficile pour un utilisateur final de faire la différence entre une application web et un site web. Ils tapent simplement l'URL et hop, les résultats s'affichent. 

Et c'est ce qui compte vraiment pour l'utilisateur – obtenir les résultats de ce qu'ils recherchent. Si cela se produit, alors c'est essentiellement ce qui les intéresse. 

Mais pour vous, en tant que développeur, vous êtes celui qui doit construire ces produits pour l'utilisateur. Vous devez donc bien connaître leurs différences. Connaissez-vous les principales différences entre un *Site Web et une Application Web* ?

Nous adopterons une approche technique de base dans cet article. Et bien qu'il existe des opinions divergentes parmi certains développeurs, je vais essayer de mettre en évidence certaines des différences clés qui m'ont aidé à comprendre comment les sites web et les applications web diffèrent.

Commençons.

## Qu'est-ce qu'un site web ?  
![site web](https://www.freecodecamp.org/news/content/images/2021/11/fresh.png)
Un site web est une collection de pages publiquement accessibles contenant des documents, des images, de l'audio, du texte ou d'autres fichiers que les utilisateurs peuvent accéder via Internet. 

Les sites web peuvent consister en une seule page ou en plusieurs pages. Pour qu'un utilisateur accède à un site web, il aura besoin d'une URL qu'il entre dans la barre de recherche de son navigateur web. Un exemple d'URL est https://www.freecodecamp.org/

### Types de sites web 

Il existe deux principaux types de sites web :

* __Sites web dynamiques__   
Ceux-ci sont plus complexes et ont des interactions back-end avec l'utilisateur. Ils utilisent des langages et des bases de données plus avancés en plus du HTML, CSS et JavaScript personnalisés. Contrairement aux sites statiques, ils affichent différents contenus en fonction de divers facteurs.

* __Sites web statiques__  
Ceux-ci sont construits en utilisant des langages simples comme HTML, CSS et JavaScript. Ils sont appelés statiques car ils n'ont pas de base de données avec laquelle un utilisateur peut interagir. Ils affichent le même contenu pour chaque utilisateur qui y accède.

En comprenant ces deux principaux types de sites web, voyons quelques exemples de sites web que nous avons aujourd'hui : 

* Sites de rencontre
* Sites de blogging
* Sites éducatifs
* Sites de portfolio
* Sites personnels 
* Sites communautaires 
* Sites gouvernementaux

### Caractéristiques des sites web  
En tant que développeur, créer un site web attrayant pour l'utilisateur est l'une des tâches principales que vous effectuerez. Cela est dû au fait que les sites sont principalement jugés par leur apparence. L'apparence est un facteur principal qui aidera à déterminer combien de temps l'utilisateur restera sur votre site. 

Pour vous aider à construire un beau site, voici quelques conseils à garder à l'esprit : 
* Il doit être __convivial__ afin que vos utilisateurs puissent facilement comprendre ce que fait le site, où se trouvent les choses, et ainsi de suite.
* Il doit contenir du __contenu de qualité__. Gardez à l'esprit que les visiteurs viendront sur votre site pour le contenu, alors assurez-vous qu'il répond à leurs besoins.
* Il doit __être attrayant__. L'utilisation de principes de design simples devrait vous aider à donner à votre site web une mise en page et un look professionnels tout en fournissant un contenu de qualité. 
* Il doit être __facile à naviguer__. Cela facilitera la recherche pour les utilisateurs. Vous pouvez le faire en ajoutant un menu reliant diverses parties du site, par exemple. 
* Il doit __se charger rapidement__. Afin d'attirer plus d'utilisateurs, assurez-vous que votre site se charge rapidement. Cela fournira une expérience positive aux visiteurs.
* Il doit être __réactif__. Assurez-vous que votre site est accessible sur tous les appareils. Gardez à l'esprit que la plupart des utilisateurs n'utilisent pas d'ordinateurs mais plutôt des téléphones mobiles.

## Pourquoi vous avez besoin d'un site web 
La question principale que vous pourriez vous poser à ce stade est : Avez-vous vraiment besoin d'un site web ? Qu'il soit personnel ou même pour votre entreprise ? Eh bien, la réponse est **OUI** – et voici pourquoi : 

* Si vous avez une entreprise, un site web vous permettra de présenter vos produits à toute personne ayant une connexion Internet, n'importe où. 
* Il vous aidera à développer votre marque en tant qu'individu ou en tant qu'entreprise. 
* Il aide à créer une preuve sociale, afin que les autres puissent voir ce que vous avez fait et ce que vous faites (ce qui aide à construire la crédibilité).
* Il aide à créer une présence en ligne où vos clients peuvent vous trouver facilement. 
* Il peut être utilisé comme source de revenus via la publicité et la monétisation.

## Qu'est-ce qu'une application web ?
![application web](https://www.freecodecamp.org/news/content/images/2021/11/fresh--1-.png)
Une application web, également appelée web app, est un programme informatique avec des fonctionnalités et des éléments interactifs. Vous utilisez des technologies web régulières pour la construire, mais elle stocke également des données et les manipule selon les besoins de l'utilisateur. 

Les applications web sont extrêmement personnalisables et peuvent effectuer une large gamme de tâches et de fonctionnalités comme la création, la lecture, la mise à jour et la suppression de données dans l'application. Elles sont plus complexes et difficiles à construire, donc elles nécessitent une équipe expérimentée de développeurs pour les créer.

Quelques exemples populaires d'applications web incluent : les paniers d'achat, les applications de montage vidéo et photo, les outils de conversion de fichiers, les applications de numérisation de fichiers, les programmes de messagerie, et ainsi de suite.

## Caractéristiques d'une application web 
* Elles sont évolutives et hébergées dans le cloud.
* Ce sont des applications multiplateformes, ce qui signifie qu'elles peuvent être exécutées sur n'importe quel système d'exploitation, qu'il s'agisse de macOS, Windows ou Linux. 
* Elles doivent être facilement testées avec l'automatisation. 

## Pourquoi vous avez besoin d'une application web
De nombreuses entreprises aujourd'hui se tournent vers l'utilisation d'applications web en plus ou à la place des sites web. Voici quelques-unes des raisons pour lesquelles : 
* Faciles à maintenir et à mettre à jour
* __Facilement personnalisables__ - vous pouvez facilement ajouter plus de designs et de fonctionnalités en fonction de vos préférences et des besoins des utilisateurs.
* __Sécurisées__ - les données sont stockées dans le cloud, ce qui signifie que vous n'avez pas à vous soucier de la perte de données si votre appareil est endommagé. Vous pouvez facilement les récupérer.
* Vous n'avez pas besoin d'approbation des différents magasins d'applications, car elles peuvent être accessibles via le navigateur.
* Elles supportent tous les navigateurs modernes.

## Différences entre les sites web et les applications web
À ce stade, vous avez déjà une compréhension de ce que sont les sites web et les applications web. Nous allons donc mettre en évidence leurs principales différences.

Faisons un résumé des différences clés pour mieux les comprendre : 

| Site Web        | Application Web    | 
|--------------|-----------|
| Ils fournissent un contenu visuel et textuel, qui peut être consulté et lu. | L'utilisateur ne se contente pas de consulter le contenu des pages, mais peut également interagir avec lui et le manipuler, par exemple en remplissant des formulaires      | 
| L'authentification n'est pas obligatoire sur les sites web réguliers, sauf si vous vous inscrivez à quelque chose comme une newsletter      | Elles nécessitent une authentification car elles offrent des services beaucoup plus larges, ce qui signifie que vous devrez avoir un nom d'utilisateur et un mot de passe pour accéder aux services  | 
| Elles afficheront ce qui a été recherché par l'utilisateur | Les tâches ici sont plus complexes, vous devrez naviguer à l'intérieur pour obtenir un produit spécifique dans une page     | 
| Les informations ici sont publiquement accessibles      | Les informations ici sont restreintes aux utilisateurs enregistrés, ce qui signifie que sans identifiants, vous ne pourrez peut-être pas accéder aux données  | 
| Il est facile à développer car il ne nécessite que les technologies web de base. | C'est plus difficile, car cela nécessite un niveau de sécurité et de fonctionnalités plus élevé en fonction de son objectif.      | 
| Il est moins cher à héberger      | Coûteux à héberger car plus de composants sont nécessaires comme une base de données.  |

## Conclusion
Eh bien, pour répondre à la question, lequel des deux dois-je choisir ? C'est simple :
* si vous avez besoin de présenter vos compétences et produits, un site web devrait être votre choix.
* si vous voulez un outil en ligne pour vous aider à automatiser votre processus et obtenir des interactions avec les visiteurs comme le e-commerce, une application web est votre choix. 

En tant que développeur, la construction de sites web est un processus relativement simple une fois que vous connaissez les outils et les langages requis. Vous avez juste besoin des bases que vous pouvez commencer à apprendre sur [freeCodeCamp](https://www.freecodecamp.org/). 

D'autre part, les applications web nécessitent des connaissances plus approfondies, plus de planification, et cela peut être plus coûteux aussi. Pour vous aider à commencer, voici une ressource que vous pouvez consulter :
