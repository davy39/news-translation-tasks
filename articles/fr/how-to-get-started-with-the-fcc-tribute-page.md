## Comprendre les instructions

### Dois-je copier l'exemple ?

L'objectif indique :

Créez une application [CodePen.io](https://codepen.io/) qui est fonctionnellement similaire à ceci : [https://codepen.io/freeCodeCamp/full/zNqgVx](https://codepen.io/freeCodeCamp/full/zNqgVx).

Les mots "fonctionnellement similaire" signifient simplement que vous devez créer un projet qui passe les tests. Le but n'est pas de copier ou de recréer l'exemple.

Vous êtes encouragé à créer votre propre contenu et style. Vous devez choisir votre propre sujet et images.

### Dois-je utiliser JavaScript ? Bootstrap ? Sass ?

La réponse est non. La seule exigence est d'utiliser HTML et CSS.

Si vous avez déjà de l'expérience avec JavaScript, Bootstrap, Sass, etc., alors vous pouvez choisir de les utiliser. Mais ce n'est pas nécessaire.

## Qu'est-ce que CodePen et comment l'utiliser ?

Pour ces projets, vous avez la possibilité d'utiliser un éditeur de code en ligne appelé [CodePen](https://codepen.io/).

### Comment créer un compte CodePen gratuit

Vous pouvez créer un compte avec les options que vous voyez ci-dessous.

![Image](https://www.freecodecamp.org/news/content/images/2021/03/codepen-signup.png)

Une fois que vous avez créé votre compte, vous devriez recevoir un email vous demandant de vérifier votre adresse email avec CodePen.

### Quelle est la différence entre un Pen et un Projet ?

Un Pen est un éditeur avec seulement trois sections pour HTML, CSS et JavaScript. C'est ce que vous utiliserez pour les projets freeCodeCamp.

![Image](https://www.freecodecamp.org/news/content/images/2021/03/pen-editor-1.png)

Vous pouvez écrire du code directement dans ces éditeurs et voir vos résultats dans la fenêtre de prévisualisation.

![Image](https://www.freecodecamp.org/news/content/images/2021/03/Screen-Shot-2021-03-21-at-6.40.28-AM.png)

Un projet supporte plusieurs fichiers HTML, CSS et JavaScript. Dans la version gratuite, vous êtes autorisé à créer un projet et autant de Pens que vous le souhaitez.

### Fonctionnalités utiles de CodePen

Une fonctionnalité utile est la possibilité de changer la vue de la disposition de votre éditeur.

![Image](https://www.freecodecamp.org/news/content/images/2021/03/view-options-1.png)

CodePen offre également l'option de formater votre code. Il est vraiment important d'avoir un code correctement formaté pour que d'autres développeurs puissent le lire facilement.

Voici avant le formatage :

![Image](https://www.freecodecamp.org/news/content/images/2021/03/before-format-1.png)

Voici après le formatage :

![Image](https://www.freecodecamp.org/news/content/images/2021/03/after-format-1.png)

Une autre fonctionnalité intéressante est l'outil d'analyse. Il vous aidera à trouver des erreurs dans votre code pendant que vous construisez votre projet.

![Image](https://www.freecodecamp.org/news/content/images/2021/03/analyze-before-option-1.png)

![Image](https://www.freecodecamp.org/news/content/images/2021/03/analyze-after-option-1.png)

Avec CodePen, vous n'avez pas besoin d'inclure les balises `<!DOCTYPE html>`, `<html>`, `<head>` ou `<body>` car elles sont déjà intégrées dans l'éditeur.

Si vous devez inclure des liens pour la section head, vous pouvez les ajouter dans les paramètres HTML de l'éditeur de pen.

![Image](https://www.freecodecamp.org/news/content/images/2021/03/Screen-Shot-2021-03-21-at-12.45.39-PM.png)

## Dois-je utiliser CodePen ou puis-je utiliser un autre éditeur de code ?

Vous n'êtes pas obligé d'utiliser CodePen pour construire vos projets. Si vous vous sentez plus à l'aise avec un autre éditeur de code, vous êtes libre de l'utiliser.

## Qu'est-ce que la suite de tests ?

Afin de s'assurer que vous passez toutes les histoires utilisateur requises, freeCodeCamp a créé une suite de tests.

![Image](https://www.freecodecamp.org/news/content/images/2021/03/Screen-Shot-2021-03-21-at-7.01.54-AM.png)

Dans le menu déroulant, vous pouvez choisir le projet sur lequel vous travaillez et cliquer sur Exécuter les tests. Il vous montrera alors combien de tests sont réussis.

![Image](https://www.freecodecamp.org/news/content/images/2021/03/Screen-Shot-2021-03-21-at-8.14.53-AM.png)

Si vous cliquez sur le bouton rouge Tests, il vous montrera les messages d'erreur complets. Assurez-vous de lire le message d'erreur entier car il fournit des informations utiles sur la façon de corriger le problème.

![Image](https://www.freecodecamp.org/news/content/images/2021/03/Screen-Shot-2021-03-21-at-8.15.06-AM.png)

### Comment ajouter la suite de tests dans CodePen

1. Connectez-vous à votre compte
2. Allez à cette URL [https://codepen.io/pen?template=MJjpwO](https://codepen.io/pen?template=MJjpwO)
3. Cliquez sur Sauvegarder

Vous devriez voir ceci comme résultat et l'URL devrait inclure votre nom d'utilisateur.

![Image](https://www.freecodecamp.org/news/content/images/2021/03/Screen-Shot-2021-03-21-at-7.12.02-AM.png)

![Image](https://www.freecodecamp.org/news/content/images/2021/03/Screen-Shot-2021-03-21-at-7.12.10-AM.png)

Il est important de s'assurer que l'URL en haut de la page inclut votre nom d'utilisateur. Si vous essayez de soumettre un projet avec la même URL que le modèle, il ne sera pas accepté.

### Comment ajouter la suite de tests dans votre éditeur de code préféré

Si vous construisez le projet en utilisant un autre éditeur comme Visual Studio Code, vous devrez utiliser le lien CDN.

CDN signifie Content Delivery Network. Vous placerez ce lien CDN `https://cdn.freecodecamp.org/testable-projects-fcc/v1/bundle.js` dans une balise script et placerez cette balise script à la fin des balises body.

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Demo Tribute page</title>
  </head>
  <body>
    <p>Je viens d'ajouter la suite de tests dans VS code</p>
    
    <script src="https://cdn.freecodecamp.org/testable-projects-fcc/v1/bundle.js"></script>
  </body>
</html>
```

## Aborder les histoires utilisateur

### Qu'est-ce que les histoires utilisateur ?

En développement logiciel, les histoires utilisateur sont des descriptions de fonctionnalités que l'utilisateur souhaite voir implémentées dans le projet. Pour la page hommage, il y a une liste de neuf histoires utilisateur que vous devez inclure dans ce projet.

![Image](https://www.freecodecamp.org/news/content/images/2021/08/Screen-Shot-2021-08-22-at-12.49.35-AM.png)

Je suggérerais de se concentrer d'abord sur le passage des histoires utilisateur, puis de s'occuper du style de la page plus tard.

Prenez votre temps et passez chaque histoire utilisateur une par une. Si vous ne comprenez pas une des histoires utilisateur, il y a un moyen de trouver la réponse.

Par exemple, si vous avez des difficultés avec la première histoire utilisateur, vous pouvez vous rendre sur le [forum freeCodeCamp](https://forum.freecodecamp.org/) et utiliser la barre de recherche. Vous n'avez pas besoin d'avoir un compte sur le forum pour utiliser la fonction de recherche.

![Image](https://www.freecodecamp.org/news/content/images/2021/03/Screen-Shot-2021-03-21-at-10.34.14-AM.png)

![Image](https://www.freecodecamp.org/news/content/images/2021/03/Screen-Shot-2021-03-21-at-10.22.33-AM.png)

Comme vous pouvez le voir, beaucoup de gens ont rencontré le même problème. Vous pouvez cliquer sur l'un des posts et lire la discussion pour mieux comprendre les instructions.

Vous pouvez également créer un compte sur le forum et demander de l'aide là-bas. Il y a beaucoup de personnes compétentes qui seront heureuses de vous aider.

### Google est votre ami

Une compétence importante pour les développeurs est de savoir utiliser Google.

Par exemple, vous pouvez utiliser Google pour vous aider à passer les deux dernières histoires utilisateur.

**Histoire utilisateur #8 :** L'élément `img` doit redimensionner de manière responsive, par rapport à la largeur de son élément parent, sans dépasser sa taille originale.

**Histoire utilisateur #9 :** L'élément `img` doit être centré dans son élément parent.

Si vous cherchez sur Google "comment rendre une image responsive", vous trouverez de nombreux résultats parmi lesquels choisir.

![Image](https://www.freecodecamp.org/news/content/images/2021/03/Screen-Shot-2021-03-21-at-10.49.01-AM.png)

Vous pouvez également chercher sur Google "comment centrer une image css" pour vous aider avec la dernière histoire utilisateur.

![Image](https://www.freecodecamp.org/news/content/images/2021/03/Screen-Shot-2021-03-21-at-10.50.43-AM.png)

## Conseils pour styliser votre page

Une fois que vous avez passé toutes les histoires utilisateur, vous pouvez vous concentrer sur le style de votre page.

Le but n'est pas de recréer un design de qualité professionnelle pour votre tout premier projet. Faites simplement de votre mieux et gardez-le simple.

Assurez-vous que votre page est facile à lire et n'a pas trop de couleurs conflictuelles ou d'animations qui nuisent à l'expérience utilisateur.

Ces projets vous donnent l'opportunité d'expérimenter avec différentes fonctionnalités en CSS. N'oubliez pas que vous pouvez utiliser Google pour apprendre à faire des choses en CSS.

Voici quelques choses que vous pourriez vouloir chercher sur Google :

* Comment créer des images arrondies en CSS
* Comment utiliser les polices Google en CSS
* Comment ajouter une image de fond en CSS
* Comment utiliser les animations CSS
* Comment créer une galerie d'images avec CSS Grid

Vous n'avez pas à vous sentir obligé d'utiliser l'une de ces suggestions. C'est entièrement à vous de décider à quel point vous voulez que votre design soit complexe.

## Comment soumettre votre projet

Une fois que vous avez terminé et que tous les tests sont réussis, vous devrez le soumettre pour obtenir le crédit pour le projet.

1. Copiez l'URL de votre projet.

![Image](https://www.freecodecamp.org/news/content/images/2021/08/Screen-Shot-2021-08-22-at-12.42.44-AM.png)

2. Allez aux instructions de la page hommage [https://www.freecodecamp.org/learn/responsive-web-design/responsive-web-design-projects/build-a-tribute-page](https://www.freecodecamp.org/learn/responsive-web-design/responsive-web-design-projects/build-a-tribute-page)

3. Collez l'URL de votre projet où il est écrit Solution Link

4. Cliquez sur I've Completed this Challenge

5. Cliquez sur Submit et passez au prochain défi

Si vous avez construit votre projet en utilisant un autre éditeur comme Visual Studio Code, vous devez copier et coller votre code dans CodePen. Ensuite, vous pouvez copier l'URL pour soumettre le projet.

Vous avez également la possibilité d'utiliser autre chose comme Github Pages si vous avez de l'expérience avec Git et GitHub. Tant que vous soumettez une URL vers un site en ligne et fonctionnel, c'est tout ce qui compte.
