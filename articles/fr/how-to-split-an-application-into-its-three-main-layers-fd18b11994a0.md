Une manière courante de séparer les responsabilités dans une application est de les diviser en trois couches principales :

* UI (aka Présentation)
* domaine (aka Métier)
* accès aux données

Certains des avantages de cette approche sont :

* une meilleure compréhension du but de chaque couche
* l'implémentation d'une couche peut être remplacée. Par exemple, nous pouvons passer d'AngularJS à VueJS ou ReactJS dans la couche UI.
* plusieurs couches UI peuvent être implémentées sur la même couche domaine
* le UI peut être difficile à tester. En déplaçant la logique vers d'autres couches, il devient plus facile à tester.
