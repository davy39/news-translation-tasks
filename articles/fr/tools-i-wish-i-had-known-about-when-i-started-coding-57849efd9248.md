Par Mario Hoyos

Dans le monde de la technologie, il existe des milliers d'outils que les gens vous diront d'utiliser. Comment êtes-vous censé savoir par où commencer ?

En tant que personne qui a commencé à coder relativement récemment, cette avalanche d'informations était trop difficile à trier. Je me suis retrouvé à installer des extensions qui ne m'aidaient pas vraiment dans mon cycle de développement, et qui souvent même entravaient mon travail.

Je ne suis en aucun cas un expert, mais avec le temps, j'ai compilé une liste d'outils qui se sont avérés extrêmement utiles pour moi. Si vous commencez tout juste à apprendre à programmer, cela vous offrira, je l'espère, quelques conseils. Si vous êtes un développeur expérimenté, j'espère que vous apprendrez quand même quelque chose de nouveau.

Je vais diviser cet article en extensions Chrome et extensions VS Code. Je sais qu'il existe d'autres navigateurs et d'autres éditeurs de texte, mais je suis prêt à parier que la plupart de ces outils sont également disponibles pour votre plateforme de choix, alors ne commençons pas une discussion religieuse sur nos préférences personnelles.

N'hésitez pas à naviguer à votre guise.

### Extensions Chrome

![Image](https://cdn-media-1.freecodecamp.org/images/kNxI-xqtP3DhSiPB0GHdV19e77slYxRLGs19)
_Photo par Caio Resende de Pexels_

Maintenant que je suis un développeur web autoproclamé, je vis pratiquement dans ma console Chrome. Voici quelques outils qui me permettent de passer moins de temps là-bas :

* [WhatFont](https://chrome.google.com/webstore/detail/whatfont/jabopobgcpjmedljpbcaablpmlmfcogm?hl=en) — Le nom dit tout. C'est un moyen facile de découvrir les polices utilisées par votre site web préféré, afin que vous puissiez les emprunter pour vos propres projets.
* [Pesticide](https://chrome.google.com/webstore/detail/pesticide-for-chrome/bblbgcheenepgnnajgfpiicnbbdmmooh?hl=en) — Utile pour voir les contours de vos `<div>` et modifier le CSS. Cela m'a sauvé la vie lorsque j'essayais d'apprendre à utiliser le modèle de boîte.
* [Colorzilla](https://chrome.google.com/webstore/detail/colorzilla/bhlhnicpbhignbdhedgjhgdocnmhomnp?hl=en) — Utile pour copier les couleurs exactes d'un site web. Cela copie une couleur directement dans votre presse-papiers afin que vous ne passiez pas des heures à essayer d'obtenir la bonne combinaison RGBA.
* [CSS Peeper](https://chrome.google.com/webstore/detail/css-peeper/mbnbehikldjhnfehhnaidhjhoofhpehk?hl=en) — Utile pour examiner les couleurs et les ressources utilisées sur un site web. Un bon exercice, surtout lorsque vous commencez, consiste à cloner des sites web que vous trouvez cool. Cela vous donne un aperçu de leur schéma de couleurs et vous permet de voir quelles autres ressources existent sur leur page.
* [Wappalyzer](https://chrome.google.com/webstore/detail/wappalyzer/gppongmhjkpfnbhagpmjfkannfbllamg?hl=en) — Utile pour voir les technologies utilisées sur un site web. Vous vous êtes déjà demandé quel type de framework un site web utilise ou sur quel service il est hébergé ? Ne cherchez pas plus loin.
* [React Dev Tools](https://chrome.google.com/webstore/detail/react-developer-tools/fmkadmapgofadopljbjfkapdkoienihi?hl=en) — Utile pour déboguer vos applications React. Il est bon de mentionner que cela n'est utile que si vous programmez une application React.
* [Redux Dev Tools](https://chrome.google.com/webstore/detail/redux-devtools/lmhkpmbekcpmknklioeibfkpmmfibljd?hl=en) — Utile pour déboguer les applications utilisant Redux. Il est bon de mentionner que cela n'est utile que si vous implémentez Redux dans votre application.
* [JSON Formatter](https://chrome.google.com/webstore/detail/json-formatter/bcjindcccaagfpapjjmafapmmgkkhgoa?hl=en) — Utile pour rendre le JSON plus lisible dans le navigateur. Avez-vous déjà fixé un blob JSON laid en face, en essayant de comprendre à quel point les informations que vous voulez sont imbriquées ? Eh bien, cela fait en sorte que cela ne prend que 2 heures au lieu de 3.
* [Vimeo Repeat and Speed](https://chrome.google.com/webstore/detail/vimeo-repeat-speed/noonakfaafcdaagngpjehilgegefdima?hl=en) — Utile pour accélérer les vidéos Vimeo. Si vous regardez des tutoriels vidéo comme la plupart des développeurs web, vous savez à quel point il est pratique de les consulter à 1,25 fois la vitesse de lecture normale. Il existe également des versions pour YouTube.

### Extensions VS Code

![Image](https://cdn-media-1.freecodecamp.org/images/nWv5yCVARZsqUeBkgfqge6jUTUxoLacUJOlC)
_Photo Courtesy de Pexels_

Visual Studio Code est mon éditeur de choix.

Les gens adorent leurs éditeurs de texte, et je ne fais pas exception. Cependant, je suis prêt à parier que la plupart de ces extensions fonctionnent également pour l'éditeur que vous utilisez. Découvrez mes extensions préférées :

* [Auto Rename Tag](https://marketplace.visualstudio.com/items?itemName=formulahendry.auto-rename-tag) — Renomme automatiquement les balises HTML appariées. Vous avez créé une balise `<p>`. Maintenant, vous voulez la changer, ainsi que sa balise de fermeture `</p>` correspondante, en quelque chose d'autre. Changez simplement l'une et l'autre suivra. Théoriquement, cela améliore votre productivité d'un facteur de 2.
* [HTML CSS Support](https://marketplace.visualstudio.com/items?itemName=ecmel.vscode-html-css) — Support CSS pour les documents HTML. Cela est utile pour obtenir une belle coloration syntaxique et des suggestions de code afin que le CSS ne vous donne envie d'arrêter de coder que quelques fois par jour.
* [HTML Snippets](https://marketplace.visualstudio.com/items?itemName=abusaidm.html-snippets) — Extraits de code utiles. Un autre gain de temps agréable. Associez cela avec [Emmet](https://emmet.io/) et vous n'aurez presque jamais à taper du vrai HTML à nouveau.
* [Babel ES6/ES7](https://marketplace.visualstudio.com/items?itemName=dzannotti.vscode-babel-coloring) — Ajoute une coloration syntaxique JavaScript Babel. Si vous utilisez Babel, cela rendra beaucoup plus facile la différenciation de ce qui se passe dans votre code. C'est bien si vous aimez jouer avec les fonctionnalités modernes de JavaScript.
* [Bracket Pair Colorizer](https://marketplace.visualstudio.com/items?itemName=CoenraadS.bracket-pair-colorizer) — Ajoute des couleurs aux crochets pour une meilleure visualisation des blocs. Cela est pratique pour ces bugs trop courants où vous n'avez pas fermé vos crochets ou parenthèses avec précision.
* [ESLint](https://marketplace.visualstudio.com/items?itemName=dbaeumer.vscode-eslint) — Intègre ESLint dans Visual Studio Code. Cela est pratique pour obtenir des indices sur les bugs pendant que vous écrivez votre code et, selon votre configuration, cela peut aider à renforcer un bon style de codage.
* [Guides](https://marketplace.visualstudio.com/items?itemName=spywhere.guides) — Ajoute des lignes de guide supplémentaires au code. C'est un autre indice visuel pour vous assurer que vous fermez vos crochets correctement. Si vous ne pouvez pas le dire, je suis une personne très visuelle.
* [JavaScript Console Utils](https://marketplace.visualstudio.com/items?itemName=whtouche.vscode-js-console-utils) — Facilite la journalisation de la console. Si vous êtes comme la plupart des développeurs, vous vous retrouverez à journaliser dans la console dans votre flux de débogage (je sais que nous sommes censés utiliser le débogueur). Cet utilitaire facilite la création de déclarations `console.log()` utiles.
* [Code Spell Checker](https://marketplace.visualstudio.com/items?itemName=streetsidesoftware.code-spell-checker) — Correcteur orthographique qui tient compte du camelCase. Une autre source courante de bugs est de mal taper un nom de variable ou de fonction. Ce correcteur orthographique recherchera des mots inhabituels et est bon pour tenir compte de la manière dont nous écrivons les choses en JavaScript.
* [Git Lens](https://marketplace.visualstudio.com/items?itemName=eamodio.gitlens) — Facilite la visualisation des modifications, quand et par qui elles ont été faites. C'est bien pour blâmer la personne appropriée lorsque le code est cassé, puisque ce n'est absolument jamais de votre faute.
* [Path Intellisense](https://marketplace.visualstudio.com/items?itemName=christian-kohler.path-intellisense) — Complétion automatique des chemins de fichiers. Cela est super pratique pour importer des choses depuis d'autres fichiers. Cela rend la navigation dans votre arborescence de fichiers un jeu d'enfant.
* [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) — Formateur de code automatique. Oubliez les jours où vous deviez indenter manuellement votre code et rendre les choses lisibles par l'homme. Prettier le fera pour vous beaucoup plus rapidement et mieux que vous ne pourriez le faire vous-même. Je ne peux pas recommander celui-ci assez.
* [VSCode-Icons](https://marketplace.visualstudio.com/items?itemName=robertohuertasm.vscode-icons) — Ajoute des icônes à l'arborescence des fichiers. Si regarder la structure de vos fichiers vous fait mal aux yeux, cela pourrait aider. Il existe une icône utile pour à peu près tout type de fichier que vous créez, ce qui facilitera la distinction de ce que vous regardez.

### **En Conclusion**

Vous avez probablement votre propre ensemble d'outils qui sont indispensables à votre cycle de développement. Espérons que certains des outils que j'ai mentionnés ci-dessus pourront rendre votre flux de travail plus efficace.

Cependant, ne tombez pas dans le piège d'installer chaque outil que vous rencontrez avant d'apprendre à utiliser ceux que vous avez déjà, car cela peut être une énorme perte de temps.
