Par Gitter

[Node.js](https://nodejs.org/en/) est un environnement d'exécution JavaScript et une bibliothèque. Il permet d'exécuter du code JavaScript en backend, en dehors d'un navigateur, en utilisant le moteur V8 de Google, le même environnement d'exécution pour JavaScript que celui utilisé par Google Chrome. Il est léger, efficace et [bénéficie d'une popularité continue](https://www.quora.com/Is-Node-js-declining-already).

L'écosystème de paquets Node.js, npm, est le plus grand écosystème de bibliothèques open source au monde. Nous avons une multitude de canaux liés à Node sur Gitter — découvrez-les et rejoignez la conversation !

* [**nodejs/node**](https://gitter.im/orgs/nodejs/rooms?utm_source=blog&utm_medium=content&utm_campaign=node) **—** Salons de discussion officiels de Node.js.
* [**gitterHQ/nodejs**](https://gitter.im/gitterHQ/nodejs?utm_source=blog&utm_medium=content&utm_campaign=node) — Un chat général sur Node.
* [**nwjs/nw.js**](https://gitter.im/nwjs/nw.js?utm_source=blog&utm_medium=content&utm_campaign=node) — NW.js est un environnement d'exécution d'applications basé sur Chromium et node.js. Vous pouvez écrire des applications natives en HTML et JavaScript avec NW.js. Il vous permet également d'appeler des modules Node.js directement depuis le DOM et offre une nouvelle façon d'écrire des applications natives avec toutes les technologies Web.
* [**linnovate/mean**](https://gitter.im/linnovate/mean?utm_source=blog&utm_medium=content&utm_campaign=node) — MEAN est un framework pour un point de départ facile avec des applications basées sur [MongoDB](https://www.mongodb.org/), [Node.js](http://www.nodejs.org/), [Express](http://expressjs.com/), et [AngularJS](https://angularjs.org/). Il est conçu pour vous donner un moyen rapide et organisé de commencer à développer des applications web basées sur MEAN avec des modules utiles comme Mongoose et Passport pré-bundlés et configurés.
* [**strongloop/loopback**](https://gitter.im/strongloop/loopback?utm_source=blog&utm_medium=content&utm_campaign=node) — LoopBack est un framework Node.js open source construit sur Express, optimisé pour la création d'API pour mobile, web et autres appareils. Connectez-vous à plusieurs sources de données, écrivez la logique métier en Node.js, superposez-vous à vos services et données existants, connectez-vous en utilisant JS, iOS et Android SDK. [http://loopback.io](http://loopback.io/)
* [**aws/aws-sdk-js**](https://gitter.im/aws/aws-sdk-js?utm_source=blog&utm_medium=content&utm_campaign=node) — AWS SDK pour JavaScript dans le navigateur et Node.js [http://aws.amazon.com/javascript](http://aws.amazon.com/javascript)
* [**balderdashy/sails**](https://gitter.im/balderdashy/sails?utm_source=blog&utm_medium=content&utm_campaign=node) — Sails.js est un framework web qui facilite la création d'applications Node.js personnalisées et de niveau entreprise. Il est conçu pour ressembler à l'architecture MVC de frameworks comme Ruby on Rails, mais avec un support pour le style plus moderne et orienté données de développement d'applications web. Il est particulièrement adapté pour construire des fonctionnalités en temps réel comme le chat.
* [**hapijs/hapi**](https://gitter.im/hapijs/hapi?utm_source=blog&utm_medium=content&utm_campaign=node) — hapi est un framework simple à utiliser, centré sur la configuration, avec un support intégré pour la validation des entrées, la mise en cache, l'authentification et d'autres fonctionnalités essentielles pour la construction d'applications web et de services. hapi permet aux développeurs de se concentrer sur l'écriture de logique d'application réutilisable dans une approche hautement modulaire et prescriptive.
* [**webdriverio/webdriverio**](https://gitter.im/webdriverio/webdriverio?utm_source=blog&utm_medium=content&utm_campaign=node) — WebdriverIO est un module [webdriver](https://w3c.github.io/webdriver/webdriver-spec.html) (automatisation de navigateur) pour Node.JS. Il permet d'écrire des tests [Selenium](https://en.wikipedia.org/wiki/Selenium_%28software%29) super faciles dans votre framework de test BDD/TDD préféré, qui s'exécuteront localement ou dans le cloud en utilisant Sauce Labs, BrowserStack ou TestingBot. WebdriverIO est agnostique quant au framework de test que vous souhaitez utiliser. Cucumber, Jasmine et Mocha+Chai sont supportés par l'assistant de configuration, et vous pouvez utiliser d'autres frameworks également — par exemple [Yadda](https://github.com/webdriverio/cucumber-boilerplate).
* [**feross/webtorrent**](https://gitter.im/feross/webtorrent?utm_source=blog&utm_medium=content&utm_campaign=node) — WebTorrent est un client torrent en streaming pour node.js et le navigateur. OUI, VOUS AVEZ BIEN LU. LE NAVIGATEUR. Il est écrit entièrement en JavaScript — le langage du web — donc le même code fonctionne dans les deux environnements d'exécution. Dans node.js, ce module est un simple client torrent, utilisant TCP et UDP pour communiquer avec d'autres clients torrent. Dans le navigateur, WebTorrent utilise WebRTC (canaux de données) pour le transport pair-à-pair. Il peut être utilisé sans plugins, extensions ou installations de navigateur. C'est Juste du JavaScript™.
* [**koajs/koa**](https://gitter.im/koajs/koa?utm_source=blog&utm_medium=content&utm_campaign=node) — Koa est un framework HTTP expressif et léger pour node.js pour rendre les applications web et les API plus agréables à écrire. Les requêtes Koa circulent à travers le middleware de manière similaire à une pile, vous permettant d'effectuer des actions en aval, puis de filtrer et manipuler la réponse en amont. L'utilisation de générateurs par Koa augmente également considérablement la lisibilité et la robustesse de votre application.
* [**nodegit/nodegit**](https://gitter.im/nodegit/nodegit?utm_source=blog&utm_medium=content&utm_campaign=node) — Liaisons natives Node pour Git. [http://www.nodegit.org/](http://www.nodegit.org/)
* [**keystonejs/keystone**](https://gitter.im/keystonejs/keystone) — [KeystoneJS](http://keystonejs.com/) est un système de gestion de contenu Node.js puissant et un framework d'application web construit sur [express](http://expressjs.com/) et [mongoose](http://mongoosejs.com/). Keystone facilite la création de sites web et d'applications sophistiqués, et est livré avec une belle interface d'administration auto-générée.
* [**mongodb/node-mongodb-native**](https://gitter.im/mongodb/node-mongodb-native?utm_source=blog&utm_medium=content&utm_campaign=node) — Le pilote officiel [MongoDB](https://www.mongodb.com/) pour Node.js. Fournit une API de haut niveau au-dessus de [mongodb-core](https://www.npmjs.com/package/mongodb-core) destinée aux utilisateurs finaux.
* [**EmergingTechnologyAdvisors/node-serialport**](https://gitter.im/EmergingTechnologyAdvisors/node-serialport?utm_source=blog&utm_medium=content&utm_campaign=node) — Package Node.js pour accéder aux ports série pour la lecture et l'écriture. Accueillez vos seigneurs robotiques JavaScript. Mieux encore, programmez-les !
* [**sass/node-sass**](https://gitter.im/sass/node-sass?utm_source=blog&utm_medium=content&utm_campaign=node) — Node-sass est une bibliothèque qui fournit des liaisons pour Node.js à [LibSass](https://github.com/sass/libsass), la version C du préprocesseur de feuilles de style populaire, Sass. Il vous permet de compiler nativement des fichiers .scss en css à une vitesse incroyable et automatiquement via un middleware connect.
* [**NodeBB/NodeBB**](https://gitter.im/NodeBB/NodeBB?utm_source=blog&utm_medium=content&utm_campaign=node) — [NodeBB Forum Software](https://nodebb.org/) est alimenté par Node.js et construit sur une base de données Redis ou MongoDB. Il utilise des web sockets pour des interactions instantanées et des notifications en temps réel. NodeBB possède de nombreuses fonctionnalités modernes dès la sortie de la boîte, telles que l'intégration des réseaux sociaux et des discussions en streaming, tout en restant compatible avec les anciens navigateurs. Des fonctionnalités supplémentaires sont activées grâce à l'utilisation de plugins tiers.
* [**restify/node-restify**](https://gitter.im/restify/node-restify?utm_source=blog&utm_medium=content&utm_campaign=node) — [restify](http://restify.com/) est un framework utilisant un middleware de style [connect](https://github.com/senchalabs/connect) pour construire des API REST. Pour plus de détails, voir [http://restify.com](http://restify.com/)
* [**nodecg/nodecg**](https://gitter.im/nodecg/nodecg?utm_source=blog&utm_medium=content&utm_campaign=node) — NodeCG est un framework et une application de graphiques de diffusion. Il est principalement destiné aux diffuseurs [Twitch](http://twitch.tv/) utilisant [Open Broadcaster Software](https://obsproject.com/), mais peut être utilisé dans tout environnement capable de rendre du HTML, y compris CasparCG. NodeCG est basé sur le système de superposition développé pour l'événement caritatif [Tip of the Hats 2014](https://www.youtube.com/watch?v=x9PzBHgN29U).
* [**glazedio/glazed**](https://gitter.im/glazedio/glazed?utm_source=blog&utm_medium=content&utm_campaign=node) — Glazed est un modèle d'application pour l'itération rapide d'un produit minimal viable. Un framework simple mais opiniâtre [http://glazed.io](http://glazed.io/)
* [**igorklopov/enclose**](https://gitter.im/igorklopov/enclose?utm_source=blog&utm_medium=content&utm_campaign=node) — Compilez votre projet Node.js en un exécutable.
* [**nodemailer/nodemailer**](https://gitter.im/nodemailer/nodemailer?utm_source=blog&utm_medium=content&utm_campaign=node) — Envoyez des e-mails avec Node.JS — facile comme bonjour ! [http://nodemailer.com/](http://nodemailer.com/)
