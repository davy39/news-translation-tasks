En tant que développeur, j'apprends constamment de nouveaux concepts et outils.

Ce processus d'apprentissage commence généralement lorsque je code et que je suis bloqué.

Je fais une recherche rapide sur Google et je me retrouve généralement sur une page Stack Overflow qui ressemble à ceci :

![Image](https://cdn-media-1.freecodecamp.org/images/hvQdTN4kO0Sxmoq4uDLucKrxu3jZ8gohytaQ)
_Une parodie de page Stack Overflow._

Puis, à l'heure du déjeuner, j'entends tous mes amis parler avec enthousiasme de quelque chose appelé programmation fonctionnelle.

« Hmm... » je me dis. « Je n'ai appris que la programmation orientée objet, et j'ai un peu honte de demander ce qu'est la programmation fonctionnelle. »

Alors je me tourne vers mon ordinateur portable. « Qu'est-ce que la programmation fonctionnelle ? » je demande à Google.

Et Google me dit de lire un article Wikipedia qui ressemble à ceci :

![Image](https://cdn-media-1.freecodecamp.org/images/CJqyEI3LiaRgofSj3fcD50nAUyVYjJ0UpTtC)
_L'entrée Wikipedia pour la programmation fonctionnelle._

Wow — c'est beaucoup de grands mots, de liens vers des articles et de notes de bas de page. Entscheidungsproblem ? Est-ce même de l'anglais ?

Je voulais juste une explication « suffisamment bonne » de ce qu'est la programmation fonctionnelle.

### Présentation du Guide freeCodeCamp — des explications suffisamment bonnes pour les personnes occupées comme vous

![Image](https://cdn-media-1.freecodecamp.org/images/xJBuMBCYc7etOId6qe62yuKt5O9Q4bmbaLm5)
_Une capture d'écran du Guide freeCodeCamp_

Ce désir d'explications « suffisamment bonnes » nous a inspiré pour créer le [Guide freeCodeCamp](https://guide.freecodecamp.org/).

Le Guide est une référence consultable qui vise à couvrir tous les concepts liés au développement logiciel.

Les articles sont suffisamment simples pour que les non-anglophones puissent les comprendre. Ils sont suffisamment courts pour que les personnes occupées puissent les lire en prenant quelques gorgées de café.

![Image](https://cdn-media-1.freecodecamp.org/images/1f1vwYWwrlMlqmreaHfWcqx7DgCiXYXlKtuo)
_Un article du Guide sur la commande Git Clone._

Actuellement, le Guide freeCodeCamp contient environ 3 000 articles. Environ la moitié de ceux-ci sont des « ébauches » — des sujets que nous savons importants, mais pour lesquels nous n'avons pas encore eu le temps de rédiger.

Ces articles couvrent des sujets qui ne font pas partie des certificats de développement web freeCodeCamp, comme SQL :

![Image](https://cdn-media-1.freecodecamp.org/images/Ls-59LGftjTEjdDWbvn9RHOG0AdisGNLsd-S)
_Une capture d'écran de l'article du Guide freeCodeCamp sur la commande ALTER TABLE de SQL_

Et Python :

Ce projet entier est complètement gratuit et open source. Donc, si vous cherchez un moyen facile de commencer à contribuer à l'open source, c'est celui-ci !

### Comment contribuer à l'open source — directement dans votre navigateur — en améliorant le Guide freeCodeCamp

![Image](https://cdn-media-1.freecodecamp.org/images/HQ3e0eXvjQD4iPi1QvgsDycp673w9l8Pte9I)

Vous pouvez contribuer au Guide freeCodeCamp directement dans votre navigateur sur GitHub. Vous pouvez également le faire sans passer par le processus habituel de clonage du dépôt GitHub sur votre ordinateur local, d'installation des packages et d'exploration de la base de code.

Au lieu de cela, vous devez simplement choisir un sujet qui vous intéresse — peut-être venez-vous de découvrir un nouveau concept de design, par exemple — et écrire à ce sujet.

Actuellement, il y a des milliers de sujets et sous-sujets. La plupart de ceux-ci sont des « ébauches » de style Wikipedia avec rien de plus qu'un nom. Vous pouvez faire quelques recherches et aider à développer ces ébauches.

Voici un court gif montrant comment vous pouvez faire cela :

![Image](https://cdn-media-1.freecodecamp.org/images/IrHd3mpNZfhRI5d8T1--HftJFkMgbmZhVabW)
_Un Gif montrant comment faire une contribution open source à un article, directement dans votre navigateur._

Les étapes sont :

1. Explorez les [dossiers du guide](https://github.com/freeCodeCamp/guides/tree/master/src/pages/) et choisissez un sujet sur lequel vous voulez écrire.
2. Ouvrez le fichier index.md de ce dossier en double-cliquant dessus.
3. Cliquez sur le symbole du stylo dans le coin supérieur droit pour le modifier.
4. Apportez vos modifications. Vous pouvez intégrer des images, des vidéos YouTube, des CodePen, des REPL.it — tout ce dont vous avez besoin pour aider à enseigner un concept.
5. Faites défiler vers le bas et décrivez vos modifications dans le message de commit.
6. Assurez-vous que le bouton radio « Create a new branch for this commit and start a pull request » est sélectionné.
7. Cliquez sur « Commit Changes. »
8. Sur la page suivante, cliquez sur « Create Pull Request. »

Nous exécuterons quelques tests pour nous assurer que vos modifications n'ont rien cassé. Ensuite, l'un de nos mainteneurs vous donnera des commentaires sur votre article. Une fois que tout sera bon, nous fusionnerons votre pull request.

Votre contribution sera automatiquement déployée sur le Guide freeCodeCamp, où des millions de personnes pourront la lire et s'y référer.

![Image](https://cdn-media-1.freecodecamp.org/images/ZYMRqMxjsvpY0qYVQaKiJTTt2LKrtgBUKkVt)

Contribuer à nos articles de guide est un bon moyen de se lancer dans la contribution à l'open source. Vous pouvez également rejoindre notre [salon de discussion des contributeurs](https://gitter.im/FreeCodeCamp/Contributors) pour discuter avec d'autres contributeurs et poser des questions.

### Comment nous avons construit le Guide freeCodeCamp

Au cours des derniers mois, nous avons utilisé une variété d'outils pour construire cette application rapide et monopage :

* React et Redux
* [Gatsby.js](https://github.com/gatsbyjs/gatsby) — un outil pour créer des sites web statiques rapides
* [Elasticsearch](https://www.elastic.co/products/elasticsearch) — la référence en matière de recherche en texte intégral
* [Netlify](https://www.netlify.com/) — un outil d'hébergement de sites web statiques haute performance
