Par Andrew Brown

Si vous êtes un développeur d'applications web cherchant à apprendre comment exploiter le cloud ou souhaitez devenir un développeur cloud, la certification ExamPro Cloud Development Environment (EXP-CDE-01) est la première étape.

## Qu'est-ce qu'un Environnement de Développement Cloud ? 

Un Environnement de Développement Cloud (CDE) est :

* un éditeur de code qui peut fonctionner dans le cloud et
* l'environnement de calcul sous-jacent fonctionne dans le cloud.

Cela diffère des environnements traditionnels, qui fonctionnent sur votre machine locale et utilisent votre puissance de calcul locale.

Un Environnement de Développement Cloud présente de nombreux avantages :

* Tout ce dont vous avez besoin est un navigateur web et une connexion internet
* Très facile de collaborer à distance dans le même environnement au sein d'une équipe
* Réduit la configuration de l'environnement de développement pour les nouveaux membres de l'équipe
* Bien adapté aux technologies cloud qui nécessitent des environnements très différents selon le cas d'utilisation
* et plus...

## Qu'est-ce que la Certification ExamPro Cloud Developer Environment (EXP-CDE-01) ? 

Cette certification cloud est conçue pour vous apprendre à exploiter un Environnement de Développement Cloud pour les rôles de Développeur Web, Développeur Cloud ou Ingénieur Cloud.

Acquérir des connaissances sur les Environnements de Développement Cloud est la première étape essentielle pour devenir un Développeur Cloud.

Cette certification est **également connue sous le nom de Certification Gitpod**, car elle se concentre fortement sur l'utilisation de l'open-source et du Software as Service (SaaS) connu sous le nom de Gitpod.

![Image](https://www.freecodecamp.org/news/content/images/2022/03/code-developer-enviroment.png)
_Une image du badge et de la certification pour l'EXP-CDE-01_

La raison pour laquelle Gitpod a été choisi est la suivante :

* Il dispose d'un généreux niveau gratuit
* Il est open-source (licence MIT)
* Il est agnostique en termes de fournisseurs (peut fonctionner avec Gitlab, Github, Bitbucket)
* Son éditeur de code est Visual Studio Code, l'éditeur de code le plus populaire
* Il est le leader du marché pour les Environnements de Développement Cloud

Les Environnements de Développement Cloud (CDE) ont des ensembles de fonctionnalités très similaires, donc les concepts de CDE appris sur Gitpod seront transférables à d'autres CDE.

### Qu'est-ce qu'une Certification ExamPro ?

ExamPro est l'entreprise qui a produit plusieurs cours d'étude vidéo gratuits publiés sur la chaîne YouTube freeCodeCamp pour les principaux fournisseurs de services cloud tels que AWS, Azure, GCP, Kubernetes pour des certifications comme le : CLF-C01, SAA-C02, SOA-C01, DVA-C01, AZ-900, DP-900, AI-900, AZ-104, GCP-CDL, le KCNA et plus.

ExamPro produit sa propre certification cloud qui correspond à des rôles cloud spécifiques. ExamPro est l'autorité de délivrance et entend réduire considérablement le coût des certifications cloud pour rendre les connaissances cloud plus accessibles.

## Qu'est-ce qu'un Développeur Cloud ?

Un Développeur Cloud est un rôle de développeur d'applications web qui intègre des services cloud lors de la création d'applications web.

Les Développeurs Cloud peuvent gagner **30 % de plus** que les développeurs d'applications web car les développeurs cloud savent comment tirer pleinement parti du cloud.

Ainsi, les applications web construites par des développeurs cloud seront plus disponibles, scalables, durables, sécurisées et permettent aux petites organisations de rivaliser sur le marché mondial contre les géants technologiques établis.

Le terme développeur cloud devient de plus en plus populaire, de nombreuses offres d'emploi qui recherchent des développeurs web ou des développeurs full-stack sont en réalité des rôles de développeur cloud. 

## Est-il difficile de devenir Développeur Cloud ?

Les compétences d'un Développeur Cloud sont similaires à celles d'un Développeur d'Applications Web, cependant, une grande partie des fonctionnalités de l'application web peut être externalisée vers des services cloud.

Ainsi, un Développeur Cloud doit avoir les connaissances suivantes :

* **Environnements de Développement Cloud** - avoir un environnement de développement éphémère spécialisé adapté pour travailler avec différents types de SDK et outils de développement
* **Programmation Cloud** - une connaissance plus large de plusieurs langages de programmation, car la disponibilité des langages des SDK de services cloud varie selon le service
* **Intégration d'Applications** - Les services cloud doivent être intégrés dans votre application web ou communiquer avec d'autres services. Un développeur cloud écrit du code ou de la configuration pour intégrer ces éléments ensemble
* **Déploiement Cloud** - Un développeur cloud a la responsabilité supplémentaire de comprendre comment les pipelines de déploiement fonctionnent dans un environnement cloud

Un développeur cloud bénéficie grandement de la connaissance de l'infrastructure et de l'architecture sous-jacentes sur lesquelles son application sera déployée. Cependant, un Développeur Cloud n'est pas responsable du cycle de vie, de l'automatisation ou de la maintenance de l'infrastructure.

Un Développeur Cloud bénéficie de la connaissance des éléments suivants concernant l'infrastructure cloud :

* **Virtualisation** — L'endroit le plus courant pour exécuter une application web est sur une machine virtuelle et les environnements virtuels sont la base de l'architecture moderne telle que les conteneurs et le serverless.
* **Linux** — Le système d'exploitation le plus courant pour exécuter les serveurs d'applications web, un Développeur Cloud doit connaître les bases de la mise en réseau Linux, le dépannage Linux
* **Windows** — Windows Server est très courant dans l'IT traditionnel, et avoir des connaissances générales sur Windows aux côtés de Linux permet à un Développeur Cloud de maximiser l'utilisation des services cloud qui fonctionnent sur Windows.
* **Conteneurs** — L'étape suivante après les Machines Virtuelles, les Conteneurs permettent une portabilité et un isolement supplémentaires, et encouragent l'architecture de micro-services qui est plus alignée avec les services cloud.
* **Serverless** — Une architecture moderne où les développeurs peuvent se concentrer sur le code, et tout le reste est géré par le fournisseur de cloud.

## Aperçu de la Certification ExamPro Cloud Developer Environment

Les questions de l'examen sont divisées en les domaines suivants :

* 7 % Domaine 1 : Concepts de l'Environnement de Développement Cloud
* 5 % Domaine 2 : Intégrations Gitpod
* 35 % Domaine 3 : Configurations Gitpod
* 35 % Domaine 4 : Workflows Gitpod
* 18 % Domaine 5 : Gitpod.io, Gitpod Enterprise, Gitpod OSS
