Par Ayo Isaiah

Si vous commencez tout juste à apprendre à coder, l'une des premières choses à faire, après avoir décidé ce que vous voulez apprendre, est de choisir un éditeur de texte pour écrire du code.

Un éditeur de texte est un programme qui vous aide à écrire du texte brut (sans aucun formatage) et à l'enregistrer dans un fichier. Un bon exemple est l'éditeur Notepad sur Windows.

![Image](https://cdn-media-1.freecodecamp.org/images/RBzyZbNYXSB1JaPThmx3SQLSyIcPVRnRSdq-)
_Notepad dans Windows 8_

Au début de votre parcours en codage, vous découvrirez rapidement que l'éditeur de texte est l'endroit où vous passerez la plupart de votre temps — puisque le travail fondamental de tout programmeur est d'écrire et de modifier du code source. C'est pourquoi il est nécessaire d'en choisir un qui vous convient et qui soutient vos efforts d'apprentissage.

Tous les éditeurs de texte ne sont pas adaptés à la programmation, comme le Notepad mentionné précédemment, car il ne fait rien pour faciliter ou automatiser l'édition du code source.

Il existe des éditeurs de texte spécialement conçus pour écrire et modifier du code source. Ceux-ci sont appelés éditeurs de code. Un éditeur de code peut être un programme autonome ou intégré à un environnement de développement intégré (IDE).

Nous avons également des éditeurs de code qui fonctionnent dans le navigateur. [CodePen](https://codepen.io/), [Web Maker](https://webmakerapp.com/), et [Plunker](https://plnkr.co/) en sont des exemples notables. Vous êtes peut-être déjà familier avec certains de ces éditeurs basés sur le web, car de nombreux programmes de codage en ligne comme [freeCodeCamp](https://www.freecodecamp.org/) recommandent leur utilisation.

Cependant, si vous souhaitez passer votre codage au niveau supérieur, vous devez envisager d'utiliser un éditeur de bureau qui va au-delà des capacités de la plupart des outils basés sur le web.

Il est facile de se sentir submergé lorsque l'on regarde les options disponibles, car il y en a tant. Demandez à cinq personnes, et vous obtiendrez probablement cinq réponses différentes.

Si vous débutez, vous devez choisir un éditeur qui vous soutient en tant que débutant. Apprendre à coder est déjà suffisamment difficile, donc vous pouvez utiliser toute l'aide possible.

Le but de cet article est de vous aider à décider quel éditeur de texte utiliser lorsque vous apprenez à coder, avec un accent particulier sur la programmation en JavaScript.

### Ce qu'il faut rechercher dans un éditeur de code

Tous les éditeurs de code peuvent écrire du texte et l'enregistrer dans un fichier. Au-delà de cela, le type d'éditeur que vous choisissez dépend entièrement de vos préférences personnelles. Cela dit, il y a quelques choses que votre éditeur doit avoir pour être considéré comme suffisamment bon pour développer en JavaScript.

#### Colorisation syntaxique

Presque tous les éditeurs de code supportent la colorisation syntaxique du code source en différentes couleurs en fonction du langage avec lequel vous travaillez. Cela améliore la lisibilité et vous aide à repérer les erreurs de syntaxe beaucoup plus rapidement.

#### Complétion automatique

La complétion automatique de code est une fonctionnalité offerte par de nombreux éditeurs de code qui aide à accélérer le processus d'édition et à réduire les fautes de frappe en complétant les noms de variables, mots-clés, classes et méthodes dans la portée de votre programme.

![Image](https://cdn-media-1.freecodecamp.org/images/4FLSUoyosYklNvYSooN9Fb2R-RcZCjjrWJBp)
_Complétion automatique de code dans Visual Studio Code_

Si vous débutez, je recommande de désactiver la complétion automatique dans votre éditeur et de tout taper manuellement à la place, afin de mémoriser la syntaxe plus rapidement.

Cependant, une fois que vous avez maîtrisé tous les mots-clés de base et les méthodes intégrées, je ne vois pas beaucoup de valeur à tout taper vous-même. La complétion automatique devient inestimable à ce stade.

#### Support pour ES2015+ ainsi que pour les frameworks et bibliothèques populaires

Un bon éditeur doit également fournir un support pour les dernières fonctionnalités de JavaScript peu après leur sortie, et avoir un écosystème de plugins solide afin que vous puissiez obtenir un support pour les frameworks et bibliothèques les plus populaires.

Avec ces considérations en tête, examinons quelques options que vous pouvez utiliser si vous débutez avec JavaScript et le développement web en général.

Pour vous faciliter la tâche, je les ai classés du plus recommandé au moins recommandé. Ce classement est subjectif, donc je vous encourage à tirer vos propres conclusions après avoir essayé chacun d'eux.

Tous les éditeurs listés ici sont multiplateformes sur Windows, macOS et Linux, donc vous n'avez pas à vous soucier de la compatibilité avec votre système d'exploitation préféré.

### Visual Studio Code — Gratuit

[Visual Studio Code](https://code.visualstudio.com/) (VS Code), à ne pas confondre avec [Visual Studio](https://www.visualstudio.com/), est le meilleur éditeur de texte pour les développeurs JavaScript en ce moment. Bien qu'il soit relativement jeune par rapport à d'autres éditeurs populaires, ayant été lancé en 2015, il s'est amélioré à pas de géant au fil des ans. Tellement, en fait, qu'il a été listé comme l'éditeur de texte le plus populaire parmi les développeurs web dans la dernière [enquête de Stack Overflow](https://insights.stackoverflow.com/survey/2018/#development-environments-and-tools).

![Image](https://cdn-media-1.freecodecamp.org/images/GaHpoiCxAAxz5QK9tHGzeJ9riymzJPONnh4H)
_Visual Studio Code a devancé Visual Studio en tant qu'outil d'environnement de développement le plus populaire pour les développeurs web_

VS Code est bien plus qu'un simple éditeur de texte. Certaines personnes disent qu'il est plus proche d'un IDE que d'un éditeur de texte, car il fournit de nombreuses fonctionnalités que vous trouverez typiquement dans un IDE.

Parmi ses fonctionnalités phares, on trouve des outils de débogage intégrés, une complétion de code intelligente ([IntelliSense](https://code.visualstudio.com/docs/editor/intellisense)), un terminal intégré et un support natif pour de nombreuses opérations Git courantes. En tant que débutant, vous pouvez tirer parti de ces fonctionnalités pour produire du code fonctionnel à un rythme beaucoup plus rapide.

La personnalisation est un autre domaine où Visual Studio Code excelle. Il supporte des milliers de plugins et de thèmes que vous pouvez installer pour améliorer encore ses capacités.

### Sublime Text 3 — 80 $

[Sublime Text 3](https://www.sublimetext.com/) était le premier éditeur que j'ai utilisé lorsque j'apprenais à coder, donc j'ai un faible pour lui. C'est un excellent éditeur qui reste populaire parmi les développeurs web grâce à sa simplicité et sa facilité d'utilisation.

Bien que son support intégré pour JavaScript soit assez limité par rapport à VS Code, vous pouvez l'augmenter en installant des plugins. Il est également très rapide et n'utilise pas beaucoup de mémoire — c'est pourquoi je le recommande souvent pour les personnes dont les ordinateurs ont moins de 4 Go de RAM.

Bien que Sublime soit coûteux à 80 $ pour une licence utilisateur unique, il peut être téléchargé et évalué sans frais. Vous recevez simplement un message contextuel de temps en temps vous rappelant d'acheter une licence pour une utilisation continue.

### Atom — Gratuit

[Atom](https://atom.io/) a été introduit en 2014 par GitHub en tant qu'alternative gratuite et open source aux outils propriétaires comme Sublime Text et Text Mate. Bien qu'il ait gagné un suivi fervent, il n'a jamais atteint les niveaux de popularité de Sublime et a maintenant été dépassé par VS Code, qui a été lancé un an plus tard.

Cela ne signifie pas qu'Atom est un mauvais éditeur — loin de là. Il est accessible pour les débutants et ne nécessite aucune configuration pour commencer. Tout comme VS Code, il est également extensible via des plugins et supporte de nombreuses fonctionnalités similaires à celles d'un IDE, ce qui peut augmenter votre vitesse et votre efficacité lors de l'écriture de code.

### Vim — Gratuit

[Vim est mon éditeur de texte actuel de choix](https://freshman.tech/from-vscode-to-vim/) après diverses périodes avec d'autres éditeurs de code. Bien que j'utilise et aime Vim, je hésiterais à le recommander pour les personnes qui commencent tout juste à apprendre à coder, car il a une approche complètement différente de l'édition de texte par rapport aux autres éditeurs de code grand public. Cela signifie que vous devez investir du temps pour apprendre comment il fonctionne, ce qui peut vous distraire de l'apprentissage réel du codage.

![Image](https://cdn-media-1.freecodecamp.org/images/mZXP6hoenEhiGlQ74RpIWeg2Xxlyv5ZrsUwZ)

Si vous êtes occupé à apprendre à programmer, il est préférable d'utiliser une interface que vous connaissez déjà afin de ne pas vous éloigner de la programmation réelle. De plus, Vim manque de nombreuses fonctionnalités agréables que des éditeurs comme VS Code ou Atom fournissent par défaut. Bien qu'il soit possible d'ajouter plus de fonctionnalités à Vim en utilisant des plugins, le processus pour le faire est un peu plus impliqué.

Si vous avez déjà atteint un niveau de compétence raisonnable dans le langage que vous apprenez et que vous êtes intéressé à essayer Vim, vous pouvez vous inspirer de la manière dont [Yehuda Katz a pu passer à Vim sans perdre de productivité](http://yehudakatz.com/2010/07/29/everyone-who-tried-to-convince-me-to-use-vim-was-wrong/) et continuer à partir de là.

### WebStorm — 129 $/an

[WebStorm](https://www.jetbrains.com/webstorm/) est une autre option intéressante que de nombreux développeurs JavaScript recommandent, bien qu'il s'agisse d'un IDE complet et non pas seulement d'un éditeur de texte. C'est aussi l'option la plus chère que je connaisse, bien que vous puissiez [l'obtenir gratuitement](https://www.jetbrains.com/student/) ainsi que d'autres produits JetBrains si vous êtes étudiant.

Je n'ai pas eu d'expérience personnelle avec WebStorm, mais la plupart des critiques sont rapides à louer son support approfondi pour JavaScript et son écosystème, comme on pourrait s'y attendre d'un IDE sophistiqué. En revanche, il a une courbe d'apprentissage plus raide que les éditeurs de texte autonomes et de nombreuses fonctionnalités dont vous n'avez pas vraiment besoin en tant que débutant.

Je pense que WebStorm pourrait être un excellent outil à apprendre lorsque vous avez un peu plus d'expérience, mais je ne vois aucune raison de commencer avec lui.

### Il n'y a pas de mauvais choix

Quel que soit l'éditeur que vous choisissez parmi ceux-ci, c'est bien. Tous offrent de bons environnements pour écrire du JavaScript, donc ce n'est vraiment pas un problème si vous décidez de ne pas opter pour le premier choix en faveur de quelque chose d'autre.

Ce que je déconseillerais, c'est de passer trop de temps à apprendre des outils compliqués comme Vim ou WebStorm. En tant que débutant, vous devriez vous concentrer davantage sur la programmation elle-même — c'est pourquoi j'ai listé Visual Studio Code, Sublime Text et Atom comme mes choix principaux, car ils sont tous accueillants pour les nouveaux venus. Je vous suggère d'essayer les trois et de choisir celui avec lequel vous êtes le plus à l'aise.
