Par Johan Rin

Vous avez probablement entendu parler d'Amazon Web Services et du défi #AWSCertified. Mais saviez-vous que Microsoft est également un fournisseur de cloud et propose également des certifications professionnelles ?

Après avoir réussi [3 certifications AWS](https://www.freecodecamp.org/news/how-i-passed-the-aws-certified-developer-associate-exam/), j'ai voulu apprendre de nouvelles choses et commencer avec un autre fournisseur de cloud : Azure.

Mes objectifs étaient de comprendre comment fonctionne Azure et également de passer une nouvelle certification. J'ai décidé de passer le premier examen, AZ-900 : Microsoft Azure Fundamentals, pour me donner une transition en douceur d'AWS à Azure.

Dans cet article, je vais expliquer les certifications Azure ainsi que comment devenir Microsoft Certified.

Commençons.

## Qu'est-ce qu'Azure ?

Avant d'expliquer les certifications Microsoft, je vais brièvement présenter Azure.

Azure est la plateforme de cloud computing de Microsoft, qui vous donne la liberté de créer, gérer et déployer des applications sur un réseau mondial massif en utilisant vos outils et frameworks préférés.

Les capacités d'Azure peuvent être divisées en 8 catégories principales :

1. Services de calcul
2. Stockage cloud
3. Réseautage
4. Hébergement d'applications
5. Intelligence artificielle
6. Internet des objets
7. Intégration
8. Sécurité

## Certifications Microsoft

Microsoft propose des certifications sur ses différents produits : Azure, Dynamics 365, Microsoft 365, Office, Power Platform, SQL Server et Windows.

Si nous considérons uniquement le produit Azure, nous avons 13 certifications regroupées en 2 catégories :

* Applications et infrastructure
* Données et IA

Faisant référence aux rôles professionnels techniques suivants :

* Administrateur
* Ingénieur IA
* Ingénieur de données
* Scientifique des données
* Administrateur de base de données
* Développeur
* Ingénieur DevOps
* Ingénieur de sécurité
* Architecte de solutions

## Focus sur AZ-900 : Microsoft Azure Fundamentals

Microsoft indique que l'objectif de cet examen est de démontrer un niveau de connaissance fondamental des services cloud et de la manière dont ces services sont fournis avec Azure.

Aucune expérience technique préalable en informatique n'est requise. Les candidats ayant un parcours technique ou non technique peuvent passer l'examen.

## Quelles compétences sont évaluées pendant l'examen ?

D'après le plan des compétences, les compétences testées pendant cet examen sont les suivantes :

### Décrire les concepts du cloud (15-20%)

* Décrire les avantages et considérations de l'utilisation des services cloud
* Décrire les différences entre Infrastructure-as-a-Service (IaaS), Platform-as-a-Service (PaaS) et Software-as-a-Service (SaaS)
* Décrire les différences entre les modèles de cloud public, privé et hybride

### Décrire les services principaux d'Azure (30-35%)

* Décrire les composants architecturaux principaux d'Azure
* Décrire certains des produits principaux disponibles dans Azure
* Décrire certaines des solutions disponibles sur Azure
* Décrire les outils de gestion Azure

### Décrire la sécurité, la confidentialité, la conformité et la confiance (25-30%)

* Décrire la sécurisation de la connectivité réseau dans Azure
* Décrire les services d'identité principaux d'Azure
* Décrire les outils et fonctionnalités de sécurité d'Azure
* Décrire les méthodologies de gouvernance Azure
* Décrire les options de surveillance et de reporting dans Azure
* Décrire les normes de confidentialité, de conformité et de protection des données dans Azure

### Décrire la tarification Azure, les accords de niveau de service et les cycles de vie (20-25%)

* Décrire les abonnements Azure
* Décrire la planification et la gestion des coûts
* Décrire les accords de niveau de service (SLA) Azure
* Comprendre le cycle de vie des services dans Azure

## Combien coûte l'examen ?

Si vous avez eu la chance de participer à la journée de formation virtuelle Microsoft Azure : Fundamentals, vous avez reçu un bon vous permettant de passer l'examen gratuitement.

Sinon, cela vous coûtera 99 $ USD, et vous pouvez [planifier votre examen en ligne](https://docs.microsoft.com/en-us/learn/certifications/exams/az-900) avec Pearson VUE.

![Image](https://www.freecodecamp.org/news/content/images/2020/05/Capture-d-e-cran-2020-05-16-a--18.02.49.png)

### À quoi ressemble l'examen ?

D'après la [FAQ](https://www.microsoft.com/en-us/learning/certification-exam-policies.aspx), vous avez 60 minutes pour répondre à entre 40 et 60 questions. Mais je n'avais que 32 questions dans mon examen. Ce fut une agréable surprise !

L'examen présente plusieurs [types de questions](https://www.microsoft.com/en-us/learning/certification-exams.aspx) :

* Choix multiples sous différentes formes : une réponse correcte et trois réponses incorrectes, sélectionner la description correcte, deux ou plusieurs réponses correctes parmi cinq options ou plus
* Glisser-déposer : déplacer la réponse vers la description correcte
* Zone interactive : répondre à des questions (capture d'écran, image ou diagramme) sous forme graphique

![Image](https://www.freecodecamp.org/news/content/images/2020/05/image-118.png)

## Comment se préparer à l'examen

Il existe une tonne de ressources pour vous aider à préparer votre examen. Microsoft recommande d'utiliser sa plateforme en ligne, Microsoft Learn, qui propose des parcours d'apprentissage et des modules que vous pouvez lire à votre propre rythme gratuitement.

![Image](https://www.freecodecamp.org/news/content/images/2020/05/Capture-d-e-cran-2020-05-17-a--10.45.43-1.png)

Le [parcours d'apprentissage des fondamentaux d'Azure](https://docs.microsoft.com/en-us/learn/paths/azure-fundamentals/) est le cours recommandé pour préparer l'examen AZ-900. C'est un cours de 10 heures contenant 12 modules :

1. Concepts du cloud - Principes de l'informatique en cloud
2. Créer un compte Azure
3. Services cloud principaux - Introduction à Azure
4. Services cloud principaux - Architecture Azure et garanties de service
5. Services cloud principaux - Gérer les services avec le portail Azure
6. Services cloud principaux - Options de calcul Azure
7. Services cloud principaux - Options de stockage de données Azure
8. Services cloud principaux - Options de réseautage Azure
9. Sécurité, responsabilité et confiance dans Azure
10. Appliquer et surveiller les normes d'infrastructure avec Azure Policy
11. Contrôler et organiser les ressources Azure avec Azure Resource Manager
12. Prédire les coûts et optimiser les dépenses pour Azure

Si vous avez l'habitude de regarder les cours d'Andrew Brown, vous serez déçu par Microsoft Learn. Les modules sont en texte brut sans aucune vidéo. Préparez-vous à lire et à prendre des notes !

![Image](https://www.freecodecamp.org/news/content/images/2020/05/Capture-d-e-cran-2020-05-17-a--11.10.45.png)

Je vous recommande fortement de :

1. Créer un [compte gratuit](https://azure.microsoft.com/en-us/)
2. Lire l'aperçu des services Azure
3. Pratiquer par vous-même avec les guides de démarrage rapide d'Azure
4. Suivre vos progrès avec le plan des compétences
5. Passer des examens pratiques

Microsoft explique ce que vous devez savoir pour chaque service dans la documentation. Les explications incluent, entre autres, un aperçu du service et des guides de démarrage rapide que vous pouvez expérimenter pour mieux comprendre.

![Image](https://www.freecodecamp.org/news/content/images/2020/05/Capture-d-e-cran-2020-05-17-a--11.57.13.png)

Note : utiliser le portail Azure est suffisant pour pratiquer car ce n'est pas un examen approfondi.

## Différences avec l'examen AWS Certified Cloud Practitioner

Vous ne devriez pas sous-estimer la quantité de préparation dont vous aurez besoin simplement parce que vous avez déjà réussi l'examen AWS Certified Cloud Practitioner.

Vous devez commencer depuis le début car 80 % de l'examen valide l'écosystème Azure, qui est différent d'AWS.

Je ne veux pas que vous fassiez la même erreur que moi. Ne comparez pas chaque nouveau service que vous apprenez avec AWS. Cela vous confondra plus que tout autre chose. Certains services Azure n'ont pas leur équivalent dans AWS et vice versa.

Oubliez AWS (pendant que vous étudiez) pour apprendre Azure !
