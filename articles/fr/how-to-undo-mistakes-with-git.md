Oh non. Vous venez d'intégrer accidentellement vos fichiers dans la branche master alors que vous vouliez les intégrer dans la branche de pré-production. Que faire maintenant ?

Presque tout le monde qui utilise Git commet une erreur à un moment donné. Qu'il s'agisse de fusionner les mauvais fichiers, de supprimer la mauvaise branche ou de quelque chose de pire, il est important de savoir comment annuler les erreurs avec Git.

Nous avons publié un cours accéléré sur la chaîne YouTube freeCodeCamp.org qui vous apprendra 17 façons différentes d'annuler les erreurs dans Git.

Tobias Günther a développé ce cours. Tobias travaille avec Git depuis de nombreuses années. Il aide même à créer Tower, qui est un client GUI Git populaire.

De nombreux développeurs ne connaissent pas toutes les façons dont ils peuvent utiliser Git pour annuler les erreurs. Ce cours vous apprendra ce que Git peut faire et comment Git le fait.

La première moitié du cours couvre des cas plus courants qui sont plus faciles à récupérer. La deuxième moitié enseigne des erreurs plus compliquées à corriger. Mais il est possible d'annuler presque toutes les erreurs dans Git.

Voici les sujets abordés dans ce cours :

* Utiliser Git comme filet de sécurité
* Abandonner toutes les modifications locales dans un fichier
* Restaurer des fichiers supprimés
* Abandonner des morceaux / lignes dans un fichier
* Abandonner toutes les modifications locales
* Corriger le dernier commit
* Que signifie "réécrire l'historique"
* Revenir à un commit au milieu
* Réinitialiser à une ancienne révision
* Réinitialiser un fichier à une ancienne révision
* Le Reflog
* Récupérer des commits supprimés
* Récupérer des branches supprimées
* Déplacer un commit vers une nouvelle branche
* Déplacer un commit vers une autre branche
* Rebase interactif : aperçu de ce que vous pouvez faire
* Rebase interactif : comment cela fonctionne en général
* Modifier les anciens messages de commit avec le Rebase interactif
* Supprimer des commits avec le Rebase interactif
* Fusionner plusieurs commits en un seul avec le Rebase interactif
* Ajouter des modifications à un ancien commit avec le Rebase interactif
* Diviser/modifier un ancien commit avec le Rebase interactif
