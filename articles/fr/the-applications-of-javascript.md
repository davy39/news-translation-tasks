Par Abiola Farounbi

Selon les résultats d'une enquête de Stack Overflow en 2020, JavaScript est le langage de programmation le plus populaire au monde, avec plus de 63 % des développeurs qui l'utilisent.

Il possède le plus grand dépôt de packages open-source au monde (npm) et il existe également de nombreux frameworks et bibliothèques différents qui sont construits avec lui.

JavaScript est facile à prendre en main car il ne nécessite aucune installation pour commencer à écrire du code.

À l'origine, JavaScript ne pouvait être exécuté que sur les navigateurs web. Cependant, en raison de sa popularité, il peut maintenant être exécuté localement et sur le serveur. Ainsi, JavaScript peut être utilisé pour construire des applications complètes, à la fois front-end et back-end.

Dans cet article, je vais expliquer certains des domaines dans lesquels JavaScript peut être utilisé.

## Développement Web Front-end

Le développement front-end concerne la partie d'une page web que les utilisateurs peuvent voir et avec laquelle ils peuvent interagir. Il est également connu sous le nom de développement côté client.

Il y a trois technologies principales que vous devez connaître en matière de développement front-end : HTML, CSS et JavaScript.

Le HTML (HyperText Markup Language) a trait à la structure et au contenu du site, le CSS (Cascading Style Sheets) gère le style.

Et enfin, JavaScript rend la page web dynamique. Et cela peut être fait de différentes manières, telles que :

### Présentation

Vous pouvez créer des carousels et des curseurs impressionnants sur une page web facilement en utilisant certaines bibliothèques JavaScript comme [Reveal JS](https://revealjs.com/), [Swiper JS](https://swiperjs.com/), et [Owl JS](https://owlcarousel2.github.io/OwlCarousel2/).

Avec ces bibliothèques, vous pouvez personnaliser la présentation de votre page web rapidement et sans tracas.

### Animations

L'une des façons de rendre votre site web vivant et attrayant est d'utiliser des animations. Vous pouvez les utiliser sur une page web de différentes manières, telles que par des mouvements, des transformations d'objets, et bien d'autres choses incroyables.

Il existe un certain nombre de bibliothèques d'animation parmi lesquelles choisir, telles que [Anime JS](https://animejs.com/), [Greensock JS](https://greensock.com/gsap/), [Mo JS](https://mojs.github.io/), et [Animate On Scroll](https://michalsnik.github.io/aos/). Ce sont toutes des bibliothèques construites en JavaScript qui facilitent la création de vos animations en utilisant de plus petits blocs de code.

### Applications Monopage (SPAs)

JavaScript peut également être utilisé pour construire des Applications Monopage (SPAs). Celles-ci impliquent des interactions dynamiques complexes, et vous pouvez les construire à l'aide de frameworks et de bibliothèques tels que [React JS](https://reactjs.org/), [Polymer JS](https://polymer-library.polymer-project.org/), [Vue JS](https://vuejs.org/), et [Angular JS](https://angular.io/).

[Voici](https://frontendmasters.com/books/front-end-handbook/2019/) une ressource incroyable pour utiliser JavaScript pour le développement web front-end.

freeCodeCamp propose également un programme complet basé sur JavaScript que vous pouvez consulter [ici](https://www.freecodecamp.org/learn/).


## Développement Web Back-end

Le développement back-end concerne la partie de la page web avec laquelle les utilisateurs n'interagissent pas visuellement. Cela est également connu sous le nom de développement côté serveur.

Le développement back-end consiste à créer des bases de données, des APIs, des serveurs de fichiers, des services cloud, et plus encore.

Et avec la création de Node.js, un moteur d'exécution JavaScript, vous pouvez maintenant utiliser JavaScript pour le développement web back-end également.

Différents frameworks JavaScript ont été construits pour simplifier le processus de développement web back-end. Certains d'entre eux incluent :

- [Express Js](https://expressjs.com/) : Express est le framework d'application web le plus connu basé sur Node.js. Il est principalement utilisé pour développer des applications web et des APIs REST.

- [Fastify](https://www.fastify.io/) : Il s'agit d'un framework web côté serveur minimaliste avec un accent massif sur l'expérience du développeur et les performances de l'application.

- [Koa](https://koajs.com/) : Il s'agit d'un framework de middleware léger, moderne, expressif et robuste pour les applications web et les APIs.

[Voici](https://www.youtube.com/playlist?list=PLlxmoA0rQ-LyMoEYRyd1L9oDaTRXOXdZa) une ressource incroyable pour utiliser JavaScript pour le développement web back-end.


## Développement de Jeux

Les jeux sont l'un des projets les plus amusants que vous pouvez créer, car vous et l'utilisateur final pouvez en profiter.

Il existe différentes façons de créer des jeux avec JavaScript en fonction de la complexité du jeu.

Vous pouvez construire des jeux 2D simples à partir de zéro en utilisant Html, CSS et JavaScript vanilla. Voici une excellente ressource qui peut vous aider à commencer avec cela :

- [Apprendre JavaScript en construisant 7 jeux par Ania Kubow](https://www.freecodecamp.org/news/learn-javascript-by-building-7-games-video-course/).

Pour construire des conceptions de jeux complexes, ainsi que des jeux plus efficaces et plus rapides, vous pouvez utiliser un moteur de jeu JavaScript pour faciliter le processus.

Certains de ces moteurs sont [Phaser Js](http://phaser.io/), [Bablyon Js](http://www.babylonjs.com/), et [Lime Js](http://www.limejs.com/). Vous pouvez en trouver plus [ici](https://github.com/collections/javascript-game-engines).

Ces moteurs de jeu sont utiles car vous écriverez moins de code pour obtenir les résultats.

Voici quelques excellentes ressources pour vous aider à commencer à construire des jeux en JavaScript :

Apprenez à construire des jeux avec JS dans [ce cours vidéo](https://www.freecodecamp.org/news/intro-to-game-development-with-javascript/) de freeCodeCamp.

Et voici un [mini-cours](https://academy.zenva.com/product/html5-game-phaser-mini-degree/?zva_src=gda-how-to-make-html5-game) de développement de jeux JS.


## Développement d'Applications Mobiles

Le développement d'applications mobiles fait référence au processus d'écriture de logiciels qui fonctionnent sur un appareil mobile. Il implique le développement d'applications qui peuvent être utilisées sur des appareils mobiles.

Vous pouvez utiliser JavaScript pour développer des applications mobiles. Différents frameworks et bibliothèques JavaScript ont été construits pour créer des applications natives, multiplateformes et hybrides en utilisant JavaScript. Certains d'entre eux sont :

- [React Native](https://reactnative.dev/) : React Native est un framework open-source qui peut être utilisé pour construire des applications natives multiplateformes. React Native utilise un style de programmation déclaratif et des composants réutilisables pour les interfaces utilisateur.

- [NativeScript](https://nativescript.org/) : NativeScript est l'un des frameworks multiplateformes les plus connus qui vous aide à développer des applications mobiles vraiment natives pour les plateformes Android et iOS. Étant un framework open-source, les développeurs peuvent ajuster l'interface utilisateur pour plusieurs écrans et appareils et utiliser également diverses dépendances.

- [Ionic](https://ionicframework.com/) : Ionic est un autre framework JavaScript populaire pour construire des applications hybrides. Pour les développeurs familiarisés avec les technologies web et le développement d'applications web, comprendre la structure d'une application Ionic est simple.

Sélectionner le bon framework JavaScript pour votre projet dépend de l'application que vous construisez et également de la plateforme à laquelle elle est destinée. Assurez-vous donc de sélectionner un framework basé sur votre cas d'utilisation si possible.

## Conclusion

JavaScript est un langage polyvalent et peut être appliqué dans de nombreux domaines différents de la technologie. Je recommande vivement d'apprendre JavaScript comme langage de programmation si vous êtes intéressé par l'un des domaines que j'ai discutés ci-dessus.
