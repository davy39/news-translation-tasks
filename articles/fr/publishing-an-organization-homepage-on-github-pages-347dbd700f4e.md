Par Daniel Deutsch

J'organise une rencontre locale de groupe de [FreeCodeCamp](https://www.freecodecamp.org/) appelée [FreeCodeCamp Vienna](https://www.meetup.com/Free-Code-Camp-Vienna/). Il y a des mois, nous avons décidé de créer une simple page d'accueil pour représenter le groupe et ses progrès en ligne. Mais nous n'avons jamais semblé trouver le temps de le faire.

Suivant l'idée « mieux vaut une page d'accueil rapide que pas de page d'accueil du tout », j'ai décidé d'en créer une et de la publier sur GitHub Pages. Cela nous permet d'héberger notre page statique et sert également d'exemple parfait pour montrer à quel point il est facile de publier quelque chose en ligne.

> _« Les opportunités ne frappent pas à la porte. Elles se présentent lorsque vous enfoncez la porte ! »_

> _— Greg Plitt_

### Nommer votre dépôt

Lors de la publication sur GitHub Pages, choisir le bon nom de dépôt est crucial.

[Cette page](https://help.github.com/articles/user-organization-and-project-pages/) de GitHub l'explique parfaitement :

![Image](https://cdn-media-1.freecodecamp.org/images/0*SQGwUsaVQJz118zd.png)

Vous devez nommer le dépôt exactement comme l'organisation avec « .github.io » à la fin :

![Image](https://cdn-media-1.freecodecamp.org/images/0*z1Hfc5CZI1Qk11sq.)

### Construire votre page d'accueil

Après avoir créé le dépôt, construisez votre page d'accueil avec HTML, CSS et JavaScript.

**Astuce** : J'ai utilisé [CodePen](https://codepen.io/ddcreationstudios/pen/yogdXX) pendant le développement. C'est un [REPL](https://stackoverflow.com/questions/13603021/what-is-a-repl-in-javascript) (Read–Eval–Print Loop). Cela permet de recevoir un retour instantané du code que vous écrivez.

Pour accélérer le processus de mise en page et ajouter des fonctionnalités sympas, j'ai utilisé [Materialize](http://materializecss.com/getting-started.html). Je l'ai utilisé par le passé et j'en ai toujours été satisfait.

### Mettre votre code sur GitHub

Ensuite, mettez vos fichiers sur GitHub en suivant les instructions.

**Assurez-vous d'avoir les fichiers sur votre branche « Master »**

![Image](https://cdn-media-1.freecodecamp.org/images/0*BkJCyYBXyjXMyHg7.png)

Et choisissez la source de vos GitHub Pages pour la branche Master.
