Vous voulez apprendre React, la bibliothèque JavaScript la plus demandée au monde. Mais quelles étapes devez-vous suivre pour y parvenir ? Passons en revue les sept compétences que vous devriez apprendre pour devenir un développeur React professionnel.

En assemblant votre parcours d'apprentissage de React, il est facile de se sentir submergé et de dire : _"C'est impossible d'apprendre tout ça. Il y a trop de choses que je dois apprendre !"_

**Pour réussir avec React, n'essayez pas d'apprendre tout. Concentrez-vous sur l'apprentissage des bonnes choses.**

Décomposons les sept compétences clés sur lesquelles vous devez vous concentrer pour construire des applications impressionnantes et devenir un développeur React recherché.

## Étape 1 : Maîtriser les fondamentaux de base de HTML, CSS et JavaScript

La première étape pour apprendre React est en quelque sorte un pas en arrière.

Les éléments de base du web et de chaque page web sont HTML, CSS et JavaScript. Tout bon développeur React devrait savoir comment les utiliser. React s'appuie sur eux pour vous aider à créer des applications qui fonctionnent sur le web.

Si vous avez déjà construit quelque chose avec HTML, CSS et JavaScript, vous êtes sur la bonne voie. Par exemple, si vous avez construit une page de destination complète ou un petit site web.

Parmi les trois technologies, **React est le plus lié à JavaScript**. React _est_ JavaScript, avec quelques fonctionnalités ajoutées. Pour cette raison, vous devrez avoir des compétences solides en JavaScript.

### Quels concepts JavaScript devez-vous connaître pour React ?

* Structures de données de base : variables, objets et tableaux
* Méthodes de tableau et travail avec les données de tableau : `.map()`, `.filter()` et `.reduce()`
* Être très familier avec les fonctions et un peu avec les classes
* JavaScript asynchrone : promesses, faire des requêtes HTTP avec Fetch API, la syntaxe async/await peut aider
* Le DOM : apprendre à créer, sélectionner et modifier des éléments HTML ainsi que leurs attributs
* La destructuration d'objets et de tableaux est utile pour travailler avec les données

De nombreux développeurs affirment que vous devriez connaître le "JavaScript ES6/ES7/ES8/ESNext" (en d'autres termes, les dernières fonctionnalités de JavaScript) pour mieux apprendre React. Connaître plus de JavaScript peut aider, mais les nouvelles fonctionnalités peuvent aussi servir de distraction pour les nouveaux apprenants.

Tout d'abord, devenez confiant avec les concepts JavaScript que j'ai listés ci-dessus en construisant quelques petits projets qui nécessitent JavaScript. Après cela, vous pouvez jeter un coup d'œil à certaines des nouvelles fonctionnalités du langage.

## Étape 2 : Apprendre les fondamentaux de React et les hooks React

Une fois que vous êtes confiant avec JavaScript, vous êtes prêt à apprendre React et ses concepts de base.

Ces fondamentaux sont tous spécifiques à React et ils existent pour nous aider à construire des applications avec React, en utilisant des motifs que JavaScript lui-même n'a pas.

### Quels fondamentaux de React devez-vous connaître ?

* Comment structurer les éléments JSX (et comment cela diffère du HTML simple)
* Comment rendre (afficher) les éléments JSX et comment afficher ou masquer des éléments en fonction de certaines conditions
* Comment diviser JSX en composants et comment réutiliser et organiser ces composants pour créer l'interface de notre application.
* Comment passer des données (+ éléments JSX et composants) aux composants en utilisant les props et quand le faire
* Comment stocker et mettre à jour des données dans les composants en utilisant l'état et comment "élever l'état" vers d'autres composants
* Comment utiliser les données d'événement dans React et gérer les événements des événements `onClick`, `onChange` et `onSubmit` (c'est-à-dire les événements des boutons, des entrées et des formulaires)

En tant que bibliothèque âgée de 7 ans, les fonctionnalités de React ont changé au fil du temps. Une question qu'on me pose souvent est **que devez-vous apprendre en premier : l'ancienne ou la nouvelle syntaxe ?** À la fin de 2018, React a reçu une grande mise à jour qui incluait des fonctionnalités appelées React Hooks.

Les Hooks étaient une grande amélioration. Ils ont rendu les applications React plus simples, plus puissantes et nous permettent d'écrire moins de code. Ce qui est important pour vous de savoir, ce sont les cinq hooks principaux de React.

### Quels sont les cinq hooks React que vous devez absolument connaître ?

* `useState` : pour stocker et gérer des données dans des composants individuels
* `useEffect` : pour effectuer des actions comme des requêtes HTTP ou travailler avec une API de navigateur
* `useRef` : pour référencer des éléments JSX
* `useContext` : pour accéder aux données du contexte React afin de partager des données entre les composants plus facilement [plutôt que de passer des props]
* `useReducer` : pour stocker et gérer des données à travers plusieurs composants

Il existe plus de hooks que ces 5, mais les autres ne sont pas nécessaires aussi souvent. Donc oui, vous devriez apprendre les dernières fonctionnalités de React. Plongez dans l'utilisation des hooks tout de suite. Ils seront la base de chaque application React que vous créerez.

## Étape 3 : Apprendre à récupérer des données à partir des API REST et GraphQL

Une application React est le frontend d'une application complète. Dans chaque application, vous aurez probablement à la fois un frontend React, avec lequel l'utilisateur interagit, ainsi qu'un backend avec lequel notre code React interagit. Le backend est l'endroit d'où nous obtenons nos données et où nous faisons d'autres choses comme authentifier nos utilisateurs.

Il existe deux façons de travailler avec les données d'un backend. La manière standard d'obtenir des données est à partir de ce qu'on appelle une API REST. **Les API REST** sont la forme d'API la plus courante. Et la manière plus récente est à partir de ce qu'on appelle une **API GraphQL**.

Vous rencontrerez les deux types d'API dans votre travail, alors familiarisez-vous avec l'utilisation de React pour interagir avec les deux.

## Étape 4 : Apprendre à styliser vos applications React avec une bibliothèque de composants ou une bibliothèque de classes utilitaires

Chaque application React a besoin de styles pour son apparence. Un choix est d'utiliser du CSS simple. Vous pouvez écrire vos propres styles et les mettre dans une feuille de style séparée.

Mais outre le CSS, de nombreux développeurs React utilisent ce qu'on appelle une bibliothèque de composants pour un stylisme plus facile. **Une bibliothèque de composants** nous donne des composants réutilisables qui ont leurs propres styles pré-faits. La bibliothèque de composants la plus populaire pour React est **Material UI**. Mais il y en a beaucoup d'autres parmi lesquelles vous pouvez choisir.

Les développeurs utilisent également des outils qui fournissent des noms de classes pré-faits appelés bibliothèques de classes utilitaires. Contrairement à une bibliothèque de composants, **une bibliothèque de classes utilitaires** vient avec des classes pré-faites pour styliser vos éléments.

Vous pouvez styliser votre application en appliquant des noms de classes à chaque élément. Cela élimine le besoin d'écrire des styles vous-même. La bibliothèque de classes utilitaires la plus populaire est **Tailwind CSS**.

Vous rencontrerez ces deux types en tant que développeur, alors familiarisez-vous avec une bibliothèque de composants et une bibliothèque de classes utilitaires.

## Étape 5 : Gérer l'état dans React avec le contexte React

**Qu'est-ce que la gestion d'état ?** C'est le processus de décision de l'emplacement des données et de la manière de travailler avec elles dans notre application. Pour gérer l'état à travers les composants de votre application, utilisez le contexte React.

**Le contexte React** est un outil intégré dans la bibliothèque principale de React et nous permet de passer des données à travers les composants de notre application sans utiliser de props. Il nous aide à résoudre le problème de "prop-drilling" qui implique de passer des props (données) à des composants qui sont profondément imbriqués les uns dans les autres.

Avec le contexte React, nous plaçons une valeur sur le contexte que nous créons, puis y accédons en utilisant le hook React `useContext()`.

**Et Redux ?** Redux est une bibliothèque populaire pour gérer l'état dans les applications React. C'est un outil bien plus complexe que ce dont vous avez besoin pour la plupart des applications que vous construirez. Bien que Redux soit encore utilisé pour des applications très grandes, le contexte React sera suffisant pour toute application que vous créerez.

De plus, vous pouvez obtenir de nombreux avantages de Redux en combinant les hooks React et le contexte React. C'est un grand avantage par rapport à l'apprentissage d'une bibliothèque supplémentaire.

## Étape 6 : Apprendre un routeur React. En particulier, react-router-dom

**Qu'est-ce qu'un routeur ?** Tout site web que nous visitons a de nombreuses pages que nous pouvons parcourir en allant en avant ou en arrière dans l'historique de notre navigateur. Pour créer ces différentes pages ou routes dans React, nous devons utiliser ce qu'on appelle un routeur.

React lui-même ne vient pas avec son propre routeur, donc nous allons devoir utiliser une bibliothèque tierce, appelée react-router-dom.

**react-router-dom** est nécessaire pour créer des pages dans notre application, ainsi que pour naviguer l'utilisateur autour de chaque page. Il nous permet de créer des liens vers différentes pages de notre application et de naviguer vers elles ou de les rediriger vers d'autres pages si nécessaire.

### Quelles fonctionnalités de react-router-dom devez-vous connaître ?

* Comment créer des routes d'application en utilisant les composants `<Route />`, `<Switch />` et `<BrowserRouter />`
* Comment naviguer les utilisateurs vers différentes pages en utilisant le composant `<Link />` et de manière programmatique en utilisant le hook `useHistory()`
* Comment créer des routes dynamiques en utilisant la prop path (c'est-à-dire `<Route path="/posts/:post-slug" component={Post} />`) et obtenir la valeur du path en utilisant le hook `useParams()`
* Comment rediriger les utilisateurs depuis un contenu protégé en utilisant le composant `<Redirect />` (voir le numéro 7)

## Étape 7 : Apprendre les modèles d'authentification dans React

Les utilisateurs authentifiés sont des personnes qui se sont connectées pour utiliser notre application. Et nous voulons donner à ces utilisateurs l'accès à différentes choses. Pour cette raison, l'authentification va de pair avec le routage. C'est parce que les utilisateurs authentifiés devraient pouvoir voir certaines pages que les utilisateurs non authentifiés ne peuvent pas.

### Que devez-vous savoir sur l'authentification en tant que développeur React ?

Il y a une chose principale. Vous devriez apprendre à montrer un certain contenu aux utilisateurs authentifiés et à cacher ce contenu aux utilisateurs non authentifiés.

En résumé, ce sont toutes les compétences de base que vous devez avoir en tant que développeur React, capable de construire des applications complètes et de travailler en tant que développeur frontend embauché.

Au-delà de ces 7 compétences, notez qu'il y en a beaucoup qui peuvent approfondir votre compréhension en tant que développeur. Par exemple, en apprenant davantage sur les navigateurs, HTTP et les API, pour n'en nommer que quelques-uns. Mais si vous suivez toutes ces étapes, vous aurez une compréhension solide de React et serez capable de construire des applications de toute envergure.
