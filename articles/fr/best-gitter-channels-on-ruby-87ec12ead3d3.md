Par Gitter

Ruby est un langage de programmation polyvalent créé dans les années 1990 par Yukihiro "Matz" Matsumoto. Il est également considéré comme l'un des meilleurs langages pour débuter lorsque vous apprenez à coder. Nous avons de nombreux canaux dédiés aux projets liés à Ruby sur Gitter, tant pour les développeurs avancés que pour les débutants. Découvrez-les !

* [**rails/rails**](https://gitter.im/rails/rails?utm_source=blog&utm_medium=content&utm_campaign=ruby) — Discussion générale autour de Ruby on Rails. [http://rubyonrails.org](http://rubyonrails.org/)
* [**voltrb/volt**](https://gitter.im/voltrb/volt?utm_medium=content&utm_campaign=ruby) — Volt est un framework web réactif où votre Ruby s'exécute à la fois sur le serveur et le client.
* [**opal/opal**](https://gitter.im/opal/opal?utm_medium=content&utm_campaign=ruby) — Opal est un compilateur de Ruby vers JavaScript. Il est source-à-source, ce qui le rend rapide comme un runtime. Opal inclut un compilateur (qui peut être exécuté dans n'importe quel navigateur), une corelib et une implémentation runtime.
* [**hanami/chat**](https://gitter.im/hanami/chat) **—** [Hanami](http://hanamirb.org/) est un framework web moderne pour Ruby.
* [**jekyll/jekyll**](https://gitter.im/jekyll/jekyll?utm_medium=content&utm_campaign=ruby) — Jekyll est un générateur de site statique, conscient des blogs, en Ruby.
* [**aws/aws-sdk-ruby**](https://gitter.im/aws/aws-sdk-ruby?utm_medium=content&utm_campaign=ruby) — Le SDK officiel AWS pour Ruby.
* [**refinery/refinerycms**](https://gitter.im/refinery/refinerycms?utm_medium=content&utm_campaign=ruby) — Refinery CMS est un CMS Ruby on Rails extensible qui supporte Rails 4.2.
* [**rubinius/rubinius**](https://gitter.im/rubinius/rubinius?utm_medium=content&utm_campaign=ruby) — Rubinius est une plateforme de langage moderne qui supporte plusieurs langages de programmation. Rubinius inclut une machine virtuelle à bytecode, un ramasse-miettes générationnel et un compilateur de code machine natif juste-à-temps (JIT). Rubinius fournit un support de concurrency via des threads OS natifs sans verrou global d'interpréteur.
* [**dev-ua/ruby-ua**](https://gitter.im/dev-ua/ruby-ua?utm_medium=content&utm_campaign=ruby) — Discussion des développeurs ukrainiens dédiée à Ruby.
* [**bbatsov/rubocop**](https://gitter.im/bbatsov/rubocop?utm_medium=content&utm_campaign=ruby) — Rubocop est un analyseur de code Ruby statique robuste, basé sur le guide de style Ruby de la communauté.
* [**puma/puma**](https://gitter.im/puma/puma?utm_medium=content&utm_campaign=ruby) _—_ Puma est un serveur web Ruby conçu pour la concurrency.
* [**cucumber/cucumber-ruby**](https://gitter.im/cucumber/cucumber-ruby?utm_medium=content&utm_campaign=ruby) _—_ Cucumber est un outil pour exécuter des tests automatisés écrits en langage naturel.
* [**mruby/mruby**](https://gitter.im/mruby/mruby?utm_medium=content&utm_campaign=ruby) — mruby est l'implémentation légère du langage Ruby conforme à (une partie de) la [norme ISO](http://www.iso.org/iso/iso_catalogue/catalogue_tc/catalogue_detail.htm?csnumber=59579). Sa syntaxe est compatible avec Ruby 1.9.
* [**mperham/sidekiq**](https://gitter.im/mperham/sidekiq?utm_medium=content&utm_campaign=ruby) **—** Sidekiq est un traitement en arrière-plan simple et efficace pour Ruby.
* [**rebelidealist/stripe-ruby-mock**](https://gitter.im/rebelidealist/stripe-ruby-mock?utm_medium=content&utm_campaign=ruby) **—** Une bibliothèque de mocking pour tester stripe ruby.
* [**trailblazer/chat**](https://gitter.im/trailblazer/chat) **—** [Trailblazer](http://trailblazer.to/) vous offre une architecture de haut niveau pour les applications web.
* [**dry-rb/chat**](https://gitter.im/dry-rb/chat) **—** dry-rb est une collection de bibliothèques Ruby de nouvelle génération, chacune destinée à encapsuler une tâche commune.
