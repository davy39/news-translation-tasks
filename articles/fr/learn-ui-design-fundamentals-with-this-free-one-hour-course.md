Par Per Harald Borgen

Je suis ravi d'annoncer que nous venons de lancer ce [cours Scrimba gratuit sur les fondamentaux de la conception UI](https://scrimba.com/g/gdesign?utm_source=dev.to&utm_medium=referral&utm_campaign=gdesign_launch_article) par Gary Simon. 

%[https://twitter.com/scrimba/status/1222141166508040192]

Gary est concepteur UI et développeur front-end depuis les 20 dernières années, et il gère également la chaîne YouTube très populaire [DesignCourse](https://www.youtube.com/user/designcourse).

![Alt Text](https://thepracticaldev.s3.amazonaws.com/i/v3xxcwitwuldrqqwo60n.png)

En seulement une heure, Gary va améliorer vos compétences en conception UI en vous aidant à comprendre les sept éléments les plus importants de toutes les bonnes interfaces utilisateur :

* L'espace blanc
* La couleur
* Le contraste
* L'échelle
* L'alignement
* La typographie
* La hiérarchie visuelle

Le cours contient un mélange de leçons et de défis de codage interactifs, ce qui signifie que vous allez réellement construire des interfaces utilisateur dans le navigateur. Vous aurez donc besoin d'une compréhension de base du HTML et du CSS avant de vous inscrire. 

Cependant, si vous débutez, ne vous inquiétez pas, il y a un excellent [cours accéléré sur le HTML et le CSS](https://scrimba.com/g/ghtmlcss?utm_source=dev.to&utm_medium=referral&utm_campaign=gdesign_launch_article) sur Scrimba que vous pouvez suivre pour poser quelques bases.

En ce qui concerne la configuration initiale pour ce cours, il n'y a pas de prérequis ! Le cours est sur Scrimba, donc vous pouvez interagir avec le code au fur et à mesure que le cours avance.

[Cliquez ici pour commencer le cours immédiatement](https://scrimba.com/g/gdesign?utm_source=dev.to&utm_medium=referral&utm_campaign=gdesign_launch_article), ou continuez à lire si vous souhaitez une description de la structure du cours.

# Chapitre 1 : L'espace blanc

Dans la première leçon, vous allez apprendre à utiliser l'espace blanc. L'espace blanc est l'espace vide entre les éléments de votre interface utilisateur.

![Alt Text](https://thepracticaldev.s3.amazonaws.com/i/udsqulct5dg98b655o9g.png)

Gary montrera comment, en utilisant seulement 3 propriétés CSS, `padding`, `margin` et `line-height`, vous pouvez améliorer considérablement l'apparence de votre texte.

# Chapitre 2 : L'alignement

Dans ce chapitre, Gary montre comment l'alignement peut améliorer les choses avec un minimum de code. L'alignement est le processus qui consiste à s'assurer que chaque élément est positionné correctement par rapport aux autres éléments. Encore une fois, avec seulement 3 propriétés : `margin`, `transform` et `text-align`.

# Chapitre 3 : Le défi des cartes

Bienvenue dans le premier défi du cours. Ce défi et les suivants sont conçus de manière à ce que vous codiez une solution et essayiez de vous dépasser un peu, avec Gary montrant sa solution par la suite pour que vous puissiez comparer.

Dans ce défi, il vous donnera quelques cartes que vous devrez améliorer en termes d'espace blanc et d'alignement.

![Défi 1](https://thepracticaldev.s3.amazonaws.com/i/tj5cgifrni71eip8h1va.png)

Lorsque vous aurez terminé, n'hésitez pas à consulter [la solution de Gary](https://scrimba.com/p/prpYaAy/cNqBVeUm?utm_source=dev.to&utm_medium=referral&utm_campaign=gdesign_launch_article).

# Chapitre 4 : Le contraste

Dans ce screencast, vous allez apprendre le contraste. Le contraste est défini comme étant dans un état "frappant" différent de quelque chose d'autre. Gary abordera les "directives de contraste" et comment s'assurer que votre design est accessible à ceux qui ont des problèmes de déficience visuelle.

![Alt Text](https://thepracticaldev.s3.amazonaws.com/i/eot42ihnfpe7sepyj9f1.png)

# Chapitre 5 : L'échelle

Il est temps de parler de l'échelle. Gary montrera qu'en exploitant l'échelle des différents éléments, vous pouvez grandement améliorer un design.

![Exemple d'échelle](https://thepracticaldev.s3.amazonaws.com/i/wgx2it9an92dewwr1j3c.png)

# Chapitre 7 : La typographie

![Alt Text](https://thepracticaldev.s3.amazonaws.com/i/l4r76dttwa5u9iy1gpa7.png)

La typographie nécessite la compréhension d'autres fondamentaux, ainsi que quelques autres considérations. Dans ce screencast, Gary va donner des conseils sur la façon de choisir efficacement les polices, leur taille, leur espacement et quelques autres éléments.

# Chapitre 8 : La couleur

La couleur est le premier fondamental de la conception UI qui façonne l'expérience de l'utilisateur ! Gary aborde quelques considérations clés lors du choix des couleurs pour le public cible, quelques règles courantes à suivre et à éviter, et quelques bases sur les combinaisons de couleurs.

![Alt Text](https://thepracticaldev.s3.amazonaws.com/i/ms14ijpd4wwdfdnib6uo.png)

Lorsque vous avez quelques options prêtes, allez-y et comparez si vous et Gary avez des designs similaires [dans sa solution](https://scrimba.com/p/prpYaAy/cyp8qBfr?utm_source=dev.to&utm_medium=referral&utm_campaign=gdesign_launch_article).

# Chapitre 10 : La hiérarchie visuelle

La hiérarchie visuelle est la manière dont vous établissez l'importance des éléments sur une interface utilisateur. Certains éléments sont plus importants que d'autres. 

![Alt Text](https://thepracticaldev.s3.amazonaws.com/i/ad47ru7l5dk4fg3recf2.png)

Dans ce chapitre, Gary utilise un simple formulaire d'inscription par e-mail pour montrer comment vous pouvez améliorer un design assez terne en mettant en évidence certains aspects clés et en améliorant sa hiérarchie visuelle.

# Réflexions finales

Si vous êtes arrivé jusqu'ici : félicitations pour avoir terminé ce cours ! Vous pouvez être fier de vous, car la plupart des gens commencent des cours sans les terminer. Vous avez maintenant une compréhension de base de ces fondamentaux de la conception UI :

* L'espace blanc
* La couleur
* Le contraste
* L'échelle
* L'alignement
* La typographie
* La hiérarchie visuelle

Ils sont tous également importants pour obtenir un design correct. Si le design manque dans l'un de ces domaines, il est vraiment facile de remarquer que quelque chose ne va pas avec la qualité du design. 

Alors, assurez-vous de penser à tous ces fondamentaux la prochaine fois que vous devrez construire une interface utilisateur.
