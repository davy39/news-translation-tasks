Par Daniel Deutsch

Voici une liste de certaines extensions que j'apprécie avec Visual Studio Code (VSCode). Comme je travaille beaucoup sur le front-end, la plupart de ces extensions seront utiles pour les développeurs web. J'ai travaillé auparavant avec Atom, Visual Studio et Webstorm, mais VSCode me convient le mieux et est tout simplement incroyable. Les développeurs ont fait un excellent travail (et continuent de le faire !). Essayez-le par vous-même et profitez-en !

> _"Pour profiter de la vie, vous n'avez pas besoin de choses compliquées, mais vous devez contrôler votre temps et réaliser que la plupart des choses ne sont pas aussi sérieuses que vous le pensez."_ — [Timothy Ferriss](https://en.wikiquote.org/wiki/Timothy_Ferriss)_

### Extensions HTML

#### Auto Close Tag

Ajoute automatiquement une balise de fermeture HTML/XML, comme le fait l'IDE Visual Studio ou Sublime Text.

![Image](https://cdn-media-1.freecodecamp.org/images/ZLFNZjla4HzI94AlQNxR0clklBRvlDCSSBoX)

[Voir plus](https://marketplace.visualstudio.com/items?itemName=formulahendry.auto-close-tag)

#### Auto Rename Tag

Renomme automatiquement une balise HTML/XML appariée, comme le fait l'IDE Visual Studio.

![Image](https://cdn-media-1.freecodecamp.org/images/5lS4Jpanix2vwzO60kfeKNsKpmTrpPvIY2FR)

[Voir plus](https://marketplace.visualstudio.com/items?itemName=formulahendry.auto-rename-tag)

#### HTML Snippets

Cette extension ajoute un support riche pour le langage HTML Markup à VS Code, incluant : les balises HTML5 complètes, la colorisation et les extraits de code.

![Image](https://cdn-media-1.freecodecamp.org/images/iVkIkUJuKHCU2cbJZenkvPc1tQelxCsweAcF)

[Voir plus](https://marketplace.visualstudio.com/items?itemName=abusaidm.html-snippets)

### Extensions Markdown

#### Auto-Open Markdown Preview

Cette extension VS Code ouvre automatiquement l'aperçu Markdown chaque fois que vous ouvrez un nouveau fichier Markdown. Si vous trouvez ennuyeux de taper "Ctrl+K V" ou "⌘+K V" (aperçu côte à côte) plusieurs fois, cette extension vous aide.

[Voir plus](https://marketplace.visualstudio.com/items?itemName=hnw.vscode-auto-open-markdown-preview)

#### Markdown TOC

Génère une table des matières (TOC) des titres à partir d'un fichier markdown analysé.

[Voir plus](https://marketplace.visualstudio.com/items?itemName=AlanWalk.markdown-toc)

#### Markdown Theme Kit

Un ensemble de thèmes basé sur SublimeText-Markdown/MarkdownEditing.

[Voir plus](https://marketplace.visualstudio.com/items?itemName=ms-vscode.Theme-MarkdownKit)

#### Code Spell Checker

Un correcteur orthographique de base qui fonctionne bien avec le code camelCase.

[Voir plus](https://marketplace.visualstudio.com/items?itemName=streetsidesoftware.code-spell-checker)

### Extensions Générales

#### Color Highlight

vscode-ext-color-highlight. Cette extension style les couleurs css/web trouvées dans votre document.

[Voir plus](https://marketplace.visualstudio.com/items?itemName=naumovs.color-highlight)

#### Custom CSS and JS Loader

CSS personnalisé pour votre VS Code. Basé sur les icônes vscode de Roberto Huertas.

![Image](https://cdn-media-1.freecodecamp.org/images/xUYM4P6SmYxosSE4yQ6kNCkEnaDSOCT0fOR1)

[Voir plus](https://marketplace.visualstudio.com/items?itemName=be5invis.vscode-custom-css)

#### EditorConfig for VS Code

EditorConfig aide les développeurs à définir et maintenir des styles de codage cohérents entre différents éditeurs et IDE. Le projet EditorConfig consiste en un format de fichier pour définir des styles de codage et une collection de plugins d'éditeur de texte qui permettent aux éditeurs de lire le format de fichier et de respecter les styles définis.

[Voir plus](https://marketplace.visualstudio.com/items?itemName=EditorConfig.EditorConfig)

#### File Templates

Extension Visual Studio code qui permet de créer rapidement de nouveaux fichiers basés sur des modèles définis.

[Voir plus](https://marketplace.visualstudio.com/items?itemName=brpaz.file-templates)

#### Git History (git log)

Voir le journal git avec le graphique et les détails. Voir l'historique d'un fichier (Git log) ou l'historique d'une ligne dans un fichier (Git Blame). Voir une copie précédente du fichier. Comparer une version précédente avec la version dans l'espace de travail ou une autre. Voir les détails du journal de validation pour une validation sélectionnée. Comparer les validations.

![Image](https://cdn-media-1.freecodecamp.org/images/e5UGQg9a0F30-oTVtKQA5bnDKIZ8DCOatgVh)

[Voir plus](https://marketplace.visualstudio.com/items?itemName=donjayamanne.githistory9)

#### Guides

Une extension Visual Studio Code pour plus de lignes de guide.

![Image](https://cdn-media-1.freecodecamp.org/images/xI4qAfwGK5qWdzNVlbXlGHrEY3DWiWIE1k7O)

[Voir plus](https://marketplace.visualstudio.com/items?itemName=spywhere.guides)

#### Material Icon Theme

Le thème Material Icon Theme fournit de nombreuses icônes basées sur Material Design pour Visual Studio Code.

[Voir plus](https://marketplace.visualstudio.com/items?itemName=PKief.material-icon-theme)

#### Rainbow Brackets

Fournit des couleurs arc-en-ciel pour les parenthèses rondes, les crochets et les accolades.

[Voir plus](https://marketplace.visualstudio.com/items?itemName=2gua.rainbow-brackets)
