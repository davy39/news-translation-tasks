JavaScript est le langage de script le plus largement utilisé sur Terre. Et il possède le plus grand écosystème de bibliothèques de tous les langages de programmation.

JavaScript est le langage central du web, et le seul langage de programmation capable de s'exécuter dans tous les navigateurs web majeurs.

Notamment, JavaScript n'a aucun lien avec Java. Consultez [JavaScript: Le langage de programmation le plus incompris au monde](http://www.crockford.com/javascript/javascript.html).

Le nom officiel de JavaScript est ECMAScript défini sous la norme [ECMA-262](https://www.ecma-international.org/publications/standards/Ecma-262.htm).

Si vous souhaitez en savoir plus sur le langage JavaScript et pourquoi il est si largement utilisé, lisez l'article de Quincy Larson - [Quel langage de programmation devrais-je apprendre en premier ?](https://www.freecodecamp.org/news/what-programming-language-should-i-learn-first-19a33b0a467d/) - ou regardez cette [vidéo inspirante de Preethi Kasireddy](https://www.youtube.com/watch?v=VqiEhZYmvKk).

## **Moteurs JavaScript autonomes**

SpiderMonkey de Mozilla, le premier moteur JavaScript jamais écrit, actuellement utilisé dans Mozilla Firefox.

V8, le moteur JavaScript de Google, utilisé dans Google Chrome.

Google Apps Script, un interpréteur côté cloud/serveur qui fournit un contrôle programmatique « macro-like » des services et documents Google Apps.

Node.js, construit sur V8, une plateforme qui permet d'écrire des applications côté serveur en JavaScript.

Windows inclut JScript, une variante de JavaScript dans Windows Script Host.

Chakra, un fork de JScript, est développé par Microsoft et utilisé dans leur navigateur Edge.

Mozilla propose également Rhino, une implémentation de JavaScript construite en Java, généralement intégrée dans les applications Java pour fournir des scripts aux utilisateurs finaux.

WebKit (à l'exception du projet Chromium) implémente le moteur JavaScriptCore.

# **Avantages et inconvénients de JavaScript**

Comme pour tous les langages de programmation, JavaScript présente certains avantages et inconvénients à considérer. Beaucoup de ces aspects sont liés à la manière dont JavaScript est souvent exécuté directement dans le navigateur du client. Mais il existe désormais d'autres façons d'utiliser JavaScript qui lui permettent de bénéficier des mêmes avantages que les langages côté serveur.

## **Avantages de JavaScript**

* **Vitesse** - JavaScript tend à être très rapide car il est souvent exécuté immédiatement dans le navigateur du client. Tant qu'il n'a pas besoin de ressources externes, JavaScript n'est pas ralenti par des appels à un serveur backend. De plus, tous les navigateurs majeurs supportent la compilation JIT (juste à temps) pour JavaScript, ce qui signifie qu'il n'est pas nécessaire de compiler le code avant de l'exécuter.
* **Simplicité** - La syntaxe de JavaScript a été inspirée par celle de Java et est relativement facile à apprendre par rapport à d'autres langages populaires comme C++.
* **Popularité** - JavaScript est partout sur le web, et avec l'avènement de Node.js, il est de plus en plus utilisé côté serveur. Il existe d'innombrables ressources pour apprendre JavaScript. À la fois StackOverflow et GitHub montrent un nombre croissant de projets utilisant JavaScript, et la traction qu'il a gagnée ces dernières années n'est censée qu'augmenter.
* **Interopérabilité** - Contrairement à PHP ou à d'autres langages de script, JavaScript peut être inséré dans n'importe quelle page web. JavaScript peut être utilisé dans de nombreux types d'applications différents grâce au support dans d'autres langages comme Perl et PHP.
* **Charge du serveur** - JavaScript est côté client, donc il réduit la demande sur les serveurs en général, et les applications simples peuvent ne pas avoir besoin d'un serveur du tout.
* **Interfaces riches** - JavaScript peut être utilisé pour créer des fonctionnalités comme le glisser-déposer et des composants tels que des curseurs, ce qui améliore grandement l'interface utilisateur et l'expérience d'un site.
* **Fonctionnalités étendues** - Les développeurs peuvent étendre les fonctionnalités des pages web en écrivant des extraits de JavaScript pour des modules complémentaires tiers comme Greasemonkey.
* **Polyvalence** - Il existe de nombreuses façons d'utiliser JavaScript via des serveurs Node.js. Si vous deviez démarrer Node.js avec Express, utiliser une base de données de documents comme MongoDB, et utiliser JavaScript côté frontend pour les clients, il est possible de développer une application JavaScript entière de bout en bout en utilisant uniquement JavaScript.
* **Mises à jour** - Depuis l'avènement d'ECMAScript 5 (la spécification de script sur laquelle JavaScript repose), ECMA International s'est engagé à mettre à jour JavaScript annuellement. Jusqu'à présent, nous avons reçu le support des navigateurs pour ES6 en 2017 et attendons avec impatience que ES7 soit supporté à l'avenir.

## **Inconvénients de JavaScript**

* **Sécurité côté client** - Puisque le code JavaScript est exécuté côté client, des bugs et des négligences peuvent parfois être exploités à des fins malveillantes. Pour cette raison, certaines personnes choisissent de désactiver entièrement JavaScript.
* **Support des navigateurs** - Alors que les scripts côté serveur produisent toujours la même sortie, différents navigateurs interprètent parfois le code JavaScript différemment. De nos jours, les différences sont minimes, et vous ne devriez pas avoir à vous en soucier tant que vous testez votre script dans tous les navigateurs majeurs.

## **ES6**

La 6ème édition d'ECMAScript s'appelle ES6.

Elle est également connue sous le nom d'ES2015.

Les changements ajoutent beaucoup de sucre syntaxique qui permet aux développeurs de créer des applications dans un style orienté objet.

Exemple ES5 :

```javascript
var User = function () {
  function User(name) {
    this._name = name;
  }

  User.prototype.getName = function getName(x) {
    return 'Mr./Mrs. ' + this._name;
  };

  return User;
}();
```

Exemple ES6 :

```javascript
class User {
  constructor(name) {
    this._name = name
  }

  getName() {
    return `Mr./Mrs. ${this._name}`
  }
}
```

Beaucoup de nouvelles fonctionnalités syntaxiques ont été introduites, notamment :

* classes
* modules
* templating
* boucles for/of
* expressions génératrices
* fonctions fléchées
* collections
* promesses

De nos jours, la plupart des fonctionnalités sont disponibles dans tous les navigateurs populaires. Le [tableau de compatibilité](https://kangax.github.io/compat-table/es6/) contient toutes les informations sur la disponibilité des fonctionnalités de tous les navigateurs modernes.

Fréquemment, de nouvelles fonctionnalités arrivent qui font partie du successeur ES7. Une méthode courante consiste à traduire le JavaScript moderne (ES6, ES7 et autres propositions expérimentales) en ES5. Cela garantit que même les anciens navigateurs peuvent exécuter le code. Il existe des outils comme [Babel](https://babeljs.io/) qui transforment le nouveau JavaScript en ES5.

Outre le sucre syntaxique provenant des normes ECMAScript, il existe des fonctionnalités qui nécessitent un [Polyfill](https://babeljs.io/docs/usage/polyfill). Habituellement, elles sont nécessaires car des implémentations complètes de classes/méthodes ont été ajoutées à la norme.

## **Instanciation d'objets**

En JavaScript et dans la plupart des autres langages, un objet contient une série de propriétés, qui sont des paires clé-valeur. Plusieurs options s'offrent à vous lorsque vous devez construire un objet.

### **Initialiser une variable d'objet**

Vous pouvez créer un objet avec des propriétés pré-définies comme suit :

```javascript
let myObject = {
  name: "Dave",
  age: 33
}
```

### **Créer un objet vide**

Cela crée un objet vide dans notre variable myObject :

```javascript
let myObject = new Object();
```

Lorsque vous souhaitez ajouter des propriétés à votre objet, vous utilisez simplement la notation par point ou la notation par crochets avec le nom de propriété de votre choix :

```javascript
myObject.name = "Johnny Mnemonic"
myObject["age"] = 55
```

### **Utiliser une fonction constructeur**

Vous pouvez définir une fonction constructeur que vous pouvez utiliser pour créer vos objets :

```javascript
function Kitten(name, cute, color) {
  this.name = name,
  this.cute = cute,
  this.color = color
}
```

Vous pouvez définir une variable contenant une instanciation de cet objet en appelant la fonction constructeur :

```javascript
let myKitten = new Kitten("Nibbles", true, "white")
```

### **Object.create()**

La méthode Object.create() (définie pour la première fois dans ECMAScript 5.1) vous permet de créer des objets. Elle vous permet de choisir l'objet prototype pour votre nouvel objet sans avoir besoin de définir une fonction constructeur au préalable.

```javascript
// Notre objet pré-défini
let kitten = {
  name: "Fluff",
  cute: true,
  color: "gray"
}
// Créer un nouvel objet en utilisant Object.create(). kitten est utilisé comme prototype
let newKitten = Object.create(kitten)

console.log(newKitten.name) // Affichera "Fluff"
```
