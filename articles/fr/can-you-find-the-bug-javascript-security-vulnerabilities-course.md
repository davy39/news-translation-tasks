Dans un monde où les menaces cybernétiques sont plus présentes que jamais, comprendre comment protéger vos applications web est d'une importance capitale.

Nous venons de publier un cours sur la chaîne YouTube de freeCodeCamp.org qui vous aidera à apprendre les vulnérabilités de sécurité courantes en JavaScript et comment les prévenir.

Brandon enseigne ce cours. Il est ingénieur chez Semgrep et enseigne à l'Université Carnegie Mellon. Semgrep a fourni une subvention qui a rendu ce cours possible.

Ce cours est centré autour de 10 exemples, chacun illustrant un type différent de vulnérabilité dans les applications web. Les exemples couvrent une variété de langages et de technologies, y compris JavaScript, MongoDB et Docker, assurant une compréhension large des problèmes de sécurité potentiels dans différents contextes de programmation.

Pour le rendre intéressant et amusant, la vidéo est présentée sous forme de quiz. Vous verrez un petit extrait de code et devrez trouver les vulnérabilités. Ensuite, vous apprendrez à propos des vulnérabilités et comment les corriger.

Le cours commence par une exploration du Cross-Site Scripting (XSS), une vulnérabilité courante dans les applications JavaScript. Il approfondit ensuite les attaques par injection NoSQL, un risque de sécurité associé aux bases de données MongoDB. Vous apprendrez comment prévenir de telles attaques en vous assurant que seuls les types de données corrects sont acceptés par la base de données.

Les attaques par déni de service par expression régulière (ReDoS) sont également discutées, démontrant comment des expressions régulières complexes peuvent potentiellement conduire à une interruption de service. Le cours illustre comment prévenir cela en utilisant des bibliothèques efficaces au lieu de créer des expressions régulières complexes.

Le cours se concentre ensuite sur Docker et les potentielles mauvaises configurations de sécurité qui peuvent survenir dans les Dockerfiles. Vous apprendrez l'importance de ne pas accorder de privilèges root dans les images Docker pour éviter des conséquences désastreuses.

L'une des leçons critiques de ce cours est les dangers des informations d'identification codées en dur dans le code de l'application. Le cours montre les avantages de l'utilisation de variables d'environnement pour stocker des données sensibles, ce qui améliore la sécurité et facilite la gestion des secrets.

Le cours aborde également le problème des attaques par assignation massive dans les applications JavaScript et fournit une solution en utilisant une assignation prudente des propriétés d'objet et le chiffrement des mots de passe.

Enfin, le cours se termine par une analyse des attaques par injection d'en-tête d'hôte. Vous apprendrez à éviter d'utiliser les en-têtes d'hôte des requêtes utilisateur, qui peuvent être manipulés pour générer des URLs malveillantes.

Ce cours complet est conçu pour fournir aux développeurs une solide compréhension des vulnérabilités courantes des applications web et comment les prévenir. En apprenant à identifier ces vulnérabilités et à mettre en œuvre des pratiques de codage sécurisées, les participants seront mieux équipés pour construire des applications web robustes et sécurisées.

Regardez le cours complet sur [la chaîne YouTube de freeCodeCamp.org](https://www.youtube.com/watch?v=ypNKKYUJE5o) (30 minutes de visionnage).
