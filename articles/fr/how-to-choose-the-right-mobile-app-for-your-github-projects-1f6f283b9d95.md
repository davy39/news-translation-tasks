Par John Lafleur

Sur GitHub, vous avez maintenant [1,5 million d'organisations avec 25 millions de dépôts actifs](https://octoverse.github.com/). Cela fait une moyenne de 17 dépôts actifs par organisation ! Les développeurs participent de plus en plus à des projets GitHub simultanément. Pourtant, GitHub n'a aucun client mobile officiel. Voyons pourquoi et quelles sont les alternatives mobiles afin de voir s'il y en a une que vous devriez absolument obtenir.

Au cours des dernières années, j'ai contribué au développement de deux plateformes pour développeurs — [CodinGame](http://codingame.com) et [Tech.io](http://tech.io). Ensemble, elles totalisent plus d'1 million de développeurs. J'ai récemment cofondé une plateforme d'Intelligence Ingénierie appelée [Anaxi](http://anaxi.com). Pensez-y comme une intelligence économique actionnable pour toute l'organisation d'ingénierie. Donc, une partie de mon travail consiste à identifier les tendances croissantes dans les outils de développement logiciel. En d'autres termes, je pense beaucoup à ce genre de choses !

Alors, comprenons d'abord pourquoi GitHub ne concentre aucune ressource sur des clients mobiles.

### Pourquoi GitHub ne créerait-il pas une application mobile ?

La raison de base pour que GitHub envisage même de créer des clients mobiles est probablement que la valeur l'emporte sur le coût à leurs yeux. Et il y a plusieurs façons d'aborder cela.

La première pensée qui vient à l'esprit : que faites-vous le plus sur GitHub ? Des revues de code, des fusions de code, tout ce qui tourne autour du codage. Mais avez-vous déjà codé sur mobile ? Pas moi. Et même si vous vouliez faire un petit changement dans le code, comment testeriez-vous le code que vous avez modifié ?

Bien sûr, vous pourriez aussi vouloir commenter des problèmes GitHub, c'est vrai. Et je pense que c'est un comportement beaucoup plus susceptible de se produire sur un appareil mobile. Tout comme il s'est avéré que tous les emails ne sont pas de la prose sérieuse, alors tout ce qui se passe sur GitHub n'est pas de la programmation significative. Parfois, vous voulez juste faire de petits ajustements à un problème.

Mais en réalité, ce n'est pas ce qui est important dans cette affaire. La question importante est : GitHub gagne-t-il quelque chose en contrôlant l'interaction de l'utilisateur ? Que vous commentiez des problèmes depuis votre mobile ou non, vous finirez presque toujours par aller sur leur application web pour les revues de code et les fusions.

L'objectif de GitHub a toujours été de devenir cette partie inévitable du processus de développement logiciel. Et pour cela, il devait construire une API puissante afin que les développeurs tiers puissent créer de grandes expériences utilisateur pour tout ce qui concerne leur service de versioning. Un client mobile n'est qu'une partie de ces expériences utilisateur.

Cela changera-t-il avec l'acquisition par Microsoft ? Peut-être. Même avant l'acquisition, GitHub avait récemment construit [de nouveaux outils de gestion de projet](https://github.com/features/project-management). Microsoft est très fort dans les outils de gestion de projet. C'est en fait le leader mondial dans ce domaine. Surtout si vous considérez Microsoft Excel comme l'alternative à tout, comme je le fais ! Vous pourriez donc penser qu'ils mettraient plus de ressources pour porter la guerre à Atlassian afin d'essayer de dominer la gestion de projet en ingénierie.

Que nous reste-t-il maintenant ? Quelles sont les meilleures alternatives que nous avons sur mobile ? Quels sont les cas d'utilisation ?

### Quelles sont les applications mobiles potentielles pour vos projets GitHub ?

Si vous êtes comme moi, vous préférez comparer les choses dans un bon vieux tableau. Pour ce benchmark, je me suis uniquement concentré sur ce que je considérais être les applications les plus notables. Il y en a en fait des dizaines d'autres. Donc, si vous pensez que cette liste manque d'autres applications dignes de soulever le marteau de Thor, n'hésitez pas à laisser un commentaire !

![Image](https://cdn-media-1.freecodecamp.org/images/0*qTQQjRuatWvXtSYN.png)

Beaucoup d'entre elles offrent le même type de fonctionnalités. Notez qu'elles sont toutes gratuites. Alors, plongeons un peu plus dans chacune de ces applications !

#### Fasthub (Android)

À mon avis, Fasthub est le meilleur client mobile Android si vous cherchez à avoir toutes les fonctionnalités de GitHub sur votre téléphone. Il dispose d'un thème personnalisable et d'une interface très propre. L'expérience utilisateur de Fasthub se concentre sur les flux (comprenez les notifications de tous vos projets), les problèmes et les demandes de tirage. Cela coïncide avec vos principaux cas d'utilisation actuels pour GitHub. Fasthub est toujours activement développé, contrairement à la plupart des autres clients Android.

![Image](https://cdn-media-1.freecodecamp.org/images/0*cheBd1EEOqh3mG0f.png)
_Fasthub (n°1 aux États-Unis pour Android)_

#### OctoDroid & Forkhub (Android)

Ils représentent les clients GitHub Android les plus téléchargés, créés par des particuliers et non développés davantage — il y a beaucoup d'applications comme celle-ci.

En comparaison avec Fasthub, il semble qu'ils aient simplement voulu reproduire les fonctionnalités de GitHub sans penser à nos cas d'utilisation. Il n'a aucun thème personnalisable. Mais ils sont toujours complets en termes de fonctionnalités.

![Image](https://cdn-media-1.freecodecamp.org/images/0*4kEAUH-PpUuxZ7w3.png)
_Octodroid (n°1 en France pour Android)_

![Image](https://cdn-media-1.freecodecamp.org/images/0*LaDj41blGZSWTKzm.png)
_Forkhub (n°1 au Canada pour Android)_

#### CodeHub (iPhone / iPad)

CodeHub est le client GitHub iOS qui offre toutes les fonctionnalités de GitHub dans une interface brique par brique et une interface utilisateur fonctionnelle. La structure de l'application n'est pas adaptée aux cas d'utilisation mobile que nous avons mentionnés précédemment — les mises à jour rapides. Mais il a tout. C'est l'application la plus populaire sur iOS en ce moment.

![Image](https://cdn-media-1.freecodecamp.org/images/1*cKxVdpf-3TYo-tW4xoBSYg.png)
_CodeHub_

#### GitPoint (iOS et Android)

Construite avec React Native, GitPoint est une alternative qui fonctionne à la fois sur Android et iPhone. Elle est assez facile à utiliser, mais moins complète que CodeHub. Cependant, notez qu'il n'y a plus de développement sur cette application :(.

![Image](https://cdn-media-1.freecodecamp.org/images/0*rTdTiU0QgJwEq6NK.png)
_GitPoint_

#### GitHawk (iOS uniquement)

L'objectif de GitHawk est de vous aider à "atteindre INBOX ZERO sur vos notifications". Son interface principale est donc centrée sur les notifications, et vous pouvez ensuite les explorer en profondeur, consulter une demande de tirage et la fusionner.

Le principal problème de cette application est qu'elle ne filtre pas les notifications, et vous pourriez ne pas être intéressé par toutes. Néanmoins, elle vous permet de les gérer facilement. Je pense que c'est en fait une excellente application, car ils se sont concentrés sur vos cas d'utilisation mobile et ont adapté l'application pour faire exactement cela.

![Image](https://cdn-media-1.freecodecamp.org/images/0*JarEmGmk93b2ZOG1.png)
_GitHawk_

#### Anaxi (iOS uniquement ; bientôt sur Android aussi)

[Anaxi](https://itunes.apple.com/us/app/anaxi/id1317529886?mt=8) se concentre sur vous aider à prendre du recul sur vos projets. Combien de bugs bloquants pour la prochaine version avons-nous encore ? Qu'en est-il des problèmes de réseau ? Tous les problèmes sont-ils bien distribués au sein de votre propre équipe ?

Anaxi vous permet de créer des listes filtrées de problèmes — appelées rapports — afin que vous puissiez suivre l'avancement de votre projet. Les utilisateurs peuvent définir des seuils, de sorte que les rapports apparaissent en rouge dans des circonstances alarmantes. Ces indicateurs de couleur vous montrent ce qui nécessite votre attention d'un seul coup d'œil. [**Anaxi**](https://itunes.apple.com/us/app/anaxi/id1317529886?mt=8) **n'est clairement pas un client GitHub** ; **il apporte des fonctionnalités que GitHub n'a pas**. Et c'est pourquoi c'est très intéressant !

![Image](https://cdn-media-1.freecodecamp.org/images/1*K2E9HPN8vD4rV6FHZY0Phw.png)
_Anaxi_

### Quelle application devriez-vous obtenir ?

Je pense qu'une application mobile peut et doit vous aider à gérer et à gérer votre projet GitHub de manière plus efficace. Elle devrait compléter l'application web.

Je préférerais commencer à utiliser une application qui comprend mes besoins et qui me fournira une certaine valeur que je n'obtiens pas en utilisant l'outil, plutôt que de simplement reproduire les fonctionnalités de l'outil sans même penser à mes cas d'utilisation.

Et dans ce cas, deux applications se démarquent.

#### Insights sur le projet avec Anaxi

Anaxi apporte de nouvelles connaissances et perspectives à travers ses rapports que l'utilisateur ne peut pas obtenir sur GitHub seul. Il vous aide à comprendre exactement où vous en êtes et potentiellement à remettre en question vos priorités. Il vous aide à avoir une meilleure rétrospective et donc **à être un contributeur plus impactant**, en tant que développeur ou manager.

#### Gestion des notifications avec GitHawk

Même si GitHawk ne représente qu'un sous-ensemble des fonctionnalités de GitHub, il simplifie l'expérience pour un cas d'utilisation spécifique adapté au mobile.

En fin de compte, GitHawk et Anaxi comprennent que les cas d'utilisation les plus probables que vous aurez sur votre téléphone sont de rester à jour sur les problèmes qui vous intéressent ou sur le projet en général. Tout tourne autour du code, mais pas de la gestion du code lui-même.

Cependant, notez que les deux applications ne sont en aucun cas comparables. Anaxi est une plateforme, tandis que GitHawk est un outil GitHub utile. Anaxi s'intégrera bientôt avec d'autres outils, tels que Jira, et aura d'autres clients — Android et web —, et apportera d'autres capacités de rapport. C'est pourquoi il ne se décrit pas comme un client GitHub, mais comme une plateforme. Il dispose d'une équipe complète qui le développe activement, tandis que GitHawk est le travail remarquable d'un individu.

Si vous finissez par tester Anaxi, n'hésitez pas à laisser un commentaire et à nous dire ce que vous en pensez. Nous ne sommes qu'au début d'un long voyage !
