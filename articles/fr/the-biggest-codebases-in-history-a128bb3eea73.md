![Image](https://cdn-media-1.freecodecamp.org/images/L1kFuwwi4jM8QDW1r2TE55-8099Tdy-uPCRO)
_Infographie offerte par [Information is Beautiful](https://fcc.im/2qlhEj1" rel="noopener" target="_blank" title=")_

Comme vous pouvez le voir, Google possède de loin la plus grande base de code. Et toutes les 2 milliards de lignes de code tiennent dans un seul dépôt de code.

Mais ce n'est pas un dépôt Git. Google [utilise son propre système de contrôle de version](https://fcc.im/2rRzwDZ), conçu spécifiquement pour que ses 62 000 employés (dont environ 1/3 sont des développeurs) puissent collaborer efficacement.

La distinction de "plus grand dépôt Git" revient à Microsoft, qui a annoncé aujourd'hui que leur dépôt Git Windows contient :

* environ 3,5 millions de fichiers qui
* résultent en un dépôt Git d'environ 300 gigaoctets
* avec 4 000 ingénieurs produisant 1 760 "builds de labo" quotidiens sur 440 branches, plus des milliers de builds de validation de pull request.

Vous pouvez lire comment l'équipe Windows gère cette complexité avec Git ici : ([11 minutes de lecture](https://fcc.im/2qdxiBT))
