Par Gitter

[Python](https://www.python.org/) est un langage dynamique et orienté objet qui combine une puissance remarquable avec une syntaxe très claire. Python est relativement facile à apprendre, disponible partout et adopté par l'industrie. C'est également l'un des langages les plus populaires pour ceux qui apprennent à coder. Le nom « Python » viendrait de Monty Python’s Flying Circus !

Django est un framework d'application web gratuit et open source, écrit en Python. Django est un framework extrêmement largement utilisé, et ce qui le rend si puissant, c'est la possibilité d'être étendu avec des plugins 'app', ainsi que la disponibilité de divers packages.

Découvrez les canaux Python & Django disponibles sur [Gitter](http://gitter.im) :

* [**django/django**](https://gitter.im/django/django) **—** [Django](https://www.djangoproject.com/) est un framework Web Python de haut niveau qui encourage le développement rapide et une conception propre et pragmatique.
* [**sympy/sympy**](https://gitter.im/sympy/sympy) **—** Un système d'algèbre informatique écrit en Python pur. [http://sympy.org/](http://sympy.org/)
* [**pydata/pandas**](https://gitter.im/pydata/pandas) **—** Une bibliothèque flexible et puissante d'analyse et de manipulation de données pour Python, fournissant des structures de données étiquetées similaires aux objets R data.frame, des fonctions statistiques, et bien plus encore [http://pandas.pydata.org](http://pandas.pydata.org/)
* [**Tornado**](https://gitter.im/tornadoweb/tornado) — Tornado est un framework web Python et une bibliothèque de réseau asynchrone, initialement développé chez FriendFeed. [http://www.tornadoweb.org/](http://www.tornadoweb.org/)
* [**Axelrod-Python/Axelrod**](https://gitter.im/Axelrod-Python/Axelrod) **—** Un outil de recherche pour exécuter des tournois du dilemme du prisonnier itéré [http://axelrod.readthedocs.org/](http://axelrod.readthedocs.org/)
* [**DjangoGirls/tutorial**](https://gitter.im/DjangoGirls/tutorial) **—** Ceci est un [tutoriel](http://tutorial.djangogirls.org/) que nous utilisons pour les ateliers Django Girls.
* [**viewflow/django-material**](https://gitter.im/viewflow/django-material) — Material Design pour les [formulaires django](http://forms.viewflow.io/) et l'admin. Basé sur des templates.
* [**potatolondon/djangae**](https://gitter.im/potatolondon/djangae) — Djangae (djan-gee) est une application Django qui vous permet d'exécuter des applications Django sur Google App Engine, y compris (si vous le souhaitez) l'utilisation des modèles de Django avec le Datastore de App Engine comme base de données sous-jacente. La meilleure façon d'exécuter Django sur Google App Engine [http://djangae.org](http://djangae.org/)
* [**lazzaro/django-scheduler**](https://gitter.im/llazzaro/django-scheduler) — Une application de calendrier pour Django (en cours de refactorisation de [https://github.com/thauber/django-schedule](https://github.com/thauber/django-schedule))
* [**django-silk/silk**](https://gitter.im/django-silk/silk) **—** Silk est un outil de profilage et d'inspection en direct pour le framework Django. Silk intercepte et stocke les requêtes HTTP et les requêtes de base de données avant de les présenter dans une interface utilisateur pour une inspection plus approfondie.
* [**kmmbvnr/django-fsm**](https://gitter.im/kmmbvnr/django-fsm?source=explore) **—** django-fsm ajoute une gestion d'états déclarative simple pour les modèles django.
* [**mariocesar/sorl-thumbnail**](https://gitter.im/mariocesar/sorl-thumbnail) **—** Miniatures pour Django
