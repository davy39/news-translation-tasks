En tant que bibliothèque JavaScript la plus populaire pour construire des applications frontend, il n'y a jamais eu de meilleure année pour apprendre React que 2023. 

Dans ce guide, je vais vous montrer les ressources et conseils les plus précieux qui, selon moi, vous aideront à apprendre React plus rapidement. Vous économiserez également beaucoup de temps et d'énergie précieux dans le processus. 

## Besoin d'apprendre JavaScript ? Vous pouvez l'apprendre tout en apprenant React

La question que l'on me pose le plus fréquemment par les débutants souhaitant apprendre React est : "dois-je apprendre JavaScript ?"

React est une bibliothèque JavaScript et est souvent présentée comme étant "juste JavaScript". Cela suggère que pour vraiment apprendre React, vous devez d'abord connaître JavaScript. La façon dont je caractériserais la relation de JavaScript avec React est la suivante : _mieux vous connaissez JavaScript, meilleur développeur React vous deviendrez_.

Cela étant dit, vous n'avez **pas** besoin d'apprendre tout JavaScript d'abord. De nombreux concepts JavaScript peuvent être appris en même temps que vous apprenez React. 

Voici une courte liste de certains des concepts en JavaScript que vous devrez comprendre pour pouvoir utiliser React efficacement. Ceux-ci incluent :

* Variables 
* Tableaux (et la fonction .map())
* Objets
* Événements JavaScript
* Fonctions et fonctions fléchées 
* Portées et fermetures 
* Promesses et syntaxe async-await.
* Gestion des erreurs de base

Ce sont tous les concepts JavaScript que, selon mon expérience, vous devez vraiment connaître pour travailler avec React à n'importe quel niveau. 

Vous rencontrerez et apprendrez plus de concepts React lorsque vous commencerez à construire vos propres projets et à regarder le code des autres. 

Si vous voulez un guide utile pour commencer à apprendre ces concepts et plus encore aujourd'hui, j'ai écrit un [guide complet sur freeCodeCamp](https://www.freecodecamp.org/news/javascript-skills-you-need-for-react-practical-examples/) qui couvre tout le JavaScript dont vous avez besoin pour être à l'aise avec React.

## Vous cherchez un tutoriel ? Utilisez la nouvelle documentation React

Lorsque vous commencez votre parcours d'apprentissage de React, votre première question sera probablement : "où puis-je apprendre toutes les informations liées à React dont j'ai besoin ?" Vous vous demandez peut-être si vous devriez regarder des cours sur YouTube ou acheter un cours sur Udemy. 

Bien qu'il existe d'excellents et extensifs cours React sur de nombreux sites différents, le premier et principal site sur lequel vous devriez vous appuyer est le site officiel de la documentation React : [reactjs.org](https://reactjs.org/). 

Ce qui est très spécial dans l'apprentissage de React en 2023 par rapport aux années précédentes, c'est que l'année dernière, la documentation React a été complètement mise à jour et améliorée. Elle est entièrement à jour avec la version actuelle de React, contient des tonnes d'exemples pratiques, et même des bac à sable de code interactifs afin que vous puissiez commencer à apprendre React dans le navigateur sans avoir à créer de projets sur votre propre machine. 

> Si vous souhaitez démarrer un nouveau projet React, vous pouvez le faire très rapidement dans le navigateur en utilisant le lien [react.new](https://react.new). Cela créera un CodeSandbox avec une application React complète avec laquelle vous pourrez vous amuser. C'est beaucoup plus rapide et plus facile que de créer un projet sur votre propre ordinateur. 

Vous pouvez trouver la nouvelle documentation React sur [beta.reactjs.org](https://beta.reactjs.org). Dans un avenir proche, vous pourrez la trouver simplement sur reactjs.org.

Une autre grande raison d'utiliser la nouvelle documentation React est qu'elle est très adaptée aux débutants et vous permettra, selon moi, d'apprendre les concepts beaucoup plus rapidement que vous ne le feriez autrement. Si vous avez été intimidé par la lecture de la documentation technique dans le passé, je pense que vous serez agréablement surpris.

## Vous n'avez pas besoin d'apprendre les composants de classe

Si vous vous demandez si vous devez apprendre les composants de classe en 2023, vous n'en avez pas besoin. 

Lorsque vous commencez à apprendre React, vous entendrez parler d'une chose appelée **composant de classe** qui est basée sur une classe JavaScript normale. Ce n'est plus nécessaire de l'apprendre en tant que développeur React, mais cela peut encore vous être bénéfique de l'apprendre.

Après l'introduction d'une fonctionnalité appelée **React hooks** en 2018, les développeurs React sont passés à l'utilisation de composants de fonction, qui sont faits avec des fonctions JavaScript. 

Les composants de classe font toujours partie de React et ils sont encore utilisés dans le code de production dans un certain nombre de cas (comme les limites d'erreur). Mais sachez simplement que vous n'avez pas besoin de les apprendre pour être compétent en React. 

En fait, je les ai rarement rencontrés sauf si je regarde une base de code plus ancienne. La plupart d'entre eux ont déjà été migrés vers des composants de fonction et des hooks React.

## Faites-vous une faveur et apprenez React Query

Toute application sérieuse nécessite des données. Souvent, les données dont vous avez besoin existeront en dehors de votre application, vous aurez donc besoin d'une stratégie pour les récupérer et les utiliser dans votre projet. 

Lorsque vous en arriverez au point de récupérer des données, je vous recommande vivement d'apprendre une bibliothèque appelée **React Query**. 

Vous pouvez installer React Query via le package npm @tanstack/react-query.

![Image](https://www.freecodecamp.org/news/content/images/2023/01/Screen-Shot-2023-01-02-at-5.28.02-AM.png)
_React Query, également appelé Tanstack Query_

React Query est devenu la bibliothèque de référence pour la récupération et la gestion des données externes dans les applications React. 

Quel est l'avantage d'utiliser React Query ? En plus d'être une bibliothèque que de nombreux développeurs React et entreprises connaissent et utilisent désormais, il vous sera très bénéfique d'apprendre React Query car c'est probablement la manière la plus puissante pour vous de récupérer et de gérer des données externes dans vos applications React. 

Le plus grand avantage d'utiliser React Query vient du fait qu'il vous donne un cache (un stockage) qui vous permet de conserver le résultat de chaque requête, afin que vous puissiez "sauvegarder" les données que vous avez récupérées. De plus, il dispose de nombreux outils pour mettre à jour ce cache exactement comme vous le souhaitez. 

Il propose également des informations pertinentes sur le statut de chaque requête que vous effectuez, comme si elle est en cours de chargement ou si elle a abouti à une erreur.

Je vous recommande d'apprendre React Query après vous être familiarisé avec la récupération manuelle de données à l'aide d'outils comme l'API Fetch ou Axios avec le hook useEffect. Une fois que vous serez à l'aise, vous verrez les avantages d'utiliser React Query dans tous vos projets React.

## Vous n'avez pas besoin d'apprendre Redux

Si vous avez essayé d'apprendre React les années passées, vous avez peut-être eu l'impression que pour apprendre React, vous deviez finalement apprendre une autre bibliothèque appelée Redux. 

**Redux** est une bibliothèque qui nous aide à gérer l'état dans nos applications React. Vous pouvez penser à l'état comme à toute donnée qui pourrait changer dans une application React. Redux nous permet de gérer une partie de l'état (comme, notre utilisateur est-il connecté) dans toutes les parties de notre application React. 

Redux n'est plus requis pour l'apprentissage comme il l'était les années passées. La première raison à cela est la sortie de l'API **React Context**. Dans de nombreux cas, les développeurs React utilisaient Redux pour simplement transmettre des données entre les composants de leur application. C'est un travail que React Context peut faire pour nous.

De plus, si vous devez changer ou mettre à jour l'état dans de nombreuses parties différentes de votre application (ce qui est une exigence différente de la simple lecture de votre état), il existe de nombreux concurrents à Redux. Contrairement à Redux, ces bibliothèques ne nécessitent pas que vous appreniez de nouveaux concepts, tels que les réducteurs, les actions, et ainsi de suite. 

![Image](https://www.freecodecamp.org/news/content/images/2023/01/image.png)
_Zustand - Gestion d'état "Bearbone"_

Certaines de ces nouvelles bibliothèques de gestion d'état, plus "légères", incluent :

* **Zustand**
* Jotai
* Recoil

Une fois que vous aurez dépassé les bases de React et que vous commencerez à construire des applications légèrement plus grandes que la standard todo app, vous comprendrez le besoin d'avoir une bibliothèque de gestion d'état. Redux est toujours une excellente bibliothèque, mais regardez d'abord une bibliothèque avec une API plus simple comme Zustand.

## Ne vous précipitez pas nécessairement vers Create React App

Une fois que vous en serez au point de créer un projet React sur votre propre ordinateur, sachez qu'il existe de nombreuses façons de le faire. 

Le développement web en général se déplace de plus en plus en ligne - tous les projets React n'ont pas besoin d'être créés sur votre ordinateur. Dans de nombreux cas, vous pouvez démarrer un tout nouveau projet dans le navigateur en utilisant des outils comme **CodeSandbox** ou **StackBlitz** entièrement gratuitement. 

Il y aura cependant un moment où vous devrez ou voudrez créer un projet React sur votre machine locale. Il est facile de se tourner vers un outil comme **Create React App**, qui vous permet de créer un projet React en exécutant une seule commande.

Create React App est toujours un excellent outil pour créer des projets React. Tout comme avec Redux, de nouvelles alternatives ont émergé pour vous offrir toutes les commodités de Create React, mais dans un package plus petit et souvent plus rapide. 

![Image](https://www.freecodecamp.org/news/content/images/2023/01/Screen-Shot-2023-01-02-at-5.30.02-AM.png)
_Vite, une alternative à Create React App_

L'exemple principal de cela est **Vite**. Vite est un outil de construction qui vous permet de démarrer un nouveau projet React ainsi que de nombreux autres types de projets JavaScript. 

L'avantage d'utiliser Vite par rapport à Create React App est qu'il vous offre un serveur de développement beaucoup plus rapide. 

Pour développer et apporter des modifications à votre application, elle doit s'exécuter sur un serveur de développement. Create React App, en comparaison, peut prendre un temps relativement long pour démarrer et répondre aux changements de code que vous apportez (cela s'appelle le **rechargement à chaud**). Vite offre une expérience beaucoup plus rapide en plus de nécessiter beaucoup moins de dépendances. 

Une autre excellente alternative à Create React App est **Next.js**. Next est techniquement un framework React, qui vient avec de nombreuses commodités pour faciliter le développement. Il vient avec son propre routeur et des outils de récupération de données, pour n'en nommer que quelques-uns. 

L'avantage d'utiliser un framework comme Next.js est que vous avez simplement à prendre moins de décisions lorsque vous construisez votre projet. Vous devez choisir beaucoup moins de dépendances sur lesquelles votre projet repose. La plupart de tout ce dont vous avez besoin pour construire votre application React est intégré à Next.js 

Cela étant dit, Next fonctionne assez différemment des projets React normaux en raison du fait qu'il est rendu côté serveur par défaut. Cela peut entraîner ses propres problèmes et pièges potentiels à éviter. Mais si vous êtes intéressé par la construction de projets React sérieux et prêts pour la production, Next est probablement le meilleur choix.
