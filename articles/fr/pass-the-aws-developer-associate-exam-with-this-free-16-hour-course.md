## Qu'est-ce qu'un ingénieur cloud ?

Un ingénieur cloud est un développeur web full stack qui sait utiliser le cloud. Les ingénieurs cloud sont des développeurs web, mais tous les développeurs web ne sont pas des ingénieurs cloud.

Contrastons ces deux rôles pour voir comment les responsabilités changent avec l'utilisation du cloud :

### Développeur Web

* La configuration d'un environnement de production nécessite une connaissance approfondie de Linux et de la configuration des packages du système d'exploitation. (Certains développeurs évitent complètement le besoin de connaissances en déploiement en utilisant des plateformes en tant que service comme Heroku.)
* Il n'est pas facile pour les développeurs web de tirer parti du Machine Learning, du stockage cloud, de l'analytique ou de la réalité virtuelle, car leurs outils sont souvent directement intégrés dans l'application sur un seul serveur.
* À mesure que la complexité d'une seule application web augmente, il devient plus difficile d'ajouter plus de développeurs web car l'application devient plus difficile à apprendre, à configurer et à maintenir.

### Ingénieur Cloud

* Un ingénieur cloud dispose de nombreuses options de déploiement basées sur l'architecture qu'il utilise. De nombreux services cloud gèrent la complexité pour vous. Ensuite – une fois que vous avez appris les outils cloud – vous pouvez gérer directement le déploiement vous-même.
* Les ingénieurs cloud ont plusieurs architectures parmi lesquelles choisir, telles que les microservices, le serverless ou l'architecture traditionnelle.
* Les ingénieurs cloud sont mieux équipés pour rendre leurs applications hautement disponibles, durables et scalables.
* Les ingénieurs cloud peuvent tirer parti des services cloud pour ajouter du Machine Learning, du stockage cloud, de l'analytique, de la VR, du temps réel, et plus encore.
* Il est plus facile de faire grandir une équipe d'ingénieurs cloud, car l'utilisation des services cloud encourage l'isolation des applications et maintient les applications petites et faciles à maintenir.

## Quels services AWS méritent une attention particulière

La plupart des certifications AWS mettent l'accent sur des services AWS spécifiques. Il est important pour vous de porter une attention particulière à ces services AWS dans vos études. Je voulais mettre en avant les 8 principaux services AWS pour la certification AWS Developer Associate et expliquer pourquoi ils sont importants.

### 1. DynamoDB

DynamoDB est une base de données NoSQL qui vous permet de scalabiliser à n'importe quelle taille. Tout ce que vous avez à faire est de lui dire combien de lectures et d'écritures par seconde, et vous avez une garantie de performance. DynamoDB est serverless, ce qui signifie qu'il se scale automatiquement, et vous pouvez choisir de payer à la demande.

Le cas d'utilisation le plus immédiat que je trouve dans mon quotidien est lorsque j'ai besoin de créer une petite application soutenue par une base de données.

Si vous devez utiliser une base de données relationnelle telle que MySQL ou Postgres, vous pourriez utiliser RDS. Cependant, le coût de départ serait de 15 $ par mois pour une t2.db.micro. Vous pourriez configurer votre propre serveur Postgres sur une t2.nano, ce qui coûterait environ ~5 $ par mois, mais vous devriez configurer, sauvegarder et gérer ce serveur.

Peut-être pensez-vous que vous pourriez utiliser Aurora Serverless, mais selon mon expérience, ce n'était pas aussi rentable que DynamoDB, où la différence était entre payer des dollars et payer des centimes.

En tant qu'ingénieur cloud, vous voulez créer des applications isolées plutôt que de grandes applications qui font tout sur un seul serveur. Ce dernier est ce qu'on appelle construire un monolithe.

L'avenir de l'architecture des applications se dirige vers les microservices. Pour découpler complètement vos services, ils doivent avoir la propriété de leur propre base de données. Avec DynamoDB, vous pouvez faire cela.

Ainsi, dans le cours gratuit AWS Developer Associate, nous avons fait un effort considérable pour nous assurer que vous compreniez DynamoDB de fond en comble. La fiche de révision fait 7 pages de long ! [En fait, nous l'avons publiée gratuitement sur freeCodeCamp afin que vous puissiez l'imprimer le jour de l'examen](https://www.freecodecamp.org/news/ultimate-dynamodb-2020-cheatsheet/).

### 2. Elastic Beanstalk

Elastic Beanstalk (EB) est le moyen le plus rapide de déployer une architecture traditionnelle sur AWS. L'architecture traditionnelle est lorsque vous utilisez des machines virtuelles configurées pour un framework web. Si vous utilisez des frameworks web traditionnels tels que Ruby on Rails, Laravel, ExpressJS, Django ou Spring, alors vous utilisez une architecture traditionnelle.

Lorsque vous utilisez des microservices ou une architecture serverless, votre code est divisé en plus petits morceaux. Beaucoup des responsabilités de votre framework web traditionnel sont transférées aux services d'intégration d'applications AWS.

Cependant, la majorité des entreprises technologiques utilisent une architecture traditionnelle car c'est ce qu'elles connaissent, et il faut du temps pour que les entreprises adoptent les microservices et le serverless.

Lorsque vous voulez déployer une application web traditionnelle, vous devez :

* Configurer une image de machine virtuelle en installant les bonnes bibliothèques et applications
* Vous devez configurer un équilibreur de charge et des groupes d'auto-scaling
* Vous devez configurer une base de données relationnelle et une connexion sécurisée
* Vous devez configurer votre réseau cloud tel que les groupes de sécurité
* Vous devrez configurer un pipeline de déploiement.

Elastic Beanstalk configurera tout ce qui précède pour vous. Tout ce que vous avez à faire est de choisir l'environnement que vous voulez et de télécharger votre code.

Elastic Beanstalk gère l'infrastructure mais ne l'abstrait pas, donc vous pouvez explorer tous les services qu'il configure pour vous. Finalement, lorsque vous serez familiarisé avec toute l'infrastructure sous-jacente, vous pourrez gérer directement ces ressources.

J'aime penser à Elastic Beanstalk comme des roues d'entraînement pour le déploiement. C'est le meilleur moyen de commencer sur AWS si vous êtes un développeur, et nous vous montrons comment déployer de différentes manières avec EB.

### 3. AWS CLI et SDK, 4. CloudFormation

Presque tous les services AWS peuvent être accessibles par programmation via l'API AWS. Cela vous permet d'écrire du code pour automatiser la création, la suppression et la configuration de n'importe quels services et ressources AWS dans votre compte.

AWS CloudFormation (CFN) est également utilisé pour automatiser la création et la configuration de l'infrastructure. Bien qu'il soit important pour les développeurs de connaître CFN, l'AWS CLI et SDK est plus important pour le Developer Associate car il permet un contrôle plus fin des services par programmation. Lorsque vous rencontrez une situation où quelque chose ne peut pas être fait avec CFN, vous pouvez être sûr de pouvoir le faire avec le CLI ou le SDK.

Pour accéder à l'API AWS, vous utilisez soit l'interface de ligne de commande AWS (AWS CLI) soit le kit de développement logiciel AWS (SDK).

L'AWS CLI fait gagner du temps aux développeurs en évitant de se connecter à la console AWS et de naviguer dans l'interface graphique.

L'AWS SDK est le moyen principal d'intégrer les services AWS dans vos applications web. Le SDK est également disponible dans la plupart des langages de programmation courants.

Dans ce cours gratuit AWS Developer Associate, nous saisissons chaque opportunité d'utiliser le CLI et le SDK, et nous avons ajouté des diapositives supplémentaires montrant les commandes CLI pour divers services. Vous devrez connaître les commandes CLI pour l'examen, et elles sont toutes importantes à connaître en tant qu'ingénieur cloud.

### 5 / 6. ECS et Fargate

Elastic Container Service (ECS) et ECS Fargate facilitent l'exécution d'applications à conteneurs uniques ou multiples. L'exécution de vos applications web devient de plus en plus populaire car elle vous permet d'emballer la configuration de votre serveur avec votre code, vous offrant une plus grande portabilité des applications.

### 7. X-Ray

Avec l'architecture de microservices, vous avez de nombreux services isolés travaillant ensemble. Il peut être difficile de surveiller les performances ou de localiser les défaillances, donc X-Ray est un service qui vous permet de tracer le chemin des requêtes HTTPS à travers divers services.

### 8. Step Functions

Les Lambdas vous permettent de payer par 100 ms de temps de calcul – vous téléchargez simplement votre code et AWS est responsable du reste. Le défi est de savoir comment organiser toutes ces fonctions Lambda en applications serverless réelles.

Step Functions est une machine d'état qui vous permet de définir quelque chose qui ressemble à un organigramme afin que vous puissiez construire des applications serverless.

### 9. CodeCommit 10. CodeBuild 11. CodeDeploy et 12. CodePipeline

Elastic Beanstalk (EB) est livré avec un pipeline de déploiement simple. Lorsque vous passez d'EB, vous devrez construire votre propre pipeline de déploiement. Nous devons donc savoir comment utiliser tous les services CI/CD AWS.
