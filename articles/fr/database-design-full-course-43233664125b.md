### Pourquoi ce cours ?

La conception de bases de données est une étape fondamentale pour construire presque n'importe quelle application logicielle. Les bases de données alimentent le backend de presque tous les sites web, applications de bureau et applications mobiles. Les bases de données sont cachées des utilisateurs afin que ces derniers ne réalisent pas leur existence. Cela est une bonne chose en termes d'expérience utilisateur et de sécurité, mais a trompé de nombreux jeunes développeurs en les incitant à négliger l'importance des bases de données et de la gestion des données.

En conséquence de cette négligence, les développeurs publient des applications avec des couches de données faibles remplies de données incorrectes, de données laides, de données redondantes et de graves failles de sécurité. Ce manque de discipline avec la base de données est particulièrement vrai avec les applications web, car les développeurs web sont souvent tenus de connaître de nombreuses technologies (HTML, CSS, JavaScript, langages côté serveur, SQL et conception de bases de données). Il est très facile de négliger la base de données car ce n'est pas une technologie frontale comme JavaScript ou HTML et CSS. Cela est une énorme erreur.

Avec l'augmentation des violations de données et des réglementations comme le RGPD, une faille dans la base de données pourrait suffire à causer la chute d'une organisation entière avec la fuite involontaire de données sensibles. La première étape pour se protéger contre les problèmes de sécurité est une base de données correctement conçue et une compréhension des principes SQL et des bases de données. Après avoir suivi ce cours, vous aurez une solide compréhension de l'intégrité et de la sécurité des données ainsi que des étapes claires à suivre pour renforcer l'intégrité.

### Conception intuitive de bases de données

Beaucoup de gens ne réalisent pas qu'il existe diverses façons de concevoir une base de données. Beaucoup de ces choix dépendent des préférences personnelles et des besoins de l'application. Ce cours expliquera certaines de ces options et discutera des avantages et des inconvénients.

Par exemple, vous comprendrez les clés de substitution et les clés naturelles, les différents types de relations et les tables de recherche, les contraintes de référence et celles à utiliser, les relations NOT NULL, les index clusterisés vs non clusterisés, les types de jointures à utiliser, et plus encore.
